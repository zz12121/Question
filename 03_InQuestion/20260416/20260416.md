211. 你写了一个"本地缓存"组件，用 `ConcurrentHashMap` + 过期时间实现：
```java
public class LocalCache<K, V> {
    private final ConcurrentHashMap<K, CacheEntry<V>> cache = new ConcurrentHashMap<>();
    private final long expireAfterWriteMillis;

    public LocalCache(long expireAfterWriteMillis) {
        this.expireAfterWriteMillis = expireAfterWriteMillis;
    }

    public void put(K key, V value) {
        cache.put(key, new CacheEntry<>(value, System.currentTimeMillis()));
    }

    public V get(K key) {
        CacheEntry<V> entry = cache.get(key);
        if (entry == null) return null;
        if (System.currentTimeMillis() - entry.timestamp > expireAfterWriteMillis) {
            cache.remove(key);  // 惰性删除
            return null;
        }
        return entry.value;
    }

    static class CacheEntry<V> {
        V value;
        long timestamp;
        CacheEntry(V value, long timestamp) {
            this.value = value;
            this.timestamp = timestamp;
        }
    }
}
```
**问题：** 这个缓存实现有什么问题？（过期的 entry 不会被主动清理，内存泄漏 / get 和 remove 之间有竞态条件）在高并发读取场景下，大量 key 同时过期，`cache.remove()` 会不会成为瓶颈？和 Caffeine 的 Window TinyLfu 策略相比差在哪里？如果要加一个"定时清理过期 key"的后台线程，用什么数据结构高效扫描？（优先级队列按过期时间排序）如果要加"淘汰策略"（LRU / LFU），如何最小化锁竞争？

public class OrderIdGenerator {
    private final AtomicInteger sequence = new AtomicInteger(0);
    private final int maxSequence = 9999;
    private long lastTimestamp;

    public synchronized String generateId() {
        long timestamp = System.currentTimeMillis();
        if (timestamp == lastTimestamp) {
            int seq = sequence.incrementAndGet();
            if (seq > maxSequence) {
                // 同一毫秒内序列号用完了，等待下一毫秒
                while (timestamp <= lastTimestamp) {
                    timestamp = System.currentTimeMillis();
                }
                sequence.set(0);

        }

    }
}

public class DataLoader<K, V> {
    private final ConcurrentHashMap<K, V> cache = new ConcurrentHashMap<>();
    private final Function<K, V> loader;

    public DataLoader(Function<K, V> loader) {
        this.loader = loader;
    }

    public V get(K key) {

        if (value == null) {

}
