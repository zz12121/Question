47. 你的批量插入：
```java
for (User user : userList) {
    jdbcTemplate.update("INSERT INTO user (name, age) VALUES (?, ?)", user.getName(), user.getAge());
}
```
**问题：** 如果 userList 有 10 万条，有什么问题？如何优化？

---
## 结论先行

**10 万次循环单条插入 = 10 万次网络往返 = 可能跑 10 分钟甚至超时。** 优化核心是**减少网络交互次数**，从 O(N) 降到 O(1)，用批量插入 + 分页 + 多线程，性能提升 100 倍起步。

---

## 一、问题分析

### 原始代码的问题

```java
for (User user : userList) {  // 10万次循环
    jdbcTemplate.update("INSERT INTO user ...", ...);  // 每次都是独立网络往返
}
```

**执行过程：**

```
应用服务器                              数据库
    │                                     │
    ├─ INSERT × 1 ──────────────────────▶│ 执行
    │◀─────────────── 返回结果 ───────────┤
    │                                     │
    ├─ INSERT × 1 ──────────────────────▶│ 执行
    │◀─────────────── 返回结果 ───────────┤
    │                                     │
    │           ... 重复 10万次 ...       │
    │                                     │
总耗时：10万 × (网络延迟 + 执行时间) ≈ 几分钟~几十分钟
```

### 具体问题

| 问题 | 影响 |
|------|------|
| 网络往返次数过多 | 每次插入都有网络延迟，总耗时巨高 |
| 数据库事务未控制 | 每条都是独立事务，事务日志开销翻倍 |
| 连接池频繁获取/释放 | 10万次连接获取，增加额外开销 |
| 事务边界不明确 | 如果中途失败，数据部分插入，难以回滚 |

---

## 二、优化方案

### 方案一：JDBC 批量插入（最常用）✅

**核心思想：攒一批再发，1 次网络往返插入 N 条**

```java
// ❌ 原始写法：每条单独发
for (User user : userList) {
    jdbcTemplate.update("INSERT INTO user (name, age) VALUES (?, ?)", 
        user.getName(), user.getAge());
}

// ✅ 优化写法：攒一批后批量发
String sql = "INSERT INTO user (name, age) VALUES (?, ?)";

try (Connection conn = dataSource.getConnection();
     PreparedStatement ps = conn.prepareStatement(sql)) {
    
    conn.setAutoCommit(false);  // 关闭自动提交，手动控制事务
    
    for (User user : userList) {
        ps.setString(1, user.getName());
        ps.setInt(2, user.getAge());
        ps.addBatch();  // 加入批次
    }
    
    ps.executeBatch();  // 一次性发送所有 SQL
    conn.commit();      // 提交事务
    
} catch (Exception e) {
    conn.rollback();
    throw e;
}
```

**MyBatis-Plus 实现：**

```java
@Mapper
public interface UserMapper extends BaseMapper<User> {
    
    // 批量插入（MyBatis-Plus 自动优化）
    @Insert("<script>" +
            "INSERT INTO user (name, age) VALUES " +
            "<foreach collection='list' item='u' separator=','>" +
            "(#{u.name}, #{u.age})" +
            "</foreach>" +
            "</script>")
    int batchInsert(@Param("list") List<User> userList);
}
```

**Spring JdbcTemplate 实现：**

```java
String sql = "INSERT INTO user (name, age) VALUES (?, ?)";

    .map(u -> new Object[]{u.getName(), u.getAge()})
    .collect(Collectors.toList());

```

### 方案二：MySQL 特殊优化 `rewriteBatchedStatements` ✅

**核心思想：让 MySQL 驱动把多条 INSERT 合并成一条**

```properties
# application.yml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/test?rewriteBatchedStatements=true
```

**效果对比：**

```sql

-- ✅ rewriteBatchedStatements=true（1次网络往返）
```

|------|-----------|

### 方案三：分页批处理（防止内存溢出）

**核心思想：分批处理，每批 5000 条，防止 JDBC 批量太大或内存溢出**

```java
private static final int BATCH_SIZE = 5000;

public void batchInsertWithPaging(List<User> userList) {
    String sql = "INSERT INTO user (name, age) VALUES (?, ?)";
    
    try (Connection conn = dataSource.getConnection()) {
        conn.setAutoCommit(false);
        
        int total = userList.size();
        for (int i = 0; i < total; i += BATCH_SIZE) {
            List<User> batch = userList.subList(i, Math.min(i + BATCH_SIZE, total));
            
            try (PreparedStatement ps = conn.prepareStatement(sql)) {
                for (User user : batch) {
                    ps.setString(1, user.getName());
                    ps.setInt(2, user.getAge());
                    ps.addBatch();
                }
                ps.executeBatch();
                conn.commit();
                
                log.info("批次插入完成, progress={}/{}", Math.min(i + BATCH_SIZE, total), total);
            }
        }
    }
}
```

### 方案四：多线程并行插入（终极优化）

**核心思想：把数据分成 N 份，N 个线程并行插入**

```java
private static final int BATCH_SIZE = 5000;
private static final int THREAD_COUNT = 8;

public void parallelBatchInsert(List<User> userList) {
    // 分成 8 个分区
    List<List<User>> partitions = Lists.partition(userList, 
        (userList.size() + THREAD_COUNT - 1) / THREAD_COUNT);
    
    ExecutorService executor = Executors.newFixedThreadPool(THREAD_COUNT);
    CountDownLatch latch = new CountDownLatch(partitions.size());
    
    for (List<User> partition : partitions) {
        executor.submit(() -> {
            try {
                batchInsert(partition);  // 每个线程独立处理自己的分区
            } finally {
                latch.countDown();
            }
        });
    }
    
    latch.await();
    executor.shutdown();
}

private void batchInsert(List<User> batch) {
    String sql = "INSERT INTO user (name, age) VALUES (?, ?)";
    try (Connection conn = dataSource.getConnection();
         PreparedStatement ps = conn.prepareStatement(sql)) {
        
        conn.setAutoCommit(false);
        for (User user : batch) {
            ps.setString(1, user.getName());
            ps.setInt(2, user.getAge());
            ps.addBatch();
        }
        ps.executeBatch();
        conn.commit();
    } catch (SQLException e) {
        throw new RuntimeException("批量插入失败", e);
    }
}
```

**效果对比：**

| 方案 | 10万条耗时 |
|------|-----------|
| 单条循环 | 5-10 分钟 |
| 批量插入 | 30-60 秒 |
| 批量 + rewriteBatchedStatements | 10-20 秒 |
| 多线程 8 并行 | 3-5 秒 |

### 方案五：LOAD DATA INFILE（最快）✅

**核心思想：数据不经过应用层，直接让 MySQL 读文件**

```java
// 1. 先把数据写成 CSV 文件
String csvPath = "/tmp/users.csv";
try (PrintWriter writer = new PrintWriter(new FileWriter(csvPath))) {
    writer.println("name,age");  // 表头
    for (User user : userList) {
        writer.println(user.getName() + "," + user.getAge());
    }
}

// 2. MySQL LOAD DATA 命令
String sql = "LOAD DATA LOCAL INFILE '" + csvPath + "' " +
             "INTO TABLE user " +
             "FIELDS TERMINATED BY ',' " +
             "LINES TERMINATED BY '\\n' " +
             "(name, age)";

jdbcTemplate.execute(sql);
```

**性能对比：**

| 方式 | 10万条 |
|------|--------|
| JDBC 批量插入 | 30-60 秒 |
| LOAD DATA INFILE | 1-3 秒 |

> **注意：** `LOAD DATA` 需要 MySQL 服务端开启 `local-infile=1`，且客户端连接要加 `allowLoadLocalInfile=true`。

## 三、生产级最佳实践

### 完整代码示例（Spring Boot）

```java
@Service
@Slf4j
public class UserBatchInsertService {

    private final DataSource dataSource;
    private static final int BATCH_SIZE = 5000;

    public void batchInsertUsers(List<User> users) {
        if (CollectionUtils.isEmpty(users)) {
            return;
        }

        log.info("开始批量插入, 总数={}", users.size());
        long start = System.currentTimeMillis();

        // 分页批处理 + 多线程
        int threadCount = Math.min(8, (users.size() + BATCH_SIZE - 1) / BATCH_SIZE);
        List<List<User>> partitions = Lists.partition(users, BATCH_SIZE);

        ExecutorService executor = Executors.newFixedThreadPool(threadCount);
        try {
            List<Future<Integer>> futures = new ArrayList<>();

            for (List<User> partition : partitions) {
                Future<Integer> future = executor.submit(() -> 
                    doBatchInsert(partition)
                );
                futures.add(future);
            }

            int totalInserted = 0;
            for (Future<Integer> future : futures) {
                totalInserted += future.get();
            }

            long cost = System.currentTimeMillis() - start;
            log.info("批量插入完成, 总数={}, 耗时={}ms, QPS={}", 
                totalInserted, cost, totalInserted * 1000.0 / cost);

        } catch (Exception e) {
            log.error("批量插入失败", e);
            throw new RuntimeException("批量插入失败", e);
        } finally {
            executor.shutdown();
        }
    }

    private int doBatchInsert(List<User> batch) throws SQLException {
        String sql = "INSERT INTO user (name, age) VALUES (?, ?)";

        try (Connection conn = dataSource.getConnection();
             PreparedStatement ps = conn.prepareStatement(sql)) {

            conn.setAutoCommit(false);

            for (User user : batch) {
                ps.setString(1, user.getName());
                ps.setInt(2, user.getAge());
                ps.addBatch();
            }

            int[] results = ps.executeBatch();
            conn.commit();

            // 统计成功插入的行数
            return Arrays.stream(results).sum();
        }
    }
}
```

## 四、面试加分话术

> **面试官追问：为什么批量插入比单条快这么多？**

> 核心是减少网络往返。单条插入 10 万次 = 10 万次网络 RTT（假设 1ms 就是 100 秒纯网络开销）。批量插入攒成一批 = 几十次网络往返。另外数据库执行 SQL 有解析、权限检查、事务开启等固定开销，一批只做一次这些操作。

> **面试官追问：批量插入会不会导致数据库主从延迟？**

> 会。批量插入产生大量 binlog，如果从库同步跟不上，确实会延迟。解决方案：分批插入 + 关注主从延迟监控。或者业务高峰期用限流，控制批量任务的并发度。

---

## 一句话总结

**10 万条循环单插 = 10 万次网络往返 = 性能灾难。** 优化路径：**批量攒发 → rewriteBatchedStatements → 分页批处理 → 多线程并行 → LOAD DATA**。生产环境用多线程 + 分页批处理，3-5 秒搞定 10 万条。
