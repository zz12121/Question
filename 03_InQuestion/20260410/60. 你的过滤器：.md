让我分析一下：

1. 问题分析：
- Filter 会对所有请求生效，包括静态资源（CSS、JS、图片等）
- 没有正确处理鉴权失败后的响应体
- 没有异步处理机制
- 可能缺少异常处理

2. 性能影响：
- 如果 Filter 执行慢，所有请求都会被阻塞
- 线程池会被占满
- 服务不可用

3. 优化方案：
- 使用 Spring Security 或 Shrio
- 配置过滤路径，只过滤必要的路径
- 异步 Filter
- 添加异常处理
## 结论先行

**同步 Filter 执行慢 = 所有请求排队等待 = Tomcat 线程耗尽 = 服务假死。** 核心问题是**Filter 是同步阻塞的，鉴权不应该拖慢正常请求**，解法是**异步 Filter + 路径过滤 + 提前退出**。

## 一、问题分析

### 代码回顾


### 问题一：过滤范围过大

```
Filter 默认对所有请求生效，包括：
├── /static/**      （静态资源，不需要鉴权）
├── /health         （健康检查，不需要鉴权）
├── /metrics        （监控指标，不需要鉴权）
├── /api/**         （业务接口，需要鉴权）
└── /actuator/**    （运维接口，不需要鉴权）

问题：静态资源请求也要走鉴权逻辑，白白浪费线程
```

### 问题二：同步阻塞拖慢所有请求

```
Filter 执行链（同步）：

请求1 ──▶ Filter（鉴权 100ms）──▶ Controller（50ms）──▶ 150ms
请求2 ──▶ Filter（等请求1完成）──▶ Controller（50ms）──▶ 200ms
请求3 ──▶ Filter（等请求2完成）──▶ Controller（50ms）──▶ 250ms
...
请求200 ──▶ 等很久很久...

如果鉴权逻辑慢：
1. 每个请求都要等鉴权完成
2. 请求越多，排队越长
3. Tomcat 线程池满 → 新请求超时 → 服务不可用
```

### 问题三：鉴权失败没有返回 JSON

```java

// 客户端收到的是空白响应，体验差
```

### 问题四：异常处理缺失

```
Filter 抛出异常 → 不会被 Controller 的异常处理器捕获 → 500 错误不可控
```

### 方案一：配置过滤路径（最小化影响范围）✅

#### 1. 注解方式配置

```java
    urlPatterns = "/api/*",  // 只过滤 /api 下的请求
    filterName = "authFilter"

    // ...
```

#### 2. 配置类方式（更灵活）

```java
@Configuration
public class FilterConfig {

    @Bean
    public FilterRegistrationBean<AuthFilter> authFilterRegistration() {
        FilterRegistrationBean<AuthFilter> registration = new FilterRegistrationBean<>();
        registration.setFilter(new AuthFilter());
        registration.addUrlPatterns("/api/*");      // 只过滤 API
        registration.addUrlPatterns("/admin/*");    // 管理接口
        registration.setOrder(1);                   // 过滤链顺序
        return registration;
    }

    @Bean
    public FilterRegistrationBean<StaticResourceFilter> staticFilterRegistration() {
        FilterRegistrationBean<StaticResourceFilter> registration = new FilterRegistrationBean<>();
        registration.setFilter(new StaticResourceFilter());
        registration.addUrlPatterns("/static/*", "*.js", "*.css", "*.png");
        registration.setOrder(0);  // 在鉴权之前执行，直接放行
        return registration;
    }
}

// 静态资源过滤器：直接放行，不做鉴权
```

### 方案二：异步 Filter（解耦耗时操作）✅

**核心思想：鉴权用异步方式执行，不阻塞请求线程**

```java
@Component
@WebFilter(
    urlPatterns = "/api/*",
    filterName = "authFilterAsync"
)
public class AuthFilterAsync implements Filter {

    @Autowired
    private AuthService authService;

    @Override
    public void doFilter(ServletRequest request, ServletResponse response, FilterChain chain) 
            throws IOException, ServletException {
        
        HttpServletRequest req = (HttpServletRequest) request;
        HttpServletResponse resp = (HttpServletResponse) response;

        // 1. 快速获取 token
        String token = req.getHeader("token");
        
        // 2. 不需要鉴权的路径直接放行
        if (isPublicPath(req.getRequestURI())) {
            chain.doFilter(request, response);
            return;
        }

        // 3. token 为空直接拒绝（不同步等待）
        if (token == null || token.isEmpty()) {
            writeUnauthorized(resp, "请先登录");
            return;
        }

        // 4. 异步处理耗时鉴权
        AsyncContext asyncContext = req.startAsync();
        asyncContext.setTimeout(5000);  // 5 秒超时
        
        // 用线程池异步鉴权
        authService.validateTokenAsync(token, new AuthCallback() {
            @Override
            public void onSuccess(User user) {
                // 鉴权成功，存入上下文，继续处理
                req.setAttribute("currentUser", user);
                asyncContext.start(() -> {
                    try {
                        chain.doFilter(req, response);
                    } catch (Exception e) {
                        log.error("请求处理异常", e);
                    } finally {
                        asyncContext.complete();
                    }
                });
            }

            @Override
            public void onFailure(String error) {
                writeUnauthorized(asyncContext.getResponse(), error);
                asyncContext.complete();
            }
        });
    }

    private boolean isPublicPath(String uri) {
        return uri.startsWith("/api/public/") 
            || uri.startsWith("/health")
            || uri.startsWith("/actuator/");
    }

    private void writeUnauthorized(ServletResponse response, String msg) throws IOException {
        HttpServletResponse resp = (HttpServletResponse) response;
        resp.setStatus(401);
        resp.setContentType("application/json");
        resp.getWriter().write("{\"code\":401,\"msg\":\"" + msg + "\"}");
    }
}
```

### 方案三：添加异常处理 ✅

```java
@Component
public class AuthFilter implements Filter {

    private static final Logger log = LoggerFactory.getLogger(AuthFilter.class);

    @Override
    public void doFilter(ServletRequest request, ServletResponse response, FilterChain chain) {
        HttpServletRequest req = (HttpServletRequest) request;
        HttpServletResponse resp = (HttpServletResponse) response;

        try {
            // 鉴权逻辑
            String token = req.getHeader("token");
            
            if (token == null) {
                writeUnauthorized(resp, "请先登录");
                return;
            }

            // 验证 token（可能抛异常）
            User user = authService.validateToken(token);
            req.setAttribute("currentUser", user);

            // 继续过滤器链
            chain.doFilter(request, response);

        } catch (AuthException e) {
            // 认证异常
            log.warn("认证失败: {}", e.getMessage());
            writeUnauthorized(resp, e.getMessage());
            
        } catch (Exception e) {
            // 其他异常，不要让异常泄露
            log.error("鉴权异常", e);
            writeError(resp, "服务异常，请稍后重试");
        }
    }

    private void writeUnauthorized(HttpServletResponse resp, String msg) throws IOException {
        resp.setStatus(401);
        resp.setContentType("application/json;charset=UTF-8");
        resp.getWriter().write("{\"code\":401,\"msg\":\"" + msg + "\"}");
    }

    private void writeError(HttpServletResponse resp, String msg) throws IOException {
        resp.setStatus(500);
        resp.setContentType("application/json;charset=UTF-8");
        resp.getWriter().write("{\"code\":500,\"msg\":\"" + msg + "\"}");
    }
}
```

### 方案四：缓存 Token 减少鉴权耗时 ✅

**核心思想：Token 验证结果缓存，避免重复验证**

```java
@Service
@Slf4j
public class AuthService {

    // 本地缓存：验证结果缓存 5 分钟
    private LoadingCache<String, User> tokenCache = 
        Caffeine.newBuilder()
            .maximumSize(10_000)
            .expireAfterWrite(5, TimeUnit.MINUTES)
            .build(key -> validateTokenFromRemote(key));

    // Redis 缓存（多实例共享）
    @Autowired
    private RedisTemplate<String, User> redisTemplate;

    public User validateToken(String token) {
        String cacheKey = "token:" + token;

        // 1. 先查本地缓存
        User cached = localCache.getIfPresent(token);
        if (cached != null) {
            return cached;
        }

        // 2. 再查 Redis
        User user = redisTemplate.opsForValue().get(cacheKey);
        if (user != null) {
            localCache.put(token, user);
            return user;
        }

        // 3. 远程验证
        user = validateTokenFromRemote(token);
        if (user != null) {
            // 写入缓存
            redisTemplate.opsForValue().set(cacheKey, user, Duration.ofMinutes(30));
            localCache.put(token, user);
        }

        return user;
    }

    // 异步验证
    public void validateTokenAsync(String token, AuthCallback callback) {
        try {
            User user = validateToken(token);
            callback.onSuccess(user);
        } catch (Exception e) {
            callback.onFailure(e.getMessage());
        }
    }

    private User validateTokenFromRemote(String token) {
        // 调用 Token 服务验证
        return tokenService.verify(token);
    }
}
```

### 推荐：Spring Security + 异步鉴权

```java
@Configuration
@EnableWebSecurity
@EnableAsync
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .csrf().disable()
            .sessionManagement()
                .sessionCreationPolicy(SessionCreationPolicy.STATELESS)  // 无状态
            .and()
            .authorizeHttpRequests()
                // 放行
                .antMatchers("/health", "/actuator/**").permitAll()
                .antMatchers("/api/public/**").permitAll()
                .antMatchers("/static/**").permitAll()
                // 其他需要认证
                .anyRequest().authenticated()
            .and()
            // 添加 JWT 过滤器
            .addFilterBefore(jwtAuthFilter(), UsernamePasswordAuthenticationFilter.class)
            // 异常处理
            .exceptionHandling()
                .authenticationEntryPoint((request, response, authException) -> {
                    response.setStatus(401);
                    response.setContentType("application/json");
                    response.getWriter().write("{\"code\":401,\"msg\":\"请先登录\"}");
                })
                .accessDeniedHandler((request, response, accessDeniedException) -> {
                    response.setStatus(403);
                    response.setContentType("application/json");
                    response.getWriter().write("{\"code\":403,\"msg\":\"权限不足\"}");
                });

        return http.build();
    }

    @Bean
    public JwtAuthFilter jwtAuthFilter() {
        return new JwtAuthFilter();
    }
}

// JWT 过滤器
@Slf4j
public class JwtAuthFilter extends OncePerRequestFilter {

    @Autowired
    private JwtService jwtService;

    @Override
    protected void doFilterInternal(HttpServletRequest request, 
                                    HttpServletResponse response, 
                                    FilterChain filterChain) 
            throws ServletException, IOException {
        
        String token = request.getHeader("Authorization");
        
        if (token != null && token.startsWith("Bearer ")) {
            token = token.substring(7);
            
            try {
                UserDetails user = jwtService.validateToken(token);
                UsernamePasswordAuthenticationToken authentication = 
                    new UsernamePasswordAuthenticationToken(
                        user, null, user.getAuthorities()
                    );
                authentication.setDetails(
                    new WebAuthenticationDetailsSource().buildDetails(request)
                );
                SecurityContextHolder.getContext().setAuthentication(authentication);
            } catch (Exception e) {
                log.warn("JWT 验证失败: {}", e.getMessage());
                SecurityContextHolder.clearContext();
            }
        }

        filterChain.doFilter(request, response);
    }

    @Override
    protected boolean shouldNotFilter(HttpServletRequest request) {
        String path = request.getRequestURI();
        // 跳过不需要认证的路径
        return path.startsWith("/health")
            || path.startsWith("/api/public/")
            || path.startsWith("/static/");
    }
}
```

## 五、面试加分话术

> **面试官追问：Filter 和 Interceptor 的区别？**

> Filter 是 Servlet 规范，在 Spring 容器之外执行，可以处理所有请求（包括静态资源）。Interceptor 是 Spring MVC 的，在 Spring 容器内执行，只能处理进入 Controller 的请求。Filter 可以处理请求和响应，Interceptor 可以获取 Handler（Controller 方法）和结果。还有一个区别是 Filter 的 doFilter 方法需要手动调用 chain.doFilter() 继续往下传，Interceptor 的 preHandle 返回 true 才会继续执行。

> **面试官追问：Filter 和 AOP 的区别？**

> Filter 处理所有进入 Web 容器的请求，是 HTTP 层面的拦截。AOP 是方法级别的拦截，可以拦截任何 Bean 的方法调用。Filter 先于 AOP 执行。Filter 适合处理鉴权、日志、跨域等横切关注点，AOP 适合处理业务方法的前后置处理、事务管理等。

---

## 一句话总结

**同步 Filter 执行慢会拖垮所有请求。** 优化套路：**只过滤必要路径（@WebFilter urlPatterns）→ 提前退出（public 路径直接放行）→ 添加异常处理 → Token 验证结果缓存 → 必要时用异步 Filter**。生产环境推荐 Spring Security，内置了完整的鉴权体系。
