55. 你用 Redis 存储 Session：
```java
redis.setex("session:" + sessionId, 1800, userInfo);
```
**问题：** 如果用户一直在操作，Session 还是 30 分钟过期，如何做到"活跃续期"？

---
用户问的是 Session 续期问题。当用户一直在操作时，Session 应该在用户每次操作时自动续期，而不是固定 30 分钟后过期。这是 Session 管理的常见问题。

核心解法：
1. 每次请求时重新设置过期时间（但是有并发问题）
2. 使用 Redis 的 TTL 特性，判断剩余时间是否小于阈值，小于则续期
3. 使用滑动窗口策略
4. 不依赖过期，用定期清理机制
## 结论先行

**固定 TTL 的问题是"断崖式过期"——用户 29 分钟在操作，第 30 分钟不操作就踢出去了。** 解法是**每次请求时检查 TTL，剩余时间不足则续期**（也叫滑动过期）。

## 二、解决方案

### 方案一：惰性续期（每次请求检查 + 续期）✅

**核心思想：每次查 Session 时，检查剩余 TTL，如果不足就续期**

```java
public UserInfo getSession(String sessionId) {
    String key = "session:" + sessionId;
    
    // 1. 查 Session
    String json = redis.get(key);
    if (json == null) {
        return null;  // Session 过期
    }
    
    // 2. 检查 TTL，剩余不足 5 分钟则续期
    Long ttl = redis.ttl(key);
    if (ttl != null && ttl < 300) {  // 5 分钟 = 300 秒
        redis.expire(key, 1800);  // 续期 30 分钟
    }
    
    return JSON.parseObject(json, UserInfo.class);
}
```

**完整包装：**

```java
@Service
@Slf4j
public class SessionService {

    private final StringRedisTemplate redisTemplate;
    
    private static final int SESSION_TTL = 1800;    // 30 分钟
    private static final int RENEW_THRESHOLD = 300; // 剩余 < 5 分钟时续期

    public void createSession(String sessionId, UserInfo userInfo) {
        String key = "session:" + sessionId;
        redisTemplate.opsForValue().set(key, JSON.toJSONString(userInfo), 
            Duration.ofSeconds(SESSION_TTL));
    }

    public UserInfo getSession(String sessionId) {
        String key = "session:" + sessionId;
        
        // 1. 查 Session
        String json = redisTemplate.opsForValue().get(key);
        if (json == null) {
            return null;
        }
        
        // 2. 惰性续期
        Long ttl = redisTemplate.getExpire(key);
        if (ttl != null && ttl < RENEW_THRESHOLD) {
            redisTemplate.expire(key, Duration.ofSeconds(SESSION_TTL));
            log.debug("Session 续期, sessionId={}, oldTTL={}", sessionId, ttl);
        }
        
        return JSON.parseObject(json, UserInfo.class);
    }

    public void updateSession(String sessionId, UserInfo userInfo) {
        String key = "session:" + sessionId;
        // 更新内容时同时续期
        redisTemplate.opsForValue().set(key, JSON.toJSONString(userInfo), 
            Duration.ofSeconds(SESSION_TTL));
    }

    public void destroySession(String sessionId) {
        redisTemplate.delete("session:" + sessionId);
    }
}
```

```java
    public void afterCompletion(HttpServletRequest request, 
                                HttpServletResponse response, 
                                Object handler, 
                                Exception ex) {
        // 清理 ThreadLocal
        UserContext.clear();
    }
}
```

### 方案三：Lua 脚本原子续期 ✅

**核心思想：get + 检查 + expire 写成原子操作**

```java
public class SessionServiceWithLua {

    private final StringRedisTemplate redisTemplate;

    // Lua 脚本：原子获取 Session 并续期
    private static final String GET_AND_RENEW_SCRIPT = """
        local key = KEYS[1]
        local ttl = tonumber(ARGV[1])
        local newTtl = tonumber(ARGV[2])
        
        local value = redis.call('GET', key)
        if value then
            local currentTtl = redis.call('TTL', key)
            if currentTtl < ttl then
                redis.call('EXPIRE', key, newTtl)
            end
        end
        return value
        """;

    public UserInfo getSession(String sessionId) {
        String key = "session:" + sessionId;
        
        RedisScript<String> script = RedisScript.of(GET_AND_RENEW_SCRIPT, String.class);
        
        String json = redisTemplate.execute(
            script,
            Collections.singletonList(key),
            "300",      // 剩余 < 5 分钟续期
            "1800"      // 续期到 30 分钟
        );
        
        if (json == null) {
            return null;
        }
        
        return JSON.parseObject(json, UserInfo.class);
    }
}
```

### 方案四：永不过期 + 定期清理 ✅

**核心思想：去掉 TTL，用定期任务清理废弃 Session**

```
┌─────────────────────────────────────────────────────────────┐
│                  永不过期策略                                 │
│                                                             │
│  Session 数据结构：                                          │
│  {                                                          │
│    "userId": 123,                                          │
│    "lastAccessTime": 1712000000000,                        │
│    "createTime": 1711990000000                             │
│  }                                                          │
│                                                             │
│  定时任务：扫描所有 Session，删除 lastAccessTime > 30分钟    │
│                                                             │
│  优点：完全避免并发续期问题                                    │
│  缺点：需要额外存储 lastAccessTime，定期扫描有延迟            │
└─────────────────────────────────────────────────────────────┘
```

```java
@Component
@Slf4j
public class SessionCleanupTask {

    @Autowired
    private StringRedisTemplate redisTemplate;

    private static final long SESSION_MAX_IDLE = 30 * 60 * 1000;  // 30 分钟

    // 每 5 分钟扫描清理
    @Scheduled(fixedDelay = 300000)
    public void cleanupExpiredSessions() {
        long now = System.currentTimeMillis();
        long expireTime = now - SESSION_MAX_IDLE;
        
        // 使用 SCAN 遍历（不能用 KEYS，生产环境禁止 KEYS）
        ScanOptions options = ScanOptions.scanOptions()
            .match("session:*")
            .count(100)
            .build();
        
        try (Cursor<String> cursor = redisTemplate.scan(options)) {
            int cleaned = 0;
            while (cursor.hasNext()) {
                String key = cursor.next();
                
                // 获取 Session 内容，检查最后访问时间
                String json = redisTemplate.opsForValue().get(key);
                if (json != null) {
                    SessionData session = JSON.parseObject(json, SessionData.class);
                    
                    if (session.getLastAccessTime() < expireTime) {
                        redisTemplate.delete(key);
                        cleaned++;
                    }
                }
            }
            
            if (cleaned > 0) {
                log.info("清理过期 Session, count={}", cleaned);
            }
        }
    }
}
```

## 四、生产级完整实现

### 推荐：拦截器 + Lua 原子续期

```java
// ========== Session Service ==========
@Service
@Slf4j
public class SessionService {

    private final StringRedisTemplate redisTemplate;
    
    private static final String SESSION_PREFIX = "session:";
    private static final int SESSION_TTL = 1800;     // 30 分钟
    private static final int RENEW_THRESHOLD = 300;  // 剩余 5 分钟续期

    // Lua 脚本：原子获取 + 续期
    private static final String GET_AND_RENEW = """
        local key = KEYS[1]
        local renewTtl = tonumber(ARGV[1])
        local ttl = redis.call('TTL', key)
        if ttl > 0 and ttl < tonumber(ARGV[2]) then
            redis.call('EXPIRE', key, renewTtl)
        end
        return redis.call('GET', key)
        """;

    public void create(String sessionId, UserInfo userInfo) {
        String key = SESSION_PREFIX + sessionId;
        SessionData data = new SessionData(userInfo, System.currentTimeMillis());
        redisTemplate.opsForValue().set(key, JSON.toJSONString(data), 
            Duration.ofSeconds(SESSION_TTL));
    }

    public SessionData get(String sessionId) {
        String key = SESSION_PREFIX + sessionId;
        
        RedisScript<String> script = RedisScript.of(GET_AND_RENEW, String.class);
        String json = redisTemplate.execute(script,
            Collections.singletonList(key),
            String.valueOf(SESSION_TTL),
            String.valueOf(RENEW_THRESHOLD)
        );
        
        if (json == null) return null;
        
        SessionData data = JSON.parseObject(json, SessionData.class);
        data.setLastAccessTime(System.currentTimeMillis());  // 更新访问时间
        
        return data;
    }

    public void update(String sessionId, UserInfo userInfo) {
        String key = SESSION_PREFIX + sessionId;
        SessionData data = new SessionData(userInfo, System.currentTimeMillis());
        redisTemplate.opsForValue().set(key, JSON.toJSONString(data),
            Duration.ofSeconds(SESSION_TTL));  // 更新时自动续期
    }

    public void destroy(String sessionId) {
        redisTemplate.delete(SESSION_PREFIX + sessionId);
    }
}

// ========== 拦截器 ==========
@Component
@Slf4j
public class AuthInterceptor implements HandlerInterceptor {

    @Autowired
    private SessionService sessionService;

    @Override
    public boolean preHandle(HttpServletRequest request, 
                             HttpServletResponse response, 
                             Object handler) throws Exception {
        
        String sessionId = request.getHeader("X-Session-Id");
        
        // 放行的路径
        String path = request.getRequestURI();
        if (path.startsWith("/api/public/") || path.equals("/api/login")) {
            return true;
        }
        
        if (sessionId == null) {
            return unauthorized(response, "请先登录");
        }
        
        SessionService.SessionData session = sessionService.get(sessionId);
        if (session == null) {
            return unauthorized(response, "登录已过期，请重新登录");
        }
        
        // 存入上下文
        UserContext.set(session.getUserInfo());
        return true;
    }

    private boolean unauthorized(HttpServletResponse response, String msg) throws IOException {
        response.setStatus(401);
        response.setContentType("application/json");
        response.getWriter().write(JSON.toJSONString(Result.fail(401, msg)));
        return false;
    }

    @Override
    public void afterCompletion(HttpServletRequest request, 
                               HttpServletResponse response, 
                               Object handler, Exception ex) {
        UserContext.clear();
    }
}
```

## 五、面试加分话术

> **面试官追问：为什么不直接设永不过期（TTL = -1）？**

> 永不过期会导致内存泄漏——用户注销/退出后，如果没手动删除 Session，Redis 里会残留大量垃圾数据。所以要定期清理或用 TTL 控制生命周期。正确做法是**活跃用户通过续期保持会话，30 分钟不活跃的用户自然过期清理**。

> **面试官追问：续期操作有什么并发问题？**

> 理论上 get + expire 不是原子操作，高并发下可能多个请求同时读到 TTL < 5 分钟，然后都去 EXPIRE，重复执行而已，不影响正确性，只是多一次网络往返。如果用 Lua 脚本就能保证原子。另外续期阈值不要设太低（比如 < 1 分钟），否则可能出现"续不上"的边界问题。

---

## 一句话总结

**固定 TTL = 用户操作了 29 分 59 秒还是会被踢出去。** 正确做法是**每次请求时检查 TTL，剩余不足 5 分钟就续期**，保证活跃用户永不过期。生产环境推荐 **Lua 脚本保证原子性 + 拦截器统一处理**。
