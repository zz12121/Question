## 先说结论

`StampedLock`是JDK 8引入的**读写锁升级版**，支持**乐观读**（无锁读，失败再转悲观读），性能比`ReentrantReadWriteLock`更高。特点是**不支持重入**、**不支持条件变量**。死锁场景：**锁升级（读锁→写锁）会死锁**，必须先释放读锁再获取写锁。乐观读通过`tryOptimisticRead()`获取戳，验证戳有效则读数据，无效则转悲观读。

## 🔥 核心概念

### StampedLock vs ReadWriteLock

| 特性 | ReentrantReadWriteLock | StampedLock |
|------|------------------------|-------------|
| **读锁** | 悲观读（阻塞） | 悲观读 + 乐观读（无锁） |
| **写锁** | 独占 | 独占 |
| **重入** | ✅ 支持 | ❌ 不支持 |
| **条件变量** | ✅ 支持 | ❌ 不支持 |
| **性能** | 一般 | 更高（乐观读无锁） |
| **中断** | 支持 | 支持 |
| **适用场景** | 通用 | 读多写少，读操作耗时短 |

### 三种模式

```
┌─────────────────────────────────────────────────────────────┐
│  1. 写锁（writeLock）                                         │
│     - 独占，排他                                               │
│     - 与读写锁的写锁相同                                        │
├─────────────────────────────────────────────────────────────┤
│  2. 悲观读锁（readLock）                                        │
│     - 共享，与写锁互斥                                          │
│     - 与读写锁的读锁相同                                        │
├─────────────────────────────────────────────────────────────┤
│  3. 乐观读（tryOptimisticRead）★ 特色功能                      │
│     - 无锁，不阻塞                                              │
│     - 获取戳（stamp），验证戳有效则数据有效                      │
│     - 写锁获取会"弄脏"戳，使验证失败                            │
│     - 失败后可转为悲观读锁                                       │
└─────────────────────────────────────────────────────────────┘
```

## 🔥 详细展开：死锁场景

### 锁升级死锁（最常见）

```java
// ❌ 错误：读锁升级写锁会死锁
public void wrongUpgrade() {
    StampedLock lock = new StampedLock();
    
    long readStamp = lock.readLock();  // 获取读锁
    try {
        if (needUpdate()) {
            // 尝试获取写锁 - 死锁！
            // 因为读锁被占用，写锁无法获取
            long writeStamp = lock.writeLock();  // 永远阻塞！
            try {
                update();
            } finally {
                lock.unlockWrite(writeStamp);
            }
        }
    } finally {
        lock.unlockRead(readStamp);
    }
}

// ✅ 正确：先释放读锁，再获取写锁
public void correctUpgrade() {
    StampedLock lock = new StampedLock();
    long readStamp = lock.readLock();
    try {
        while (needUpdate()) {
            // 1. 先释放读锁
            long ws = lock.tryConvertToWriteLock(readStamp);
            if (ws != 0L) {
                // 转换成功，直接升级
                readStamp = ws;
                update();
                break;
            } else {
                // 转换失败，释放读锁
                lock.unlockRead(readStamp);
                
                // 2. 获取写锁
                readStamp = lock.writeLock();
                try {
                    // 双重检查（可能其他线程已更新）
                    if (needUpdate()) {
                        update();
                    }
                    break;
                } finally {
                    // 3. 降级为读锁
                    readStamp = lock.tryConvertToReadLock(readStamp);
                }
            }
        }
    } finally {
        lock.unlock(readStamp);
    }
}
```

### 不支持重入导致的"伪死锁"

```java
// ❌ 错误：StampedLock不支持重入
public class NonReentrantProblem {

    } finally {

}
```

## 🔥 详细展开：乐观读使用

### 乐观读原理

```
┌─────────────────────────────────────────────────────────────┐
│  乐观读流程：                                                  │
│                                                              │
│  1. 获取乐观读戳（stamp）                                      │
│     stamp = lock.tryOptimisticRead()                          │
│     此时没有加锁！                                             │
│                                                              │
│  2. 读取数据                                                   │
│     data = readData()                                         │
│                                                              │
│  3. 验证戳是否有效                                             │
│     if (lock.validate(stamp)) {                               │
│         // 有效，数据正确                                       │
│     } else {                                                   │
│         // 无效，被写锁修改过，转为悲观读                        │
│         stamp = lock.readLock();                               │
│     }                                                          │
│                                                              │
│  注意：验证和读取数据之间不能有耗时操作！                         │
└─────────────────────────────────────────────────────────────┘
```

### 乐观读代码示例

```java
public class OptimisticReadExample {
    private final StampedLock lock = new StampedLock();
    private volatile Data data = new Data();
    
    /**
     * 乐观读示例
     */
    public Data readWithOptimistic() {
        // 1. 获取乐观读戳（无锁）
        long stamp = lock.tryOptimisticRead();
        
        // 2. 读取数据（可能不一致）
        Data current = data;
        int value = current.getValue();
        
        // 3. 验证戳（检查期间是否有写操作）
        if (!lock.validate(stamp)) {
            // 验证失败，转为悲观读
            stamp = lock.readLock();
            try {
                current = data;
                value = current.getValue();
            } finally {
                lock.unlockRead(stamp);
            }
        }
        
        return current;
    }
    
    /**
     * 写操作
     */
    public void write(int newValue) {
        long stamp = lock.writeLock();
        try {
            data = new Data(newValue);
        } finally {
            lock.unlockWrite(stamp);
        }
    }
}
```

### 乐观读 + 缓存示例

```java
@Service
public class CacheService {
    private final StampedLock lock = new StampedLock();
    private volatile Map<String, Object> cache = new HashMap<>();
    
    /**
     * 乐观读获取缓存（读多写少场景）
     */
    @SuppressWarnings("unchecked")
    public <T> T get(String key) {
        long stamp = lock.tryOptimisticRead();
        Map<String, Object> currentCache = cache;
        T value = (T) currentCache.get(key);
        
        // 验证期间是否有写操作
        if (!lock.validate(stamp)) {
            // 有写操作，转为悲观读
            stamp = lock.readLock();
            try {
                currentCache = cache;
                value = (T) currentCache.get(key);
            } finally {
                lock.unlockRead(stamp);
            }
        }
        
        return value;
    }
    
    /**
     * 更新缓存（写操作少）
     */
    public void put(String key, Object value) {
        long stamp = lock.writeLock();
        try {
            Map<String, Object> newCache = new HashMap<>(cache);
            newCache.put(key, value);
            cache = newCache;  // volatile写，保证可见性
        } finally {
            lock.unlockWrite(stamp);
        }
    }
}
```

## 一句话收尾

`StampedLock`是**读多写少场景的性能利器**，特点是**乐观读无锁**、**不支持重入和条件变量**。死锁主要发生在**锁升级**（读锁内获取写锁），正确做法是**先释放读锁再获取写锁**。乐观读通过**戳验证**保证数据一致性，验证失败转悲观读，适合**读操作远多于写**且**读操作耗时短**的场景。
