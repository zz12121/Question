33. 你用 `ExecutorService` 提交了一批任务：
```java
ExecutorService executor = Executors.newFixedThreadPool(10);
for (int i = 0; i < 1000; i++) {
    executor.submit(() -> {
        // 处理任务
    });
}
executor.shutdown();
```
**问题：** 如果任务执行过程中抛出异常，会发生什么？如何捕获异常？

---
## 先说结论

`submit()`提交的任务抛出异常时，**异常被吞掉，不会打印堆栈，调用方无感知**。这是因为异常被包装在`Future`对象中，而代码没有获取`Future`结果。解决方案：**① 用`execute()`替代`submit()`**（异常会抛给UncaughtExceptionHandler）；**② 在任务内部try-catch**；**③ 获取`Future.get()`结果**（会抛出ExecutionException包裹的原始异常）。

---

## 🔥 核心概念

### submit() vs execute() 异常处理差异

```
┌─────────────────────────────────────────────────────────────┐
│  submit(Runnable)                                           │
│  ├── 返回Future对象                                          │
│  ├── 异常被捕获并包装在Future中                               │
│  ├── 如果不调用future.get()，异常被"吞掉"                     │
│  └── 线程不会终止，继续执行下一个任务                          │
├─────────────────────────────────────────────────────────────┤
│  execute(Runnable)                                          │
│  ├── 无返回值                                                │
│  ├── 异常抛出到ThreadPoolExecutor                            │
│  ├── 调用UncaughtExceptionHandler处理                        │
│  └── 默认打印堆栈，线程终止，线程池创建新线程替换                │
└─────────────────────────────────────────────────────────────┘
```

---

## 🔥 详细展开：问题演示

### 异常被"吞掉"的场景

```java
ExecutorService executor = Executors.newFixedThreadPool(10);

// 提交1000个任务，部分会抛异常
for (int i = 0; i < 1000; i++) {
    final int taskId = i;
    executor.submit(() -> {
        if (taskId % 10 == 0) {
            throw new RuntimeException("Task " + taskId + " failed!");
        }
        System.out.println("Task " + taskId + " success");
    });
}

executor.shutdown();
executor.awaitTermination(1, TimeUnit.MINUTES);

// 输出结果：
// Task 1 success
// Task 2 success
// ...
// 没有任何异常信息！100个异常被"吞掉"了！
```

### 为什么异常被吞？

```java
// submit()源码简析：
public Future<?> submit(Runnable task) {
    if (task == null) throw new NullPointerException();
    // 包装为FutureTask
    RunnableFuture<Void> ftask = newTaskFor(task, null);
    execute(ftask);  // 最终还是execute
    return ftask;
}

// FutureTask.run()内部：
public void run() {
    try {
        callable.call();  // 执行任务
    } catch (Throwable ex) {
        setException(ex);  // 异常被捕获，保存到Future中
        // 不抛出！不打印！
    }
}
```

## 🔥 详细展开：解决方案

### 方案1：使用execute() + UncaughtExceptionHandler

```java
// 自定义线程工厂，设置异常处理器
ThreadFactory threadFactory = new ThreadFactory() {
    private final AtomicInteger counter = new AtomicInteger(0);
    
    @Override
    public Thread newThread(Runnable r) {
        Thread thread = new Thread(r, "worker-" + counter.incrementAndGet());
        // 设置未捕获异常处理器
        thread.setUncaughtExceptionHandler((t, e) -> {
            System.err.println("Thread " + t.getName() + " threw exception: " + e.getMessage());
            e.printStackTrace();
            // 可以在这里发送告警
            alertService.send("Task failed", e);
        });
        return thread;
    }
};

ExecutorService executor = Executors.newFixedThreadPool(10, threadFactory);

// 使用execute提交
for (int i = 0; i < 1000; i++) {
    final int taskId = i;
    executor.execute(() -> {
        if (taskId % 10 == 0) {
            throw new RuntimeException("Task " + taskId + " failed!");
        }
        System.out.println("Task " + taskId + " success");
    });
}

// 现在异常会被UncaughtExceptionHandler捕获并打印
```

### 方案2：任务内部try-catch（最常用）

```java
for (int i = 0; i < 1000; i++) {
    final int taskId = i;
    executor.submit(() -> {
        try {
            // 业务逻辑
            if (taskId % 10 == 0) {
                throw new RuntimeException("Task " + taskId + " failed!");
            }
            System.out.println("Task " + taskId + " success");
            
        } catch (Exception e) {
            // 捕获并处理异常
            System.err.println("Task " + taskId + " error: " + e.getMessage());
            // 记录日志、发送告警、补偿操作等
            log.error("Task execution failed, taskId={}", taskId, e);
        }
    });
}
```

### 方案3：获取Future结果（需要返回值时）

```java
List<Future<String>> futures = new ArrayList<>();

for (int i = 0; i < 1000; i++) {
    final int taskId = i;
    Future<String> future = executor.submit(() -> {
        if (taskId % 10 == 0) {
            throw new RuntimeException("Task " + taskId + " failed!");
        }
        return "Result of task " + taskId;
    });
    futures.add(future);
}

// 获取结果（会抛出异常）
for (int i = 0; i < futures.size(); i++) {
    try {
        String result = futures.get(i).get();  // 阻塞等待
        System.out.println("Task " + i + " result: " + result);
    } catch (ExecutionException e) {
        // 包装在ExecutionException中的原始异常
        Throwable cause = e.getCause();
        System.err.println("Task " + i + " failed: " + cause.getMessage());
    } catch (InterruptedException e) {
        Thread.currentThread().interrupt();
    }
}
```

### 方案4：CompletableFuture（推荐Java 8+）

```java
for (int i = 0; i < 1000; i++) {
    final int taskId = i;
    
    CompletableFuture
        .supplyAsync(() -> {
            if (taskId % 10 == 0) {
                throw new RuntimeException("Task " + taskId + " failed!");
            }
            return "Result of task " + taskId;
        }, executor)
        .thenAccept(result -> {
            System.out.println("Success: " + result);
        })
        .exceptionally(ex -> {
            // 统一处理异常
            System.err.println("Task " + taskId + " error: " + ex.getMessage());
            return null;
        });
}
```

## 🔥 扩展追问 1：批量任务异常处理最佳实践

```java
/**
 * 批量任务执行器，带异常收集
 */
@Component
public class BatchTaskExecutor {
    
    @Autowired
    private ExecutorService executor;
    
    public <T> BatchResult<T> executeBatch(List<Callable<T>> tasks) {
        List<Future<T>> futures = new ArrayList<>();
        List<T> successes = new ArrayList<>();
        List<TaskError> errors = new ArrayList<>();
        
        // 提交所有任务
        for (Callable<T> task : tasks) {
            futures.add(executor.submit(task));
        }
        
        // 收集结果
        for (int i = 0; i < futures.size(); i++) {
            try {
                T result = futures.get(i).get();
                successes.add(result);
            } catch (ExecutionException e) {
                errors.add(new TaskError(i, e.getCause()));
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
                errors.add(new TaskError(i, e));
            }
        }
        
        return new BatchResult<>(successes, errors);
    }
    
    @Data
    public static class BatchResult<T> {
        private final List<T> successes;
        private final List<TaskError> errors;
        
        public boolean hasError() {
            return !errors.isEmpty();
        }
        
        public double getSuccessRate() {
            int total = successes.size() + errors.size();
            return total == 0 ? 0 : (double) successes.size() / total;
        }
    }
    
    @Data
    public static class TaskError {
        private final int taskIndex;
        private final Throwable cause;
    }
}
```

@Configuration
@EnableAsync
public class AsyncConfig implements AsyncConfigurer {
    
    @Override
    @Bean(name = "taskExecutor")

        ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
        executor.setCorePoolSize(10);

    }

        // 处理void返回值的@Async方法异常
        return (ex, method, params) -> {
            log.error("Async method exception: {}.{}",
                method.getDeclaringClass().getName(),
                method.getName(), ex);
        };
    }
}

        }

    }
}

## 🔥 扩展追问 3：优雅关闭线程池

```java
/**
 * 线程池优雅关闭，等待任务完成
 */
public void gracefulShutdown(ExecutorService executor, long timeout, TimeUnit unit) {
    // 第一阶段：优雅关闭，等待已提交任务完成
    executor.shutdown();
    try {
        // 等待指定时间
        if (!executor.awaitTermination(timeout, unit)) {
            // 超时，强制关闭
            List<Runnable> remaining = executor.shutdownNow();
            System.err.println("Force shutdown, remaining tasks: " + remaining.size());
            
            // 再等待一段时间，让任务响应中断
            if (!executor.awaitTermination(timeout, unit)) {
                System.err.println("Executor did not terminate");
            }
        }
    } catch (InterruptedException e) {
        // 当前线程被中断，强制关闭
        executor.shutdownNow();
        Thread.currentThread().interrupt();
    }
}
```

## 一句话收尾

`ExecutorService.submit()`的异常陷阱是**"异常被包装在Future中，不get就吞掉"**。解决方案：**`execute()`配合UncaughtExceptionHandler**、**任务内try-catch**、**Future.get()获取结果**、或用**CompletableFuture.exceptionally()**统一处理。生产环境务必做好异常收集和监控，避免"静默失败"。
