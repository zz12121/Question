39. 你写了一个日志工具类，频繁创建大对象：
```java
public void log(String msg) {
    byte[] data = msg.getBytes(StandardCharsets.UTF_8);
    // 写入日志
}
```
**问题：** 高并发下有什么问题？如何优化减少 GC 压力？

---
## 先说结论

高并发下频繁`getBytes()`会**大量创建byte数组**，导致**Young GC频繁**甚至**Full GC**。优化方案：**① 使用ThreadLocal复用byte数组**；**② 使用StringBuilder/Buffer预分配**；**③ 批量异步写入**；**④ 直接使用Slf4j/Logback等成熟框架**（内部已优化）。

---

## 🔥 核心概念

### 问题分析

```
┌─────────────────────────────────────────────────────────────┐
│  原代码问题：                                                  │
│  byte[] data = msg.getBytes(StandardCharsets.UTF_8);         │
│                                                              │
│  每次调用都创建新的byte数组：                                   │
│  - 假设msg平均1KB，QPS 10000                                  │
│  - 每秒创建 10MB 垃圾对象                                      │
│  - 每分钟 600MB，每小时 36GB                                   │
│                                                              │
│  后果：                                                       │
│  - Young区迅速填满 → 频繁Young GC                             │
│  - GC线程占用CPU → 业务线程停顿                                │
│  - 大对象直接进入老年代 → 频繁Full GC                          │
│  - 最终：吞吐量下降，延迟增加，甚至OOM                         │
└─────────────────────────────────────────────────────────────┘
```

## 🔥 详细展开：优化方案

### 方案1：ThreadLocal复用byte数组

```java
public class OptimizedLogger {
    // ThreadLocal复用byte数组（注意：只适用于同步处理）
    private static final ThreadLocal<byte[]> BUFFER = 
        ThreadLocal.withInitial(() -> new byte[8192]);  // 8KB初始
    
    public void log(String msg) {
        byte[] buffer = BUFFER.get();
        byte[] data = msg.getBytes(StandardCharsets.UTF_8);
        
        // 如果数据小，复用buffer
        if (data.length <= buffer.length) {
            System.arraycopy(data, 0, buffer, 0, data.length);
            write(buffer, 0, data.length);
        } else {
            // 大数据直接写
            write(data, 0, data.length);
        }
    }
    
    private void write(byte[] data, int offset, int length) {
        // 写入日志
    }
}
```

### 方案2：对象池复用（更通用）

```java
@Component
public class PooledLogger {
    
    // 使用Apache Commons Pool
    private final GenericObjectPool<byte[]> bufferPool;
    
    public PooledLogger() {
        GenericObjectPoolConfig<byte[]> config = new GenericObjectPoolConfig<>();
        config.setMaxTotal(100);          // 最大对象数
        config.setMaxIdle(50);            // 最大空闲
        config.setMinIdle(10);            // 最小空闲
        
        bufferPool = new GenericObjectPool<>(
            new BasePooledObjectFactory<byte[]>() {
                @Override
                public byte[] create() {
                    return new byte[4096];  // 4KB buffer
                }
                
                @Override
                public PooledObject<byte[]> wrap(byte[] obj) {
                    return new DefaultPooledObject<>(obj);
                }
            },
            config
        );
    }
    
    public void log(String msg) {
        byte[] buffer = null;
        try {
            buffer = bufferPool.borrowObject();
            byte[] data = msg.getBytes(StandardCharsets.UTF_8);
            
            if (data.length <= buffer.length) {
                System.arraycopy(data, 0, buffer, 0, data.length);
                write(buffer, 0, data.length);
            } else {
                write(data, 0, data.length);
            }
        } catch (Exception e) {
            // 降级：直接创建
            write(msg.getBytes(StandardCharsets.UTF_8));
        } finally {
            if (buffer != null) {
                bufferPool.returnObject(buffer);
            }
        }
    }
}
```

### 方案3：批量异步写入（减少IO次数）

```java
@Component
public class BatchLogger {
    
    private final BlockingQueue<LogEntry> buffer = 
        new LinkedBlockingQueue<>(10000);
    
    private final List<LogEntry> batch = new ArrayList<>(100);
    
    @PostConstruct
    public void init() {
        // 启动批量写入线程
        ScheduledExecutorService executor = 
            Executors.newSingleThreadScheduledExecutor();
        executor.scheduleAtFixedRate(this::flush, 100, 100, TimeUnit.MILLISECONDS);
    }
    
    public void log(String msg) {
        // 只创建轻量对象入队，不立即序列化
        buffer.offer(new LogEntry(msg, System.currentTimeMillis()));
    }
    
    /**
     * 批量刷盘（单线程，减少锁竞争和IO次数）
     */
    private void flush() {
        buffer.drainTo(batch, 100);  // 批量取出
        
        if (batch.isEmpty()) {
            return;
        }
        
        // 一次性序列化和写入
        StringBuilder sb = new StringBuilder();
        for (LogEntry entry : batch) {
            sb.append(format(entry)).append("\n");
        }
        
        // 一次IO写入
        writeToFile(sb.toString().getBytes(StandardCharsets.UTF_8));
        
        batch.clear();
    }
}
```

// JVM参数优化日志场景

## 🔥 扩展追问 3：零拷贝技术

```java
// 使用FileChannel零拷贝写入（减少数据复制）
public class ZeroCopyLogger {
    
    private FileChannel channel;
    
    public void log(String msg) throws IOException {
        byte[] data = msg.getBytes(StandardCharsets.UTF_8);
        ByteBuffer buffer = ByteBuffer.wrap(data);
        
        // 直接写入Channel，避免用户态到内核态的拷贝
        channel.write(buffer);
    }
    
    // 或使用内存映射文件（更快）
    public void logWithMmap(String msg) throws IOException {
        byte[] data = msg.getBytes(StandardCharsets.UTF_8);
        MappedByteBuffer buffer = channel.map(
            FileChannel.MapMode.READ_WRITE, 
            position, 
            data.length
        );
        buffer.put(data);
    }
}
```

@Component
public class SampledLogger {
    
    private final AtomicLong counter = new AtomicLong(0);
    
    // 只记录1%的日志
    public void logWithSampling(String msg) {
        if (counter.incrementAndGet() % 100 == 0) {
            log.info(msg);
        }
    }
    
    // 或按时间采样（每秒最多100条）
    private final RateLimiter rateLimiter = RateLimiter.create(100);
    
    public void logWithRateLimit(String msg) {
        if (rateLimiter.tryAcquire()) {
            log.info(msg);
        }
    }
}

## 一句话收尾

高并发日志优化的核心是**"减少对象创建"**和**"异步批量写入"**：用**ThreadLocal/对象池复用buffer**，**占位符延迟字符串拼接**，**批量异步刷盘减少IO**，生产环境直接用**Slf4j+Logback/Log4j2**（已内置这些优化）。记住**"日志是辅助，不能影响主业务"**，极端场景下要**采样降级**。
