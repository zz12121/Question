32. 你用 `ConcurrentHashMap` 做缓存，并给每个 key 加了过期时间：
```java
ConcurrentHashMap<String, ValueWithExpire> cache = new ConcurrentHashMap<>();

public V get(K key) {
    ValueWithExpire v = cache.get(key);
    if (v != null && v.isExpired()) {
        cache.remove(key);
        return null;
    }
    return v != null ? v.getValue() : null;
}
```
**问题：** 这段代码有什么问题？并发场景下会出现什么异常？

---
## 先说结论

这段代码存在**"竞态条件（Race Condition）"**问题：`get`和`remove`不是原子操作，并发场景下会导致**"误删"**（删掉其他线程刚放入的新值）或**"返回过期值"**。此外还存在**"内存泄漏"**（过期key不会自动清理，只有被访问时才删）。正确方案是用`ConcurrentHashMap`的`computeIfPresent`原子操作，或直接用`Caffeine`等成熟缓存框架。

### 问题1：竞态条件（Race Condition）

## 🔥 详细展开：修复方案

### 方案1：使用`computeIfPresent`原子操作

```java
public class SafeCache<K, V> {
    private final ConcurrentHashMap<K, ValueWithExpire<V>> cache = new ConcurrentHashMap<>();
    
    public V get(K key) {
        // 原子操作：如果key存在，执行函数；如果不存在，返回null
        ValueWithExpire<V> v = cache.computeIfPresent(key, (k, oldVal) -> {
            // 在原子块内判断是否过期
            if (oldVal.isExpired()) {
                return null;  // 返回null表示删除
            }
            return oldVal;  // 返回原值表示保留
        });
        
        return v != null ? v.getValue() : null;
    }
    
    public void put(K key, V value, long expireMillis) {
        cache.put(key, new ValueWithExpire<>(value, System.currentTimeMillis() + expireMillis));
    }
    
    // 内部类
    private static class ValueWithExpire<V> {
        private final V value;
        private final long expireTime;
        
        ValueWithExpire(V value, long expireTime) {
            this.value = value;
            this.expireTime = expireTime;
        }
        
        boolean isExpired() {
            return System.currentTimeMillis() > expireTime;
        }
        
        V getValue() {
            return value;
        }
    }
}
```

### 方案2：定时清理 + 惰性删除（双保险）

```java
@Component
public class CacheWithCleanup<K, V> {
    private final ConcurrentHashMap<K, ValueWithExpire<V>> cache = new ConcurrentHashMap<>();
    
    public CacheWithCleanup() {
        // 启动定时清理线程
        ScheduledExecutorService scheduler = Executors.newSingleThreadScheduledExecutor();
        scheduler.scheduleAtFixedRate(this::cleanup, 1, 1, TimeUnit.MINUTES);
    }
    
    public V get(K key) {
        // 惰性删除（访问时检查）
        ValueWithExpire<V> v = cache.get(key);
        if (v != null && v.isExpired()) {
            // 使用remove(key, value)避免误删
            cache.remove(key, v);  // 只有值匹配时才删除
            return null;
        }
        return v != null ? v.getValue() : null;
    }
    
    // 定时清理（防止内存泄漏）
    private void cleanup() {
        // 使用迭代器安全删除
        Iterator<Map.Entry<K, ValueWithExpire<V>>> it = cache.entrySet().iterator();
        while (it.hasNext()) {
            Map.Entry<K, ValueWithExpire<V>> entry = it.next();
            if (entry.getValue().isExpired()) {
                it.remove();  // 安全删除
            }
        }
    }
}
```

## 🔥 扩展追问 1：ConcurrentHashMap的线程安全边界

```java
// ConcurrentHashMap保证的是"单个操作"的线程安全
// 不保证"复合操作"的线程安全！

// ❌ 复合操作（非线程安全）
if (!cache.containsKey(key)) {  // 操作1
    cache.put(key, value);       // 操作2：可能被其他线程插入
}

// ✅ 使用原子操作
cache.putIfAbsent(key, value);

// ❌ 先get后remove（本题的情况）
V v = cache.get(key);
if (v.isExpired()) {
    cache.remove(key);  // 非原子
}

// ✅ 使用compute原子操作
cache.computeIfPresent(key, (k, v) -> v.isExpired() ? null : v);
```

## 🔥 扩展追问 2：延迟队列实现精确过期

```java
/**
 * 使用DelayedQueue实现精确过期通知
 */
public class PreciseExpireCache<K, V> {
    private final ConcurrentHashMap<K, V> cache = new ConcurrentHashMap<>();
    private final DelayQueue<DelayedItem<K>> delayQueue = new DelayQueue<>();
    
    public PreciseExpireCache() {
        // 启动过期处理线程
        Thread cleaner = new Thread(() -> {
            while (!Thread.interrupted()) {
                try {
                    DelayedItem<K> item = delayQueue.take();  // 阻塞等待到期
                    cache.remove(item.getKey());  // 精确删除
                } catch (InterruptedException e) {
                    break;
                }
            }
        });
        cleaner.setDaemon(true);
        cleaner.start();
    }
    
    public void put(K key, V value, long expireMillis) {
        cache.put(key, value);
        delayQueue.put(new DelayedItem<>(key, expireMillis));
    }
    
    public V get(K key) {
        return cache.get(key);
    }
    
    // 延迟项
    private static class DelayedItem<K> implements Delayed {
        private final K key;
        private final long expireTime;
        
        DelayedItem(K key, long delayMillis) {
            this.key = key;
            this.expireTime = System.currentTimeMillis() + delayMillis;
        }
        
        @Override
        public long getDelay(TimeUnit unit) {
            return unit.convert(expireTime - System.currentTimeMillis(), TimeUnit.MILLISECONDS);
        }
        
        @Override
        public int compareTo(Delayed o) {
            return Long.compare(expireTime, ((DelayedItem<?>) o).expireTime);
        }
        
        K getKey() { return key; }
    }
}
```

## 一句话收尾

`ConcurrentHashMap`做缓存的核心问题是**"复合操作非原子"**导致竞态条件，以及**"惰性删除"导致内存泄漏**。修复方案是用`computeIfPresent`原子操作替代get+remove，配合定时清理线程，生产环境推荐直接用`Caffeine`或`Redis`。记住**"ConcurrentHashMap保证单操作原子，不保证复合操作原子"**。
