34. 你用 `ReentrantLock` 实现了一个带超时的锁：
```java
ReentrantLock lock = new ReentrantLock();
public boolean tryLockWithTimeout() throws InterruptedException {
    return lock.tryLock(5, TimeUnit.SECONDS);
}
```
**问题：** 如果获取锁超时返回 false，后续如何处理？如果一直获取不到锁，可能是什么原因？

---
## 先说结论

`tryLock`超时返回`false`后，应该**快速失败或降级处理**，而不是重试或阻塞。一直获取不到锁的原因通常是**死锁**（循环等待）或**锁饥饿**（公平锁下某些线程长期拿不到锁）。排查思路：**`jstack`打印线程栈找死锁 → 检查锁的获取顺序 → 考虑锁超时自动释放或改用无锁设计**。

### tryLock超时后的处理策略

```

│  获取锁超时返回false后，常见处理策略：                          │
│                                                              │

│     → 适合可用性优先场景                                      │

│                                                              │
```

### 方案1：快速失败

```java
public class OrderService {
    private final ReentrantLock lock = new ReentrantLock();
    
    public void createOrder(OrderRequest request) {
        boolean locked = false;
        try {
            // 尝试获取锁，超时5秒
            locked = lock.tryLock(5, TimeUnit.SECONDS);
            if (!locked) {
                // 快速失败，抛出自定义异常
                throw new LockAcquireTimeoutException(
                    "获取订单锁超时，请稍后重试");

            // 执行业务逻辑
            doCreateOrder(request);

            Thread.currentThread().interrupt();
            throw new RuntimeException("线程被中断", e);
        } finally {
            if (locked) {
                lock.unlock();
            }
        }
    }
}
```

### 方案3：带重试的锁获取

```java
public class RetryableLock {
    private final ReentrantLock lock = new ReentrantLock();
    private static final int MAX_RETRY = 3;
    private static final long BASE_WAIT = 100;  // 基础等待时间ms
    
    public boolean tryLockWithRetry() throws InterruptedException {
        for (int i = 0; i < MAX_RETRY; i++) {
            // 指数退避
            long waitTime = BASE_WAIT * (1 << i);
            
            if (lock.tryLock(waitTime, TimeUnit.MILLISECONDS)) {
                return true;
            }
            
            log.warn("第{}次获取锁失败，等待{}ms后重试", i + 1, waitTime);
        }
        
        return false;  // 最终失败
    }
}
```

## 🔥 详细展开：一直获取不到锁的原因

### 原因1：死锁（Deadlock）

```java
// 典型死锁场景：两个线程互相等待对方持有的锁
public class DeadlockDemo {
    private final ReentrantLock lockA = new ReentrantLock();
    private final ReentrantLock lockB = new ReentrantLock();
    
    public void method1() {
        lockA.lock();
        try {
            lockB.lock();  // 等待lockB
            try {
                // do something
            } finally {
                lockB.unlock();
            }
        } finally {
            lockA.unlock();
        }
    }
    
    public void method2() {
        lockB.lock();
        try {
            lockA.lock();  // 等待lockA → 死锁！
            try {
                // do something
            } finally {
                lockA.unlock();
            }
        } finally {
            lockB.unlock();
        }
    }
}
```

```java
// 方案1：统一锁顺序
public void safeMethod1() {
    ReentrantLock first = lockA.hashCode() < lockB.hashCode() ? lockA : lockB;
    ReentrantLock second = lockA.hashCode() < lockB.hashCode() ? lockB : lockA;
    
    first.lock();
    try {
        second.lock();
        try {
            // safe
        } finally {
            second.unlock();
        }
    } finally {
        first.unlock();
    }
}

// 方案2：使用tryLock避免死锁
public boolean safeMethodWithTimeout() {
    boolean lockA = false, lockB = false;
    try {
        lockA = lockA.tryLock(1, TimeUnit.SECONDS);
        lockB = lockB.tryLock(1, TimeUnit.SECONDS);
        
        if (lockA && lockB) {
            // do something
            return true;
        }
        return false;
    } catch (InterruptedException e) {
        Thread.currentThread().interrupt();
        return false;
    } finally {
        if (lockA) lockA.unlock();
        if (lockB) lockB.unlock();
    }
}
```

### 原因3：锁未释放

```java
// 异常导致锁未释放
public void badMethod() {
    lock.lock();
    // 如果这里抛出异常，锁永远不会释放！
    riskyOperation();
    lock.unlock();  // 可能执行不到
}

// 正确做法：try-finally
public void goodMethod() {
    lock.lock();
    try {
        riskyOperation();
    } finally {
        lock.unlock();  // 确保释放
    }
}
```

```java
@Component
public class MonitoredLock {
    private final ReentrantLock lock = new ReentrantLock();
    private final MeterRegistry meterRegistry;
    
    // 监控指标
    private final Counter acquireSuccess;
    private final Counter acquireTimeout;
    private final Timer lockHoldTime;
    
    public MonitoredLock(MeterRegistry meterRegistry) {
        this.meterRegistry = meterRegistry;
        this.acquireSuccess = Counter.builder("lock.acquire.success")
            .tag("lock", "orderLock")
            .register(meterRegistry);
        this.acquireTimeout = Counter.builder("lock.acquire.timeout")
            .tag("lock", "orderLock")
            .register(meterRegistry);
        this.lockHoldTime = Timer.builder("lock.hold.time")
            .tag("lock", "orderLock")
            .register(meterRegistry);
    }
    
    public void execute(Runnable task) {
        boolean acquired = false;
        long startTime = System.currentTimeMillis();
        
        try {
            acquired = lock.tryLock(5, TimeUnit.SECONDS);
            
            if (!acquired) {
                acquireTimeout.increment();
                throw new LockTimeoutException();
            }
            
            acquireSuccess.increment();
            long lockStart = System.currentTimeMillis();
            
            try {
                task.run();
            } finally {
                lockHoldTime.record(System.currentTimeMillis() - lockStart, 
                    TimeUnit.MILLISECONDS);
                lock.unlock();
            }
            
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
            throw new RuntimeException(e);
        }
    }
}
```

```java
AtomicInteger counter = new AtomicInteger(0);
public void increment() {
    int expect, update;
    do {
        expect = counter.get();
        update = expect + 1;
    } while (!counter.compareAndSet(expect, update));
}
```

## 一句话收尾

`tryLock`超时后应**快速失败或降级**，避免无限重试。一直拿不到锁要排查**死锁**（`jstack`找循环等待）、**锁饥饿**（考虑公平锁）或**锁泄漏**（确保finally释放）。生产环境建议加**锁监控**（获取成功率、持有时间），必要时用**无锁设计**（CAS、分段锁）替代。
