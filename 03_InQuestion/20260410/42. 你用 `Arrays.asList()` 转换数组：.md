42. 你用 `Arrays.asList()` 转换数组：
```java
String[] arr = {"a", "b", "c"};
List<String> list = Arrays.asList(arr);
list.add("d");
```
**问题：** 这段代码有什么问题？运行时会发生什么异常？

---
## 结论先行

**`Arrays.asList()` 返回的是一个**固定大小**的 List，调用 `add()` 直接抛 `UnsupportedOperationException`。**

---

## 三、另一个坑：共享原始数组

```java
String[] arr = {"a", "b", "c"};
List<String> list = Arrays.asList(arr);

// 修改 list 元素
list.set(0, "X");
System.out.println(arr[0]);     // 输出 "X" ！！！

// 修改原始数组
arr[1] = "Y";
System.out.println(list.get(1)); // 输出 "Y" ！！！
```

```
┌─────────────────────────────────────────────────────────────────────┐
│                    共享引用陷阱                                      │
│                                                                     │
│  arr ────────────────────────┐                                      │
│                              ▼                                      │
│                         ["a","b","c"]   ← 原始数组（堆内存）         │
│                              ▲                                      │
│  list.a ─────────────────────┘                                      │
│                                                                     │
│  两者指向同一个数组！修改任意一方，另一方立即感知。                   │
└─────────────────────────────────────────────────────────────────────┘
```

## 四、正确解决方案

### 方式一：`new ArrayList<>(Arrays.asList(...))` ✅

```java
String[] arr = {"a", "b", "c"};
List<String> list = new ArrayList<>(Arrays.asList(arr));  // 包一层
list.add("d");        // ✅ 正常
list.remove("a");     // ✅ 正常
System.out.println(list);  // [b, c, d]
```

### 方式二：`List.of(...)` — Java 9+（不可变列表）

```java
List<String> list = List.of("a", "b", "c");
list.add("d");   // 💥 UnsupportedOperationException
// ⚠️ List.of() 完全不可变，连 set() 都不行
```

### 方式三：`Stream` 转换 ✅

```java
String[] arr = {"a", "b", "c"};
List<String> list = Arrays.stream(arr)
    .collect(Collectors.toCollection(ArrayList::new));
list.add("d");   // ✅
```

### 方式四：`Collections.addAll()` ✅

```java
String[] arr = {"a", "b", "c"};
List<String> list = new ArrayList<>();
Collections.addAll(list, arr);
list.add("d");   // ✅
```

### 方式五：Java 8+ `Stream` 快速转换

```java
// 基本类型数组注意：int[] 不能直接 asList
int[] arr = {1, 2, 3};

// ❌ 这样是 List<int[]>，不是 List<Integer>
List<int[]> wrong = Arrays.asList(arr);  // 整个数组当一个元素！

// ✅ 正确
List<Integer> list = Arrays.stream(arr)
    .boxed()
    .collect(Collectors.toList());
```

## 五、对比汇总

| 方式 | 可 add/remove | 与原数组共享 | 适用场景 |
|------|:------------:|:-----------:|---------|
| `Arrays.asList()` | ❌ | ✅ 共享 | 只读遍历 |
| `new ArrayList<>(Arrays.asList())` | ✅ | ❌ 独立 | 需要增删 |
| `List.of()` (Java 9+) | ❌ | ❌ 独立 | 不可变常量 |
| `Stream.collect()` | ✅ | ❌ 独立 | 流式处理 |
| `Collections.addAll()` | ✅ | ❌ 独立 | 逐个追加 |

## 六、面试追问

> **`int[]` 用 `Arrays.asList()` 会怎样？**

```java
int[] arr = {1, 2, 3};
List<int[]> list = Arrays.asList(arr);
System.out.println(list.size());     // 1，整个数组当成一个元素！
System.out.println(list.get(0));     // [I@7852e922（数组对象地址）
```

基本类型数组不会自动拆箱，要用 `Integer[]` 才正常：

```java
Integer[] arr = {1, 2, 3};
List<Integer> list = Arrays.asList(arr);
System.out.println(list.size());     // 3 ✅
```

> **`List.of()` 和 `Arrays.asList()` 有什么区别？**

```java
// Arrays.asList：可以 set，不能 add/remove，允许 null
list.set(0, null);  // ✅
list.add(null);     // ❌ UnsupportedOperationException

// List.of：完全不可变，不允许 null
List<String> immutable = List.of("a", null);  // ❌ NullPointerException
immutable.set(0, "b");                        // ❌ UnsupportedOperationException
```

## 一句话收尾

**`Arrays.asList()` 是"披着 List 皮的数组"，大小固定、共享内存——需要增删时必须套一层 `new ArrayList<>()`。**

## 七、连环坑速记

```java
// 坑1：subList 同样是视图，修改会影响原 List
List<String> list = new ArrayList<>(Arrays.asList("a","b","c","d"));
List<String> sub  = list.subList(1, 3);   // [b, c]
sub.clear();
System.out.println(list);  // [a, d]  原 list 也变了！

// 坑2：遍历时 remove 会 ConcurrentModificationException
for (String s : list) {
    list.remove(s);  // ❌ 抛异常
}
// ✅ 正确：用 Iterator 或 removeIf
list.removeIf(s -> s.equals("a"));

// 坑3：Collections.unmodifiableList 同样不能增删
List<String> unmod = Collections.unmodifiableList(list);
unmod.add("x");  // ❌ UnsupportedOperationException
```

记住一句口诀：**`asList` 能 set 不能 add，`List.of` 啥都不能改，`unmodifiableList` 读专用。**
