**问题：** 如果 user-service 挂了，会发生什么？如何配置降级？

---
用户问的是 Feign 调用的降级处理，这是微服务架构中非常重要的高可用话题。需要讲解：
1. Feign 默认没有降级时的问题

## 结论先行

**user-service 挂了 = Feign 抛异常 = 调用方线程阻塞 = 线程池耗尽 = 级联崩溃。** 必须配置**降级熔断**，让失败时快速返回兜底数据而不是无限等待。

## 一、问题分析

### 默认 Feign 挂了会怎样？

```java
@FeignClient(name = "user-service")
public interface UserClient {
    @GetMapping("/user/{id}")
    User getUser(@PathVariable Long id);  // user-service 挂了会抛异常
}
```

### 调用链崩溃过程

```
用户请求
    │
    ▼
OrderController ──▶ UserClient.getUser()
                           │
                           ▼
                      user-service 超时/不可用
                           │
                           ▼
                      Feign 抛异常
                           │
                           ▼
                      异常冒泡到 Controller
                           │
                           ▼
                      线程阻塞等响应
                           │
                           ▼
                      100 个请求 → 100 个线程阻塞
                           │
                           ▼
                      Tomcat 线程池耗尽
                           │
                           ▼
                      服务不可用 💥💥💥

这不是 order-service 的问题，
是 user-service 的问题拖垮了整个系统
```

### Feign 默认行为

| 情况 | 默认行为 | 结果 |
|------|---------|------|
| 超时 | 等待（无超时）| 线程阻塞到天荒地老 |

| 部分失败 | 抛异常 | 无法返回部分数据 |

## 二、解决方案

### 方案一：配置超时 + Hystrix 熔断 ✅

**核心思想：设置超时，超时后快速失败；配置熔断器，失败率高时短路**

#### 1. 引入依赖

```xml
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-netflix-hystrix</artifactId>
</dependency>
```

#### 2. 启用 Hystrix

```java
@SpringBootApplication
@EnableFeignClients
@EnableCircuitBreaker  // 启用熔断器
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}
```

#### 3. 配置超时

```yaml
# application.yml
feign:
  client:
    config:
      default:  # 全局配置
        connect-timeout: 2000   # 连接超时 2秒
        read-timeout: 5000       # 读取超时 5秒
        logger-level: basic      # 日志级别
      
      user-service:  # 针对特定服务配置
        connect-timeout: 1000    # 更短的超时
        read-timeout: 2000
```

#### 4. 配置 Hystrix

```yaml
hystrix:
  command:
    default:  # 默认配置
      execution:
        timeout:
          enabled: true
        isolation:
          thread:
            timeoutInMilliseconds: 6000  # Hystrix 超时（比 Feign 超时稍长）
          strategy: THREAD  # 线程池隔离
      circuitBreaker:
        enabled: true
        requestVolumeThreshold: 20        # 10秒内至少20个请求才熔断
        sleepWindowInMilliseconds: 5000   # 熔断后5秒尝试恢复
        errorThresholdPercentage: 50       # 错误率50%时熔断
      fallback:
        enabled: true  # 启用降级
```

### 方案二：实现降级（Fallback）✅

**核心思想：远程调用失败时，返回兜底数据而不是抛异常**

#### 1. 定义降级类

```java
@Component
@Slf4j
public class UserClientFallback implements UserClient {

    // 保存触发降级时的异常信息
    private final Throwable cause;

    public UserClientFallback() {
        this(null);
    }

    public UserClientFallback(Throwable cause) {
        this.cause = cause;
    }

    @Override
    public User getUser(Long id) {
        log.warn("UserService 调用失败，触发降级, userId={}, error={}", 
            id, cause != null ? cause.getMessage() : "unknown");
        
        // 返回兜底数据
        return User.builder()
            .id(id)
            .name("未知用户")  // 兜底值
            .defaultAvatar(true)
            .fromFallback(true)  // 标记是降级数据
            .build();
    }

    @Override
    public List<User> getUsers(List<Long> ids) {
        log.warn("批量获取用户失败，返回空列表, ids={}", ids);
        return Collections.emptyList();  // 兜底空列表
    }
}
```

#### 2. 配置 Fallback

```java
@FeignClient(
    name = "user-service",
    fallback = UserClientFallback.class  // 指定降级类
)
public interface UserClient {
    @GetMapping("/user/{id}")
    User getUser(@PathVariable Long id);

    @GetMapping("/users")
    List<User> getUsers(@RequestParam("ids") List<Long> ids);
}
```

### 方案三：Sentinel 降级（更现代）✅

**核心思想：用 Sentinel 替代 Hystrix，功能更丰富**

#### 1. 引入依赖

```xml
<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-sentinel</artifactId>
</dependency>
```

#### 2. 配置降级规则

```yaml
spring:
  cloud:
    sentinel:
      eager: true  # 启动时加载规则
      transport:
        dashboard: localhost:8080  # Sentinel 控制台地址

feign:
  sentinel:
    enabled: true  # 启用 Sentinel 替代 Hystrix
```

#### 3. 定义 Sentinel 降级处理

```java
@Component
public class UserClientFallbackFactory implements FallbackFactory<UserClient> {

    @Override
    public UserClient create(Throwable cause) {
        return new UserClient() {
            @Override
            public User getUser(Long id) {
                log.warn("Sentinel 降级触发, userId={}", id);
                
                return User.builder()
                    .id(id)
                    .name("用户(" + id + ")")
                    .status("OFFLINE")  // 标记降级状态
                    .build();
            }

            @Override
            public List<User> getUsers(List<Long> ids) {
                log.warn("批量用户查询降级, count={}", ids.size());
                return ids.stream()
                    .map(id -> User.builder()
                        .id(id)
                        .name("用户" + id)
                        .status("OFFLINE")
                        .build())
                    .collect(Collectors.toList());
            }
        };
    }
}

@FeignClient(
    name = "user-service",
    fallbackFactory = UserClientFallbackFactory.class
)
public interface UserClient {
    @GetMapping("/user/{id}")
    User getUser(@PathVariable Long id);
}
```

## 三、生产级完整实现

### 推荐：Hystrix + Fallback + 线程隔离

```java
// ========== Feign Client ==========
@FeignClient(
    name = "user-service",
    fallback = UserClientFallback.class,
    configuration = FeignConfig.class
)
public interface UserClient {
    
    @GetMapping("/user/{id}")
    @SentinelResource(value = "getUser", fallback = "getUserFallback")
    User getUser(@PathVariable Long id);

    @GetMapping("/users")
    List<User> getUsers(@RequestParam("ids") List<Long> ids);
}

// ========== Fallback 实现 ==========
@Component
@Slf4j
public class UserClientFallback implements UserClient {

    @Override
    public User getUser(Long id) {
        log.warn("UserService 调用失败，降级返回, userId={}", id);
        return User.defaultUser(id);
    }

    @Override
    public List<User> getUsers(List<Long> ids) {
        log.warn("批量用户查询降级, count={}", ids.size());
        return ids.stream()
            .map(User::defaultUser)
            .collect(Collectors.toList());
    }
}

// ========== Feign 配置类 ==========
@Configuration
public class FeignConfig {

    @Bean
    @Scope("prototype")
    public Feign.Builder feignBuilder() {
        return Feign.builder()
            .retryer(Retryer.NEVER_RETRY)  // 禁用默认重试，用 Hystrix 控制
            .options(new Options(2000, 5000));  // connect, read 超时
}
```

### 全局降级策略

```java
/**
 * 全局 Feign 降级工厂
 */
@Component
@Slf4j
public class GlobalFeignFallbackFactory implements FallbackFactory<Client> {

    @Override
    public Client create(Throwable cause) {
        return (target, template, options) -> {
            String url = template.url().toString();
            log.error("Feign 调用失败, url={}, error={}", url, cause.getMessage());
            
            // 返回 503 Service Unavailable
            return Response.builder()
                .status(503)
                .reason("Service Unavailable")
                .request(template.request())
                .body("Fallback response".getBytes())
                .build();
        };
    }
}
```

### 配置多级降级策略

```java
@FeignClient(
    name = "user-service",
    fallback = UserClientFallback.class
)
public interface UserClient {
    
    // 主方法：查询用户
    @GetMapping("/user/{id}")
    @SentinelResource(
        value = "getUser",
        blockHandler = "getUserBlockHandler",  // 限流/熔断降级
        fallback = "getUserFallback"            // 业务异常降级
    )
    User getUser(@PathVariable Long id);

    // ========== 限流降级处理 ==========
    public User getUserBlockHandler(Long id, BlockException e) {
        log.warn("用户查询被限流, userId={}", id);
        return User.builder()
            .id(id)
            .name("用户" + id)
            .rateLimited(true)
            .build();
    }

    // ========== 异常降级处理 ==========
    public User getUserFallback(Long id, Throwable e) {
        log.warn("用户查询异常降级, userId={}, error={}", id, e.getMessage());
        return User.builder()
            .id(id)
            .name("用户" + id)
            .error(true)
            .build();
    }
}
```

## 六、面试加分话术

> **面试官追问：Hystrix 和 Sentinel 的区别？**

> Hystrix 是 Netflix 出的，功能全面但侵入性强，已经停止维护。Sentinel 是阿里开源的，轻量级、无代码侵入、配置更灵活，还带流量控制台。Sentinel 还支持黑白名单、系统自适应保护等高级功能。Spring Cloud Alibaba 现在推荐用 Sentinel 替代 Hystrix。

> **面试官追问：线程隔离和信号量隔离的区别？**

> 线程隔离：每个依赖用独立线程池，线程池满时不影响其他服务，但开销大（线程切换）。适合慢服务和不可信服务。信号量隔离：在调用线程执行，无线程开销，适合快速失败场景。Hystrix 默认用线程隔离，Sentinel 默认用信号量隔离（更快）。

---

## 一句话总结

**Feign 不配降级 = 服务挂了拖垮整个系统。** 必须配置 **超时（快速失败）+ Fallback（降级返回兜底数据）+ 熔断器（防止级联崩溃）**。生产环境推荐 Sentinel，功能强且轻量。
