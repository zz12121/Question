35. 你用 `BlockingQueue` 实现生产者消费者：
```java
BlockingQueue<Task> queue = new LinkedBlockingQueue<>(100);

// 生产者
queue.put(task);

// 消费者
while (true) {
    Task task = queue.take();
    process(task);
}
```
**问题：** 如果消费者处理任务很慢，队列满了会怎样？如果消费者挂了，队列里的任务怎么处理？

---
## 先说结论

队列满时`put()`会**阻塞生产者线程**，直到有空间；消费者挂了队列中的任务会**丢失（内存中）**。解决方案：**① 用`offer()`替代`put()`配合拒绝策略**（避免阻塞）；**② 队列持久化到Redis/DB**（防丢失）；**③ 多消费者 + 消费超时监控**（高可用）；**④ 优雅关闭时持久化剩余任务**。

## 🔥 核心概念

### BlockingQueue行为

```
┌─────────────────────────────────────────────────────────────┐
│  生产者方法行为对比：                                          │
│                                                              │
│  put(e)      ──► 队列满时阻塞，直到有空间                      │
│  offer(e)    ──► 队列满时返回false，不阻塞                    │
│  offer(e, t) ──► 队列满时等待t时间，超时返回false              │
│  add(e)      ──► 队列满时抛IllegalStateException              │
├─────────────────────────────────────────────────────────────┤
│  消费者方法行为对比：                                          │
│                                                              │
│  take()      ──► 队列空时阻塞，直到有元素                      │
│  poll()      ──► 队列空时返回null，不阻塞                     │
│  poll(t)     ──► 队列空时等待t时间，超时返回null               │
└─────────────────────────────────────────────────────────────┘
```

## 🔥 详细展开：队列满的处理

### 问题1：put()阻塞导致生产者线程耗尽

```java
// 问题场景：生产者线程池被阻塞
ExecutorService producerPool = Executors.newFixedThreadPool(10);
BlockingQueue<Task> queue = new LinkedBlockingQueue<>(100);

// 100个生产者同时put，队列满后全部阻塞
for (int i = 0; i < 100; i++) {
    producerPool.submit(() -> {
        try {
            queue.put(new Task());  // 队列满时阻塞！
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
    });
}
// 结果：100个线程全部阻塞，生产者线程池耗尽，系统假死
```

### 解决方案1：非阻塞offer() + 降级

```java
@Component
public class TaskProducer {
    
    private final BlockingQueue<Task> queue = new LinkedBlockingQueue<>(100);
    
    public boolean submit(Task task) {
        // 非阻塞提交，超时1秒
        boolean offered = queue.offer(task, 1, TimeUnit.SECONDS);
        
        if (!offered) {
            // 降级处理：持久化到DB，异步补偿
            persistenceService.save(task);
            log.warn("队列已满，任务持久化到DB: {}", task.getId());
            
            // 或丢弃并告警
            // metrics.counter("queue.full.drop").increment();
            // alertService.send("队列已满", task);
        }
        
        return offered;
    }
}
```

### 解决方案2：CallerRunsPolicy（生产者优先级降低）

```java
// 使用ThreadPoolExecutor的拒绝策略
ThreadPoolExecutor executor = new ThreadPoolExecutor(
    4, 8, 60, TimeUnit.SECONDS,
    new ArrayBlockingQueue<>(100),
    new ThreadFactoryBuilder().setNameFormat("worker-%d").build(),
    new ThreadPoolExecutor.CallerRunsPolicy()  // 队列满时，提交任务的线程自己执行
);

// 效果：生产者线程自己执行task，自然降低了生产速度
// 相当于背压（Back Pressure）机制
```

### 解决方案3：动态队列扩容

```java
@Component
public class ElasticQueue {
    
    private volatile BlockingQueue<Task> queue = new LinkedBlockingQueue<>(100);
    
    public void put(Task task) throws InterruptedException {
        if (!queue.offer(task)) {
            // 队列满，扩容
            expandQueue();
            queue.put(task);
        }
    }
    
    private synchronized void expandQueue() {
        int newCapacity = queue.size() * 2;
        BlockingQueue<Task> newQueue = new LinkedBlockingQueue<>(newCapacity);
        queue.drainTo(newQueue);  // 转移旧数据
        queue = newQueue;
        log.info("队列扩容至: {}", newCapacity);
    }
}
```

## 🔥 详细展开：消费者挂了怎么办

### 问题：内存队列数据丢失

```java
// 消费者线程
while (true) {
    Task task = queue.take();  // 从内存队列取
    process(task);             // 处理
}

// 如果JVM崩溃或消费者线程异常退出：
// - 内存队列中的任务全部丢失
// - 已take但未处理完的任务丢失
// - 没有任何持久化保障
```

### 解决方案1：Redis持久化队列

```java
@Service
public class RedisQueueService {
    
    @Autowired
    private StringRedisTemplate redisTemplate;
    
    private static final String QUEUE_KEY = "task:queue";
    
    // 生产者
    public void produce(Task task) {
        redisTemplate.opsForList().rightPush(QUEUE_KEY, 
            JSON.toJSONString(task));
    }
    
    // 消费者（阻塞弹出）
    public Task consume() {
        String json = redisTemplate.opsForList()
            .leftPop(QUEUE_KEY, 0, TimeUnit.SECONDS);  // 阻塞等待
        return JSON.parseObject(json, Task.class);
    }
    
    // 优点：
    // - Redis持久化，消费者挂了任务不丢
    // - 多消费者竞争消费（原子弹出）
    // - 可查看队列长度、监控积压
}
```

### 解决方案2：数据库+状态机（强可靠）

```java
@Entity
public class Task {
    @Id
    private Long id;
    
    @Enumerated(EnumType.STRING)
    private TaskStatus status;  // PENDING, PROCESSING, COMPLETED, FAILED
    
    private Integer retryCount;
    private LocalDateTime createTime;
    private LocalDateTime processTime;
}

@Service
public class ReliableTaskService {
    
    @Scheduled(fixedRate = 5000)  // 每5秒拉取
    public void consume() {
        // 拉取待处理任务（乐观锁防止重复消费）
        List<Task> tasks = taskRepo.findTop10ByStatusOrderByCreateTimeAsc(
            TaskStatus.PENDING);
        
        for (Task task : tasks) {
            // CAS更新状态
            int updated = taskRepo.updateStatus(
                task.getId(), 
                TaskStatus.PENDING, 
                TaskStatus.PROCESSING);
            
            if (updated == 1) {  // 抢到了
                process(task);
            }
        }
    }
    
    @Transactional
    public void process(Task task) {
        try {
            // 执行业务
            doBusiness(task);
            task.setStatus(TaskStatus.COMPLETED);
        } catch (Exception e) {
            task.setStatus(TaskStatus.FAILED);
            task.setRetryCount(task.getRetryCount() + 1);
        }
        taskRepo.save(task);
    }
}
```

@Component
public class GracefulShutdownHandler {
    
    private final BlockingQueue<Task> queue = new LinkedBlockingQueue<>();
    
    @PreDestroy
    public void onShutdown() {
        // 持久化剩余任务
        List<Task> remaining = new ArrayList<>();
        queue.drainTo(remaining);
        
        if (!remaining.isEmpty()) {
            persistenceService.batchSave(remaining);

        }
    }
}

## 🔥 扩展追问 1：消费者高可用设计

```java
@Service
public class ConsumerManager {
    
    private final BlockingQueue<Task> queue = new LinkedBlockingQueue<>();
    private final List<ConsumerThread> consumers = new ArrayList<>();
    
    @PostConstruct
    public void init() {
        // 启动多个消费者
        int consumerCount = Runtime.getRuntime().availableProcessors();
        for (int i = 0; i < consumerCount; i++) {
            ConsumerThread consumer = new ConsumerThread(queue);
            consumer.start();
            consumers.add(consumer);
        }
    }
    
    // 消费者线程
    private static class ConsumerThread extends Thread {
        private final BlockingQueue<Task> queue;
        private volatile boolean running = true;
        
        @Override
        public void run() {
            while (running) {
                try {
                    Task task = queue.poll(1, TimeUnit.SECONDS);
                    if (task != null) {
                        process(task);
                    }
                } catch (Exception e) {
                    log.error("消费异常", e);
                    // 异常不退出，继续消费
                }
            }
        }
        
        public void shutdown() {
            running = false;
        }
    }
    
    // 监控消费者健康
    @Scheduled(fixedRate = 60000)
    public void healthCheck() {
        for (ConsumerThread consumer : consumers) {
            if (!consumer.isAlive()) {
                log.warn("消费者线程死亡，重启");
                ConsumerThread newConsumer = new ConsumerThread(queue);
                newConsumer.start();
                consumers.remove(consumer);
                consumers.add(newConsumer);
            }
        }
    }
}
```

## 🔥 扩展追问 2：背压（Back Pressure）机制

```java
/**
 * 响应式背压：根据消费者处理能力调整生产速度
 */
public class BackPressureQueue {
    
    private final BlockingQueue<Task> queue = new LinkedBlockingQueue<>(100);
    private final Semaphore semaphore = new Semaphore(100);  // 许可证
    
    // 生产者
    public void produce(Task task) throws InterruptedException {
        semaphore.acquire();  // 获取许可证，满了就阻塞
        queue.put(task);
    }
    
    // 消费者
    public void consume() throws InterruptedException {
        Task task = queue.take();
        try {
            process(task);
        } finally {
            semaphore.release();  // 归还许可证
        }
    }
}
```

## 一句话收尾

`BlockingQueue`生产者的`put()`会在队列满时**阻塞**，消费者挂了会导致**内存数据丢失**。生产环境建议：**`offer()`非阻塞+降级**、**Redis/DB持久化**、**多消费者+健康检查**、**优雅关闭持久化**。高可靠场景直接用**专业MQ**（RocketMQ/RabbitMQ）替代。
