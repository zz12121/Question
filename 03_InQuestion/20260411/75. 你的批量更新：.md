75. 你的批量更新：
```java
for (User user : users) {
    jdbcTemplate.update("UPDATE user SET name = ? WHERE id = ?", user.getName(), user.getId());
}
```
**问题：** 如何优化为批量更新？

---
## 先说结论

逐条 `update()` 会产生**大量网络往返（RTT）**和**重复 SQL 解析开销**，性能极差。优化方向：**JDBC 批量执行（`addBatch`/`executeBatch`）**、**多条合并为一条（`CASE WHEN`）**、或**数据库方言的批量语法（MySQL `INSERT ... ON DUPLICATE KEY UPDATE`、PostgreSQL `unnest`）**，通常能提升 **10-100 倍** 性能。

---

## 🔥 问题分析：逐条更新的开销

```java
// 1000 条数据，逐条执行
for (User user : users) {
    jdbcTemplate.update("UPDATE user SET name = ? WHERE id = ?", 
        user.getName(), user.getId());
}
```

**实际发生了什么：**
```
循环 1000 次：
  1. 发送 SQL 到数据库（网络 RTT ≈ 1-5ms）
  2. 数据库解析 SQL（软/硬解析）
  3. 执行 UPDATE（可能 0.1ms）
  4. 返回结果（网络 RTT）

总时间 ≈ 1000 × (5ms + 0.1ms) = 5.1 秒
```

**瓶颈：网络往返和 SQL 解析，不是数据库执行。**

---

## 🔥 优化方案 1：JDBC 批量执行（推荐通用方案）

```java
// ✅ 使用 JdbcTemplate 的 batchUpdate
String sql = "UPDATE user SET name = ? WHERE id = ?";

// 方式 1：JdbcTemplate 批量（内部用 PreparedStatement.addBatch）
int[] results = jdbcTemplate.batchUpdate(sql, users, 1000, 
    (ps, user) -> {
        ps.setString(1, user.getName());
        ps.setLong(2, user.getId());
    });

// 方式 2：手动控制 BatchSize（防止单批次过大内存溢出）
public void batchUpdate(List<User> users) {
    String sql = "UPDATE user SET name = ? WHERE id = ?";
    
    jdbcTemplate.execute((Connection conn) -> {
        try (PreparedStatement ps = conn.prepareStatement(sql)) {
            int count = 0;
            int batchSize = 500;  // 每 500 条提交一次
            
            for (User user : users) {
                ps.setString(1, user.getName());
                ps.setLong(2, user.getId());
                ps.addBatch();
                
                if (++count % batchSize == 0) {
                    ps.executeBatch();      // 执行批量
                    ps.clearBatch();        // 清空批次
                }
            }
            
            // 剩余不足 batchSize 的
            if (count % batchSize != 0) {
                ps.executeBatch();
            }
            return null;
        }
    });
}
```

**性能提升：**
```
逐条：1000 次网络往返
批量：2 次网络往返（假设 batchSize=500）

提升 ≈ 500 倍（网络层面）
```

## 🔥 优化方案 2：单条 SQL 批量更新（MySQL）

```java
// ✅ 一条 SQL 更新多条（MySQL 语法）
// UPDATE user SET name = CASE id 
//   WHEN 1 THEN 'Alice' 
//   WHEN 2 THEN 'Bob' 
//   ...
// END WHERE id IN (1, 2, ...)

public void batchUpdateCaseWhen(List<User> users) {
    if (users.isEmpty()) return;
    
    StringBuilder sql = new StringBuilder("UPDATE user SET name = CASE id ");
    List<Object> params = new ArrayList<>();
    List<Long> ids = new ArrayList<>();
    
    for (User user : users) {
        sql.append("WHEN ? THEN ? ");
        params.add(user.getId());
        params.add(user.getName());
        ids.add(user.getId());
    }
    
    sql.append("END WHERE id IN (");
    sql.append(String.join(",", Collections.nCopies(ids.size(), "?")));
    sql.append(")");
    
    params.addAll(ids);
    
    jdbcTemplate.update(sql.toString(), params.toArray());
}
```

**限制：**
- SQL 长度有限制（`max_allowed_packet`，默认 4MB）
- 参数过多可能超过数据库限制（`max_prepared_stmt_count`）
- 适合 **100-1000 条** 的批量

## 🔥 优化方案 3：INSERT ON DUPLICATE KEY UPDATE（MySQL）

```java
// ✅ 利用 MySQL 的 UPSERT 语法
// INSERT INTO user (id, name) VALUES (1, 'Alice'), (2, 'Bob'), ...
// ON DUPLICATE KEY UPDATE name = VALUES(name)

public void upsertBatch(List<User> users) {
    String sql = "INSERT INTO user (id, name) VALUES " +
        String.join(",", Collections.nCopies(users.size(), "(?, ?)")) +
        " ON DUPLICATE KEY UPDATE name = VALUES(name)";
    
    List<Object> params = new ArrayList<>();
    for (User user : users) {
        params.add(user.getId());
        params.add(user.getName());
    }
    
    jdbcTemplate.update(sql, params.toArray());
}
```

**优势：**
- 一条 SQL，一次网络往返
- 自动处理 INSERT 或 UPDATE
- 比 `CASE WHEN` 更简洁

## 🔥 优化方案 4：PostgreSQL 批量语法

```java
// ✅ PostgreSQL 的 unnest 函数
// UPDATE user SET name = data.name
// FROM (SELECT unnest(?) as id, unnest(?) as name) as data
// WHERE user.id = data.id

public void batchUpdatePostgres(List<User> users) {
    Long[] ids = users.stream().map(User::getId).toArray(Long[]::new);
    String[] names = users.stream().map(User::getName).toArray(String[]::new);
    
    String sql = "UPDATE user SET name = data.name " +
        "FROM (SELECT unnest(?) as id, unnest(?) as name) as data " +
        "WHERE user.id = data.id";
    
    jdbcTemplate.update(sql, 
        conn -> conn.createArrayOf("bigint", ids),
        conn -> conn.createArrayOf("varchar", names)
    );
}
```

## 🔥 扩展追问 1：批量更新的事务控制

```java
// ❌ 问题：批量太大，一个事务持有锁太久
@Transactional
public void updateAll(List<User> users) {
    batchUpdate(users);  // 10万条，锁表 10 秒，其他查询被阻塞
}

// ✅ 优化：分批提交，减少锁持有时间
public void updateInBatches(List<User> users, int batchSize) {
    for (int i = 0; i < users.size(); i += batchSize) {
        List<User> batch = users.subList(i, Math.min(i + batchSize, users.size()));
        
        // 每批独立事务
        transactionTemplate.execute(status -> {
            batchUpdate(batch);
            return null;
        });
        
        // 可选：批次间休眠，让其他事务有机会执行
        if (i + batchSize < users.size()) {
            Thread.sleep(100);
        }
    }
}
```

## 🔥 扩展追问 2：异步批量更新（削峰）

```java
// ✅ 高并发场景：先写 MQ，异步批量消费
@Service
public class UserUpdateService {
    
    @Autowired
    private KafkaTemplate<String, UserUpdateEvent> kafka;
    
    // 接口层：只发 MQ，立即返回
    public void updateUserAsync(User user) {
        kafka.send("user-update", new UserUpdateEvent(user));
    }
}

// 消费者：批量聚合，定时刷盘
@Component
public class UserUpdateConsumer {
    
    private final List<User> buffer = new ArrayList<>();
    private static final int BATCH_SIZE = 500;
    
    @KafkaListener(topics = "user-update")
    public void onMessage(UserUpdateEvent event) {
        buffer.add(event.getUser());
        
        if (buffer.size() >= BATCH_SIZE) {
            flush();
        }
    }
    
    @Scheduled(fixedRate = 1000)  // 每秒兜底刷盘
    public void flush() {
        if (buffer.isEmpty()) return;
        
        List<User> batch = new ArrayList<>(buffer);
        buffer.clear();
        
        // 批量更新
        batchUpdate(batch);
    }
}
```

## 一句话收尾

> 逐条 `update()` 是**性能杀手**，优先用 **`jdbcTemplate.batchUpdate()`**（通用）或 **数据库方言批量语法**（MySQL 用 `INSERT ODKU`，PostgreSQL 用 `unnest`），配合 **合理批次大小（500-1000）** 和 **分批事务提交**，可提升 **10-100 倍** 性能；高并发场景用 **MQ 异步聚合批量消费** 削峰。
