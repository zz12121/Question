67. 你用 `CopyOnWriteArrayList` 实现读多写少的列表：
```java
CopyOnWriteArrayList<String> list = new CopyOnWriteArrayList<>();

// 读
for (String s : list) {
    System.out.println(s);
}

// 写
list.add("new");
```
**问题：** 写操作的开销大吗？适合什么场景？迭代器有什么特点？

---
## 先说结论

`CopyOnWriteArrayList` 的**写操作开销极大**——每次修改都复制整个底层数组，时间复杂度 O(n)，空间瞬间翻倍。它只适合**读多写极少、数据量小**的场景（如配置列表、白名单）。迭代器是**快照迭代**，不会抛出 `ConcurrentModificationException`，但也看不到其他线程的实时修改。

---

## 🔥 写操作的开销有多大

```java
public boolean add(E e) {
    final ReentrantLock lock = this.lock;
    lock.lock();
    try {
        Object[] elements = getArray();      // 1. 获取原数组
        int len = elements.length;
        Object[] newElements = Arrays.copyOf(elements, len + 1);  // 2. 复制整个数组！
        newElements[len] = e;                 // 3. 修改新数组
        setArray(newElements);                // 4. 原子替换引用
        return true;
    } finally {
        lock.unlock();
    }
}
```

**开销分析：**
| 操作 | 时间复杂度 | 空间开销 |
|---|---|---|
| `add()` | O(n) — 复制整个数组 | 原数组 + 新数组共存，瞬间翻倍 |
| `set()` | O(n) — 同上 | 同上 |
| `remove()` | O(n) — 复制 + 元素移动 | 同上 |

**1000 个元素的列表，每次 add 都要复制 1000 个引用：**
```
原数组: [a, b, c, ..., z]  (1000个元素)
新数组: [a, b, c, ..., z, new]  (复制1000 + 新增1)

如果频繁写：GC 压力爆炸，Young GC 频繁，甚至晋升到老年代
```

---

## 🔥 适合什么场景

### ✅ 推荐场景

```java
// 1. 配置列表 —— 启动时加载，运行期几乎不修改
CopyOnWriteArrayList<String> whiteList = new CopyOnWriteArrayList<>();
// 启动加载
whiteList.addAll(loadFromDB());
// 运行期大量读取
if (whiteList.contains(ip)) { ... }

// 2. 事件监听器列表 —— 注册/注销少，触发遍历多
CopyOnWriteArrayList<Listener> listeners = new CopyOnWriteArrayList<>();
// 注册（很少）
listeners.add(listener);
// 触发（频繁）
for (Listener l : listeners) {
    l.onEvent(event);  // 遍历快照，不怕其他线程修改
}
```

### ❌ 禁用场景

```java
// 1. 高频写 —— 日志收集、消息队列
CopyOnWriteArrayList<LogEntry> logs = new CopyOnWriteArrayList<>();
logs.add(log);  // 每秒几千次？GC 直接崩

// 2. 大数据量 —— 10万条数据，每次复制 800KB
CopyOnWriteArrayList<Item> items = new CopyOnWriteArrayList<>();
// 内存抖动严重，频繁 Full GC

// 3. 需要强一致性的场景
// 写完立刻读，可能读到旧数据（数组引用还没切换）
```

## 🔥 迭代器的快照特性

```java
CopyOnWriteArrayList<String> list = new CopyOnWriteArrayList<>();
list.add("A");
list.add("B");

Iterator<String> it = list.iterator();  // 迭代器创建时，拿到当前数组快照
list.add("C");  // 新增元素，不影响迭代器

while (it.hasNext()) {
    System.out.println(it.next());  // 输出 A, B —— 看不到 C！
}

// 新迭代器才能看到 C
for (String s : list) {
    System.out.println(s);  // 输出 A, B, C
}
```

**关键特点：**
| 特性 | 说明 |
|---|---|
| 快照隔离 | 迭代器基于创建时刻的数组副本 |
| 无 CME | 不会 `ConcurrentModificationException` |
| 弱一致性 | 不保证读到最新数据 |
| `remove()` 不支持 | 迭代器的 `remove()` 抛 `UnsupportedOperationException` |

## 🔥 与 `Collections.synchronizedList` 对比

```java
// 方案 1：CopyOnWriteArrayList —— 读无锁，写有锁+复制
CopyOnWriteArrayList<String> cow = new CopyOnWriteArrayList<>();
// 读：无锁，直接读数组
// 写：加锁，复制数组

// 方案 2：SynchronizedList —— 所有操作加同一把锁
List<String> sync = Collections.synchronizedList(new ArrayList<>());
// 读/写都加 synchronized，并发读也串行
```

| 场景 | 推荐选择 |
|---|---|
| 读极多、写极少、数据量小 | `CopyOnWriteArrayList` |
| 读写均衡、数据量大 | `Collections.synchronizedList` 或 `ReadWriteLock` |
| 高并发读写 | `ConcurrentHashMap.newKeySet()` 或自定义分段锁 |

## 🔥 扩展追问 1：`CopyOnWriteArraySet` 是什么？

```java
// 底层就是 CopyOnWriteArrayList，去重靠 equals
CopyOnWriteArraySet<String> set = new CopyOnWriteArraySet<>();
set.add("A");
set.add("A");  // 第二次 add 会遍历整个数组检查是否存在，O(n)！

// 大数据量时，add 性能极差
```

**`CopyOnWriteArraySet.add()` 是 O(n) 的**，因为每次都要遍历检查重复，比 `CopyOnWriteArrayList` 还慢。

## 🔥 扩展追问 2：为什么叫"写时复制"？

```
读操作：
  直接读 array 引用指向的数组，无锁，多线程并发安全

写操作：
  1. 加锁（保证只有一个线程在写）
  2. 复制新数组
  3. 修改新数组
  4. 原子替换 array 引用（volatile 保证可见性）
  5. 解锁

读线程永远读的是"某一时刻的快照"，不会被写线程干扰
```

**核心思想：读写分离，用空间换时间，用不可变性保证并发安全。**

## 🔥 扩展追问 3：实际生产中的替代方案

```java
// 1. 读多写少 + 数据量大 —— ReadWriteLock
ReadWriteLock rwLock = new ReentrantReadWriteLock();
List<String> list = new ArrayList<>();

// 读
rwLock.readLock().lock();
try {
    for (String s : list) { ... }
} finally {
    rwLock.readLock().unlock();
}

// 写
rwLock.writeLock().lock();
try {
    list.add("new");
} finally {
    rwLock.writeLock().unlock();
}

// 2. 纯配置场景 —— 不可变列表 + volatile 切换
private volatile List<String> config = loadConfig();

// 读：直接读 volatile 引用
for (String s : config) { ... }

// 写：新建列表，原子替换
public void updateConfig() {
    List<String> newConfig = new ArrayList<>(loadFromDB());
    this.config = newConfig;  // volatile 保证可见性
}
```

## 一句话收尾

> `CopyOnWriteArrayList` 写操作 O(n) 复制数组，**仅适合读多写极少、数据量小的场景**（如配置、监听器列表）；迭代器是快照视图，弱一致性；大数据量或高频写时，用 `ReadWriteLock` 或 `volatile` + 不可变对象替代。
