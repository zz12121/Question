84. 你用 Redis 存储热点数据：
```java
public String getHotData(String key) {
    String value = redis.get(key);
    if (value == null) {
        value = db.query(key);
        redis.setex(key, 60, value);
    }
    return value;
}
```
**问题：** 如果热点数据过期瞬间，大量请求同时打到数据库，如何解决？

---
## 先说结论

热点数据过期瞬间，大量请求同时穿透缓存打到数据库，这就是**缓存击穿**。解决方案：**互斥锁（单线程重建）**、**逻辑过期（永不过期）**、或**热点数据预加载**，生产环境推荐 **互斥锁 + 异步重建** 的组合方案。

---

## 🔥 问题分析：缓存击穿

```
时间线（key 在 T0 时刻过期）：

T0: redis.get(key) 返回 null（key 过期）
    ↓
    1000 个并发请求同时发现缓存未命中
    ↓
    1000 个线程同时查数据库
    ↓
    数据库 QPS 从 100 飙升到 1000，被打挂
    ↓
    每个线程查完后都 setex，重复写缓存 1000 次
```

**与缓存穿透、缓存雪崩的区别：**

| 问题 | 原因 | 现象 |
|---|---|---|
| **缓存穿透** | 查询不存在的数据 | 每次都查 DB，如攻击者查 id=-1 |
| **缓存击穿** | 热点 key 过期 | 过期瞬间大量请求查 DB |
| **缓存雪崩** | 大量 key 同时过期 | 整个缓存层失效，DB 被打爆 |

---

## 🔥 解决方案 1：互斥锁（单线程重建）

```java
public String getHotData(String key) {
    String value = redis.get(key);
    if (value != null) {
        return value;
    }
    
    // 缓存未命中，尝试获取锁
    String lockKey = "lock:" + key;
    boolean locked = tryLock(lockKey, 10);
    
    if (locked) {
        try {
            // 双重检查，防止其他线程已重建
            value = redis.get(key);
            if (value != null) {
                return value;
            }
            
            // 查 DB 并重建缓存
            value = db.query(key);
            redis.setex(key, 60, value);
            
        } finally {
            unlock(lockKey);
        }
    } else {
        // 没抢到锁，短暂等待后重试
        Thread.sleep(50);
        return getHotData(key);  // 递归重试
    }
    
    return value;
}

private boolean tryLock(String key, int seconds) {
    return redisTemplate.opsForValue()
        .setIfAbsent(key, "1", seconds, TimeUnit.SECONDS);
}

private void unlock(String key) {
    redisTemplate.delete(key);
}
```

**执行流程：**

```
1000 个请求同时到达：
  - 第 1 个：获取锁成功，查 DB，写缓存，释放锁（耗时 100ms）
  - 第 2-1000 个：获取锁失败，sleep 50ms 后重试
    - 重试时缓存已存在，直接返回
  
结果：只有 1 次 DB 查询，999 次等待后走缓存
```

## 🔥 解决方案 2：逻辑过期（永不过期）

```java
@Data
public class RedisData {
    private String value;
    private long expireTime;  // 逻辑过期时间戳
}

public String getHotData(String key) {
    RedisData data = redisTemplate.opsForValue().get(key);
    
    if (data == null) {
        // 缓存不存在，查 DB 并写入（带逻辑过期时间）
        String value = db.query(key);
        data = new RedisData(value, System.currentTimeMillis() + 60000);
        redisTemplate.opsForValue().set(key, data);  // 永不过期
        return value;
    }
    
    // 缓存存在，检查逻辑过期时间
    if (data.getExpireTime() > System.currentTimeMillis()) {
        // 未过期，直接返回
        return data.getValue();
    }
    
    // 已过期，启动异步重建，但先返回旧值
    CompletableFuture.runAsync(() -> rebuildCache(key));
    return data.getValue();  // 返回过期数据，保证可用性
}

private void rebuildCache(String key) {
    String lockKey = "lock:" + key;
    if (tryLock(lockKey, 10)) {
        try {
            // 双重检查
            RedisData data = redisTemplate.opsForValue().get(key);
            if (data != null && data.getExpireTime() > System.currentTimeMillis()) {
                return;  // 其他线程已重建
            }
            
            String value = db.query(key);
            RedisData newData = new RedisData(value, System.currentTimeMillis() + 60000);
            redisTemplate.opsForValue().set(key, newData);
            
        } finally {
            unlock(lockKey);
        }
    }
}
```

**优势：**
- 永不过期，不存在击穿问题
- 过期后异步重建，用户无感知
- 极端情况下返回旧数据，保证可用性

```java
    private RedisTemplate<String, String> redis;

    }
```

## 🔥 扩展追问 1：Redisson 实现互斥锁

```java
@Autowired
private RedissonClient redisson;

public String getHotData(String key) {
    String value = redis.get(key);
    if (value != null) {
        return value;
    }
    
    RLock lock = redisson.getLock("lock:" + key);
    try {
        // 尝试获取锁，最多等待 10 秒，锁持有 30 秒
        boolean locked = lock.tryLock(10, 30, TimeUnit.SECONDS);
        if (!locked) {
            throw new RuntimeException("获取锁超时");
        }
        
        // 双重检查
        value = redis.get(key);
        if (value != null) {
            return value;
        }
        
        value = db.query(key);
        redis.setex(key, 60, value);
        return value;
        
    } catch (InterruptedException e) {
        Thread.currentThread().interrupt();
        throw new RuntimeException(e);
    } finally {
        if (lock.isHeldByCurrentThread()) {
            lock.unlock();
        }
    }
}
```

**Redisson 优势：**
- 可重入锁
- 自动续期（Watchdog）
- 锁超时保护

## 🔥 扩展追问 4：热点 Key 的自动发现

```java
@Component
public class HotKeyDetector {
    
    // 滑动窗口统计访问频率
    private final ConcurrentHashMap<String, LongAdder> counter = new ConcurrentHashMap<>();
    
    // 每 10 秒分析一次
    @Scheduled(fixedRate = 10000)
    public void analyze() {
        // 找出 Top 100 热点 key
        List<String> hotKeys = counter.entrySet().stream()
            .sorted((e1, e2) -> Long.compare(e2.getValue().sum(), e1.getValue().sum()))
            .limit(100)
            .map(Map.Entry::getKey)
            .collect(toList());
        
        // 推送到本地缓存预加载
        hotKeys.forEach(this::preloadToLocalCache);
        
        // 清空计数器
        counter.clear();
    }
    
    public void recordAccess(String key) {
        counter.computeIfAbsent(key, k -> new LongAdder()).increment();
    }
}
```
