90. 你的接口返回：
```java
@GetMapping("/user/{id}")
public User getUser(@PathVariable Long id) {
    return userService.getById(id);
}
```
**问题：** 如果用户不存在，返回什么？如何统一包装返回结果？

---
## 先说结论

用户不存在时**直接返回 null**，前端收到 `200 OK` 但数据为空，无法区分"用户不存在"和"系统错误"。正确做法是：**不存在抛异常或返回明确状态**，并通过 **ResponseBodyAdvice 统一包装** 所有接口返回，保证格式一致。

---

## 🔥 当前代码的问题

### 1. 返回 null，语义不明

```java
@GetMapping("/user/{id}")
public User getUser(@PathVariable Long id) {
    return userService.getById(id);  // 不存在返回 null
}
```

**前端收到的响应：**
```json
// 用户存在
{ "id": 1, "name": "张三" }

// 用户不存在
null  // 或者空响应体，HTTP 200

// 系统报错
null  // 同样的 null，无法区分！
```

### 2. 每个接口格式不一致

```java
// 接口 A 直接返回实体
@GetMapping("/user/{id}")
public User getUser(...) { }

// 接口 B 手动包装
@GetMapping("/order/{id}")
public Result<Order> getOrder(...) { 
    return Result.ok(orderService.getById(id));
}

// 接口 C 返回 Map
@GetMapping("/stats")
public Map<String, Object> getStats() { }
```

前端需要写三套解析逻辑。

## 🔥 正确做法：统一包装 + 明确状态

### 方案一：ResponseBodyAdvice 全局包装（推荐）

```java
@Data
public class Result<T> {
    private Integer code;
    private String message;
    private T data;
    private Long timestamp;
    
    public static <T> Result<T> success(T data) {
        Result<T> r = new Result<>();
        r.code = 200;
        r.message = "success";
        r.data = data;
        r.timestamp = System.currentTimeMillis();
        return r;
    }
    
    public static <T> Result<T> error(int code, String message) {
        Result<T> r = new Result<>();
        r.code = code;
        r.message = message;
        r.timestamp = System.currentTimeMillis();
        return r;
    }
}
```

**全局包装器：**

```java
@RestControllerAdvice(basePackages = "com.example.controller")
public class ResponseWrapper implements ResponseBodyAdvice<Object> {
    
    @Override
    public boolean supports(MethodParameter returnType, 
                          Class<? extends HttpMessageConverter<?>> converterType) {
        // 已经是 Result 类型或 String 类型不处理（String 需特殊处理）
        return !returnType.getParameterType().equals(Result.class);
    }
    
    @Override
    public Object beforeBodyWrite(Object body,
                                  MethodParameter returnType,
                                  MediaType selectedContentType,
                                  Class<? extends HttpMessageConverter<?>> selectedConverterType,
                                  ServerHttpRequest request,
                                  ServerHttpResponse response) {
        
        // String 类型特殊处理（否则会报类型转换错误）
        if (body instanceof String) {
            return JSON.toJSONString(Result.success(body));
        }
        
        // 已经是 Result 不包装
        if (body instanceof Result) {
            return body;
        }
        
        // null 包装成空 data
        if (body == null) {
            return Result.success(null);
        }
        
        return Result.success(body);
    }
}
```

**Controller 保持简洁：**

```java
@RestController
public class UserController {
    
    @GetMapping("/user/{id}")
    public User getUser(@PathVariable Long id) {
        User user = userService.getById(id);
        if (user == null) {
            throw new BusinessException(404, "用户不存在");
        }
        return user;  // 自动包装成 Result.success(user)
    }
    
    @GetMapping("/users")
    public List<User> listUsers() {
        return userService.list();  // 自动包装
    }
}
```

**统一响应格式：**
```json
// 成功
{
    "code": 200,
    "message": "success",
    "data": { "id": 1, "name": "张三" },
    "timestamp": 1713100800000
}

// 用户不存在
{
    "code": 404,
    "message": "用户不存在",
    "data": null,
    "timestamp": 1713100800000
}
```

```java
public class UserController {
```

### 追问 1：HTTP 状态码怎么定？

| 场景 | HTTP 状态码 | Result.code | 说明 |
|-----|------------|-------------|------|
| 成功 | 200 | 200 | 正常返回 |
| 参数错误 | 400 | 400001 | 业务错误码细分 |
| 未认证 | 401 | 401 | 需登录 |
| 无权限 | 403 | 403 | 权限不足 |
| 资源不存在 | 404 | 404 | 用户/订单不存在 |
| 系统错误 | 500 | 500 | 服务端异常 |

```java
// 统一在异常处理器中设置 HTTP 状态码
@ExceptionHandler(BusinessException.class)
public ResponseEntity<Result<?>> handleBusiness(BusinessException e) {
    HttpStatus status = HttpStatus.resolve(e.getCode());
    if (status == null) status = HttpStatus.BAD_REQUEST;
    
    return ResponseEntity.status(status)
        .body(Result.error(e.getCode(), e.getMessage()));
}
```

### 追问 3：文件下载/特殊响应怎么跳过包装？

```java
// 自定义注解标记跳过包装
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface SkipWrapper {}

// 在 ResponseBodyAdvice 中判断
@Override
public boolean supports(MethodParameter returnType, ...) {
    // 有 SkipWrapper 注解不包装
    if (returnType.hasMethodAnnotation(SkipWrapper.class)) {
        return false;
    }
    return !returnType.getParameterType().equals(Result.class);
}

// 使用
@GetMapping("/export")
@SkipWrapper
public void exportExcel(HttpServletResponse response) {
    // 直接写响应流
}
```

### 追问 4：Feign 调用怎么统一处理？

```java
// Feign 配置解码器
public class FeignResultDecoder implements Decoder {
    
    @Override
    public Object decode(Response response, Type type) throws IOException {
        String body = Util.toString(response.body().asReader(StandardCharsets.UTF_8));
        Result<?> result = JSON.parseObject(body, Result.class);
        
        if (!result.isSuccess()) {
            throw new BusinessException(result.getCode(), result.getMessage());
        }
        
        // 提取 data 反序列化
        return JSON.parseObject(JSON.toJSONString(result.getData()), type);
    }
}
```
