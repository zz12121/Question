69. 你用 `String.intern()` 节省内存：
```java
String s1 = new String("hello").intern();
String s2 = new String("hello").intern();
System.out.println(s1 == s2);
```
**问题：** 输出是什么？`intern()` 的原理是什么？JDK 6 和 JDK 7+ 有什么区别？

---
## 先说结论

输出是 **`true`**。`intern()` 把字符串放入**字符串常量池**，后续相同的字符串直接返回池中的引用。JDK 6 和 JDK 7+ 的核心区别：**JDK 6 常量池在永久代（PermGen），JDK 7+ 移到堆内存**，且 JDK 7+ 对堆中已存在的字符串引用优化，避免了重复拷贝。

---

## 🔥 代码执行流程拆解

```java
String s1 = new String("hello").intern();
// 步骤1: new String("hello") 在堆中创建新对象
// 步骤2: "hello" 字面量编译期已在常量池
// 步骤3: intern() 发现常量池已有 "hello"，返回常量池引用
// s1 指向常量池中的 "hello"

String s2 = new String("hello").intern();
// 步骤1: 堆中又创建一个 String 对象
// 步骤2: intern() 还是返回常量池的同一个引用
// s2 也指向常量池中的 "hello"

System.out.println(s1 == s2);  // true，同一个引用
```

---

## 🔥 `intern()` 的原理

```
调用 intern() 时：
  1. 检查字符串常量池（StringTable）是否存在 equals 相等的字符串
  2. 存在 → 返回常量池中的引用
  3. 不存在 → 将当前字符串加入常量池，然后返回该引用
```

**关键点：intern() 返回的永远是常量池中的引用，不是堆中的对象。**

---

## 🔥 JDK 6 vs JDK 7+ 的区别

| 特性 | JDK 6 | JDK 7+ |
|---|---|---|
| **常量池位置** | 永久代（PermGen） | 堆内存（Heap） |
| **内存限制** | PermGen 默认很小（64MB），易 OOM | 堆内存大，可 GC 回收 |
| **intern 实现** | 拷贝字符串到永久代 | 常量池存**引用**，指向堆中对象 |
| **GC 影响** | PermGen 满才 Full GC，字符串难回收 | 堆中字符串可被 Young GC |
| **性能** | 大字符串 intern 导致 PermGen OOM | 更安全，但仍可能撑大 StringTable |

### JDK 6 的问题

```java
// JDK 6：大量不同字符串 intern → PermGen OOM
List<String> list = new ArrayList<>();
for (int i = 0; i < 10000000; i++) {
    list.add(("key-" + i).intern());  // 每个都拷贝到 PermGen
}
// java.lang.OutOfMemoryError: PermGen space
```

### JDK 7+ 的优化

```java
// JDK 7+：常量池存引用，不拷贝字符串内容
String s = new StringBuilder("hello").toString();
String interned = s.intern();
// 常量池中的 entry 指向堆中的 s，没有额外拷贝
```

**但注意：如果常量池已存在字面量，intern 返回的是那个引用**

```java
String s1 = "hello";              // 常量池已有 "hello"
String s2 = new String("hello");  // 堆中新对象
String s3 = s2.intern();          // 返回常量池的引用（即 s1 的引用）

System.out.println(s1 == s3);  // true
System.out.println(s2 == s3);  // false！s2 还是堆中对象
```

## 🔥 实战：什么时候用 intern？

### ✅ 推荐场景：大量重复字符串，内存敏感

```java
// 场景：用户地址去重，1000万条记录，实际只有 100 个不同城市
List<String> cities = fetchFromDB();  // ["北京", "上海", "北京", "北京", "上海"...]

// 优化前：1000万个 String 对象
// 优化后：100个 String 对象 + 1000万个引用
List<String> optimized = cities.stream()
    .map(String::intern)
    .collect(toList());
```

**效果：内存从几百 MB 降到几 MB。**

### ❌ 禁用场景

```java
// 1. 字符串生命周期短
String temp = request.getParam("keyword").intern();  // 马上 GC，intern 白做

// 2. 字符串唯一性高
String uuid = UUID.randomUUID().toString().intern();  // 100万个不同 UUID，撑爆 StringTable

// 3. 长字符串
String bigJson = loadBigJson().intern();  // 几 MB 的字符串进常量池，GC 难回收
```

## 🔥 扩展追问 1：StringTable 的大小调优

```java
// JDK 7+ 参数调优
-XX:StringTableSize=100000  // 默认约 60013，调大减少哈希冲突

// 查看 StringTable 统计
jcmd <pid> VM.stringtable
```

**StringTable 是哈希表，大小固定，太多不同字符串会导致链表过长，intern 性能下降。**

---

## 🔥 扩展追问 2：`new String("a") + new String("b")` 的 intern 陷阱

```java
String s1 = new String("a") + new String("b");  // 堆中 "ab"
s1.intern();                                     // 常量池引用指向堆中 s1
String s2 = "ab";                                // 常量池已有，直接引用
System.out.println(s1 == s2);  // true（JDK 7+）/ false（JDK 6）

// 但如果反过来：
String s3 = "cd";                                // 编译期入常量池
String s4 = new String("c") + new String("d");
s4.intern();                                     // 返回 s3 的引用
System.out.println(s3 == s4);  // false！s4 还是堆中对象
```

**规律：谁先把字符串放入常量池，intern 就返回谁的引用。**

---

## 🔥 扩展追问 3：G1 的 String Deduplication

```java
// JDK 8u20+ G1 垃圾收集器的优化
-XX:+UseStringDeduplication

// 不需要手动 intern()，GC 时自动去重
// 原理：Young GC 时扫描堆，发现重复 char[]，共享同一个
```

**和 intern() 的区别：**
| 特性 | `intern()` | String Deduplication |
|---|---|---|
| 时机 | 运行时立即去重 | GC 时才去重 |
| 范围 | 常量池 | 整个堆 |
| 开销 | 立即有 CPU 开销 | GC 时才有开销 |
| 控制 | 手动 | 自动 |
| 生命周期 | 常量池引用难回收 | 随对象 GC |

---

## 🔥 扩展追问 4：字符串常量池的面试陷阱题

```java
String s1 = new String("1") + new String("1");  // 堆中 "11"
s1.intern();
String s2 = "11";
System.out.println(s1 == s2);  // true（JDK 7+）

// 但：
String s3 = new String("2") + new String("2");
String s4 = "22";  // 先定义字面量！
s3.intern();
System.out.println(s3 == s4);  // false
```

**核心：执行顺序决定谁先入常量池，intern 返回先入者的引用。**

---

## 一句话收尾

> `intern()` 返回常量池引用实现字符串复用，**JDK 6 在 PermGen 易 OOM，JDK 7+ 移到堆中且存引用而非拷贝**；适合大量重复字符串的内存优化，但唯一性高的字符串 intern 会撑大 StringTable，配合 `-XX:StringTableSize` 调优，或直接用 G1 的 String Deduplication 自动去重。
