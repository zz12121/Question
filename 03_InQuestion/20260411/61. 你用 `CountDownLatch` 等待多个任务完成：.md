61. 你用 `CountDownLatch` 等待多个任务完成：
```java
CountDownLatch latch = new CountDownLatch(3);
for (int i = 0; i < 3; i++) {
    new Thread(() -> {
        doWork();
        latch.countDown();
    }).start();
}
latch.await();
```
**问题：** 如果某个任务执行异常，`countDown()` 没有被调用，会发生什么？如何处理异常？

---
## 先说结论

如果 `doWork()` 抛出异常导致 `countDown()` 未被调用，`latch.await()` 将**永久阻塞**（或超时），主线程挂死。解决方案是用 **try-finally 保证 countDown 必被执行**，同时通过**异常收集 + CompletableFuture / ExecutorService** 做更完善的错误处理。

---

## 🔥 核心概念

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| `await()` 永久阻塞 | countDown 未执行，计数无法归零 | try-finally 保证 countDown |
| 异常静默丢失 | 子线程异常不传播到主线程 | 收集异常，主线程处理 |
| 无法感知哪个任务失败 | 只知道 await 结束，不知道成功失败 | AtomicReference 存储异常 |

---

## 🔥 详细展开

### 1. 问题复现

```java
// ❌ 危险代码：任务异常 → countDown 丢失 → await 永久阻塞
CountDownLatch latch = new CountDownLatch(3);
for (int i = 0; i < 3; i++) {
    final int taskId = i;
    new Thread(() -> {
        doWork(taskId);  // 任务2 抛出 RuntimeException
        latch.countDown();  // 永远不会执行！
    }).start();
}
latch.await();  // 计数停留在 1，主线程永久挂死
```

---

### 2. 修复方案一：try-finally（最基础）

```java
// ✅ try-finally 保证 countDown 必须执行
CountDownLatch latch = new CountDownLatch(3);
for (int i = 0; i < 3; i++) {
    final int taskId = i;
    new Thread(() -> {
        try {
            doWork(taskId);
        } catch (Exception e) {
            log.error("任务 {} 执行失败", taskId, e);
            // 吞掉异常，记录日志，主线程继续
        } finally {
            latch.countDown();  // 无论成功失败，一定执行
        }
    }).start();
}
latch.await();
System.out.println("所有任务已完成（含失败）");
```

**问题**：主线程不知道哪些任务失败了，适用于"容忍部分失败"的场景。

---

### 3. 修复方案二：收集异常（推荐）

```java
// ✅ 收集异常，主线程感知任务失败
CountDownLatch latch = new CountDownLatch(3);
List<Exception> errors = new CopyOnWriteArrayList<>();

for (int i = 0; i < 3; i++) {
    final int taskId = i;
    new Thread(() -> {
        try {
            doWork(taskId);
        } catch (Exception e) {
            errors.add(e);  // 收集异常
        } finally {
            latch.countDown();
        }
    }).start();
}

latch.await();

// 主线程检查是否有失败
if (!errors.isEmpty()) {
    log.error("有 {} 个任务失败", errors.size());
    errors.forEach(e -> log.error("失败原因: {}", e.getMessage()));
    throw new RuntimeException("部分任务执行失败", errors.get(0));
}
```

---

### 4. 修复方案三：带超时的 await（防死锁兜底）

```java
// ✅ await(timeout) 防止永久阻塞
CountDownLatch latch = new CountDownLatch(3);
List<Exception> errors = new CopyOnWriteArrayList<>();

for (int i = 0; i < 3; i++) {
    final int taskId = i;
    new Thread(() -> {
        try {
            doWork(taskId);
        } catch (Exception e) {
            errors.add(e);
        } finally {
            latch.countDown();
        }
    }).start();
}

// 最多等待 30 秒
boolean finished = latch.await(30, TimeUnit.SECONDS);
if (!finished) {
    // 超时，说明有任务卡住了（死锁/网络超时等）
    throw new TimeoutException("任务执行超时，剩余计数: " + latch.getCount());
}
if (!errors.isEmpty()) {
    throw new RuntimeException("部分任务失败", errors.get(0));
}
```

---

### 5. 修复方案四：用线程池 + Future（最佳实践）

```java
// ✅ ExecutorService + Future：最优雅的异常处理
ExecutorService pool = Executors.newFixedThreadPool(3);
CountDownLatch latch = new CountDownLatch(3);

List<Future<?>> futures = new ArrayList<>();
for (int i = 0; i < 3; i++) {
    final int taskId = i;
    Future<?> future = pool.submit(() -> {
        try {
            doWork(taskId);
        } finally {
            latch.countDown();
        }
    });
    futures.add(future);
}

latch.await();

// 检查所有任务执行结果
for (int i = 0; i < futures.size(); i++) {
    try {
        futures.get(i).get(0, TimeUnit.SECONDS);  // 非阻塞，已完成
    } catch (ExecutionException e) {
        log.error("任务 {} 失败: {}", i, e.getCause().getMessage());
    }
}

pool.shutdown();
```

---

### 6. 修复方案五：CompletableFuture（现代写法）

```java
// ✅ CompletableFuture：最现代的方案，不需要 CountDownLatch
List<CompletableFuture<Void>> futures = new ArrayList<>();

for (int i = 0; i < 3; i++) {
    final int taskId = i;
    CompletableFuture<Void> future = CompletableFuture
        .runAsync(() -> doWork(taskId), pool)
        .exceptionally(e -> {
            log.error("任务 {} 失败: {}", taskId, e.getMessage());
            return null;  // 异常已处理，继续
        });
    futures.add(future);
}

// 等待所有任务完成（包含异常处理后的任务）
CompletableFuture.allOf(futures.toArray(new CompletableFuture[0])).join();
System.out.println("所有任务完成");
```

|------|--------|---------|-----------|---------|

## 🔥 扩展追问 2：UncaughtExceptionHandler 捕获子线程异常

```java
// 设置线程的 UncaughtExceptionHandler
CountDownLatch latch = new CountDownLatch(3);
AtomicBoolean hasError = new AtomicBoolean(false);

for (int i = 0; i < 3; i++) {
    final int taskId = i;
    Thread thread = new Thread(() -> {
        try {
            doWork(taskId);
        } finally {
            latch.countDown();
        }
    });
    
    thread.setUncaughtExceptionHandler((t, e) -> {
        log.error("线程 {} 发生未捕获异常: {}", t.getName(), e.getMessage());
        hasError.set(true);
        latch.countDown();  // 未捕获异常也要 countDown！
    });
    
    thread.start();
}

latch.await();
if (hasError.get()) {
    log.warn("部分任务执行失败");
}
```

> ⚠️ 注意：UncaughtExceptionHandler 里也需要调用 `countDown()`，因为 finally 在抛出后不一定执行（如果异常从 Thread.run() 逃逸）。

## 🔥 扩展追问 3：CountDownLatch vs CyclicBarrier 异常处理区别

| 维度 | CountDownLatch | CyclicBarrier |
|------|---------------|---------------|
| **异常传播** | 子线程异常不影响主线程 await | 某个线程在 await() 时被中断，BrokenBarrierException 传播给所有等待线程 |
| **重置** | 不可重置，一次性 | 可重置（reset() 方法） |
| **损坏状态** | 无 broken 概念 | 一旦 broken，所有 await 抛异常 |

```java
// CyclicBarrier 异常处理示例
CyclicBarrier barrier = new CyclicBarrier(3, () -> {
    System.out.println("所有线程到达屏障，执行汇总逻辑");
});

new Thread(() -> {
    try {
        doWork();
        barrier.await();
    } catch (BrokenBarrierException e) {
        // 其他线程出异常导致屏障损坏
        log.error("屏障已损坏，任务取消");
    } catch (InterruptedException e) {
        Thread.currentThread().interrupt();
    }
}).start();
```

## 一句话收尾

`countDown()` 必须放在 **try-finally** 里保证执行，用 **CopyOnWriteArrayList 收集异常**让主线程感知失败，生产环境首选 **CompletableFuture.allOf()** 或 **Future + 线程池**替代裸 CountDownLatch，代码更健壮。

1. **防死锁**：`countDown()` 必须在 `try-finally` 里，这是最基础的要求
2. **感知失败**：用 `CopyOnWriteArrayList` 在子线程中收集异常，主线程 `await()` 后检查
3. **现代写法**：直接用 `CompletableFuture.allOf()` 替代 CountDownLatch，天然支持异常处理，代码也更简洁
