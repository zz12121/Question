87. 你的配置属性：
```java
@Value("${server.timeout:5000}")
private int timeout;
```
**问题：** 如果配置项不存在且没有默认值，会发生什么？如何配置必须项？

---
## 先说结论

`@Value` 配置项**不存在且无默认值**时，启动直接抛 `IllegalArgumentException` 报错。配置必填项有两种方式：去掉默认值让 Spring 强制校验，或用 `@ConfigurationProperties` 配合 `@Validated` 进行结构化校验。

---

## 🔥 场景演示

### 场景 1：无默认值，配置缺失 → 启动失败

```java
@Value("${server.timeout}")  // 没有 :默认值
private int timeout;
```

**配置文件中没配：**
```yaml
# application.yml 里 server.timeout 不存在
```

**启动报错：**
```
IllegalArgumentException: Could not resolve placeholder 'server.timeout' in value "${server.timeout}"
```

---

### 场景 2：有默认值，配置缺失 → 使用默认值

```java
@Value("${server.timeout:5000}")  // 冒号后是默认值
private int timeout;  // timeout = 5000
```

---

### 场景 3：配置存在但类型不匹配 → 启动失败

```yaml
server:
  timeout: "not_a_number"  # 字符串，不是数字
```

```java
@Value("${server.timeout:5000}")
private int timeout;
```

**报错：**
```
ConversionFailedException: Failed to convert from type [java.lang.String] to type [int]
```

---

## 🔥 配置必填项的两种方式

### 方式一：去掉默认值（简单场景）

```java
@Component
public class AppConfig {
    
    @Value("${server.timeout}")  // 无默认值 = 必填
    private int timeout;
    
    @Value("${db.password}")     // 无默认值 = 必填
    private String dbPassword;
}
```

**缺失时启动即报错**，强制开发者配置。

---

### 方式二：@ConfigurationProperties + @Validated（推荐）

**配置类：**

```java
@Data
@Component
@ConfigurationProperties(prefix = "server")
@Validated  // 开启校验
public class ServerProperties {
    
    @NotNull(message = "timeout 必须配置")
    @Min(value = 1000, message = "timeout 最小 1000ms")
    private Integer timeout;
    
    @NotBlank(message = "host 必须配置")
    private String host;
    
    @Range(min = 1, max = 65535, message = "port 范围 1-65535")
    private int port;
}
```

**启用配置属性：**

```java
@SpringBootApplication
@EnableConfigurationProperties(ServerProperties.class)  // 显式启用
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}
```

**或加 `@ConfigurationPropertiesScan`：**

```java
@SpringBootApplication
@ConfigurationPropertiesScan  // 自动扫描 @ConfigurationProperties
public class Application { }
```

**使用：**

```java
@Service
@RequiredArgsConstructor
public class OrderService {
    
    private final ServerProperties serverProperties;
    
    public void doSomething() {
        int timeout = serverProperties.getTimeout();  // 类型安全
    }
}
```

---

## 🔥 @Value vs @ConfigurationProperties 对比

| 特性 | @Value | @ConfigurationProperties |
|------|--------|-------------------------|
| **类型安全** | ❌ String 需转换 | ✅ 自动类型绑定 |
| **批量注入** | ❌ 一个个写 | ✅ 前缀批量映射 |
| **校验支持** | ❌ 需手动校验 | ✅ @Validated 注解 |
| **松散绑定** | ❌ 不支持 | ✅ `server-timeout` = `serverTimeout` |
| **IDE 提示** | ❌ 无 | ✅ 配置元数据提示 |
| **SpEL 支持** | ✅ 支持 | ❌ 不支持 |
| **适用场景** | 单个简单值 | 结构化配置对象 |

## 🔥 扩展追问

### 追问 1：SpEL 表达式在 @Value 中的妙用

```java
@Component
public class SpelDemo {
    
    // 数学运算
    @Value("#{10 * 60}")  // 600
    private int cacheTtl;
    
    // 引用其他 Bean 的属性
    @Value("#{serverProperties.timeout}")
    private int timeout;
    
    // 条件表达式
    @Value("#{systemProperties['os.name'].contains('Windows') ? 'D:/tmp' : '/tmp'}")
    private String tempDir;
    
    // 列表/Map
    @Value("#{'${app.white-list}'.split(',')}")
    private List<String> whiteList;
    
    @Value("#{${app.rate-limit:{'default':100}}}")
    private Map<String, Integer> rateLimit;
}
```

### 追问 2：配置热刷新怎么实现？

**@Value 不支持热刷新**（除非加 `@RefreshScope`，但有代价）：

```java
@Component
@RefreshScope  // 配置刷新时重建 Bean
public class OrderService {
    @Value("${server.timeout:5000}")
    private int timeout;  // 刷新后生效，但 Bean 重建
}
```

**@ConfigurationProperties 天然支持：**

```java
@Data
@Component
@ConfigurationProperties(prefix = "server")
public class ServerProperties {
    private int timeout;  // 配合 Actuator /refresh 自动刷新
}
```

**注意：** 刷新后已注入的实例不会自动更新，需重新获取或配合 `@RefreshScope`。

### 追问 3：复杂配置怎么写？（List/Map）

```yaml
# application.yml
app:
  endpoints:
    - name: user-service
      url: http://user:8080
      timeout: 3000
    - name: order-service
      url: http://order:8080
      timeout: 5000
  
  feature-flags:
    new-checkout: true
    beta-coupon: false
```

```java
@Data
@Component
@ConfigurationProperties(prefix = "app")
public class AppProperties {
    
    private List<Endpoint> endpoints;
    private Map<String, Boolean> featureFlags;
    
    @Data
    public static class Endpoint {
        private String name;
        private String url;
        private int timeout;
    }
}
```
