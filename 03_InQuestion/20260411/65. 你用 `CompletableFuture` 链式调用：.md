65. 你用 `CompletableFuture` 链式调用：
```java
CompletableFuture.supplyAsync(() -> getUser())
    .thenApply(user -> getOrders(user))
    .thenAccept(orders -> process(orders));
```
**问题：** 如果 `getUser()` 抛出异常，后续操作还会执行吗？如何捕获异常并处理？

---
## 先说结论

`CompletableFuture` 链式调用中，**任一环节抛出异常，后续所有操作都会跳过（短路）**，除非显式使用异常处理方法。捕获异常的核心方法是 `exceptionally()`（兜底值）和 `handle()`（统一处理正常/异常结果）。

---

## 🔥 异常传播机制演示

```java
CompletableFuture.supplyAsync(() -> {
    System.out.println("Step 1: getUser");
    throw new RuntimeException("用户服务挂了");
})
.thenApply(user -> {
    System.out.println("Step 2: getOrders");  // ❌ 不会执行！
    return getOrders(user);
})
.thenAccept(orders -> {
    System.out.println("Step 3: process");    // ❌ 不会执行！
    process(orders);
});

// 输出：
// Step 1: getUser
// （Step 2、3 被跳过，异常被吞掉，你可能永远不知道出错了）
```

**关键问题：异常被吞掉，没有堆栈打印，调试困难！**

---

## 🔥 异常处理三剑客

### 1. `exceptionally()` — 异常时返回兜底值

```java
CompletableFuture.supplyAsync(() -> getUser())
    .exceptionally(ex -> {
        log.error("获取用户失败", ex);
        return User.GUEST;  // 异常时返回默认用户
    })
    .thenApply(user -> getOrders(user))  // ✅ 继续执行，用 GUEST 用户
    .thenAccept(orders -> process(orders));
```

**特点：**
- 只在异常时执行，正常时跳过
- 必须返回与上游相同的类型（`User`）
- 适合有默认值的降级场景

---

### 2. `handle()` — 统一处理正常和异常

```java
CompletableFuture.supplyAsync(() -> getUser())
    .thenApply(user -> getOrders(user))
    .handle((orders, ex) -> {  // 两个参数：正常结果 或 异常
        if (ex != null) {
            log.error("处理失败", ex);
            return Collections.emptyList();  // 异常返回空列表
        }
        return orders;  // 正常直接返回
    })
    .thenAccept(orders -> process(orders));  // 总是执行
```

**特点：**
- 无论正常/异常都会执行
- 可以转换结果类型
- 适合需要统一收尾的场景

---

### 3. `whenComplete()` — 只观察不转换

```java
CompletableFuture.supplyAsync(() -> getUser())
    .thenApply(user -> getOrders(user))
    .whenComplete((orders, ex) -> {  // 纯副作用，不返回值
        if (ex != null) {
            metrics.recordFailure("get_orders");
        } else {
            metrics.recordSuccess("get_orders", orders.size());
        }
    })
    .thenAccept(orders -> process(orders));  // 异常时这里仍会被跳过！
```

**⚠️ 注意：`whenComplete()` 只是观察，不会阻止异常继续传播！**

## 🔥 实战：完整的异常处理链路

```java
public CompletableFuture<OrderResult> processOrder(Long userId) {
    return CompletableFuture.supplyAsync(() -> getUser(userId))
        // 第一层：用户服务异常降级
        .exceptionally(ex -> {
            log.error("用户服务异常，使用缓存用户", ex);
            return userCache.get(userId);
        })
        
        // 第二层：获取订单
        .thenCompose(user -> 
            CompletableFuture.supplyAsync(() -> getOrders(user))
                // 订单服务独立异常处理
                .exceptionally(ex -> {
                    log.error("订单服务异常", ex);
                    return Collections.emptyList();
                })
        )
        
        // 第三层：处理结果
        .thenApply(orders -> calculateTotal(orders))
        
        // 最终统一处理
        .handle((result, ex) -> {
            if (ex != null) {
                // 前面都没兜住的异常
                alertService.send("订单处理完全失败: " + ex.getMessage());
                return OrderResult.error("系统繁忙，请稍后重试");
            }
            return result;
        });
}
```

## 🔥 扩展追问 1：`exceptionally()` 和 `handle()` 的区别

| 方法 | 执行时机 | 能否转换类型 | 是否继续传播异常 |
|---|---|---|---|
| `exceptionally()` | 仅异常时 | ❌ 保持原类型 | ❌ 异常被消化 |
| `handle()` | 正常/异常都执行 | ✅ 可以 | ❌ 异常被消化 |
| `whenComplete()` | 正常/异常都执行 | ❌ 保持原类型 | ✅ 异常继续传播 |

## 🔥 扩展追问 2：异常被吞掉怎么排查？

```java
// 问题代码：异常被吞，没有任何日志
CompletableFuture.runAsync(() -> {
    throw new RuntimeException("出错了");
});
// 程序继续跑，好像什么都没发生

// 修复：设置默认异常处理器
Thread.setDefaultUncaughtExceptionHandler((t, e) -> {
    log.error("未捕获异常 in " + t.getName(), e);
});

// 或者：CompletableFuture 版本
CompletableFuture.runAsync(() -> {
    throw new RuntimeException("出错了");
}).exceptionally(ex -> {
    log.error("任务失败", ex);
    return null;
});
```

## 🔥 扩展追问 3：多个 CF 组合时的异常处理

```java
// allOf：任一失败，整体失败
CompletableFuture<Void> all = CompletableFuture.allOf(cf1, cf2, cf3);
// 如果 cf1 异常，all 也会异常，cf2/cf3 继续跑但结果拿不到

// anyOf：任一成功即可
CompletableFuture<Object> any = CompletableFuture.anyOf(cf1, cf2, cf3);

// 实战：allOf + 单独处理每个异常
List<CompletableFuture<Result>> futures = tasks.stream()
    .map(task -> CompletableFuture.supplyAsync(() -> task.call())
        .exceptionally(ex -> {
            log.error("单个任务失败", ex);
            return Result.empty();  // 失败不影响其他任务
        }))
    .toList();

CompletableFuture.allOf(futures.toArray(new CompletableFuture[0]))
    .thenApply(v -> futures.stream()
        .map(CompletableFuture::join)  // 这里不会抛异常，因为都 exceptionally 了
        .collect(toList())
    );
```

## 🔥 扩展追问 4：超时 + 异常联合处理

```java
CompletableFuture.supplyAsync(() -> getUser())
    .orTimeout(3, TimeUnit.SECONDS)  // 超时抛 TimeoutException
    .exceptionally(ex -> {
        if (ex instanceof TimeoutException) {
            log.warn("获取用户超时");
            return User.CACHE_FALLBACK;
        }
        log.error("获取用户失败", ex);
        return User.GUEST;
    })
    .thenApply(user -> getOrders(user));
```

## 一句话收尾

> `CompletableFuture` 异常会短路后续操作，**必须用 `exceptionally()` 或 `handle()` 显式捕获**，否则异常被静默吞掉；生产环境推荐每层都加 `exceptionally` 兜底，最外层用 `handle` 统一收尾，配合 `orTimeout` 防挂死。
