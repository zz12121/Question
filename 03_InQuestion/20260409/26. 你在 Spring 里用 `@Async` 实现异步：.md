**26. 你在 Spring 里用 `@Async` 实现异步：**

```java
@Service
public class OrderService {
    @Async
    public void sendNotify(Order order) {
        // 发送通知
    }
}
```

**问题：** 默认情况下用的什么线程池？如果 `sendNotify()` 里抛出异常，调用方知道吗？如何配置自定义线程池？

---
## 先说结论

Spring `@Async` 默认使用 `SimpleAsyncTaskExecutor`（**每任务新建线程，无上限！**），生产环境绝对不能用；异步方法异常**调用方感知不到**，需要通过 `AsyncUncaughtExceptionHandler` 或返回 `Future` 来捕获；自定义线程池需实现 `AsyncConfigurer` 或指定 `@Async("beanName")`。

---

## 🔥 核心概念

**解决什么问题：**
- 避免同步调用阻塞主线程，提升接口响应速度
- 解耦非核心逻辑（如发送通知、记录日志）

**基本原理：**
- Spring AOP 拦截 `@Async` 方法，将调用提交到线程池异步执行
- 默认线程池是 `SimpleAsyncTaskExecutor`，每次创建新线程，无复用、无上限

---

## 🔥 详细展开

### 1. 默认线程池的问题

```java
// Spring 默认实现
public class SimpleAsyncTaskExecutor implements AsyncTaskExecutor {
    public void execute(Runnable task) {
        // 每次 new Thread()，没有线程复用！
        Thread thread = new Thread(task);
        thread.start();
    }
}
```

**风险：**
- 高并发下瞬间创建大量线程 → OOM
- 线程无复用，上下文切换开销大
- 无拒绝策略，任务无限堆积

### 2. 异常感知问题

```java
@Service
public class OrderService {
    @Async
    public void sendNotify(Order order) {
        // 异常在这里抛出，但调用方完全不知道！
        throw new RuntimeException("发送失败");
    }
}

// 调用方
orderService.sendNotify(order);  // 立即返回，异常被"吞掉"
```

**为什么感知不到？**
- 异步方法在另一个线程执行
- 调用方拿到的是 `void`，无法获取执行结果
- 异常被 Spring 捕获，默认只打印日志

### 3. 自定义线程池配置

**方式一：实现 AsyncConfigurer（推荐）**

```java
@Configuration
@EnableAsync
public class AsyncConfig implements AsyncConfigurer {
    
    @Override
    @Bean(name = "taskExecutor")
    public Executor getAsyncExecutor() {
        ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
        executor.setCorePoolSize(10);
        executor.setMaxPoolSize(50);
        executor.setQueueCapacity(200);
        executor.setThreadNamePrefix("async-");
        executor.setRejectedExecutionHandler(new ThreadPoolExecutor.CallerRunsPolicy());
        executor.setWaitForTasksToCompleteOnShutdown(true);
        executor.setAwaitTerminationSeconds(60);
        executor.initialize();
        return executor;
    }
    
    @Override
    public AsyncUncaughtExceptionHandler getAsyncUncaughtExceptionHandler() {
        return (ex, method, params) -> {
            // 统一处理异步方法异常
            log.error("Async method {} failed: {}", method.getName(), ex.getMessage());
            // 可以发送告警、记录到数据库等
        };
    }
}
```

**方式二：指定特定线程池**

```java
@Configuration
public class AsyncConfig {
    
    @Bean("notifyExecutor")
    public Executor notifyExecutor() {
        ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
        executor.setCorePoolSize(5);
        executor.setMaxPoolSize(20);
        executor.setQueueCapacity(100);
        executor.setThreadNamePrefix("notify-");
        executor.initialize();
        return executor;
    }
}

@Service
public class OrderService {
    // 指定使用 notifyExecutor
    @Async("notifyExecutor")
    public void sendNotify(Order order) {
        // ...
    }
}
```

### 4. 让调用方感知异常

**方案 A：返回 Future**

```java
@Async
public Future<String> sendNotify(Order order) {
    try {
        // 发送逻辑
        return new AsyncResult<>("success");
    } catch (Exception e) {
        return new AsyncResult<>(e.getMessage());  // 异常信息包装返回
    }
}

// 调用方
Future<String> future = orderService.sendNotify(order);
try {
    String result = future.get(5, TimeUnit.SECONDS);  // 阻塞等待结果
} catch (ExecutionException e) {
    // 捕获异步执行异常
    log.error("Notify failed", e.getCause());
}
```

**方案 B：CompletableFuture（更现代）**

```java
@Async
public CompletableFuture<Void> sendNotify(Order order) {
    return CompletableFuture.runAsync(() -> {
        // 发送逻辑
    }, CompletableFuture.delayedExecutor(0, TimeUnit.MILLISECONDS));
}

// 调用方
orderService.sendNotify(order)
    .exceptionally(ex -> {
        log.error("Failed", ex);
        return null;
    });
```

## 🔥 扩展追问 1：同一个类内部调用 `@Async` 方法为什么不生效？

```java
@Service
public class OrderService {
    public void createOrder() {
        // 内部调用，@Async 不生效！
        this.sendNotify(order);
    }
    
    @Async
    public void sendNotify(Order order) { }
}
```

**原因：** Spring AOP 基于代理实现，内部调用 `this.sendNotify()` 不走代理，直接调用目标方法。

**解决：**
```java
// 方式1：注入自身代理
@Autowired
private OrderService self;

public void createOrder() {
    self.sendNotify(order);  // 走代理
}

// 方式2：使用 AopContext
@EnableAspectJAutoProxy(exposeProxy = true)
public class OrderService {
    public void createOrder() {
        ((OrderService) AopContext.currentProxy()).sendNotify(order);
    }
}
```

## 🔥 扩展追问 2：`@Async` 方法的事务怎么保证？

**问题：** 异步方法里抛异常，主事务会回滚吗？

```java
@Transactional
public void createOrder() {
    orderMapper.insert(order);
    sendNotify(order);  // 异步发送
}

@Async
public void sendNotify(Order order) {
    // 这里抛异常，主事务不会回滚！
    throw new RuntimeException();
}
```

**答案：不会回滚。** 因为 `@Async` 方法在另一个线程执行，与主事务不在同一个事务上下文。

**正确做法：**
```java
@Transactional
public void createOrder() {
    orderMapper.insert(order);
    
    // 先提交事务，再异步
    TransactionSynchronizationManager.registerSynchronization(
        new TransactionSynchronizationAdapter() {
            @Override
            public void afterCommit() {
                // 事务提交后才发送通知
                asyncService.sendNotify(order);
            }
        }
    );
}
```

## 🔥 扩展追问 3：如何优雅关闭线程池，避免任务丢失？

```java
@Bean
public Executor taskExecutor() {
    ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
    // ... 配置
    
    // 优雅关闭关键配置
    executor.setWaitForTasksToCompleteOnShutdown(true);  // 等待任务完成
    executor.setAwaitTerminationSeconds(60);             // 最长等待60秒
    executor.setPrestartAllCoreThreads(true);            // 预启动核心线程
    
    return executor;
}
```

**配合 Spring 生命周期：**
```java
@Component
public class GracefulShutdown implements ApplicationListener<ContextClosedEvent> {
    
    @Autowired
    private ThreadPoolTaskExecutor taskExecutor;
    
    @Override
    public void onApplicationEvent(ContextClosedEvent event) {
        taskExecutor.setAwaitTerminationMillis(30000);
        taskExecutor.shutdown();
    }
}
```

## 一句话收尾

`@Async` 默认线程池是"线程炸弹"，生产环境必须自定义；异常感知要配合 `Future` 或 `AsyncUncaughtExceptionHandler`；同类内部调用不生效是 AOP 代理机制决定的，面试常考！
