**25. 你用 Redis Pipeline 批量操作：**

```java
Pipeline pipeline = jedis.pipelined();
for (String key : keys) {
    pipeline.get(key);
}
List<Object> results = pipeline.syncAndReturnAll();
```

**问题：** Pipeline 和事务（MULTI/EXEC）有什么区别？如果其中某个命令失败了，其他命令会执行吗？
# Redis Pipeline 与事务——区别与失败处理

## 先说结论

Pipeline 是**批量发送命令减少 RTT**，不保证原子性，某个命令失败不影响其他命令执行。事务（MULTI/EXEC）是**原子性执行**，但无回滚能力，语法错误会全部失败，运行时错误不影响其他命令。两者都不支持跨命令依赖。

---

## 🔥 核心概念——Pipeline vs 事务

```
┌─────────────────────────────────────────────────────────────┐
│  Pipeline（管道）                                            │
│  - 目的：减少网络往返（RTT），批量发送命令                     │
│  - 原理：客户端缓冲命令，一次性发送，服务端依次执行             │
│  - 原子性：❌ 无，每个命令独立执行                             │
│  - 失败处理：某个命令失败，其他命令继续执行                     │
│  - 适用：批量读写，无依赖关系                                  │
├─────────────────────────────────────────────────────────────┤
│  事务（MULTI/EXEC）                                          │
│  - 目的：保证一组命令原子性执行                                 │
│  - 原理：MULTI 开启事务，命令入队，EXEC 原子执行                │
│  - 原子性：✅ 是，但无回滚（不是关系型事务）                    │
│  - 失败处理：                                                 │
│     - 入队错误（语法错误）：EXEC 失败，全部不执行               │
│     - 执行错误（运行时）：其他命令继续执行                      │
│  - 适用：需要原子性的简单操作                                  │
└─────────────────────────────────────────────────────────────┘
```

---

## 🔥 详细展开

### 问题一：Pipeline 失败处理

```java
Pipeline pipeline = jedis.pipelined();
pipeline.set("key1", "value1");
pipeline.hget("key2", "field");  // key2 不是 hash，会失败
pipeline.get("key3");

List<Object> results = pipeline.syncAndReturnAll();
// results: [OK, JedisDataException, value3]
// 结果：
//   - key1 设置成功
//   - key2 报错（类型错误），但 key3 仍然执行
```

**特点：**
- 命令按顺序执行
- 失败命令返回异常，不影响后续命令
- 无原子性，中间失败不会回滚已执行命令

### 问题二：事务失败处理

```java
// 场景1：入队错误（语法错误）
jedis.multi();
jedis.set("key1", "value1");
jedis.lpush("key2");  // 语法错误，参数不足
jedis.get("key3");

List<Object> results = jedis.exec();
// 结果：null，全部不执行（EXEC 失败）

// 场景2：运行时错误
jedis.multi();
jedis.set("key1", "value1");
jedis.hget("key2", "field");  // key2 是 string，不是 hash
jedis.get("key3");

List<Object> results = jedis.exec();
// 结果：[OK, JedisDataException, value3]
// key1 成功，key2 失败，key3 成功
// 无回滚！
```

## 🔥 扩展追问 1：Lua 脚本实现真正原子性

```java
// Redis 保证 Lua 脚本原子执行（单线程）
// 且支持跨命令逻辑

String lua = 
    "local val = redis.call('get', KEYS[1]) " +
    "if val == ARGV[1] then " +
    "  redis.call('set', KEYS[1], ARGV[2]) " +
    "  return 1 " +
    "else " +
    "  return 0 " +
    "end";

Long result = jedis.eval(lua, 
    Collections.singletonList("key1"), 
    Arrays.asList("oldValue", "newValue"));
// CAS 操作：只有当前值是 oldValue 时才更新
```

**优势：**
- 真正原子性（单线程执行）
- 支持跨命令逻辑判断
- 减少网络往返

## 🔥 扩展追问 2：Pipeline 事务组合

```java
// Pipeline + 事务：批量原子操作
Pipeline pipeline = jedis.pipelined();
for (String userId : userIds) {
    pipeline.multi();                    // 每个用户一个事务
    pipeline.hincrBy("user:" + userId, "score", 10);
    pipeline.expire("user:" + userId, 3600);
    pipeline.exec();
}
pipeline.sync();
```

**注意：**
- 事务在 Pipeline 中发送，减少 RTT
- 每个事务独立，互不影响

## 🔥 扩展追问 3：生产环境最佳实践

```java
@Component
public class RedisBatchService {
    
    @Autowired
    private JedisPool jedisPool;
    
    // 批量查询（Pipeline）
    public List<String> batchGet(List<String> keys) {
        try (Jedis jedis = jedisPool.getResource()) {
            Pipeline pipeline = jedis.pipelined();
            for (String key : keys) {
                pipeline.get(key);
            }
            return pipeline.syncAndReturnAll().stream()
                .map(Object::toString)
                .collect(Collectors.toList());
        }
    }
    
    // 原子性操作（Lua）
    public boolean atomicUpdate(String key, String expect, String update) {
        String lua = 
            "if redis.call('get', KEYS[1]) == ARGV[1] then " +
            "  redis.call('set', KEYS[1], ARGV[2]) " +
            "  return 1 else return 0 end";
        
        try (Jedis jedis = jedisPool.getResource()) {
            Long result = (Long) jedis.eval(lua, 
                Collections.singletonList(key), 
                Arrays.asList(expect, update));
            return result == 1;
        }
    }
}
```

## 🔥 扩展追问 4：对比总结

| 特性 | Pipeline | MULTI/EXEC | Lua 脚本 |
|------|----------|------------|----------|
| 目的 | 减少 RTT | 原子性 | 原子性+逻辑 |
| 原子性 | ❌ | ✅ | ✅ |
| 回滚 | ❌ | ❌ | ❌ |
| 跨命令依赖 | ❌ | ❌ | ✅ |
| 性能 | 高 | 中 | 高 |
| 复杂度 | 低 | 低 | 中 |
| 适用 | 批量读写 | 简单原子操作 | 复杂原子操作 |

---

## 一句话收尾

> Pipeline 是**批量提速**（无原子性），事务是**简单原子**（无回滚），复杂原子操作用 **Lua 脚本**，三者根据场景选择，核心原则是**能批量不单个，能原子不拆分**。
