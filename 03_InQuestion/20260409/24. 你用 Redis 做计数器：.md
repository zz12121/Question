**24. 你用 Redis 做计数器：**

```java
jedis.incr("counter:product:100");
```

**问题：** 如果要求"每天 0 点清零"，怎么实现？如果 Redis 挂了，计数会丢失吗？如何保证最终准确？
## 先说结论

`INCR` 计数器默认永不过期，需用**过期时间+预清零**或**滑动窗口**实现每天清零；Redis 挂了计数会丢失，需**异步持久化到数据库**或**双写**保证最终准确。

---

## 🔥 核心概念——当前代码的问题

```java
jedis.incr("counter:product:100");
// 问题：
// 1. key 永不过期，计数一直累加，不会每天清零
// 2. Redis 挂了，内存数据丢失，计数归零
// 3. 无持久化机制，无法恢复历史计数
```

## 🔥 详细展开

### 问题一：每天清零的实现

```java
// ❌ 错误方案：定时任务遍历所有key清零
// 问题：key太多（百万级），遍历耗时，阻塞Redis

// ✅ 正确方案1：带日期的key，自然过期
public void incrCounter(String productId) {
    String date = LocalDate.now().format(DateTimeFormatter.BASIC_ISO_DATE);
    String key = "counter:" + productId + ":" + date;  // counter:100:20260411
    jedis.incr(key);
    jedis.expire(key, 2 * 24 * 3600);  // 48小时后自动删除
}

// 查询今日计数
public String getTodayCounter(String productId) {
    String date = LocalDate.now().format(DateTimeFormatter.BASIC_ISO_DATE);
    return jedis.get("counter:" + productId + ":" + date);
}
```

### 问题二：Redis 挂了数据丢失

```
场景：
  - 今日计数已累加到 10000
  - Redis 挂了，重启
  - 计数归零，从 0 开始
  - 数据丢失，业务异常（如库存超卖）
```

### 更优方案：滑动窗口计数

```java
@Component
public class SlidingWindowCounter {
    
    @Autowired
    private StringRedisTemplate redis;
    
    // 滑动窗口计数（每小时一个slot，保留24小时）
    public void incr(String productId) {
        long now = System.currentTimeMillis();
        long hourSlot = now / (3600 * 1000);  // 当前小时
        
        String key = "sw:counter:" + productId;
        redis.opsForZSet().incrementScore(key, String.valueOf(hourSlot), 1);
        redis.expire(key, 25 * 3600);  // 25小时过期
    }
    
    // 查询近24小时计数
    public long get24hCount(String productId) {
        long now = System.currentTimeMillis();
        long currentHour = now / (3600 * 1000);
        
        String key = "sw:counter:" + productId;
        // 查询24个slot的和
        return redis.opsForZSet().range(key, currentHour - 23, currentHour)
            .stream()
            .mapToLong(Long::parseLong)
            .sum();
    }
}
```

## 🔥 扩展追问 2：Redis+数据库双写保证最终一致

```java
@Service
public class ReliableCounter {
    
    @Autowired
    private StringRedisTemplate redis;
    
    @Autowired
    private CounterMapper counterMapper;  // 数据库
    
    // 双写：Redis + 异步落库
    public void incr(String productId) {
        String date = LocalDate.now().toString();
        String key = "counter:" + productId + ":" + date;
        
        // 1. Redis 快速计数
        long count = redis.opsForValue().increment(key);
        redis.expire(key, 2, TimeUnit.DAYS);
        
        // 2. 异步落库（MQ或线程池）
        asyncSaveToDB(productId, date, count);
    }
    
    @Async
    public void asyncSaveToDB(String productId, String date, long count) {
        // 幂等更新：UPSERT
        counterMapper.upsert(productId, date, count);
    }
    
    // 查询：优先Redis，失败查DB
    public long get(String productId) {
        String date = LocalDate.now().toString();
        String key = "counter:" + productId + ":" + date;
        
        String value = redis.opsForValue().get(key);
        if (value != null) {
            return Long.parseLong(value);
        }
        
        // Redis挂了，查数据库兜底
        return counterMapper.select(productId, date);
    }
}
```

## 🔥 扩展追问 3：Redis 持久化配置

```bash
# RDB 快照（默认）
save 900 1      # 900秒内1次修改则快照
save 300 10     # 300秒内10次修改则快照
save 60 10000   # 60秒内10000次修改则快照

# AOF 持久化（更可靠，推荐）
appendonly yes
appendfsync everysec  # 每秒刷盘，性能与可靠性平衡

# 混合持久化（Redis 4.0+）
aof-use-rdb-preamble yes  # RDB+AOF混合，重启快且数据完整
```

**注意：** 即使开启 AOF，也可能丢失 1 秒数据，关键计数必须双写数据库。

## 🔥 扩展追问 4：高并发计数优化

```java
// 方案：本地缓存 + Redis 批量合并
@Component
public class BatchCounter {
    
    private final ConcurrentHashMap<String, AtomicLong> localCache = 
        new ConcurrentHashMap<>();
    
    @Scheduled(fixedRate = 1000)  // 每秒批量同步
    public void syncToRedis() {
        localCache.forEach((key, count) -> {
            long value = count.getAndSet(0);  // 清零并获取
            if (value > 0) {
                redis.opsForValue().increment(key, value);
            }
        });
    }
    
    public void incr(String productId) {
        localCache.computeIfAbsent(productId, k -> new AtomicLong(0))
                  .incrementAndGet();
    }
}
```

## 一句话收尾

> 每天清零用**日期后缀key自然过期**或**滑动窗口**，Redis挂了用**双写数据库**兜底，高并发用**本地缓存批量合并**，核心原则是**快速响应+异步持久化+数据库兜底**。
