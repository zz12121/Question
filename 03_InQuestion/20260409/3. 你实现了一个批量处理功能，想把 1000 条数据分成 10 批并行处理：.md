### 1. 问题一：资源泄漏（最严重）

```java

    new Thread(() -> process(batch)).start();  // 线程创建后不回收
```

**隐患**：
- 如果这个方法被频繁调用（如每秒 100 次），瞬间创建 1000 个线程

**正确做法：使用线程池**

```java
// 定义线程池（复用线程）
private final ExecutorService executor = Executors.newFixedThreadPool(10);

public void batchProcess(List<Data> dataList) {
    List<Future<?>> futures = new ArrayList<>();
    
    for (int i = 0; i < 10; i++) {
        List<Data> batch = dataList.subList(i * 100, (i + 1) * 100);
        Future<?> future = executor.submit(() -> process(batch));
        futures.add(future);
    }
    
    // 等待所有任务完成
    for (Future<?> f : futures) {
        f.get();  // 阻塞等待
    }
}
```

### 4. 完整优化方案

#### 方案一：CompletableFuture（推荐）

```java
@Service
public class BatchProcessor {
    
    private final ExecutorService executor = Executors.newFixedThreadPool(8);
    
    public void processBatch(List<Data> dataList) {
        // 1. 分批（创建副本）
        List<List<Data>> batches = Lists.partition(dataList, 100);  // Guava
        
        // 2. 并行处理
        List<CompletableFuture<Void>> futures = batches.stream()
            .map(batch -> CompletableFuture.runAsync(
                () -> process(batch), 
                executor
            ))
            .collect(Collectors.toList());
        
        // 3. 等待全部完成（带超时）
        CompletableFuture.allOf(futures.toArray(new CompletableFuture[0]))
            .orTimeout(30, TimeUnit.SECONDS)  // 超时控制
            .join();
    }
    
    private void process(List<Data> batch) {
        // 业务处理
    }
}
```

**优点**：
- 线程池复用，资源可控
- 支持超时、异常处理、结果组合
- 代码简洁，函数式风格

### 2. 正确实现

#### 方案一：线程池 + CountDownLatch（基础版）

```java
public void processAll(List<Data> dataList, int batchSize) {
    int totalBatches = (dataList.size() + batchSize - 1) / batchSize;
    ExecutorService executor = Executors.newFixedThreadPool(10);
    CountDownLatch latch = new CountDownLatch(totalBatches);
    
    List<Throwable> errors = Collections.synchronizedList(new ArrayList<>());
    
    for (int i = 0; i < totalBatches; i++) {
        final int start = i * batchSize;
        final int end = Math.min(start + batchSize, dataList.size());
        
        // 🔑 copy 一份，防止 subList 视图问题
        final List<Data> batch = new ArrayList<>(dataList.subList(start, end));
        
        executor.submit(() -> {
            try {
                process(batch);
            } catch (Exception e) {
                errors.add(e);  // 收集异常
            } finally {
                latch.countDown();
            }
        });
    }
    
    try {
        // 等待全部完成，超时控制
        boolean completed = latch.await(5, TimeUnit.MINUTES);
        if (!completed) {
            throw new RuntimeException("批量处理超时");
        }
    } catch (InterruptedException e) {
        Thread.currentThread().interrupt();
        throw new RuntimeException("批量处理被中断");
    }
    
    // 🔑 检查异常
    if (!errors.isEmpty()) {
        log.error("处理失败 {} 条", errors.size());
        throw new RuntimeException("部分批次处理失败", errors.get(0));
    }
}
```

#### 方案二：分段并行 + 结果聚合（生产版）

```java
public BatchResult processAll(List<Data> dataList) {
    int parallelism = Math.min(10, Runtime.getRuntime().availableProcessors() * 2);
    
    // 🔑 并行流，分片处理
    List<List<Data>> batches = Lists.partition(dataList, 100);
    
    List<CompletableFuture<BatchResult>> futures = batches.stream()
        .map(batch -> CompletableFuture.supplyAsync(() -> {
            BatchResult result = new BatchResult();
            for (Data d : batch) {
                try {
                    processOne(d);
                    result.successCount++;
                } catch (Exception e) {
                    result.failCount++;
                    result.errors.add(new ErrorRecord(d.getId(), e));
                }
            }
            return result;
        }, executor))
        .collect(Collectors.toList());
    
    // 🔑 聚合所有批次结果
    BatchResult total = new BatchResult();
    for (CompletableFuture<BatchResult> f : futures) {
        BatchResult r = f.join();
        total.successCount += r.successCount;
        total.failCount += r.failCount;
        total.errors.addAll(r.errors);
    }
    
    return total;
}

public class BatchResult {
    public int successCount = 0;
    public int failCount = 0;
    public List<ErrorRecord> errors = new ArrayList<>();
}
```

## 🔥 扩展追问 1：线程池大小怎么定？

**CPU 密集型**：
```
线程数 = CPU 核心数 + 1
```

**IO 密集型**（数据库、网络、文件）：
```
线程数 = CPU 核心数 × (1 + 等待时间 / 计算时间)
≈ CPU 核心数 × 2 ~ 10
```

**计算公式**：
```java
// N = CPU 核心数，U = 目标 CPU 利用率，W/C = 等待时间/计算时间
线程数 = N × U × (1 + W/C)

例：8 核，目标是 80% 利用率，等待 100ms，计算 20ms
线程数 = 8 × 0.8 × (1 + 100/20) = 8 × 0.8 × 6 = 38.4 ≈ 40
```
