**22. 你用 Redis 缓存热点数据：**

```java
String value = redis.get(key);
if (value == null) {
    value = db.query(key);
    redis.setex(key, 3600, value);
}
return value;
```

**问题：** 缓存击穿、穿透、雪崩分别是什么场景？这段代码存在哪些问题？如何改进？
## 先说结论

这段代码存在**缓存击穿**（热点key过期瞬间并发查DB）、**缓存穿透**（查询不存在key直接打DB）、**缓存雪崩**（大量key同时过期）风险。改进方案：**互斥锁防击穿**、**布隆过滤器/空值防穿透**、**随机过期时间防雪崩**。

## 🔥 核心概念——缓存三问题

```
┌─────────────────────────────────────────────────────────────┐
│  缓存击穿（Cache Breakdown）                                  │
│  场景：热点key过期瞬间，大量请求同时查DB                       │
│  后果：DB压力剧增，可能打挂                                   │
│  这段代码：✅ 存在（无锁保护）                                 │
├─────────────────────────────────────────────────────────────┤
│  缓存穿透（Cache Penetration）                                │
│  场景：查询不存在的key，每次都穿透到DB                         │
│  后果：DB无效查询，攻击者可恶意构造不存在的key                  │
│  这段代码：✅ 存在（直接查DB）                                 │
├─────────────────────────────────────────────────────────────┤
│  缓存雪崩（Cache Avalanche）                                  │
│  场景：大量key同时过期，或Redis挂了                            │
│  后果：所有请求打DB，DB瞬间崩溃                               │
│  这段代码：✅ 存在（固定3600秒，批量key可能同时过期）           │
└─────────────────────────────────────────────────────────────┘
```

### 问题一：缓存击穿

## 🔥 扩展追问 1：互斥锁防击穿

```java
public String getWithLock(String key) {
    String value = redis.get(key);
    if (value != null) {
        return value;
    }
    
    // 获取分布式锁，只有一个线程查DB
    String lockKey = "lock:" + key;
    String requestId = UUID.randomUUID().toString();
    
    try {
        Boolean locked = redis.opsForValue()
            .setIfAbsent(lockKey, requestId, 10, TimeUnit.SECONDS);
        
        if (Boolean.TRUE.equals(locked)) {
            // 双重检查
            value = redis.get(key);
            if (value == null) {
                value = db.query(key);
                if (value != null) {
                    redis.setex(key, 3600, value);
                }
            }
            return value;
        } else {
            // 没拿到锁，短暂等待后重试
            Thread.sleep(100);
            return getWithLock(key);  // 递归或循环重试
        }
    } catch (InterruptedException e) {
        Thread.currentThread().interrupt();
        throw new RuntimeException(e);
    } finally {
        // 释放锁
        unlock(lockKey, requestId);
    }
}
```

### Redisson 实现（更优雅）

RBucket<String> bucket = redisson.getBucket(key);
String value = bucket.get();
if (value == null) {
    // trySet 原子操作，只有一个线程成功
    RLock lock = redisson.getLock("lock:" + key);
    try {
        lock.lock();
        // 双重检查
        value = bucket.get();
        if (value == null) {
            value = db.query(key);
            bucket.set(value, 3600, TimeUnit.SECONDS);
        }
    } finally {
        lock.unlock();
    }
}

## 🔥 扩展追问 2：布隆过滤器防穿透

```java
@Component
public class BloomFilterCache {
    
    @Autowired
    private RedissonClient redisson;
    
    private RBloomFilter<String> bloomFilter;
    
    @PostConstruct
    public void init() {
        bloomFilter = redisson.getBloomFilter("cache:bloom");
        // 预期100万个元素，误判率0.01
        bloomFilter.tryInit(1000000L, 0.01);
        
        // 预热：加载所有有效key
        loadAllKeys().forEach(bloomFilter::add);
    }
    
    public String get(String key) {
        // 1. 布隆过滤器判断
        if (!bloomFilter.contains(key)) {
            return null;  // 一定不存在，直接返回
        }
        
        // 2. 查缓存
        String value = redis.get(key);
        if (value != null) {
            return value;
        }
        
        // 3. 查DB（布隆过滤器可能有误判，但概率低）
        value = db.query(key);
        if (value != null) {
            redis.setex(key, 3600, value);
        } else {
            // 缓存空值，防穿透
            redis.setex(key, 60, "NULL");  // 短过期时间
        }
        return value;
    }
}
```

public void setWithRandomExpire(String key, String value) {

### 多级缓存防雪崩

```java
@Component
public class MultiLevelCache {
    
    private final Caffeine<String, String> localCache = Caffeine.newBuilder()
        .maximumSize(10000)
        .expireAfterWrite(10, TimeUnit.MINUTES)  // 本地缓存10分钟
        .build();
    
    @Autowired
    private StringRedisTemplate redis;
    
    public String get(String key) {
        // 1. 查本地缓存
        String value = localCache.getIfPresent(key);
        if (value != null) {
            return value;
        }
        
        // 2. 查Redis
        value = redis.opsForValue().get(key);
        if (value != null) {
            localCache.put(key, value);  // 回填本地缓存
            return value;
        }
        
        // 3. 查DB（带锁防击穿）
        value = getFromDBWithLock(key);
        if (value != null) {
            redis.opsForValue().set(key, value, 
                3600 + ThreadLocalRandom.current().nextInt(300), 
                TimeUnit.SECONDS);
            localCache.put(key, value);
        }
        return value;
    }
}
```

## 🔥 扩展追问 4：Redis 挂了怎么办？

```java
// 熔断降级：Redis 不可用时，直接查DB或返回默认值
@Component
public class CacheService {
    
    @Autowired
    private CircuitBreakerRegistry circuitBreakerRegistry;
    
    public String get(String key) {
        CircuitBreaker cb = circuitBreakerRegistry.circuitBreaker("redis");
        
        return cb.executeSupplier(() -> {
            // 尝试从Redis获取
            return redis.get(key);
        }, throwable -> {
            // 熔断时降级
            log.warn("Redis不可用，降级查DB: {}", key);
            return db.query(key);
        });
    }
}
```
