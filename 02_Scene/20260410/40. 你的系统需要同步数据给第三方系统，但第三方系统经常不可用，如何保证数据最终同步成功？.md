## 先说结论

第三方系统不稳定时的数据同步，核心方案是**本地消息表 + 重试机制 + 死信队列**：数据变更先持久化到本地消息表，通过定时任务或 MQ 异步发送，失败时按指数退避重试，最终失败入死信队列人工介入，确保数据最终一致。

## 🔥 详细展开：核心实现

### 1. 同步任务表设计

```sql
CREATE TABLE sync_task (
    id              BIGINT          NOT NULL AUTO_INCREMENT,
    biz_id          VARCHAR(64)     NOT NULL COMMENT '业务单号',
    biz_type        VARCHAR(32)     NOT NULL COMMENT '业务类型：ORDER/PAYMENT',
    target_system   VARCHAR(32)     NOT NULL COMMENT '目标系统：ERP/WMS/PAY',
    target_url      VARCHAR(512)    NOT NULL COMMENT '目标URL',
    payload         TEXT            NOT NULL COMMENT '请求体JSON',
    
    -- 状态流转：PENDING -> PROCESSING -> SUCCESS/FAILED/DEAD
    status          VARCHAR(16)     NOT NULL DEFAULT 'PENDING',
    
    -- 重试控制
    retry_count     INT             NOT NULL DEFAULT 0,
    max_retry       INT             NOT NULL DEFAULT 5,
    next_retry_time DATETIME        COMMENT '下次重试时间',
    
    -- 执行记录
    last_error      VARCHAR(1024)   COMMENT '上次错误信息',
    response_body   TEXT            COMMENT '第三方响应',
    
    -- 时间戳
    create_time     DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP,
    update_time     DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    PRIMARY KEY (id),
    UNIQUE KEY uk_biz_target (biz_id, target_system),  -- 同一业务同一目标只同步一次
    KEY idx_status_retry (status, next_retry_time),     -- 定时扫描索引
    KEY idx_create_time (create_time)
) ENGINE=InnoDB COMMENT='第三方同步任务表';
```

```java
@Service
public class OrderService {
    
    @Autowired
    private OrderMapper orderMapper;
    
    @Autowired
    private SyncTaskMapper syncTaskMapper;
    
    /**
     * 支付成功：更新订单 + 记录同步任务（同一事务）
     */
    @Transactional
    public void onPaymentSuccess(String orderNo) {
        // 1. 更新订单状态
        orderMapper.updateStatus(orderNo, OrderStatus.PAID);
        
        // 2. 查询订单详情
        Order order = orderMapper.findByOrderNo(orderNo);
        
        // 3. 写入 ERP 同步任务
        syncTaskMapper.insert(SyncTask.builder()
            .bizId(orderNo)
            .bizType("ORDER")
            .targetSystem("ERP")
            .targetUrl("https://erp.example.com/api/orders")
            .payload(JSON.toJSONString(buildErpPayload(order)))
            .status("PENDING")
            .maxRetry(5)
            .nextRetryTime(LocalDateTime.now())
            .build());
        
        // 4. 写入物流同步任务
        syncTaskMapper.insert(SyncTask.builder()
            .bizId(orderNo)
            .bizType("ORDER")
            .targetSystem("WMS")
            .targetUrl("https://wms.example.com/api/shipments")
            .payload(JSON.toJSONString(buildWmsPayload(order)))
            .status("PENDING")
            .maxRetry(5)
            .nextRetryTime(LocalDateTime.now())
            .build());
        
        // 事务提交后，两个同步任务都持久化了
    }
}
```

### 3. 定时扫描 + 异步发送

```java
@Component
public class SyncTaskScheduler {
    
    @Autowired
    private SyncTaskMapper syncTaskMapper;
    
    @Autowired
    private SyncExecutor syncExecutor;
    
    /**
     * 定时扫描待同步任务（1分钟一次）
     */
    @Scheduled(fixedRate = 60000)
    public void scanAndDispatch() {
        // 查询待处理任务（PENDING 或 到达重试时间的 FAILED）
        List<SyncTask> tasks = syncTaskMapper.findPendingTasks(100);
        
        for (SyncTask task : tasks) {
            // 更新为处理中，防止被其他线程取到
            int affected = syncTaskMapper.casUpdateStatus(
                task.getId(), "PENDING", "PROCESSING");
            
            if (affected > 0) {
                // 异步执行，不阻塞定时任务
                syncExecutor.execute(task);
            }
        }
    }
}
```

### 4. 同步执行器（含重试逻辑）

```java
@Component
public class SyncExecutor {
    
    @Autowired
    private SyncTaskMapper syncTaskMapper;
    
    @Autowired
    private RestTemplate restTemplate;
    
    /**
     * 执行同步任务
     */
    @Async("syncTaskExecutor")
    public void execute(SyncTask task) {
        try {
            // 1. 发送 HTTP 请求
            ResponseEntity<String> response = sendRequest(task);
            
            // 2. 处理响应
            if (response.getStatusCode().is2xxSuccessful()) {
                // 成功
                markSuccess(task, response.getBody());
            } else if (isRetryable(response.getStatusCode())) {
                // 可重试错误（5xx, 429, 超时）
                scheduleRetry(task, "HTTP " + response.getStatusCode());
            } else {
                // 不可重试错误（4xx 客户端错误）
                markDeadLetter(task, "Client error: " + response.getStatusCode());
            }
            
        } catch (ResourceAccessException e) {
            // 网络超时，可重试
            scheduleRetry(task, "Network timeout: " + e.getMessage());
        } catch (Exception e) {
            // 其他异常，可重试
            scheduleRetry(task, "Exception: " + e.getMessage());
        }
    }
    
    /**
     * 指数退避重试
     */
    private void scheduleRetry(SyncTask task, String error) {
        int retryCount = task.getRetryCount() + 1;
        
        if (retryCount > task.getMaxRetry()) {
            // 超过最大重试次数，入死信队列
            markDeadLetter(task, "Max retry exceeded: " + error);
            return;
        }
        
        // 指数退避：1min, 2min, 4min, 8min, 16min
        long delayMinutes = (long) Math.pow(2, retryCount - 1);
        LocalDateTime nextRetry = LocalDateTime.now().plusMinutes(delayMinutes);
        
        syncTaskMapper.updateForRetry(task.getId(), retryCount, nextRetry, error);
        
        // 告警：连续失败多次
        if (retryCount >= 3) {
            alertService.send(AlertLevel.P2, 
                String.format("同步任务连续失败%d次: %s -> %s", 
                    retryCount, task.getBizId(), task.getTargetSystem()));
        }
    }
    
    /**
     * 判断错误是否可重试
     */
    private boolean isRetryable(HttpStatusCode status) {
        return status.is5xxServerError() 
            || status.value() == 429  // Too Many Requests
            || status.value() == 408; // Request Timeout
    }
    
    private void markSuccess(SyncTask task, String response) {
        syncTaskMapper.updateStatus(task.getId(), "SUCCESS", response);
    }
    
    private void markDeadLetter(SyncTask task, String error) {
        syncTaskMapper.updateStatus(task.getId(), "DEAD", error);
        
        // 死信告警，需人工介入
        alertService.send(AlertLevel.P1, 
            String.format("同步任务进入死信队列: %s -> %s, error: %s",
                task.getBizId(), task.getTargetSystem(), error));
    }
}
```

## 🔥 扩展追问 1：重试策略对比

| 策略 | 公式 | 适用场景 |
|------|------|---------|
| **固定间隔** | 每 5 分钟重试 | 简单场景 |
| **线性退避** | 间隔 = n × 5 分钟 | 一般场景 |
| **指数退避** | 间隔 = 2ⁿ 分钟 | **推荐**，避免雪崩 |
| **随机抖动** | 间隔 ± 随机值 | 防止惊群效应 |

```java
// 指数退避 + 随机抖动
long baseDelay = (long) Math.pow(2, retryCount - 1);  // 1, 2, 4, 8, 16...
long jitter = ThreadLocalRandom.current().nextLong(baseDelay / 2);  // ±50% 抖动
long delay = baseDelay + jitter;
```

## 🔥 扩展追问 2：第三方限流应对

```java
@Component
public class RateLimitHandler {
    
    // 令牌桶限流：每秒 10 个请求
    private final RateLimiter rateLimiter = RateLimiter.create(10.0);
    
    /**
     * 带限流的发送
     */
    public ResponseEntity<String> sendWithRateLimit(SyncTask task) {
        // 获取令牌，阻塞等待
        rateLimiter.acquire();
        
        return restTemplate.exchange(
            task.getTargetUrl(),
            HttpMethod.POST,
            buildRequest(task),
            String.class
        );
    }
    
    /**
     * 应对 429 限流：自适应退避
     */
    public ResponseEntity<String> sendWithAdaptiveBackoff(SyncTask task) {
        int attempt = 0;
        while (attempt < 3) {
            try {
                ResponseEntity<String> response = sendWithRateLimit(task);
                
                if (response.getStatusCode().value() == 429) {
                    // 获取 Retry-After 头
                    String retryAfter = response.getHeaders()
                        .getFirst("Retry-After");
                    long waitSeconds = retryAfter != null ? 
                        Long.parseLong(retryAfter) : (long) Math.pow(2, attempt);
                    
                    Thread.sleep(waitSeconds * 1000);
                    attempt++;
                    continue;
                }
                
                return response;
                
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
                throw new RuntimeException(e);
            }
        }
        
        throw new RuntimeException("Rate limit exceeded after retries");
    }
}
```

```java
@Component
public class ReconciliationService {
    
    @Autowired
    private SyncTaskMapper syncTaskMapper;
    
    /**
     * 每日对账：检查是否有遗漏的同步
     */
    @Scheduled(cron = "0 0 2 * * ?")  // 每天凌晨2点
    public void dailyReconcile() {
        // 1. 查询昨日业务数据
        List<Order> yesterdayOrders = orderMapper
            .findByCreateTimeBetween(getYesterdayStart(), getYesterdayEnd());
        
        for (Order order : yesterdayOrders) {
            // 2. 检查是否已同步到 ERP
            boolean synced = syncTaskMapper.existsSuccess(
                order.getOrderNo(), "ERP");
            
            if (!synced) {
                // 3. 遗漏，补录同步任务
                log.error("对账发现遗漏同步: {}", order.getOrderNo());
                createSyncTask(order, "ERP");
                
                // 4. 告警
                alertService.send(AlertLevel.P1, 
                    "对账发现遗漏同步: " + order.getOrderNo());
            }
        }
    }
}
```

## 一句话收尾

第三方同步不可靠场景的核心解法是**"落库为先，异步重试"**：数据变更时**同步写入任务表**保证不丢，通过**指数退避重试**应对临时故障，**死信队列+告警**处理最终失败，配合**每日对账**兜底，确保数据最终一致送达。
