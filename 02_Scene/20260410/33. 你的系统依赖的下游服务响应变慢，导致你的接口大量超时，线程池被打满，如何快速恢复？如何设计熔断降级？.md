## 先说结论

下游响应变慢导致线程池打满，本质是**级联故障（雪崩）**。快速恢复靠**熔断（Circuit Breaker）**：检测到失败率超阈值自动切断对下游的调用，返回降级结果，保住本系统可用性；等下游恢复后半开放探测，自动恢复。

```
│                                                             │
```

### 2. Sentinel 熔断降级实现

```java
// ============ application.yml 配置 ============
spring:
  cloud:
    sentinel:
      transport:
        dashboard: localhost:8080
      datasource:
        # 动态推送规则（生产必须）
        ds1:
          nacos:
            server-addr: localhost:8848
            data-id: ${spring.application.name}-sentinel-rules
            rule-type: degrade

// ============ 熔断规则定义 ============
@Configuration
public class SentinelConfig {
    
    @PostConstruct
    public void initDegradeRules() {
        List<DegradeRule> rules = new ArrayList<>();
        
        DegradeRule rule = new DegradeRule();
        rule.setResource("userService#getUser");  // 资源名
        
        // 策略1：慢调用比例熔断
        rule.setGrade(RuleConstant.DEGRADE_GRADE_RT);
        rule.setCount(500);        // RT > 500ms 认为是慢调用
        rule.setSlowRatioThreshold(0.5);  // 慢调用比例 > 50% 触发熔断
        rule.setStatIntervalMs(10000);    // 统计窗口 10s
        rule.setMinRequestAmount(5);      // 最少5个请求才统计
        rule.setTimeWindow(10);           // 熔断持续 10s
        
        // 策略2：异常比例熔断（二选一）
        // rule.setGrade(RuleConstant.DEGRADE_GRADE_EXCEPTION_RATIO);
        // rule.setCount(0.5);  // 异常比例 > 50% 触发
        
        // 策略3：异常数熔断（二选一）
        // rule.setGrade(RuleConstant.DEGRADE_GRADE_EXCEPTION_COUNT);
        // rule.setCount(10);  // 异常数 > 10 触发
        
        rules.add(rule);
        DegradeRuleManager.loadRules(rules);
    }
}
```

### 3. 业务接口熔断降级

```java
// ============ 方式1：注解方式（简洁） ============
@Service
public class UserService {
    
    @Autowired
    private UserClient userClient;  // Feign Client
    
    // @SentinelResource：资源名 + 降级方法
    @SentinelResource(
        value = "userService#getUser",
        blockHandler = "getUser_blocked",    // 被限流/熔断时调用
        fallback = "getUser_fallback"        // 业务异常时调用
    )
    public UserDTO getUser(Long userId) {
        return userClient.getUser(userId);
    }
    
    // 熔断/限流降级（BlockException）
    public UserDTO getUser_blocked(Long userId, BlockException ex) {
        log.warn("getUser 被熔断, userId={}, rule={}", userId, ex.getRule());
        // 返回缓存数据或默认值
        return getUserFromCache(userId);
    }
    
    // 业务异常降级（Throwable）
    public UserDTO getUser_fallback(Long userId, Throwable t) {
        log.error("getUser 异常降级, userId={}", userId, t);
        return UserDTO.defaultUser(userId);
    }
    
    private UserDTO getUserFromCache(Long userId) {
        // 先查本地缓存
        UserDTO cached = localCache.get(userId);
        if (cached != null) return cached;
        
        // 再查 Redis
        String json = redisTemplate.opsForValue().get("user:" + userId);
        if (StringUtils.isNotBlank(json)) {
            return JSON.parseObject(json, UserDTO.class);
        }
        
        // 兜底返回默认用户（保证接口不报错）
        return UserDTO.anonymous();
    }
}

// ============ 方式2：编程式（灵活） ============
@Service
public class OrderService {
    
    public OrderDetailVO getOrderDetail(Long orderId) {
        // 查订单（核心，不降级）
        Order order = orderMapper.findById(orderId);

            .then(() -> userClient.getUser(order.getUserId()))
            .onException(e -> UserDTO.anonymous())  // 降级
            .run();
        
        // 查物流信息（可降级）

    }
}
```

```java
// Feign Client + Fallback Factory
@FeignClient(
    name = "user-service",
    fallbackFactory = UserClientFallbackFactory.class
)
public interface UserClient {
    
    @GetMapping("/users/{id}")
    UserDTO getUser(@PathVariable Long id);
    
    @GetMapping("/users/batch")
    List<UserDTO> batchGetUsers(@RequestParam List<Long> ids);
}

// Fallback Factory（可获取异常原因）
@Component
public class UserClientFallbackFactory implements FallbackFactory<UserClient> {
    
    @Override
    public UserClient create(Throwable cause) {
        return new UserClient() {
            
            @Override
            public UserDTO getUser(Long id) {
                log.error("调用 user-service 失败, 降级返回默认值, id={}", id, cause);
                // 尝试从缓存读
                return UserDTO.anonymous();
            }
            
            @Override
            public List<UserDTO> batchGetUsers(List<Long> ids) {
                log.error("批量查用户失败, 降级, ids={}", ids, cause);
                return Collections.emptyList();
            }
        };
    }
}
```

### 5. 线程池隔离（舱壁模式）

```java
// Hystrix 线程池隔离（Sentinel 主要靠信号量）
@Service
public class OrderService {
    
    // 不同下游使用不同线程池，互相隔离
    private final ExecutorService userServicePool = new ThreadPoolExecutor(
        10, 20, 60, TimeUnit.SECONDS,
        new ArrayBlockingQueue<>(100),
        new ThreadFactoryBuilder().setNameFormat("user-pool-%d").build(),
        new ThreadPoolExecutor.CallerRunsPolicy()
    );
    
    private final ExecutorService logisticsPool = new ThreadPoolExecutor(
        5, 10, 60, TimeUnit.SECONDS,
        new ArrayBlockingQueue<>(50),
        new ThreadFactoryBuilder().setNameFormat("logistics-pool-%d").build(),
        new ThreadPoolExecutor.AbortPolicy()
    );
    
    public OrderDetailVO getOrderDetail(Long orderId) {
        Order order = orderMapper.findById(orderId);
        
        // 并发查询，各自用独立线程池
        Future<UserDTO> userFuture = userServicePool.submit(() -> 
            userClient.getUser(order.getUserId()));
        
        Future<LogisticsDTO> logisticsFuture = logisticsPool.submit(() -> 
            logisticsClient.getTracking(orderId));
        
        UserDTO user;
        try {
            user = userFuture.get(500, TimeUnit.MILLISECONDS);  // 限时等待
        } catch (TimeoutException e) {
            userFuture.cancel(true);
            user = UserDTO.anonymous();  // 超时降级
        }
        
        LogisticsDTO logistics;
        try {
            logistics = logisticsFuture.get(500, TimeUnit.MILLISECONDS);
        } catch (Exception e) {
            logistics = LogisticsDTO.unknown();  // 物流查不到不影响主流程
        }
        
        return buildVO(order, user, logistics);
    }
}
```

## 🔥 扩展追问 1：Sentinel vs Hystrix 对比

| 特性 | Sentinel | Hystrix |
|------|---------|---------|
| **隔离方式** | 信号量（轻量） | 线程池（重量） |
| **熔断策略** | 慢调用/异常比例/异常数 | 异常比例 |
| **限流** | ✅ 支持多种算法 | ❌ 不支持 |
| **实时监控** | ✅ Dashboard | ✅ Hystrix Dashboard |
| **动态规则** | ✅ 接入 Nacos/ZK | ❌ 不支持 |
| **维护状态** | 活跃（Alibaba维护） | 停止维护 |
| **Spring Cloud** | Spring Cloud Alibaba | Spring Cloud Netflix |

**推荐**：新项目用 Sentinel，老项目 Hystrix 按需迁移。

```java
// 监控告警：熔断开启时发告警
@EventListener
public void onDegradeEvent(DegradeEvent event) {
    if (event.isBlocking()) {
        // 触发熔断，发钉钉/邮件告警
        alertService.send(AlertLevel.P1,
            String.format("熔断告警: 资源[%s]已熔断，规则: %s", 
                event.getResourceName(), event.getRule()));
    }
}
```

**关键监控指标：**

| 指标 | 含义 | 告警阈值 |
|------|------|---------|
| RT（响应时间） | 下游平均耗时 | > 1000ms 告警 |
| 异常率 | 调用异常比例 | > 30% 告警 |
| 线程池使用率 | 隔离线程池占用 | > 80% 告警 |
| 熔断次数 | 单位时间熔断触发数 | > 0 告警 |

## 一句话收尾

下游变慢导致雪崩的核心应对是**快速失败**：通过熔断器在检测到异常后立即切断调用链，配合**线程池隔离**防止扩散，**多级降级缓存**保住核心功能可用；生产环境推荐 **Sentinel + Nacos 动态规则推送**，实现秒级介入、分钟级恢复。
