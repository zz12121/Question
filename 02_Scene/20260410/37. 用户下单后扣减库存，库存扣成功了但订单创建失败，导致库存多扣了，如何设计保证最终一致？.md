## 先说结论

库存扣成功但订单创建失败，本质是**分布式事务不一致**问题。解决方案：**本地消息表**或**TCC 模式**保证最终一致；推荐做法是**先创建订单（本地事务），再异步扣库存**，配合**库存预占+超时释放**机制，避免库存多扣。

## 🔥 核心概念

### 问题分析

```
┌─────────────────────────────────────────────────────────┐
│                  库存多扣场景分析                          │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  错误流程（先扣库存再创建订单）：                           │
│                                                         │
│  1. 扣减库存（DB 或 Redis）                               │
│     └── UPDATE inventory SET stock = stock - 1           │
│     └── ✅ 成功，库存已扣                                 │
│                                                         │
│  2. 创建订单                                             │
│     └── INSERT INTO orders ...                           │
│     └── ❌ 失败（网络超时/主键冲突/校验失败）               │
│                                                         │
│  3. 结果：库存少了，订单没有                              │
│     └── 用户没买到东西，库存被占用                        │
│     └── 如果是秒杀场景，严重影响其他用户购买               │
│                                                         │
│  根本原因：两个操作非原子，无事务保障                      │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 正确顺序：先订单，后库存

```
┌─────────────────────────────────────────────────────────┐
│              推荐流程：先创建订单                           │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. 本地事务：创建订单 + 记录扣库存消息                     │
│     └── BEGIN                                            │
│     └── INSERT INTO orders (status=PENDING)              │
│     └── INSERT INTO local_message (type=DEDUCT_STOCK)    │
│     └── COMMIT                                           │
│     └── ✅ 订单创建成功，消息持久化                        │
│                                                         │
│  2. 异步扣库存（MQ 消费）                                  │
│     └── 消费 local_message                                │
│     └── 扣减库存（Redis/DB）                              │
│     └── 更新订单状态为 PAID                               │
│     └── ✅ 最终一致                                       │
│                                                         │
│  3. 补偿机制：库存扣失败                                   │
│     └── 订单状态保持 PENDING                              │
│     └── 定时任务扫描，重试扣库存                           │
│     └── 或超时自动关单，释放库存                           │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

```java
// ============ 订单服务 ============
@Service
public class OrderService {
    
    @Autowired
    private OrderMapper orderMapper;
    
    @Autowired
    private MessageMapper messageMapper;
    
    /**
     * 创建订单（本地事务）
     */
    @Transactional
    public Order createOrder(CreateOrderRequest request) {
        // 1. 生成订单
        Order order = Order.builder()
            .orderNo(generateOrderNo())
            .userId(request.getUserId())
            .skuId(request.getSkuId())
            .quantity(request.getQuantity())
            .status(OrderStatus.PENDING)  // 待支付状态
            .expireTime(LocalDateTime.now().plusMinutes(15))
            .build();
        
        orderMapper.insert(order);
        
        // 2. 同一事务中写入消息表
        LocalMessage message = LocalMessage.builder()
            .topic("DEDUCT_STOCK")
            .body(JSON.toJSONString(new DeductStockMessage(
                order.getOrderNo(),
                order.getSkuId(),
                order.getQuantity()
            )))
            .status(0)  // 待发送
            .build();
        
        messageMapper.insert(message);
        
        // 事务提交后：订单创建成功，消息持久化
        
        return order;
    }
}

// ============ 消息投递服务 ============
@Component
public class MessagePublisher {
    
    @Autowired
    private MessageMapper messageMapper;
    
    @Autowired
    private KafkaTemplate<String, String> kafkaTemplate;
    
    /**
     * 定时扫描消息表，投递到 MQ
     */
    @Scheduled(fixedRate = 5000)  // 5秒扫描一次
    public void scanAndPublish() {
        // 查询待发送消息（最多100条）
        List<LocalMessage> messages = messageMapper
            .findByStatusAndCreateTimeBefore(0, LocalDateTime.now().minusSeconds(10));
        
        for (LocalMessage msg : messages) {
            try {
                kafkaTemplate.send(msg.getTopic(), msg.getBody())
                    .addCallback(
                        success -> messageMapper.updateStatus(msg.getId(), 1),  // 标记已发送
                        failure -> {
                            // 重试次数+1
                            messageMapper.incrementRetry(msg.getId());
                            if (msg.getRetryCount() > 5) {
                                // 告警，人工介入
                                alertService.send("消息投递失败: " + msg.getId());
                            }
                        }
                    );
            } catch (Exception e) {
                log.error("投递消息失败: {}", msg.getId(), e);
            }
        }
    }
}

// ============ 库存消费者 ============
@Component
@KafkaListener(topics = "DEDUCT_STOCK", groupId = "stock-group")
public class StockConsumer {
    
    @Autowired
    private StockService stockService;
    
    @Autowired
    private OrderMapper orderMapper;
    
    @Transactional
    public void onMessage(String message) {
        DeductStockMessage msg = JSON.parseObject(message, DeductStockMessage.class);
        
        // 幂等检查：该订单是否已扣过库存
        if (stockService.isDeducted(msg.getOrderNo())) {
            return;  // 已处理，跳过
        }
        
        try {
            // 1. 扣减库存（Redis + Lua 原子操作）
            boolean success = stockService.deduct(msg.getSkuId(), msg.getQuantity());
            
            if (success) {
                // 2. 更新订单状态为已支付
                orderMapper.updateStatus(msg.getOrderNo(), OrderStatus.PAID);
                
                // 3. 记录已扣库存（幂等）
                stockService.markDeducted(msg.getOrderNo());
            } else {
                // 库存不足，关单
                orderMapper.updateStatus(msg.getOrderNo(), OrderStatus.CANCELLED);
            }
            
        } catch (Exception e) {
            // 消费失败，抛异常让 MQ 重试
            throw new RuntimeException("扣库存失败", e);
        }
    }
}
```

### 方案二：TCC 模式（Seata）

```java
// ============ TCC 接口定义 ============
@LocalTCC
public interface OrderTccAction {
    
    @TwoPhaseBusinessAction(name = "orderTcc", 
        commitMethod = "commit", rollbackMethod = "rollback")
    boolean tryCreate(@BusinessActionContextParameter(paramName = "orderNo") String orderNo,
                      @BusinessActionContextParameter(paramName = "skuId") Long skuId,
                      @BusinessActionContextParameter(paramName = "quantity") Integer quantity);
    
    boolean commit(BusinessActionContext context);
    
    boolean rollback(BusinessActionContext context);
}

@LocalTCC
public interface StockTccAction {
    
    @TwoPhaseBusinessAction(name = "stockTcc",
        commitMethod = "commit", rollbackMethod = "rollback")
    boolean tryDeduct(@BusinessActionContextParameter(paramName = "skuId") Long skuId,
                      @BusinessActionContextParameter(paramName = "quantity") Integer quantity);
    
    boolean commit(BusinessActionContext context);
    
    boolean rollback(BusinessActionContext context);
}

// ============ 订单 TCC 实现 ============
@Component
public class OrderTccActionImpl implements OrderTccAction {
    
    @Autowired
    private OrderMapper orderMapper;
    
    @Override
    public boolean tryCreate(String orderNo, Long skuId, Integer quantity) {
        // Try：创建订单，状态为 CREATING（创建中）
        Order order = Order.builder()
            .orderNo(orderNo)
            .skuId(skuId)
            .quantity(quantity)
            .status(OrderStatus.CREATING)  // 中间状态
            .build();
        
        return orderMapper.insert(order) > 0;
    }
    
    @Override
    public boolean commit(BusinessActionContext context) {
        String orderNo = context.getActionContext("orderNo");
        // Confirm：更新为最终状态 PENDING
        return orderMapper.updateStatus(orderNo, OrderStatus.PENDING) > 0;
    }
    
    @Override
    public boolean rollback(BusinessActionContext context) {
        String orderNo = context.getActionContext("orderNo");
        // Cancel：删除订单（或标记为 CANCELLED）
        return orderMapper.deleteByOrderNo(orderNo) > 0;
    }
}

// ============ 库存 TCC 实现 ============
@Component
public class StockTccActionImpl implements StockTccAction {
    
    @Autowired
    private StringRedisTemplate redisTemplate;
    
    @Autowired
    private StockFreezeMapper freezeMapper;
    
    @Override
    public boolean tryDeduct(Long skuId, Integer quantity) {
        // Try：冻结库存（预留）
        String freezeKey = "stock:freeze:" + skuId;
        
        // Redis 预扣 + DB 记录冻结
        Long stock = redisTemplate.opsForValue().decrement(freezeKey, quantity);
        if (stock == null || stock < 0) {
            // 库存不足，回补
            redisTemplate.opsForValue().increment(freezeKey, quantity);
            return false;
        }
        
        // 记录冻结明细
        freezeMapper.insert(new StockFreeze(skuId, quantity));
        return true;
    }
    
    @Override
    public boolean commit(BusinessActionContext context) {
        Long skuId = Long.valueOf(context.getActionContext("skuId"));
        Integer quantity = Integer.valueOf(context.getActionContext("quantity"));
        
        // Confirm：真正扣减库存
        // 1. 从冻结移到实际扣减
        redisTemplate.opsForValue().increment("stock:freeze:" + skuId, quantity);
        inventoryMapper.decrement(skuId, quantity);
        
        // 2. 删除冻结记录
        freezeMapper.delete(skuId, quantity);
        
        return true;
    }
    
    @Override
    public boolean rollback(BusinessActionContext context) {
        Long skuId = Long.valueOf(context.getActionContext("skuId"));
        Integer quantity = Integer.valueOf(context.getActionContext("quantity"));
        
        // Cancel：释放冻结库存
        redisTemplate.opsForValue().increment("stock:freeze:" + skuId, quantity);
        freezeMapper.delete(skuId, quantity);
        
        return true;
    }
}

// ============ 业务使用 ============
@Service
public class OrderService {
    
    @Autowired
    private OrderTccAction orderTccAction;
    
    @Autowired
    private StockTccAction stockTccAction;
    
    @GlobalTransactional
    public void createOrder(CreateOrderRequest request) {
        String orderNo = generateOrderNo();
        
        // Try 阶段：预留资源
        boolean orderOk = orderTccAction.tryCreate(orderNo, request.getSkuId(), request.getQuantity());
        if (!orderOk) {
            throw new BusinessException("创建订单失败");
        }
        
        boolean stockOk = stockTccAction.tryDeduct(request.getSkuId(), request.getQuantity());
        if (!stockOk) {
            // 库存不足，TCC 自动回滚订单
            throw new BusinessException("库存不足");
        }
        
        // 所有 Try 成功，Seata 自动调用 Confirm
        // 任一 Try 失败，Seata 自动调用 Cancel
    }
}
```

```java
     */
    public SeckillResult createOrder(Long userId, Long skuId) {
        String orderNo = generateOrderNo();
        
        // 1. Redis Lua 预占库存（15分钟有效期）
        String script = 
            "local stock = redis.call('get', KEYS[1]) " +
            "if stock and tonumber(stock) > 0 then " +
            "    redis.call('decr', KEYS[1]) " +
            "    redis.call('setex', KEYS[2], 900, ARGV[1]) " +  // 15分钟过期
            "    return 1 " +
            "else " +
            "    return 0 " +
            "end";
        
        Long result = redisTemplate.execute(
            new DefaultRedisScript<>(script, Long.class),
            Arrays.asList("stock:" + skuId, "preorder:" + skuId + ":" + userId),
            orderNo
        );
        
        if (result == null || result == 0) {
            return SeckillResult.fail("库存不足");
        }
        
        // 2. 发送 MQ 异步创建订单
        kafkaTemplate.send("seckill-order", JSON.toJSONString(
            new CreateOrderMessage(orderNo, userId, skuId)
        ));
        
        // 3. 返回排队中（前端轮询结果）
        return SeckillResult.queuing(orderNo);
```

## 🔥 扩展追问 2：补偿机制设计

```java
@Component
public class CompensationService {
    
    @Autowired
    private OrderMapper orderMapper;
    
    @Autowired
    private StockService stockService;
    
    /**
     * 定时扫描异常订单，执行补偿
     */
    @Scheduled(fixedRate = 300000)  // 5分钟
    public void compensate() {
        // 1. 查找 PENDING 超过 30 分钟的订单（可能库存扣失败）
        List<Order> pendingOrders = orderMapper
            .findByStatusAndCreateTimeBefore(
                OrderStatus.PENDING, 
                LocalDateTime.now().minusMinutes(30)
            );
        
        for (Order order : pendingOrders) {
            // 2. 检查库存是否已扣
            boolean deducted = stockService.isDeducted(order.getOrderNo());
            
            if (!deducted) {
                // 3. 重试扣库存
                boolean success = stockService.deduct(
                    order.getSkuId(), order.getQuantity());
                
                if (success) {
                    orderMapper.updateStatus(order.getOrderNo(), OrderStatus.PAID);
                } else {
                    // 4. 库存不足，关单退款
                    closeOrderAndRefund(order);
                }
            }
        }
    }
    
    /**
     * 关单并回补库存
     */
    public void closeOrderAndRefund(Order order) {
        // 1. 更新订单状态
        orderMapper.updateStatus(order.getOrderNo(), OrderStatus.CANCELLED);
        
        // 2. 回补库存
        stockService.increment(order.getSkuId(), order.getQuantity());
        
        // 3. 退款（如果有支付）
        refundService.refund(order.getOrderNo());
    }
}
```

## 一句话收尾

库存多扣问题的根本解法是**调整操作顺序**：先创建订单（本地事务保证），再异步扣库存；配合**本地消息表或 TCC 模式**实现最终一致，加上**库存预占+超时释放**机制兜底，确保库存不会永久丢失。
