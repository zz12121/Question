## 先说结论

DDoS 攻击防护是**分层防御**——最外层云厂商高防（清洗/黑洞）扛住大流量，接入层 Nginx/Gateway 限流熔断削峰，应用层验证码/人机校验过滤 bot，核心业务降级保关键链路。核心思路：**能挡在系统外的绝不放进系统，能丢的绝不硬扛**。

---

## 🔥 核心概念

**1. 解决什么问题**
- DDoS（分布式拒绝服务）通过海量请求耗尽系统资源，导致正常用户无法访问
- 100 倍流量瞬间打满带宽、连接数、CPU，服务完全不可用

**2. 基本原理/思想**
- **分层防御**：每层过滤一部分，层层削弱攻击流量
- **资源隔离**：核心业务和非核心业务隔离，确保关键功能可用
- **快速失败**：识别恶意流量后直接丢弃，不消耗系统资源处理

            # 每秒10请求，突发20

                end

            proxy_pass http://backend;
        }
    }
}

#### Layer 4：应用层防护（人机识别）

```java
@Component
public class DDoSProtectionFilter implements Filter {
    
    @Autowired
    private StringRedisTemplate redisTemplate;
    
    @Override
    public void doFilter(ServletRequest request, ServletResponse response, 
                        FilterChain chain) throws IOException, ServletException {
        
        HttpServletRequest httpRequest = (HttpServletRequest) request;
        HttpServletResponse httpResponse = (HttpServletResponse) response;
        String ip = getClientIP(httpRequest);
        String uri = httpRequest.getRequestURI();
        
        // 1. IP黑名单检查
        if (isBlacklisted(ip)) {
            httpResponse.setStatus(403);
            return;
        }
        
        // 2. 单IP访问频率检查（滑动窗口）
        String key = "access:" + ip + ":" + uri;
        Long count = redisTemplate.opsForValue().increment(key);
        if (count == 1) {
            redisTemplate.expire(key, 1, TimeUnit.MINUTES);
        }
        
        if (count > 100) {  // 1分钟超过100次
            // 加入黑名单，10分钟
            redisTemplate.opsForSet().add("blacklist:ip", ip);
            redisTemplate.expire("blacklist:ip", 10, TimeUnit.MINUTES);
            
            httpResponse.setStatus(429); // Too Many Requests
            return;
        }
        
        // 3. 可疑行为检测（无Cookie、无Referer、UA异常）
        if (isSuspicious(httpRequest)) {
            // 要求验证码
            httpResponse.sendRedirect("/captcha?redirect=" + uri);
            return;
        }
        
        chain.doFilter(request, response);
    }
    
    private boolean isSuspicious(HttpServletRequest request) {
        String ua = request.getHeader("User-Agent");
        String referer = request.getHeader("Referer");
        
        // UA为空或包含脚本特征
        if (ua == null || ua.contains("curl") || ua.contains("python")) {
            return true;
        }
        
        // 关键接口无Referer（直接访问API）
        if (referer == null && isSensitiveApi(request.getRequestURI())) {
            return true;
        }
        
        return false;
    }
}
```

## 🔥 扩展追问 1：如何区分正常流量高峰和 DDoS 攻击？

| 特征 | 正常流量高峰 | DDoS 攻击 |
|-----|-------------|----------|
| 时间分布 | 有规律（如促销开始） | 突发，无预警 |
| IP 分布 | 分散，各地都有 | 集中，某些IP段异常多 |
| 请求特征 | 正常浏览路径 | 直接打API，无页面访问 |
| User-Agent | 正常浏览器 | 为空或脚本工具 |
| 转化率 | 正常 | 接近0% |
| 请求间隔 | 有思考时间 | 固定间隔，机器特征 |

**检测算法：**

```java
@Component
public class AttackDetector {
    
    @Autowired
    private MeterRegistry meterRegistry;
    
    /**
     * 基于多维度检测攻击
     */
    public boolean isUnderAttack() {
        // 1. QPS突增检测（相比基线10倍）
        double currentQps = meterRegistry.counter("http.requests").count();
        double baselineQps = getBaselineQps(); // 历史同期
        if (currentQps > baselineQps * 10) {
            return true;
        }
        
        // 2. 错误率检测（大量无效请求）
        double errorRate = getErrorRate();
        if (errorRate > 0.5) { // 错误率超过50%
            return true;
        }
        
        // 3. 新IP比例检测（大量陌生IP）
        double newIpRatio = getNewIpRatio();
        if (newIpRatio > 0.8) { // 80%是新IP
            return true;
        }
        
        return false;
    }
}
```
