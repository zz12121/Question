1000 万用户的实时排行榜，核心方案是 **Redis ZSet（有序集合）**：天然支持按分数排序、O(log N) 插入更新、O(log N + K) 查询 Top K，完全满足实时排行榜需求。配合 **MQ 异步更新分数 + DB 定时快照**，实现高性能、高可靠的排行榜系统。

### 3. 核心服务实现

```java
@Service
public class LeaderboardService {

    private static final String GLOBAL_RANK_KEY = "leaderboard:global";
    private static final int TOP_N = 100;

    @Autowired
    private StringRedisTemplate redisTemplate;

    @Autowired
    private UserMapper userMapper;

    // ─────────────────────────────────────────
    // 更新用户积分（MQ 消费者调用）
    // ─────────────────────────────────────────
    public void updateScore(Long userId, int delta) {
        // ZINCRBY：原子增加，不存在则创建，O(logN)
        redisTemplate.opsForZSet().incrementScore(
            GLOBAL_RANK_KEY,
            String.valueOf(userId),
            delta
        );
    }

    // ─────────────────────────────────────────
    // 查询 Top 100（含用户详情）
    // ─────────────────────────────────────────
    public List<RankItemVO> getTop100() {
        // 1. 从 Redis 获取 Top 100（分数从高到低）
        Set<ZSetOperations.TypedTuple<String>> tuples =
            redisTemplate.opsForZSet()
                .reverseRangeWithScores(GLOBAL_RANK_KEY, 0, TOP_N - 1);

        if (CollectionUtils.isEmpty(tuples)) {
            return Collections.emptyList();
        }

        // 2. 批量查用户信息（避免 N+1）
        List<Long> userIds = tuples.stream()
            .map(t -> Long.valueOf(t.getValue()))
            .collect(Collectors.toList());

        Map<Long, UserInfo> userMap = userMapper.batchFindByIds(userIds)
            .stream().collect(Collectors.toMap(UserInfo::getId, u -> u));

        // 3. 组装结果
        int rank = 1;
        List<RankItemVO> result = new ArrayList<>();
        for (ZSetOperations.TypedTuple<String> tuple : tuples) {
            Long userId = Long.valueOf(tuple.getValue());
            UserInfo user = userMap.getOrDefault(userId, UserInfo.unknown(userId));

            result.add(RankItemVO.builder()
                .rank(rank++)
                .userId(userId)
                .nickname(user.getNickname())
                .avatar(user.getAvatar())
                .score((long) tuple.getScore().doubleValue())
                .build());
        }
        return result;
    }

    // ─────────────────────────────────────────
    // 查询当前用户排名与分数
    // ─────────────────────────────────────────
    public UserRankVO getUserRank(Long userId) {
        String member = String.valueOf(userId);

        // 排名（0-based → +1 变成 1-based）
        Long rank = redisTemplate.opsForZSet()
            .reverseRank(GLOBAL_RANK_KEY, member);

        // 分数
        Double score = redisTemplate.opsForZSet()
            .score(GLOBAL_RANK_KEY, member);

        if (rank == null || score == null) {
            return UserRankVO.notRanked(userId);
        }

        return UserRankVO.builder()
            .userId(userId)
            .rank(rank + 1)
            .score(score.longValue())
            .build();
    }

    // ─────────────────────────────────────────
    // 查询用户附近排名（上下各5名）
    // ─────────────────────────────────────────
    public List<RankItemVO> getNearbyRank(Long userId, int range) {
        Long rank = redisTemplate.opsForZSet()
            .reverseRank(GLOBAL_RANK_KEY, String.valueOf(userId));

        if (rank == null) return Collections.emptyList();

        long start = Math.max(0, rank - range);
        long end   = rank + range;

        Set<ZSetOperations.TypedTuple<String>> tuples =
            redisTemplate.opsForZSet()
                .reverseRangeWithScores(GLOBAL_RANK_KEY, start, end);

        // 组装（同上）...
        return buildRankItems(tuples, (int) start);
    }
}
```

```java
@Service
public class ScoreEventPublisher {

    @Autowired
    private KafkaTemplate<String, String> kafkaTemplate;

    /**
     * 用户完成任务，发送积分变更事件
     */
    public void publishScoreEvent(Long userId, int delta, String reason) {
        ScoreEvent event = ScoreEvent.builder()
            .userId(userId)
            .delta(delta)
            .reason(reason)
            .timestamp(System.currentTimeMillis())
            .build();

        // 按 userId 分区，保证同一用户的事件顺序消费
        kafkaTemplate.send("score-events",
            String.valueOf(userId),   // partition key
            JSON.toJSONString(event)
        );
    }
}

// ─── 积分事件消费 ───
@Component
@KafkaListener(
    topics = "score-events",
    groupId = "leaderboard-group",
    concurrency = "8"           // 8个分区，8个消费者线程
)
public class ScoreEventConsumer {

    @Autowired
    private LeaderboardService leaderboardService;

    @Autowired
    private UserScoreMapper scoreMapper;

    @Override
    public void onMessage(ConsumerRecord<String, String> record) {
        ScoreEvent event = JSON.parseObject(record.value(), ScoreEvent.class);

        // 1. 更新 Redis ZSet（实时排行）
        leaderboardService.updateScore(event.getUserId(), event.getDelta());

        // 2. 更新 DB（持久化，异步落库）
        scoreMapper.incrementScore(event.getUserId(), event.getDelta());
    }
}
```

### 5. 同分排名处理

```java
/**
 * 同分时，先达到该分数的用户排名靠前
 * 编码方案：score = 积分值 * 1e9 + (Long.MAX_VALUE - createTimeMs / 1000)
 *
 * 积分越高 → score 越大 → 排名越前
 * 同积分时，时间戳越小（越早） → (MAX - ts) 越大 → score 越大 → 排名越前
 */
public double encodeScore(long score, long timestampMs) {
    // 假设积分最大 1亿，时间戳秒级（10位）
    // score 占高位，timestamp 占低位
    return score * 1_000_000_000L + (Long.MAX_VALUE / 1000 - timestampMs / 1000);
}

// 解码原始积分
public long decodeScore(double encodedScore) {
    return (long) encodedScore / 1_000_000_000L;
}
```

## 🔥 扩展追问 1：1000 万数据 ZSet 内存占用多大？

```
Redis ZSet 内存估算：

每个成员：
├── member: userId（8字节 long → string约10字节）
├── score: 8字节 double
├── ZSet skiplist 节点开销：约 64 字节
└── 合计：约 80-100 字节/成员

1000 万用户：
1000万 × 100字节 = 1GB

结论：
- 单个 ZSet 存 1000万用户 ≈ 1GB Redis 内存
- 完全可接受（生产 Redis 通常 8-32GB）
- 如需节省：只存活跃用户（近30天有积分变更的）
```

```java
@Service
public class MultiPeriodLeaderboard {

    // 不同周期独立 Key
    private String getDailyKey()   { return "rank:daily:"   + LocalDate.now(); }
    private String getWeeklyKey()  { return "rank:weekly:"  + getWeekOfYear(); }
    private String getMonthlyKey() { return "rank:monthly:" + YearMonth.now(); }
    private String getGlobalKey()  { return "rank:global";                     }

    // 更新积分时，同时更新所有榜单
    public void updateAllBoards(Long userId, int delta) {
        // Pipeline 批量执行，减少网络往返
        redisTemplate.executePipelined((RedisCallback<?>) conn -> {
            String uid = String.valueOf(userId);
            conn.zIncrBy(getDailyKey().getBytes(),   delta, uid.getBytes());
            conn.zIncrBy(getWeeklyKey().getBytes(),  delta, uid.getBytes());
            conn.zIncrBy(getMonthlyKey().getBytes(), delta, uid.getBytes());
            conn.zIncrBy(getGlobalKey().getBytes(),  delta, uid.getBytes());
            return null;
        });
    }

    // 日榜每天凌晨自动过期
    @PostConstruct
    public void setTTL() {
        redisTemplate.expire(getDailyKey(),   25, TimeUnit.HOURS);
        redisTemplate.expire(getWeeklyKey(),  8,  TimeUnit.DAYS);
        redisTemplate.expire(getMonthlyKey(), 32, TimeUnit.DAYS);
    }
}
```

## 一句话收尾

1000 万用户实时排行榜的最优解是 **Redis ZSet**：O(logN) 更新、O(logN+K) 查 Top K，配合 **MQ 异步写分**削峰、**本地缓存快照**抗高并发读、**DB 持久化**保证数据安全；多周期榜单用独立 Key + TTL 自动过期，同分排名通过**时间戳编码到 score 高位**解决。
这道题是经典的**系统设计题**，重点考察：

1. **技术选型能力**：为什么选 ZSet 而不是 MySQL/ES
2. **数据结构设计**：同分排名怎么处理
3. **高并发优化**：本地缓存、Pipeline、分片
4. **容灾降级**：Redis 挂了怎么办

如果面试官追问"ZSet 内存占用太大怎么办"，可以答：
- 只存活跃用户的 Top N（如 100 万）
- 冷数据归档到 DB，查询时合并
- 使用 Redis Cluster 分片存储
