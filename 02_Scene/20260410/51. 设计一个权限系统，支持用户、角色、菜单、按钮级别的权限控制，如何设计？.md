## 先说结论

权限系统设计的核心是**RBAC模型+资源抽象**——用户通过角色间接关联权限，支持多角色继承；资源统一抽象为菜单、按钮、API、数据四类，用位运算或权限码标识；数据权限通过数据范围（全部/本部门/本人）控制；高性能靠Redis缓存权限树+本地缓存权限码。

---

## 🔥 核心概念

**1. 解决什么问题**
- 细粒度权限控制：不同用户看到不同菜单、按钮、数据
- 权限管理便捷性：批量授权、角色继承、动态调整
- 权限校验高性能：不影响接口响应速度

**2. 基本原理/思想**
- **RBAC**（Role-Based Access Control）：用户→角色→权限，解耦灵活
- **资源抽象**：统一抽象权限资源，支持多种类型
- **数据权限**：独立于功能权限，通过数据范围控制可见性

### 三、核心代码实现

#### 1. 权限加载与缓存

```java
@Service
public class PermissionService {
    
    @Autowired
    private SysMenuMapper menuMapper;
    
    @Autowired
    private SysRoleMapper roleMapper;
    
    @Autowired
    private StringRedisTemplate redisTemplate;
    
    // 本地缓存（权限码）
    private final Cache<Long, Set<String>> permissionCache = Caffeine.newBuilder()
        .maximumSize(10000)
        .expireAfterWrite(10, TimeUnit.MINUTES)
        .build();
    
    /**
     * 加载用户权限（登录时调用）
     */
    public UserPermission loadUserPermission(Long userId) {
        String cacheKey = "permission:user:" + userId;
        
        // 1. 查Redis缓存
        String cached = redisTemplate.opsForValue().get(cacheKey);
        if (cached != null) {
            return JSON.parseObject(cached, UserPermission.class);
        }
        
        // 2. 查询用户角色
        List<SysRole> roles = roleMapper.selectRolesByUserId(userId);
        
        // 3. 查询角色对应的菜单权限
        Set<Long> roleIds = roles.stream()
            .map(SysRole::getRoleId)
            .collect(Collectors.toSet());
        
        List<SysMenu> menus = menuMapper.selectMenusByRoleIds(roleIds);
        
        // 4. 构建权限树
        List<MenuTreeVO> menuTree = buildMenuTree(menus);
        
        // 5. 提取权限码集合
        Set<String> permissions = menus.stream()
            .map(SysMenu::getPermission)
            .filter(StringUtils::hasText)
            .collect(Collectors.toSet());
        
        // 6. 确定数据范围（取最大范围）
        Integer dataScope = roles.stream()
            .map(SysRole::getDataScope)
            .min(Integer::compare)
            .orElse(5); // 默认仅本人
        
        // 7. 组装权限对象
        UserPermission userPermission = new UserPermission();
        userPermission.setUserId(userId);
        userPermission.setRoles(roles);
        userPermission.setMenuTree(menuTree);
        userPermission.setPermissions(permissions);
        userPermission.setDataScope(dataScope);
        
        // 8. 写入缓存
        redisTemplate.opsForValue().set(cacheKey, 
            JSON.toJSONString(userPermission), 30, TimeUnit.MINUTES);
        
        // 9. 写入本地缓存
        permissionCache.put(userId, permissions);
        
        return userPermission;
    }
    
    /**
     * 构建菜单树
     */
    private List<MenuTreeVO> buildMenuTree(List<SysMenu> menus) {
        Map<Long, MenuTreeVO> map = menus.stream()
            .map(this::convertToVO)
            .collect(Collectors.toMap(MenuTreeVO::getMenuId, v -> v));
        
        List<MenuTreeVO> tree = new ArrayList<>();
        
        for (MenuTreeVO node : map.values()) {
            if (node.getParentId() == 0) {
                tree.add(node);
            } else {
                MenuTreeVO parent = map.get(node.getParentId());
                if (parent != null) {
                    parent.getChildren().add(node);
                }
            }
        }
        
        // 排序
        tree.sort(Comparator.comparing(MenuTreeVO::getSortOrder));
        return tree;
    }
}
```

```java
/**
 * 权限注解
 */
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface RequiresPermission {
    String value();  // 权限标识，如 "user:list"
    Logical logical() default Logical.AND;  // AND/OR
}

/**
 * 权限切面
 */
@Aspect
@Component
public class PermissionAspect {
    
    @Autowired
    private PermissionService permissionService;
    
    @Around("@annotation(requiresPermission)")
    public Object around(ProceedingJoinPoint point, RequiresPermission requiresPermission) 
            throws Throwable {
        
        // 1. 获取当前用户
        Long userId = SecurityUtils.getCurrentUserId();
        
        // 2. 获取用户权限（本地缓存）
        Set<String> permissions = permissionService.getPermissionsFromCache(userId);
        
        // 3. 校验权限
        String required = requiresPermission.value();
        
        if (!permissions.contains(required)) {
            throw new AccessDeniedException("无权限：" + required);
        }
        
        // 4. 继续执行
        return point.proceed();
    }
}

// 使用示例
@RestController
@RequestMapping("/api/user")
public class UserController {
    
    @GetMapping("/list")
    @RequiresPermission("user:list")
    public Result list(UserQuery query) {

    }
    
    @PostMapping("/add")
    @RequiresPermission("user:add")

        return Result.ok(userService.add(user));
    }
    
    @DeleteMapping("/{id}")
    @RequiresPermission("user:delete")
```

#### 3. 数据权限实现（SQL拦截）

```java
/**
 * 数据权限注解
 */
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface DataScope {
    String deptAlias() default "";   // 部门表别名
    String userAlias() default "";   // 用户表别名
}

/**
 * MyBatis拦截器：自动拼接数据权限SQL
 */
@Component
@Intercepts({
    @Signature(type = Executor.class, method = "query", 
        args = {MappedStatement.class, Object.class, RowBounds.class, ResultHandler.class})
})
public class DataScopeInterceptor implements Interceptor {
    
    @Override
    public Object intercept(Invocation invocation) throws Throwable {
        // 1. 获取方法上的@DataScope注解
        DataScope dataScope = getDataScopeAnnotation(invocation);
        if (dataScope == null) {
            return invocation.proceed();
        }
        
        // 2. 获取当前用户数据范围
        Long userId = SecurityUtils.getCurrentUserId();
        Integer scope = permissionService.getDataScope(userId);
        
        // 3. 根据范围拼接SQL
        String dataScopeSql = buildDataScopeSql(scope, userId, dataScope);
        
        // 4. 修改原始SQL
        if (StringUtils.hasText(dataScopeSql)) {
            modifySql(invocation, dataScopeSql);
        }
        
        return invocation.proceed();
    }
    
    /**
     * 构建数据权限SQL
     */
    private String buildDataScopeSql(Integer scope, Long userId, DataScope dataScope) {
        StringBuilder sql = new StringBuilder();
        
        switch (scope) {
            case 1: // 全部数据
                return "";
                
            case 2: // 本部门及以下
                SysDept dept = deptMapper.selectByUserId(userId);
                sql.append(String.format(" AND %s.dept_id IN (SELECT dept_id FROM sys_dept WHERE ancestors LIKE '%%%d%%')",
                    dataScope.deptAlias(), dept.getDeptId()));
                break;
                
            case 3: // 本部门
                dept = deptMapper.selectByUserId(userId);
                sql.append(String.format(" AND %s.dept_id = %d",
                    dataScope.deptAlias(), dept.getDeptId()));
                break;
                
            case 4: // 仅本人
                sql.append(String.format(" AND %s.user_id = %d",
                    dataScope.userAlias(), userId));
                break;
                
            case 5: // 自定义
                List<Long> deptIds = roleMapper.selectCustomDeptsByUserId(userId);
                if (!deptIds.isEmpty()) {
                    sql.append(String.format(" AND %s.dept_id IN (%s)",
                        dataScope.deptAlias(), StringUtils.join(deptIds, ",")));
                }
                break;
        }
        
        return sql.toString();
    }
}

// 使用示例
@Service
public class OrderService {
    
    // 自动根据用户数据范围过滤订单
    @DataScope(deptAlias = "o", userAlias = "o")
    public List<Order> list(OrderQuery query) {
        // 实际执行的SQL会自动拼接 AND o.dept_id IN (...) 或 AND o.user_id = ?
        return orderMapper.selectList(query);
    }
}
```

```java
/**
 * API权限控制：通过URL+Method控制
 */
@Component
public class ApiPermissionService {
    
    @Autowired
    private SysApiMapper apiMapper;
    
    /**
     * 初始化API权限（项目启动时扫描所有Controller）
     */
    @PostConstruct
    public void initApiPermissions() {
        // 扫描所有@RequestMapping方法
        Map<RequestMappingInfo, HandlerMethod> handlerMethods = 
            requestMappingHandlerMapping.getHandlerMethods();
        
        handlerMethods.forEach((info, method) -> {
            // 提取URL和Method
            Set<String> patterns = info.getPatternsCondition().getPatterns();
            Set<RequestMethod> methods = info.getMethodsCondition().getMethods();
            
            // 保存到数据库
            patterns.forEach(url -> {
                methods.forEach(httpMethod -> {
                    SysApi api = new SysApi();
                    api.setUrl(url);
                    api.setMethod(httpMethod.name());
                    api.setPermission(generatePermission(url, httpMethod));
                    apiMapper.insertOrUpdate(api);
                });
            });
        });
    }
    
    /**
     * 网关层统一鉴权
     */
    @Component
    public class GatewayAuthFilter implements GlobalFilter {
        
        @Override
        public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {
            String path = exchange.getRequest().getURI().getPath();
            String method = exchange.getRequest().getMethodValue();
            
            // 查询API所需权限
            String requiredPermission = apiMapper
                .selectPermissionByUrlAndMethod(path, method);
            
            if (requiredPermission == null) {
                // 未配置的接口，放行或拒绝（根据策略）
                return chain.filter(exchange);
            }
            
            // 校验用户是否有该权限
            Long userId = extractUserId(exchange);
            if (!hasPermission(userId, requiredPermission)) {
                return unauthorized(exchange);
            }
            
            return chain.filter(exchange);
        }
    }
}
```

```java
/**
 * 权限变更监听器

@Component
public class PermissionChangeListener {
    
    @Autowired
    private StringRedisTemplate redisTemplate;
    
    /**
     * 角色权限变更时，清除相关用户缓存
     */
    @EventListener
    public void onRoleMenuChange(RoleMenuChangeEvent event) {
        Long roleId = event.getRoleId();
        
        // 1. 查询拥有该角色的所有用户
        List<Long> userIds = userMapper.selectUserIdsByRoleId(roleId);
        
        // 2. 清除这些用户的权限缓存
        for (Long userId : userIds) {
            String cacheKey = "permission:user:" + userId;
            redisTemplate.delete(cacheKey);
        }
        
        // 3. 发布消息，通知各节点清除本地缓存
        redisTemplate.convertAndSend("permission:clear", 
            JSON.toJSONString(userIds));
    }
    
    /**
     * 监听清除消息
     */
    @RedisListener(channel = "permission:clear")
    public void onClearMessage(String message) {
        List<Long> userIds = JSON.parseArray(message, Long.class);
        for (Long userId : userIds) {
            permissionCache.invalidate(userId);
        }
    }
}
```

```java
/**
 * 复杂数据权限：多维度组合
 */
@Data
public class DataPermissionRule {
    private String tableAlias;           // 表别名
    private String column;               // 字段
    private DataPermissionType type;     // 规则类型
    
    public enum DataPermissionType {
        DEPT_AND_CHILD,  // 本部门及以下
        DEPT_ONLY,       // 仅本部门
        SELF_ONLY,       // 仅本人
        CUSTOM_DEPTS,    // 自定义部门
        CUSTOM_USERS,    // 自定义用户
        FIELD_VALUE      // 字段值匹配（如：creator_id = ${userId}）
    }
}

/**
 * 多规则组合
 */
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface ComplexDataScope {
    DataPermissionRule[] rules();
    Logical logical() default Logical.AND;
}

// 使用
@ComplexDataScope({
    @DataPermissionRule(tableAlias = "o", column = "dept_id", type = DEPT_AND_CHILD),
    @DataPermissionRule(tableAlias = "o", column = "status", type = FIELD_VALUE, value = "1")
})
public List<Order> listComplex(OrderQuery query) {
    // 生成的SQL: 
    // SELECT * FROM orders o 
    // WHERE o.dept_id IN (SELECT dept_id FROM sys_dept WHERE ancestors LIKE '%1%')
    // AND o.status = 1
    return orderMapper.selectList(query);
}
```

## 一句话收尾

**权限系统的核心是"RBAC解耦+资源抽象+分层缓存"**——用户通过角色间接拥有权限，菜单/按钮/API/数据统一抽象，功能权限用AOP拦截，数据权限用SQL拼接，Redis+本地缓存保证性能，权限变更时异步清除缓存实现实时生效。
第51题完成！权限系统是后端开发的基础能力，面试时要区分清楚**功能权限**和**数据权限**。

几个核心要点：
- **RBAC模型**：用户→角色→权限，解耦灵活，支持多角色
- **功能权限**：能做什么（菜单/按钮/API），用AOP拦截
- **数据权限**：能看到什么数据，用SQL拼接
- **数据范围**：全部/部门及以下/本部门/本人/自定义，数字越小范围越大
