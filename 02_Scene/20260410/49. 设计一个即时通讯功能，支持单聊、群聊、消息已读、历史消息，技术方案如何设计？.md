## 先说结论

即时通讯设计的核心是**长连接+消息队列+分层存储**——WebSocket维护客户端长连接实现实时推送，消息先写MQ保证可靠投递，再异步落库；存储分层用Redis缓存最近消息、MySQL存历史、对象存储存文件；单聊走点对点投递，群聊用扩散读+在线状态优化，已读通过ACK回执+读扩散实现。

---

## 🔥 核心概念

**1. 解决什么问题**
- 实时性：消息秒级到达
- 可靠性：不丢消息、不乱序
- 扩展性：支持万人群、百万在线

**2. 基本原理/思想**
- **长连接保活**：WebSocket替代HTTP轮询，降低延迟和服务器压力
- **消息可靠投递**：先确认收到，再确认已读，双重ACK
- **读写分离**：写消息快，读历史走单独链路
- **扩散优化**：单聊写扩散（各存一份），群聊读扩散（共享一份）

### 三、核心流程实现

#### 1. WebSocket连接管理

```java
@Component
public class WebSocketServer {
    
    // 本地连接映射：userId -> Channel
    private static final ConcurrentHashMap<Long, Channel> userChannelMap = 
        new ConcurrentHashMap<>();
    
    // 用户多设备支持：userId -> Set<Channel>
    private static final ConcurrentHashMap<Long, Set<Channel>> userMultiDeviceMap = 
        new ConcurrentHashMap<>();
    
    @Autowired
    private RedisTemplate<String, String> redisTemplate;
    
    @Autowired
    private UserSessionService sessionService;
    
    /**
     * 连接建立
     */
    @OnOpen
    public void onOpen(Session session, @PathParam("userId") Long userId, 
                      @PathParam("token") String token) {
        // 1. 校验token
        if (!validateToken(userId, token)) {
            session.close();
            return;
        }
        
        // 2. 绑定用户到当前Channel
        Channel channel = (Channel) session.getUserProperties().get("channel");
        userChannelMap.put(userId, channel);
        
        // 3. 更新用户在线状态到Redis（全局可见）
        String serverId = getCurrentServerId();
        redisTemplate.opsForHash().put("im:user:online", userId.toString(), serverId);
        
        // 4. 记录会话到DB
        sessionService.updateUserSession(userId, serverId, "Web");
        
        // 5. 推送离线期间的消息
        pushOfflineMessages(userId);
    }
    
    /**
     * 收到消息
     */
    @OnMessage
    public void onMessage(String message, Session session) {
        ImMessagePacket packet = JSON.parseObject(message, ImMessagePacket.class);
        
        switch (packet.getMsgType()) {
            case SINGLE_CHAT:  // 单聊
                handleSingleChat(packet);
                break;
            case GROUP_CHAT:   // 群聊
                handleGroupChat(packet);
                break;
            case ACK:          // 消息确认
                handleAck(packet);
                break;
            case READ_RECEIPT: // 已读回执
                handleReadReceipt(packet);
                break;
        }
    }
    
    /**
     * 连接关闭
     */
    @OnClose
    public void onClose(Session session, @PathParam("userId") Long userId) {
        userChannelMap.remove(userId);
        redisTemplate.opsForHash().delete("im:user:online", userId.toString());
        sessionService.updateOffline(userId);
    }
}
```

#### 2. 单聊消息投递

```java
@Service
public class SingleChatService {
    
    @Autowired
    private RocketMQTemplate rocketMQTemplate;
    
    @Autowired
    private MessageRepository messageRepository;
    
    @Autowired
    private WebSocketServer webSocketServer;
    
    /**
     * 处理单聊消息
     */
    public void sendSingleMessage(SendMessageRequest request) {
        Long fromUser = request.getFromUser();
        Long toUser = request.getToUser();
        
        // 1. 生成消息ID和序列号
        Long msgId = snowflake.nextId();
        Long seq = generateConvSeq(getConvId(fromUser, toUser));
        
        // 2. 构建消息体
        ImMessage message = new ImMessage();
        message.setMsgId(msgId);
        message.setConvId(getConvId(fromUser, toUser));
        message.setSenderId(fromUser);
        message.setReceiverId(toUser);
        message.setMsgType(request.getMsgType());
        message.setContent(request.getContent());
        message.setSeq(seq);
        message.setSendTime(LocalDateTime.now());
        
        // 3. 先入库（保证消息不丢）
        messageRepository.save(message);
        
        // 4. 更新会话最后消息
        updateConversationLastMsg(message);
        
        // 5. 推送给发送方（ACK）
        webSocketServer.sendToUser(fromUser, buildAckMessage(msgId, seq));
        
        // 6. 推送给接收方（如果在线）
        boolean online = webSocketServer.sendToUser(toUser, buildPushMessage(message));
        
        if (!online) {
            // 离线：发送推送通知
            sendPushNotification(toUser, message);
        }
        
        // 7. 异步写入接收方的收件箱（写扩散）
        rocketMQTemplate.asyncSend("im:inbox:topic", 
            new InboxMessage(toUser, msgId, seq), new SendCallback() {
                @Override
                public void onSuccess(SendResult sendResult) {
                    // 投递成功
                }
                @Override
                public void onException(Throwable e) {
                    // 投递失败，记录日志，定时补偿
                }
            });
    }
}
```

```java
@Service
public class GroupChatService {
    
    @Autowired
    private GroupMemberRepository groupMemberRepository;
    
    @Autowired
    private MessageRepository messageRepository;
    
    @Autowired
    private WebSocketServer webSocketServer;
    
    /**
     * 群聊消息 - 读扩散模式
     * 只存一份消息，成员通过last_ack_msg_id拉取未读
     */
    public void sendGroupMessage(SendMessageRequest request) {
        Long groupId = request.getToGroup();
        Long senderId = request.getFromUser();
        
        // 1. 生成消息
        Long msgId = snowflake.nextId();
        Long seq = generateGroupSeq(groupId);
        
        ImMessage message = new ImMessage();
        message.setMsgId(msgId);
        message.setConvId(groupId);
        message.setSenderId(senderId);
        message.setMsgType(request.getMsgType());
        message.setContent(request.getContent());
        message.setSeq(seq);
        message.setSendTime(LocalDateTime.now());
        
        // 2. 只存一份（读扩散核心）
        messageRepository.save(message);
        
        // 3. 更新群会话最后消息
        updateGroupConversation(groupId, message);
        
        // 4. 获取在线成员，实时推送
        List<Long> members = groupMemberRepository.findMemberIdsByGroupId(groupId);
        
        for (Long memberId : members) {
            if (memberId.equals(senderId)) continue; // 不发给自己
            
            // 查询成员是否在线
            String serverId = (String) redisTemplate.opsForHash()
                .get("im:user:online", memberId.toString());
            
            if (serverId != null) {
                // 在线：直接推送
                webSocketServer.sendToUser(memberId, buildPushMessage(message));
            } else {
                // 离线：发送推送通知（批量聚合）
                addToPendingPush(memberId, message);
            }
        }
    }
}
```

```java
@Service
public class ReadReceiptService {
    
    @Autowired
    private MessageReadRepository readRepository;
    
    @Autowired
    private WebSocketServer webSocketServer;
    
    /**
     * 处理已读回执
     */
    public void handleReadReceipt(ReadReceiptRequest request) {
        Long userId = request.getUserId();
        Long convId = request.getConvId();
        Long lastReadMsgId = request.getLastReadMsgId();
        
        // 1. 更新用户的last_ack_msg_id
        conversationMemberRepository.updateLastAckMsgId(convId, userId, lastReadMsgId);
        
        // 2. 批量插入已读记录（可选，用于精确统计）
        List<Long> unreadMsgIds = messageRepository
            .findUnreadMsgIds(convId, userId, lastReadMsgId);
        
        if (!unreadMsgIds.isEmpty()) {
            List<MessageRead> reads = unreadMsgIds.stream()
                .map(msgId -> {
                    MessageRead read = new MessageRead();
                    read.setMsgId(msgId);
                    read.setConvId(convId);
                    read.setReaderId(userId);
                    read.setReadTime(LocalDateTime.now());
                    return read;
                }).collect(Collectors.toList());
            
            readRepository.batchInsert(reads);
        }
        
        // 3. 通知发送方（如果在线）
        notifySenderReadStatus(convId, userId, lastReadMsgId);
    }
    
    /**
     * 查询会话已读状态
     */
    public ReadStatusVO getReadStatus(Long msgId) {
        ImMessage message = messageRepository.findById(msgId);
        
        if (message.getConvType() == SINGLE_CHAT) {
            // 单聊：查对方是否已读
            boolean read = readRepository.existsByMsgIdAndReaderId(
                msgId, message.getReceiverId());
            return new ReadStatusVO(read ? 1 : 0);
        } else {
            // 群聊：统计已读人数
            long readCount = readRepository.countByMsgId(msgId);
            long totalCount = groupMemberRepository.countByGroupId(message.getConvId());
            return new ReadStatusVO(readCount, totalCount);
        }
    }
}
```

```java
@Service
public class MessageHistoryService {
    
    @Autowired
    private MessageRepository messageRepository;
    
    @Autowired
    private StringRedisTemplate redisTemplate;
    
    /**
     * 拉取历史消息（分页）
     */
    public List<ImMessageVO> pullHistory(Long userId, Long convId, 
                                         Long lastSeq, int limit) {
        // 1. 先查Redis缓存（最近100条）
        String cacheKey = "im:msg:recent:" + convId;
        List<String> cached = redisTemplate.opsForList()
            .range(cacheKey, 0, limit - 1);
        
        if (!cached.isEmpty()) {
            return cached.stream()
                .map(json -> JSON.parseObject(json, ImMessageVO.class))
                .collect(Collectors.toList());
        }
        
        // 2. 查DB
        List<ImMessage> messages = messageRepository
            .findByConvIdAndSeqLessThanOrderBySeqDesc(convId, lastSeq, limit);
        
        // 3. 回填Redis
        if (lastSeq == Long.MAX_VALUE) { // 首次加载
            List<String> jsonList = messages.stream()
                .map(JSON::toJSONString)
                .collect(Collectors.toList());
            redisTemplate.opsForList().rightPushAll(cacheKey, jsonList);
            redisTemplate.expire(cacheKey, 7, TimeUnit.DAYS);
        }
        
        return messages.stream().map(this::convertToVO).collect(Collectors.toList());
    }
}
```

```
- 消息按Seq排序，天然有序
```

## 🔥 扩展追问 2：万人群如何优化？

| 优化点 | 实现方式 | 效果 |
|-------|---------|------|
| **读扩散** | 只存一份消息 | 写入O(1)，不随群人数增长 |
| **在线状态筛选** | 只推送给在线用户 | 减少无效推送 |
| **批量推送** | 聚合离线通知 | 减少推送API调用 |
| **消息分级** | @消息优先推送 | 提升核心体验 |
| **懒加载** | 不活跃成员不推送 | 节省资源 |

```java
/**
 * 万人群推送优化
 */
public void pushToLargeGroup(Long groupId, ImMessage message, int memberCount) {
    if (memberCount > 1000) {
        // 1. 只查询在线成员
        List<Long> onlineMembers = getOnlineMembers(groupId);
        
        // 2. 分批推送（每批100人）
        Lists.partition(onlineMembers, 100).forEach(batch -> {
            asyncPushBatch(batch, message);
        });
        
        // 3. 离线成员发聚合推送（每小时一次）
        schedulePendingPush(groupId);
    } else {
        // 小群直接全量推送
        pushToAllMembers(groupId, message);
    }
}
```

**IM设计的核心是"长连接保实时、消息队列保可靠、分层存储保性能"**——WebSocket维护长连接实现秒级推送，RocketMQ保证消息至少一次投递，Redis+MySQL+OSS分层存储平衡性能和成本，单聊写扩散简单可靠，群聊读扩散支撑万人群，已读回执用last_ack_msg_id优化读扩散。
第49题完成！IM是面试中的高频复杂系统设计题，重点要理解**写扩散vs读扩散**的权衡。

几个核心要点：
- **写扩散**（单聊）：各存一份，查询快，适合人少场景
- **读扩散**（群聊）：存一份，通过last_ack_msg_id拉取，适合人多场景
- **消息有序**：会话内自增Seq，客户端检测丢包
- **可靠投递**：三重ACK（发送ACK、送达ACK、已读ACK）
