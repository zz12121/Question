# 业务场景题 - 补充（B211-B235）

> 以下为对前7天630题中原有领域内部遗漏点的补充，编号从211起，严格只涉及高并发、数据一致性、系统稳定性、业务设计、排查优化、架构演进六大原有场景。

## 高并发场景（4题）

211. 你的电商系统要做"秒杀"活动，商品库存只有 100 件，但预计瞬间请求量 100 万。你用了 Redis 预扣库存 + Lua 脚本原子扣减的方案，但压测发现 Redis 的 QPS 到了 5 万就上不去了，CPU 只有 30%。可能的原因有哪些？（Lua 脚本太复杂 / 单线程瓶颈 / 网络带宽 / 大 key 操作）如何把扣库存拆分到多个 key 上并行执行？库存扣减后如何异步落库？最终一致性怎么保证？

212. 你的系统在做"大批量数据导出"功能，用户点击导出按钮后，系统需要查询 500 万条订单数据并生成 Excel。如果同步导出，接口会超时、服务器 OOM。如何设计异步导出方案？（提交导出任务 → 后台异步生成 → 生成完毕通知用户下载）文件存储在哪里？（OSS/S3/MinIO）如果同时有 100 个用户发起导出，后台资源怎么隔离？导出进度怎么让用户实时感知？

213. 你的系统要接入第三方支付，支付回调接口需要做到"高可用 + 幂等 + 顺序"。高可用：支付平台可能对同一笔订单发送多次回调；幂等：同一笔订单的重复回调不能重复处理；顺序：如果同一笔订单先后收到"支付成功"和"退款成功"两个回调，必须按时间顺序处理。如何设计这个回调处理系统？回调接口的签名验证流程是什么？

214. 你的系统在做"整点抢购"活动，使用了 Redis + Lua 做库存扣减，但发现有个别用户利用"时间差"在活动开始前就提前下单成功了（请求时间戳被篡改）。如何防止这种"时间窗口作弊"？服务端如何校验请求时间？除了 Redis 扣减，数据库层面的库存校验如何保证不超卖？（乐观锁 / FOR UPDATE 悲观锁）如果 Redis 和数据库的库存不一致怎么办？

## 数据一致性场景（4题）

215. 你的系统用了"先更新数据库，再删除缓存"的策略保证缓存一致性，但发现偶尔会出现"删除缓存失败"的情况。引入消息队列做异步重试后，又发现消息队列本身也可能丢失消息。如何设计一个"最终一定成功删除缓存"的方案？（本地消息表 + 定时扫描重试 + 对账补偿）如果数据库回滚了但缓存已经删除了，怎么处理？（"读时再加载"策略天然支持这种场景）

216. 你的系统做了读写分离（主库写、从库读），在"创建订单"接口中，先写主库然后立即从从库查询刚创建的订单，但查不到。Spring 的 `@Transactional` 注解配合读写分离中间件时，事务内的读操作可能走从库吗？如何保证"事务内强制走主库读"？（ShardingSphere 的 Hint / 自定义事务切面 / AOP 设置数据源标记）Binlog 延迟导致的脏读如何监控和告警？

217. 你的系统从"单体"拆分为微服务后，原来的一个数据库事务变成了跨服务的分布式事务。订单服务创建订单成功后，调用库存服务扣减库存、调用积分服务增加积分，如果积分服务失败，订单和库存都已经成功了。如何在"不引入额外中间件"的情况下实现补偿？（本地消息表 + 定时任务扫描 + 幂等重试）如果补偿任务本身也失败了怎么办？（重试次数限制 + 死信队列 + 人工告警）

218. 你的系统要做一个"用户余额转账"功能：从用户 A 的账户扣 100 元，加到用户 B 的账户。在分布式环境下，A 和 B 可能在不同的数据库甚至不同的微服务中。如果用 TCC 模式，Try 阶段冻结金额、Confirm 阶段实际扣减、Cancel 阶段解冻——这个过程中如果 Confirm 执行成功了但 A 的本地事务提交失败（比如数据库连接超时），怎么办？如何设计"幂等的 Confirm"和"幂等的 Cancel"？补偿任务的执行频率怎么定？

## 系统稳定性场景（4题）

219. 你的线上服务突然"大面积超时"——所有接口的 P99 延迟从 200ms 飙升到 30s，但 CPU、内存、网络都正常。排查发现是某个下游服务（日志服务）的线程池被慢 SQL 打满了，导致所有调用链路阻塞。这种"非核心服务拖垮核心服务"的问题如何从架构层面解决？（线程池隔离 / 信号量隔离 / 异步化非核心路径 / Bulkhead 模式）Hystrix/Sentinel 的线程池隔离和信号量隔离有什么区别？

220. 你的服务部署了 3 个实例，使用 Nginx 做负载均衡。某次发布后，发现新版本有一个严重 bug（某个接口报 500），但只有 1/3 的用户受影响（因为流量被均匀分配到 3 个实例）。运维需要"只回滚新版本的那个实例"而不影响其他实例。技术方案怎么做？（蓝绿部署 / 金丝雀发布 / 滚动发布中的 pause + 回滚）K8s 的 RollingUpdate 中 `maxUnavailable` 和 `maxSurge` 参数的含义是什么？

221. 你的 Java 服务在做"JDK 升级"（从 8 升到 17）后，发现 Young GC 频率明显增高，但每次 GC 时间很短（10ms 左右），总体吞吐量反而下降了。这可能是什么原因？（JDK 17 的 G1 Region 默认大小变化 / String 的底层实现变化 / 某些被内联的方法在内联后反而增大了 Code Cache）如何用 JFR（Java Flight Recorder）对比升级前后的性能差异？升级过程中如何做"灰度验证"？

222. 你的微服务系统有 A → B → C 三级调用链路，C 服务突然变慢（P99 从 50ms 涨到 5s），但 C 服务的错误率并没有升高。A 服务很快就报了大量超时错误。这种"慢但不报错"的场景，常规的熔断器（基于错误率）能触发吗？Sentinel 的"慢调用熔断"（慢调用比例熔断）怎么配置？如果 B 服务也做了降级返回默认值，A 服务拿到的是降级数据而不是错误——这种"静默降级"有什么隐患？

## 业务设计场景（4题）

223. 设计一个"延时任务"系统：用户下单后 30 分钟未支付自动取消订单。你的系统已经有 Redis，但不引入额外中间件（如 RabbitMQ 的死信队列、RocketMQ 的延迟消息）。如何用 Redis 实现延迟任务？（ZSet 以到期时间为 score / 轮询扫描 + 内存计算）轮询扫描的时间间隔怎么定？如果 1 秒扫描一次，30 秒内有 100 万个订单到期，怎么高效处理？和 RocketMQ 的延迟消息方案相比，Redis 方案的精度和可靠性差在哪里？

224. 设计一个"分布式 ID 生成器"服务：全局唯一、趋势递增、高可用、高性能（QPS 10万+）。如果不用雪花算法，用数据库自增 ID + 号段模式（Segment）怎么设计？（一次从数据库取一批 ID 缓存在内存中）号段模式的"双 Buffer"是怎么工作的？（一个号段用完了无缝切换到下一个号段）数据库宕机时号段模式怎么保证 ID 不中断？（持久化当前号段 + 启动时校验）美团 Leaf 的号段模式和雪花算法分别适合什么场景？

225. 设计一个"多级缓存"架构：本地缓存（Caffeine）+ 分布式缓存（Redis）+ 数据库。缓存更新策略怎么做？（Cache Aside 模式：先更新数据库再删除缓存）本地缓存和 Redis 缓存的一致性怎么保证？（发布订阅模式：数据库变更 → Redis 删除 → 广播通知所有节点清除本地缓存）如果 Redis 缓存和本地缓存同时失效（缓存雪崩），大量请求同时打到数据库怎么办？（本地缓存设置不同的过期时间 + 缓存预热 + 互斥锁只允许一个线程回源）

226. 设计一个"订单导出"功能，需求如下：支持多维度筛选（时间范围、状态、用户ID）、支持字段自定义（用户选择导出哪些列）、支持异步导出大文件（百万级数据）、导出完成后通知用户下载。技术方案怎么设计？（EasyExcel / SXSSFWorkbook 流式写入 + 异步任务队列 + OSS 存储）如果用户频繁点击"导出"按钮，怎么限制？（防重复提交 + 导出频率限制）导出文件的安全性问题（其他用户不能通过 URL 访问到别人的导出文件）怎么解决？

## 排查优化场景（4题）

227. 你的微服务系统中，用户反馈"提交表单偶尔会超时"但大多数时候正常。后端日志中看到偶尔出现 `Read timed out`，但下游服务的监控显示 P99 耗时只有 100ms。可能的原因有哪些？（DNS 解析偶发超时 / 网络抖动 / GC STW / 连接池获取连接超时）如何用分布式追踪系统（如 SkyWalking）定位是哪个环节导致的超时？如果发现是 DNS 解析的问题，Java 的 `networkaddress.cache.ttl` 参数怎么调优？

228. 你的 MySQL 数据库突然 CPU 飙到 100%，但慢查询日志里没有新增的慢 SQL。可能的原因有哪些？（大量短 SQL 高并发 / 锁等待 / 临时表创建 / 内存表使用 / 连接数暴增）如何用 `SHOW PROCESSLIST` 和 `SHOW ENGINE INNODB STATUS` 快速定位？（看 State 列和 Time 列，找长时间运行的查询 / 看事务锁等待链）`innodb_thread_concurrency` 参数的作用是什么？

229. 你的 Java 服务上线后发现 JVM 堆内存持续增长，但不是每次 GC 后都增长（有回收但回收不完全）。这种"缓慢内存泄漏"如何排查？（MAT 分析 Heap Dump → 查看支配树 Dominator Tree → 找到 GC Root 引用链）如果 Heap Dump 文件太大（8GB+），怎么快速分析？（只 dump 活跃对象 `jmap -dump:live` / Arthas 的 `dashboard` + `heapdump` / 只看特定类的实例）如何在不重启服务的情况下"在线查看"内存占用 Top N 的对象？

230. 你的 Redis 集群中，某个节点的内存使用率明显高于其他节点，但 key 的数量差不多。可能的原因有哪些？（某些 key 的 value 特别大（大 key）/ 某些 key 的过期时间特别长没有被动清理 / ZSet 的 member 数量特别多）如何发现大 key？（`redis-cli --bigkeys` / `MEMORY USAGE key` / SCAN + DEBUG OBJECT）发现大 key 后怎么安全删除？（`UNLINK` 异步删除 / 分批删除）

## 架构演进场景（4题）

231. 你的单体应用拆分为微服务后，发现服务间 RPC 调用的平均链路深度达到了 5 层（A → B → C → D → E），每次请求的累计延迟 300ms+。如何减少调用链路深度？（BFF 模式合并前端请求 / 把同步调用改为异步消息 / 服务合并 / 数据冗余避免跨服务查询）"数据冗余"和"数据不一致"之间怎么平衡？（最终一致性 + 对账补偿）哪些场景适合用消息队列替代同步 RPC？

232. 你的系统从单库单表演进到了分库分表架构（ShardingSphere），现在面临"跨库 JOIN"的痛点：订单表按 user_id 分片，商品表按 shop_id 分片，需要关联查询。如何解决？（数据冗余到 ES / 宽表反范式 / 应用层组装）分库分表后的"全局排序"怎么做？（在应用层做归并排序，不能在数据库层 ORDER BY）分库分表后如何做"唯一性约束"？（分布式 ID + 唯一索引建在每个分片上 + 数据库层面无法保证跨分片唯一）

233. 你的系统要从"自建机房"迁移到"阿里云"，数据库从 MySQL 迁移到 RDS，Redis 从自建迁移到 Redis 云服务。迁移期间要求"零停机"且"数据不丢失"。数据库迁移方案怎么做？（DTS 数据传输服务 / 双写方案）数据一致性怎么验证？（对比源库和目标库的行数和 checksum）DNS 切换怎么做才能做到用户无感知？（TTL 设置、逐步缩小 TTL、监控异常回切）

234. 你的微服务系统要支持"多版本 API 共存"：旧版本客户端（v1）和新版本客户端（v2）同时在线，需要保证 API 向下兼容。如何设计多版本 API 的路由？（URL 路径版本号 `/api/v1/` 和 `/api/v2/` / Header 版本号 `Accept-Version: v2`）如果 v2 API 改了字段名（比如 `userName` 改成 `nickname`），后端怎么做兼容？（Jackson 的 `@JsonAlias` / 不同版本返回不同的 DTO）API 版本的生命周期怎么管理？（v1 上线后维持多久？什么条件下可以下线 v1？）
