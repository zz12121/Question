# 业务场景题 - 第5天（121-150）

## 高并发场景（6题）

121. 你的系统要做一个"直播弹幕"功能，直播间同时在线 10 万人，每秒 5000 条弹幕，要求所有观众实时看到（延迟 < 1 秒），如何设计？WebSocket 还是其他方案？消息怎么分发？

122. 用户在 App 上操作后，偶尔会看到"数据回滚"——刚才改好的设置又变回去了，可能是什么原因？涉及多级缓存、数据库主从、CDN 缓存时，如何定位是哪一层导致的数据不一致？

123. 你的系统在高峰期接口 P99 延迟突然从 200ms 升至 3s，但错误率没有升高，可能是什么原因？如何定位？

124. 要设计一个"流量削峰"方案，把瞬时 10 万 QPS 平摊到 1 分钟内处理，有哪些技术手段？如果用消息队列做缓冲，消费者能力不足时如何动态扩容？

125. 你的系统某个服务节点 CPU 持续 90%+，但其他节点只有 30%，请求量看起来也差不多，可能是哈希不均匀还是热点 key 导致的？怎么排查？

126. 要实现一个"全局唯一序列号"生成服务，要求：单机 QPS 100 万、全局有序递增、不能有空洞（跳号），同时还要保证高可用。雪花算法能满足吗？如果不能，还需要做什么改进？

## 数据一致性场景（5题）

127. 用户充值成功，但账户余额没有增加，可能是什么原因？如何排查分布式事务问题？

128. 你的系统使用 Canal 监听 MySQL binlog 同步数据到 ES，如果 Canal 挂了，数据如何补偿？

129. 消息队列中有消息积压，消费者重启后部分消息被跳过，如何保证消息不丢不重？

130. 你的系统用 Redis 缓存用户权限信息，权限更新后如何通知集群内所有节点刷新本地缓存？

131. 数据库主库写入成功，但从库查询一直查不到，排查了半天发现是主从同步线程卡死，如何处理？

---

## 系统稳定性场景（5题）

132. 你的服务在每天凌晨 2 点会出现一次接口超时，其他时间正常，可能是什么原因？（定时任务？GC？连接池回收？）

133. 线上服务突然大量报 `Too many open files` 错误，如何快速定位和解决？Linux 的 ulimits 和 JVM 的文件描述符有什么关系？

134. 你的服务调用链路很长（A→B→C→D），每个节点耗时都正常但总延迟很高，可能是网络问题还是序列化开销？怎么快速定位哪个环节是瓶颈？

135. 系统做了熔断降级，但触发熔断后用户体验很差（直接返回错误），你有哪些"优雅降级"的思路？（返回缓存数据、降级为简化版本、引导用户稍后重试等）

136. 你的 K8s Pod 在高峰期频繁触发 OOMKilled，但 `java -XX:+PrintFlagsFinal` 显示堆只用了 2G，容器限制是 4G，剩下的 2G 哪去了？怎么排查堆外内存占用？

## 业务设计场景（5题）

137. 设计一个"红包"系统，支持随机红包、拼手气红包，要求高并发抢红包不超发，如何设计？

138. 设计一个"广告投放"系统，根据用户标签精准推送，支持 A/B 测试，如何设计？

139. 设计一个"订阅通知"系统，用户订阅某个商品到货后自动通知，支持百万用户，如何设计？

140. 设计一个"数据脱敏"系统，对手机号、身份证、银行卡等敏感信息自动脱敏，如何设计？

141. 设计一个"操作日志"系统，记录用户所有操作，支持审计查询，如何设计？

## 排查优化场景（5题）

142. 系统的接口在高并发下偶尔返回脏数据（旧数据），但 Redis 缓存已经更新，可能是什么原因？

143. 你的服务日志中频繁出现 `Connection refused`，如何快速定位是哪个服务或端口有问题？

144. 压测时发现接口的吞吐量上不去，增加线程数反而变慢，可能是什么原因？

145. 你的服务重启后前几分钟大量超时，稳定后恢复正常，可能是什么原因？如何解决预热问题？

146. 线上某个批量任务跑了 3 小时还没完成，如何快速排查瓶颈？

## 架构演进场景（4题）

147. 你的系统要从 RPC 调用改为消息驱动架构，哪些场景适合改造？哪些场景不适合？改造过程中如何保证"正在飞行中"的请求不丢失？

148. 你的系统要做数据归档，把 3 年前的订单数据迁移到冷存储（如 OSS/S3），查询时如何做到用户无感知？是应用层路由还是数据库层透明代理？

149. 你的系统要支持多租户 SaaS 化，不同租户要求：数据隔离、自定义字段、独立计费、独立部署（大客户）。怎么设计一套既能满足小客户共享、又能满足大客户独占的混合架构？

150. 你的系统要从 JDK 8 升级到 JDK 17，除了 API 变更，还有哪些隐含风险？（模块化系统 JPMS、反射限制、垃圾收集器默认变更、Nashorn 移除）如何制定分阶段升级策略？
