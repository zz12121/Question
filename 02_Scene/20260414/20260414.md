# 业务场景题 - 第6天（151-180）

## 高并发场景（6题）

151. 你负责的"即时通讯"系统需要支持"消息已读回执"功能：群聊 500 人，每条消息最多产生 500 个已读标记，如果群里活跃用户多，已读回执的写入量可能远超消息本身。如何设计已读回执的数据结构和存储方案？是实时更新还是批量聚合？

152. 系统要做"限流"功能，但需求比较复杂：不仅要限制"单用户 QPS"，还要限制"单接口总 QPS"、"单 IP 频率"、"全局限流"。而且规则要能动态调整（运营随时改阈值），如何设计一套灵活的限流架构？（令牌桶 / 滑动窗口 / Redis + Lua / Sentinel / Gateway 层面）

153. 你的系统在"双十一零点"面临流量洪峰，但和往年不同的是今年引入了直播带货——主播喊"3-2-1 上链接"时，瞬间流量可能达到日常峰值的 100 倍。这种"可预期的极端瞬时峰值"如何应对？预热、缓存、降级分别怎么做？

154. 系统有一个"全局配置中心"的需求：所有微服务实例都需要实时感知配置变更，且要求变更在 3 秒内生效到所有节点。如果用 Nacos 做配置推送，Nacos 本身挂了会怎样？长轮询 vs WebSocket vs gRPC Stream 各自优劣？

155. 你发现线上系统的线程池队列经常积压，导致任务执行延迟很高。但增加线程数后 CPU 又打满。你怀疑是有慢任务阻塞了整个线程池。如何在线上快速定位是哪些任务慢？有没有办法对任务进行"超时熔断"或"隔离"？

156. 系统要做一个"短链接生成器"服务，要求：生成的短链全局唯一、支持自定义短码后缀（如品牌名）、高可用（SLA 99.99%）、防恶意刷量。62 进制编码够用吗？发号冲突怎么处理？如何防止别人通过遍历短码来爬取原始 URL？

## 数据一致性场景（5题）

157. 你的系统涉及"跨行转账"：A 银行扣款成功后调用 B 银行的加款接口，B 银行返回超时。此时钱已经从 A 用户账户扣了但 B 用户没收到。这种"部分失败"的场景，TCC（Try-Confirm-Cancel）事务模式如何处理？Confirm 和 Cancel 的幂等性怎么保证？

158. 你的系统用了 Elasticsearch 存储商品搜索数据，数据源来自 MySQL 的 binlog 同步。某天发现 ES 中某个商品的库存数和 MySQL 不一致，排查发现是 Canal 在重启期间漏掉了几条 binlog。如何做数据的全量和增量校验？离线对账方案怎么设计？

159. 消息队列消费者处理订单逻辑时，需要同时操作数据库和 Redis（更新订单状态 + 清除缓存）。如果数据库更新成功但清除缓存失败了（Redis 挂了），怎么办？"本地消息表"方案在这种场景下怎么落地？

160. 你的系统有多个数据中心（北京、上海、广州），用户数据需要在三个中心之间同步。如果网络抖动导致同步延迟，同一个用户先后在北京和上海修改了昵称，最后以谁为准？如何解决多活架构下的数据冲突？（向量时钟、最后写入获胜 LWW、业务层合并策略等）

161. 系统在做"数据库迁移"：要从 MySQL 迁移到 TiDB（兼容 MySQL 协议），要求迁移期间服务不中断、数据不丢失。双写方案怎么实施？如何验证两边数据的一致性？什么时候可以切掉旧的 MySQL？

## 系统稳定性场景（5题）

162. 你的 K8s 集群中某个 Service 的 Pod 数量从 10 个扩到了 50 个，但整体 QPS 并没有提升多少，反而 Pod 之间互相抢资源。你怀疑是"连接池数量膨胀"导致的——每个Pod 都和下游建立了大量连接。如何计算合理的连接池大小？连接池大小和 Pod 数量的关系是什么？

163. 线上系统出现"间歇性 Full GC 停顿超过 10 秒"的问题，但不是每次 GC 都这样，一天可能出现 2-3 次。G1 日志里看到 `to-space exhausted` 和 `to-space exhausted, triggering full GC`，这通常意味着什么？如何从根本上避免 G1 的退化 Full GC？

164. 你的系统依赖了一个第三方 API，对方没有 SLA 承诺，偶尔响应时间长达 30 秒。你的服务如果用默认的超时配置（比如 5 秒），大量请求会在等待后超时抛异常。如何在代码层面做到"快速失败 + 自动重试 + 降级兜底"？重试策略怎么设计不会造成"重试风暴"?

165. 系统做了"优雅关闭"（Graceful Shutdown），但实际测试时发现有少量请求在关闭过程中丢失了。Spring Boot 2.3+ 的 `server.shutdown=graceful` 是怎么工作的？Tomcat 的 Connector 怎么判断"不再接收新请求"但允许"正在处理的请求继续完成"？如果有异步线程池中的任务还没跑完呢？

166. 监控告警显示你的服务在某段时间内 "5xx 错误率突然飙到 15%"，但你查日志发现应用层面并没有报错，都是下游服务返回的错误。问题是：你的服务把下游的 4xx/5xx 直接透传给了前端，导致前端也报了一堆错误。这种情况下，网关/服务层应该怎么做"错误隔离"和"友好报错"？

## 业务设计场景（5题）

167. 设计一个"审批流程引擎"系统，支持：多级审批、会签/或签、条件分支（金额 > 5 万走总监审批）、驳回/撤回/转办、审批历史记录可追溯。工作流引擎怎么选？（Activiti / Flowable / 自研）状态机怎么设计？

168. 设计一个"实时日志采集与分析"平台：每天产生 5 TB 日志数据，需要支持全文检索、实时聚合统计（QPS 分布、错误率趋势）、日志告警（关键字匹配 + 阈值触发）。技术选型怎么定？（Filebeat → Kafka → Flink/Spark → ES/ClickHouse）各组件的瓶颈在哪里？

169. 设计一个"优惠券"系统，支持满减券、折扣券、立减券、兑换券多种类型；券的领取、核销、退款（退券）全生命周期管理；还要防止薅羊毛（一人领多张、机器刷券）。库存怎么管理？并发抢券怎么做？退单后的券回收逻辑是什么？

170. 设计一个"IM 消息存储"方案，要求：消息永久保存（聊天记录可翻看多年历史）、支持未读数计数、支持多媒体消息（图片/视频/文件）、单聊和群聊统一存储。消息 ID 怎么生成有序？分表策略怎么定？（按用户ID哈希？按时间范围？）消息索引怎么建？

171. 设计一个"灰度发布"平台，支持按比例灰度（10% 流量到新版本）、按用户标签灰度（VIP 用户优先体验新功能）、按地区灰度（先开放北上广深）、灰度效果实时对比（新旧版本的关键指标 A/B 测试数据）。流量路由怎么做？指标采集怎么确保准确？

## 排查优化场景（5题）

172. 你的服务在压测中发现，当并发线程数增加到 200 后，TPS 不再上升反而开始下降，CPU 利用率却只有 40%。使用 `jstack` 发现大量线程处于 `RUNNABLE` 状态但堆栈都在 `socketRead0` 或 `socketWrite0`。这是什么原因？可能是连接池不够还是网络带宽瓶颈？如何进一步定位？

173. 线上 MySQL 出现了一条"慢 SQL"，执行时间 12 秒，`EXPLAIN` 显示走了索引但扫描了 80 万行。这条 SQL 平时只需要 200ms。可能的原因有哪些？（参数嗅探问题、统计信息过时、Buffer Pool 命中率下降、锁等待、TempIO 等）如何一步步排查？

174. 你的 Spring Cloud 微服务系统中，Feign 调用偶尔出现 "Read timed out" 错误，但下游服务的 P99 耗时正常。抓包发现是 DNS 解析偶发超时（几十秒！）。Java 默认的 DNS 缓存机制是什么？为什么会出现 DNS 解析超时？如何优化 JVM 层面的 DNS 配置？

175. 系统 Redis 集群中某个 slot 的数据量是其他 slot 的 10 倍（数据倾斜），导致该节点 CPU 和内存都比其他节点高很多。为什么会数据倾斜？（key 设计不合理、hash tag 使用不当）如何在不影响业务的前提下做数据再平衡（rebalance）？

176. 你的服务使用了 MyBatis-Plus 的 `saveBatch()` 方法做批量插入，但发现插入 5000 条数据耗时 30 秒。开启 MySQL 的 general log 后发现是一条一条 INSERT 语句逐条执行的！MyBatis-Plus 的批处理什么时候会退化成逐条插入？`rewriteBatchedStatements=true` 参数的作用是什么？

## 架构演进场景（4题）

177. 你的单体应用要拆分为微服务，拆完后发现服务之间的 RPC 调用链路很深（A→B→C→D→E），一次请求经过 5 个服务才能返回结果。每个服务增加 20ms 延迟，累计就是 100ms+。如何减少链路深度？哪些调用应该改为异步消息？哪些可以做"服务合并"（Backend for Frontend, BFF）？

178. 你的系统目前部署在一个机房，现在要扩展到两个异地机房实现"同城双活"或"异地多活"。数据库层面的数据同步怎么搞？DNS 流量调度怎么做？如果一个机房整体故障，切换到另一个机房的 RTO（恢复时间目标）能做到多少秒？

179. 你的系统要接入一套新的"统一认证中心"（SSO / OAuth2 / OIDC），所有微服务都要改造为通过 Token 认证。现有系统用的是 Session + Cookie 方案。改造过程中如何保证平滑过渡？（双写模式：新老方案并行运行一段时间）Token 的刷新机制怎么设计（Access Token + Refresh Token）？

180. 你的团队要建立一套"研发效能平台"，包括 CI/CD 流水线、代码质量检测（SonarQube）、自动化测试、镜像构建与发布、环境管理等。基于 GitLab CI / Jenkins / ArgoCD / Tekton 如何选型和组合？K8s 多环境（dev/test/staging/prod）的配置管理和权限控制怎么做？
