## 三、文本审核实现

```java
@Service
public class TextAuditService {
    
    @Autowired private SensitiveWordMapper wordMapper;
    @Autowired private TextModerationClient textClient;  // 第三方审核SDK
    
    public AuditResult auditText(Long userId, String content, String scene) {
        String taskId = generateTaskId();
        
        // 1. 内容去重（hash检素）
        if (isDuplicate(content)) {
            return AuditResult.skip(taskId, "重复内容");
        }
        
        // 2. 敏感词初筛（正则 + HashMap）
        List<SensitiveWord> hitWords = matchSensitiveWords(content);
        if (hitWords.stream().anyMatch(w -> w.getLevel() == 2)) {
            return reject(taskId, "含敏感词: " + hitWords);
        }
        
        // 3. 机审模型调用（异步+超时控制）
        CompletableFuture<TextModerationClient.Result> future = 
            CompletableFuture.supplyAsync(() -> textClient.moderate(content));
        
        try {
            TextModerationClient.Result machineResult = 
                future.get(3, TimeUnit.SECONDS);  // 3秒超时
            
            // 4. 风险评估决策
            return decide(taskId, machineResult, hitWords);
            
        } catch (TimeoutException e) {
            // 超时走降级：保守拦截
            return AuditResult.manualReview(taskId, "机审超时");
        }
    }
    
    private AuditResult decide(String taskId, 
            TextModerationClient.Result result, List<SensitiveWord> hitWords) {
        
        int riskScore = result.getRiskScore();
        List<String> labels = result.getLabels();
        
        // 分数阈值策略（可配置）
        if (riskScore >= 90 || labels.contains("politics")) {
            return reject(taskId, "高风险内容", AuditStatus.MACHINE_REJECT);
        }
        
        if (riskScore >= 70 || labels.contains("suspicious")) {
            return manualReview(taskId, "需人工复核");
        }
        
        // 轻度敏感词替换后放行
        if (!hitWords.isEmpty()) {
            String filtered = replaceSensitiveWords(content, hitWords);
            return pass(taskId, filtered);
        }
        
        return pass(taskId, content);
    }
}
```

## 四、图片审核实现

```java
@Service
public class ImageAuditService {
    
    @Autowired private OSSClient ossClient;  // 阿里云/七牛
    
    public AuditResult auditImage(MultipartFile file, Long userId) {
        String taskId = generateTaskId();
        
        // 1. 上传到临时目录
        String tempUrl = ossClient.uploadTemp(file);
        
        // 2. 计算图片hash（防盗链+去重）
        String hash = calculateHash(file);
        if (isDuplicateHash(hash)) {
            return AuditResult.skip(taskId, "图片已审核");
        }
        
        // 3. 并行调用多个审核维度
        CompletableFuture<ImageCheckResult> nsfwFuture = 
            CompletableFuture.supplyAsync(() -> checkNSFW(tempUrl));
        CompletableFuture<ImageCheckResult> logoFuture = 
            CompletableFuture.supplyAsync(() -> checkLogo(tempUrl));
        CompletableFuture<ImageCheckResult> textFuture = 
            CompletableFuture.supplyAsync(() -> checkTextInImage(tempUrl));
        
        CompletableFuture.allOf(nsfwFuture, logoFuture, textFuture).join();
        
        ImageCheckResult nsfw = nsfwFuture.get();
        ImageCheckResult logo = logoFuture.get();
        ImageCheckResult text = textFuture.get();
        
        // 4. 综合决策
        if (nsfw.isBlocked() || logo.isBlocked()) {
            return reject(taskId, "图片违规", AuditStatus.MACHINE_REJECT);
        }
        
        if (nsfw.getConfidence() > 0.7 || text.isSuspect()) {
            return manualReview(taskId, "需人工复核");
        }
        
        // 5. Move to 正式存储
        String formalUrl = ossClient.moveToFormal(tempUrl);
        return pass(taskId, formalUrl);
    }
    
    // OCR识别图片内文字
    private ImageCheckResult checkTextInImage(String url) {
        OcrResult ocr = ocrClient.detectText(url);
        return textAuditService.auditText(userId, ocr.getText(), "image_ocr");
    }
}
```

## 五、视频审核实现

```java
@Service  
public class VideoAuditService {
    
    @Autowired private FFmpegClient ffmpegClient;
    
    public AuditResult auditVideo(MultipartFile file, Long userId) {
        String taskId = generateTaskId();
        
        // 1. 异步上传 + 切片
        String videoId = videoStorageService.asyncUpload(file);
        
        // 2. 提交视频审核任务（离线任务，不阻塞）
        videoAuditProducer.sendAuditTask(VideoAuditMessage.builder()
            .taskId(taskId)
            .videoId(videoId)
            .userId(userId)
            .build());
        
        // 3. 立即返回，告知用户"审核中"
        return AuditResult.processing(taskId);
    }
    
    // 异步处理（消费者）
    @KafkaListener(topics = "video-audit-tasks")
    public void processVideoAudit(VideoAuditMessage msg) {
        String videoId = msg.getVideoId();
        
        // 1. 截帧（每10秒抽1帧）
        List<String> frames = ffmpegClient.extractFrames(videoId, 10);
        
        // 2. 并行审核所有帧
        List<CompletableFuture<ImageCheckResult>> futures = frames.stream()
            .map(frame -> CompletableFuture.supplyAsync(() -> 
                imageAuditService.checkFrame(frame)))
            .collect(Collectors.toList());
        
        // 3. 汇总帧审核结果
        List<ImageCheckResult> results = futures.stream()
            .map(CompletableFuture::join)
            .collect(Collectors.toList());
        
        // 4. 音频审核（单独处理）
        AudioCheckResult audioResult = audioAuditService.check(videoId);
        
        // 5. 综合决策
        decideAndNotify(msg.getTaskId(), results, audioResult);
    }
    
    private void decideAndNotify(String taskId, 
            List<ImageCheckResult> frames, AudioCheckResult audio) {
        
        boolean hasBlockedFrame = frames.stream()
            .anyMatch(ImageCheckResult::isBlocked);
        
        if (hasBlockedFrame || audio.isBlocked()) {
            updateTaskStatus(taskId, AuditStatus.MACHINE_REJECT);
            notifyUser(taskId, "VIDEO_REJECT", "含违规内容");
        } else if (hasRiskFrames(frames) || audio.isSuspect()) {
            updateTaskStatus(taskId, AuditStatus.MANUAL_REVIEW);
            addToManualQueue(taskId);  // 加入人工队列
        } else {
            updateTaskStatus(taskId, AuditStatus.MACHINE_PASS);
            notifyUser(taskId, "VIDEO_PASS", null);
        }
    }
}
```

```java
@RestController
@RequestMapping("/admin/audit")
public class ManualReviewController {
    
    // 领取任务（抢单模式）
    @PostMapping("/claim")
    public AuditTask claimTask(@CurrentUser Reviewer reviewer) {
        // 优先领取高优先级任务
        return manualReviewQueue.pop(reviewer.getCategory())
            .withReviewerId(reviewer.getId())
            .withClaimTime(LocalDateTime.now());
    }
    
    // 审核操作
    @PostMapping("/decide")
    public void decide(@RequestBody AuditDecision decision) {
        // 1. 更新任务状态
        auditTaskService.updateStatus(decision.getTaskId(), 
            decision.getPass() ? AuditStatus.MANUAL_PASS : AuditStatus.MANUAL_REJECT);
        
        // 2. 记录操作日志
        auditRecordService.save(decision);
        
        // 3. 通知用户
        notificationService.notifyUser(decision);
        
        // 4. 如果是误判，反馈给模型训练
        if (decision.isFalsePositive()) {
            feedbackProducer.send(decision.getTaskId(), 
                FeedbackType.FALSE_POSITIVE);
        }
    }
}
```

## 🔥 扩展追问 1：如何处理高并发审核请求？

> **A:** 核心思路是**分层削峰 + 异步处理**：

```
┌─────────────────────────────────────────────────────────────┐
│                        请求洪峰                              │
└──────────────────────────┬──────────────────────────────────┘
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  L1: 限流层 (Sentinel/SGuava RateLimiter)                    │
│      → 单用户限流：每分钟10次                                │
│      → 全局限流：QPS 10万                                   │
└──────────────────────────┬──────────────────────────────────┘
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  L2: 快速通道（文本+小图）                                   │
│      → 本地缓存敏感词库，毫秒级响应                          │
│      → 直接放行 / 直接拦截                                  │
└──────────────────────────┬──────────────────────────────────┘
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  L3: 异步队列（视频+大批量图片）                              │
│      → Kafka/RocketMQ 缓冲                                 │
│      → 用户感知：正在审核中                                  │
└─────────────────────────────────────────────────────────────┘
```

> **A:** 常见绕过手段及对策：

| 绕过手段 | 示例 | 应对策略 |
|---------|------|---------|

| 图片水印 | 联系方式打在水印上 | OCR + 水印检测模型 |

| 截图拼接 | 多张图片拼成完整信息 | 图片关联分析 |

```java
// 谐音词匹配示例
public class PinyinMatcher {
    private final Map<String, List<String>> pinyinMap = loadPinyinDict();
    
    public boolean containsSensitivePinyin(String text) {
        String pinyin = convertToPinyin(text);  // "微信" → "weixin"
        for (Map.Entry<String, List<String>> entry : pinyinMap.entrySet()) {
            if (entry.getValue().contains(pinyin)) {
                return true;
            }
        }
        return false;
    }
}
```

## 🔥 扩展追问 3：审核结果如何与业务系统解耦？

> **A:** 用 MQ 回调 + 双向检查机制：

```java
// 1. 业务系统发起审核时注册回调
@RestController
public class ContentController {
    
    @PostMapping("/publish")
    public Response publish(@RequestBody Content content) {
        // 1. 创建草稿状态
        Content draft = contentService.createDraft(content);
        
        // 2. 提交审核（携带回调地址）
        auditClient.submit(AuditRequest.builder()
            .contentId(draft.getId())
            .contentType(content.getType())
            .callbackUrl("https://your-api.com/audit/callback")
            .build());
        
        return Response.success("审核中", draft.getId());
    }
    
    // 3. 接收回调
    @PostMapping("/audit/callback")
    public void onAuditCallback(@RequestBody AuditCallback callback) {
        if (callback.isPass()) {
            contentService.publish(callback.getContentId());
        } else {
            contentService.reject(callback.getContentId(), 
                callback.getReason());
        }
    }
}

// 4. 兜底：定时拉取未回调的任务（防止消息丢失）
@Scheduled(fixedDelay = 60000)
public void checkPendingAudits() {
    List<Long> pendingIds = auditService.findPendingTasks(30); // 30分钟未回调
    for (Long id : pendingIds) {
        auditClient.queryStatus(id);  // 主动查询
    }
}
```

## 🔥 扩展追问 4：如何防止审核人员放水/误判？

> **A:** 核心是**交叉复核 + 质量监控**：

```
┌──────────────────────────────────────────────────────────┐
│                    人工审核质量体系                        │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  1️⃣ 任务分配                                             │
│     ├── 同一内容分配给 2 名审核员（A/B岗）                │
│     ├── A通过 + B拒绝 → 进入第三审                        │
│     └── A/B结论一致 → 直接生效                           │
│                                                          │
│  2️⃣ 实时监控                                             │
│     ├── 通过率 > 95% → 标红预警                           │
│     ├── 审核速度异常（过快/过慢）→ 重点抽查              │
│     └── 单一账号操作异常 → 冻结                           │
│                                                          │
│  3️⃣ 事后审计                                             │
│     ├── 每日随机抽样 5% 复审                              │
│     └── 抽审准确率纳入 KPI                               │
│                                                          │
│  4️⃣ 灰度机制                                             │
│     └── 新模型上线，先用 10% 流量对比验证                 │
│                                                          │
└──────────────────────────────────────────────────────────┘
```
