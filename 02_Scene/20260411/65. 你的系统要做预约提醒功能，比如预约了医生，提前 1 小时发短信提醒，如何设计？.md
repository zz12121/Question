### 2.2 预约服务实现

```java
// ========== 1. 预约服务 ==========
@Service
public class AppointmentService {
    
    @Autowired
    private AppointmentMapper appointmentMapper;
    @Autowired
    private ReminderMessageProducer messageProducer;
    
    /**
     * 创建预约
     */
    @Transactional
    public Appointment createAppointment(CreateAppointmentDTO dto) {
        // 1. 检查是否重复预约（同一用户、同一医生、同一时间）
        int conflictCount = appointmentMapper.countConflict(
            dto.getUserId(), 
            dto.getDoctorId(), 
            dto.getAppointmentTime()
        );
        if (conflictCount > 0) {
            throw new BizException("该时间段已有预约");
        }
        
        // 2. 创建预约记录
        Appointment appointment = new Appointment();
        appointment.setAppointmentNo(generateAppointmentNo());
        appointment.setUserId(dto.getUserId());
        appointment.setDoctorId(dto.getDoctorId());
        appointment.setDoctorName(dto.getDoctorName());
        appointment.setHospitalName(dto.getHospitalName());
        appointment.setAppointmentTime(dto.getAppointmentTime());
        
        // 提醒时间 = 预约时间 - 1小时
        appointment.setRemindTime(dto.getAppointmentTime().minusHours(1));
        appointment.setStatus(AppointmentStatus.PENDING);
        appointment.setRemindStatus(RemindStatus.PENDING);
        
        appointmentMapper.insert(appointment);
        
        // 3. 发送延迟消息
        ReminderMessage message = new ReminderMessage();
        message.setMessageId(UUID.randomUUID().toString());
        message.setAppointmentNo(appointment.getAppointmentNo());
        message.setUserId(dto.getUserId());
        message.setAppointmentTime(dto.getAppointmentTime());
        
        // 延迟时间 = (预约时间 - 1小时) - 当前时间
        long delayMs = Duration.between(LocalDateTime.now(), appointment.getRemindTime()).toMillis();
        if (delayMs > 0) {
            // 只发送还没到提醒时间的消息
            messageProducer.sendDelayMessage(message, delayMs);
        } else {
            // 已经过了提醒时间，立即发送
            messageProducer.sendImmediate(message);
        }
        
        return appointment;
    }
    
    /**
     * 取消预约
     */
    @Transactional
    public void cancelAppointment(String appointmentNo, Long userId) {
        Appointment appointment = appointmentMapper.selectByNo(appointmentNo);
        
        // 校验：只能取消自己的预约
        if (!appointment.getUserId().equals(userId)) {
            throw new BizException("无权取消该预约");
        }
        
        // 校验：只能取消待提醒状态的预约
        if (appointment.getStatus() != AppointmentStatus.PENDING) {
            throw new BizException("该预约状态不可取消");
        }
        
        // 更新状态
        appointmentMapper.updateStatus(appointmentNo, AppointmentStatus.CANCELLED);
    }
}
```

### 2.3 延迟消息生产者

```java
// ========== 2. RabbitMQ 延迟消息生产者 ==========
@Service
public class ReminderMessageProducer {
    
    @Autowired
    private RabbitTemplate rabbitTemplate;
    
    // 正常提醒交换机
    private static final String REMINDER_EXCHANGE = "appointment.reminder.exchange";
    // 死信交换机
    private static final String DLX_EXCHANGE = "appointment.reminder.dlx.exchange";
    // 队列
    private static final String REMINDER_QUEUE = "appointment.reminder.queue";
    // 死信队列（真正消费的队列）
    private static final String DLX_QUEUE = "appointment.reminder.dlx.queue";
    
    /**
     * 发送延迟消息（使用 RabbitMQ 延迟插件 或 TTL + DLX）
     */
    public void sendDelayMessage(ReminderMessage message, long delayMs) {
        // 设置消息属性
        MessageProperties props = new MessageProperties();
        props.setMessageId(message.getMessageId());
        props.setExpiration(String.valueOf(delayMs));  // 延迟时间
        
        Message msg = MessageBuilder
            .withBody(JSON.toJSONString(message).getBytes())
            .andProperties(props)
            .build();
        
        // 发送到延迟交换机
        rabbitTemplate.convertAndSend(
            REMINDER_EXCHANGE, 
            "remind", 
            msg
        );
    }
    
    /**
     * 立即发送
     */
    public void sendImmediate(ReminderMessage message) {
        rabbitTemplate.convertAndSend(
            DLX_EXCHANGE, 
            "remind", 
            message
        );
    }
}

// ========== RabbitMQ 配置 ==========
@Configuration
public class RabbitMQConfig {
    
    @Bean
    public CustomExchange reminderExchange() {
        // 延迟交换机（需要 rabbitmq_delayed_message_exchange 插件）
        Map<String, Object> args = new HashMap<>();
        return new CustomExchange("appointment.reminder.exchange", "x-delayed-message", true, false, args);
    }
    
    @Bean
    public DirectExchange dlxExchange() {
        // 死信交换机
        return new DirectExchange("appointment.reminder.dlx.exchange");
    }
    
    @Bean
    public Queue dlxQueue() {
        // 死信队列（真正处理消息的队列）
        return QueueBuilder.durable("appointment.reminder.dlx.queue").build();
    }
    
    @Bean
    public Binding dlxBinding() {
        return BindingBuilder
            .bind(dlxQueue())
            .to(dlxExchange())
            .with("remind")
            .noargs();
    }
}
```

### 2.4 消息消费者

```java
// ========== 3. 提醒消息消费者 ==========
@Component
@Slf4j
public class ReminderMessageConsumer {
    
    @Autowired
    private AppointmentMapper appointmentMapper;
    @Autowired
    private SmsService smsService;
    @Autowired
    private RemindMessageLogMapper messageLogMapper;
    
    @RabbitListener(queues = "appointment.reminder.dlx.queue", concurrency = "5-10")
    public void processReminder(ReminderMessage message, Channel channel, 
                               @Header(AmqpHeaders.DELIVERY_TAG) long tag) {
        String messageId = message.getMessageId();
        
        try {
            // 1. 幂等检查：消息是否已处理
            RemindMessageLog existLog = messageLogMapper.selectByMessageId(messageId);
            if (existLog != null && existLog.getStatus() == MessageStatus.SUCCESS) {
                log.info("消息已处理，跳过: messageId={}", messageId);
                channel.basicAck(tag, false);
                return;
            }
            
            // 2. 记录处理开始
            RemindMessageLog logRecord = new RemindMessageLog();
            logRecord.setMessageId(messageId);
            logRecord.setAppointmentNo(message.getAppointmentNo());
            logRecord.setStatus(MessageStatus.PROCESSING);
            messageLogMapper.insert(logRecord);
            
            // 3. 查询预约信息
            Appointment appointment = appointmentMapper.selectByNo(message.getAppointmentNo());
            
            if (appointment == null) {
                log.warn("预约不存在，跳过: appointmentNo={}", message.getAppointmentNo());
                messageLogMapper.updateStatus(messageId, MessageStatus.SUCCESS);
                channel.basicAck(tag, false);
                return;
            }
            
            // 4. 状态检查
            if (appointment.getStatus() == AppointmentStatus.CANCELLED) {
                log.info("预约已取消，跳过提醒: {}", message.getAppointmentNo());
                messageLogMapper.updateStatus(messageId, MessageStatus.SUCCESS);
                channel.basicAck(tag, false);
                return;
            }
            
            if (appointment.getRemindStatus() == RemindStatus.SENT) {
                log.info("已发送过提醒，跳过: {}", message.getAppointmentNo());
                messageLogMapper.updateStatus(messageId, MessageStatus.SUCCESS);
                channel.basicAck(tag, false);
                return;
            }
            
            // 5. 发送短信
            String smsContent = buildSmsContent(appointment);
            boolean sendSuccess = smsService.send(appointment.getUserId(), smsContent);
            
            if (sendSuccess) {
                // 6. 更新预约提醒状态
                appointmentMapper.updateRemindStatus(
                    message.getAppointmentNo(), 
                    RemindStatus.SENT
                );
                messageLogMapper.updateStatus(messageId, MessageStatus.SUCCESS);
                
                log.info("提醒发送成功: appointmentNo={}", message.getAppointmentNo());
            } else {
                throw new BizException("短信发送失败");
            }
            
            channel.basicAck(tag, false);
            
        } catch (Exception e) {
            log.error("处理提醒消息失败: messageId={}", messageId, e);
            
            // 记录失败
            messageLogMapper.updateError(messageId, e.getMessage());
            
            // 判断是否需要重试
            RemindMessageLog msgLog = messageLogMapper.selectByMessageId(messageId);
            if (msgLog.getRetryCount() < 3) {
                // 重试次数 < 3，拒绝并重新入队
                channel.basicNack(tag, false, true);
            } else {
                // 超过重试次数，标记失败并确认
                messageLogMapper.updateStatus(messageId, MessageStatus.FAILED);
                channel.basicAck(tag, false);
                
                // 发送告警
                sendAlert("提醒消息处理失败超过3次", message);
            }
        }
    }
    
    /**
     * 构建短信内容
     */
    private String buildSmsContent(Appointment appointment) {
        return String.format(
            "【就医提醒】您预约的%s医生（%s）将于1小时后开始，" +
            "请提前到院候诊。预约编号：%s，如有疑问请致电咨询。",
            appointment.getDoctorName(),
            appointment.getHospitalName(),
            appointment.getAppointmentNo()
        );
    }
}
```

### 2.5 兜底定时任务

```java
// ========== 4. 兜底定时任务（防止消息丢失）==========
@Service
public class ReminderBackupTask {
    
    @Autowired
    private AppointmentMapper appointmentMapper;
    @Autowired
    private ReminderMessageProducer messageProducer;
    @Autowired
    private SmsService smsService;
    
    /**
     * 每5分钟执行一次，扫描未发送的提醒
     * 兜底消息队列可能丢失的消息
     */
    @Scheduled(cron = "0 */5 * * * ?")
    public void scanUnsentReminders() {
        // 查出提醒时间已过，但状态还是 PENDING 的记录
        List<Appointment> unsentList = appointmentMapper.selectUnsentReminders(
            LocalDateTime.now().minusMinutes(30),  // 提醒时间超过30分钟还没发
            LocalDateTime.now()
        );
        
        for (Appointment appt : unsentList) {
            try {
                // 检查是否已发送（可能消息刚好在处理中）
                Appointment fresh = appointmentMapper.selectByNo(appt.getAppointmentNo());
                if (fresh.getRemindStatus() == RemindStatus.SENT) {
                    continue;
                }
                
                // 重新发送消息
                ReminderMessage message = new ReminderMessage();
                message.setMessageId(UUID.randomUUID().toString());
                message.setAppointmentNo(appt.getAppointmentNo());
                message.setUserId(appt.getUserId());
                
                messageProducer.sendImmediate(message);
                
                log.warn("兜底重发提醒消息: {}", appt.getAppointmentNo());
                
            } catch (Exception e) {
                log.error("兜底发送失败: appointmentNo={}", appt.getAppointmentNo(), e);
            }
        }
    }
    
    /**
     * 每天凌晨清理过期预约
     */
    @Scheduled(cron = "0 0 2 * * ?")
    public void cleanupExpiredAppointments() {
        LocalDateTime threshold = LocalDateTime.now().minusDays(7);
        appointmentMapper.deleteExpired(threshold);
    }
}
```

## 🔥 扩展追问 1：定时任务和延迟消息各适合什么场景？

> **A:** 看数据量和时效性要求。

| 场景 | 推荐方案 | 原因 |
|------|---------|------|
| 数据量大（百万级） | 定时任务 | 消息队列压力大 |
| 时效性要求高 | 延迟消息 | 精确到毫秒触发 |
| 定时任务 | 延迟消息 | 不依赖外部调度系统 |
| 业务查询较多 | 定时任务 | 可以复用查询条件 |

**方案对比：**


## 🔥 扩展追问 3：多渠道通知（短信+小程序+App推送）怎么做？

> **A:** 模板模式 + 降级策略。

```java
// ========== 多渠道通知 ==========
@Service
public class MultiChannelNotifier {
    
    @Autowired
    private SmsService smsService;
    @Autowired
    private MiniProgramService miniProgramService;
    @Autowired
    private AppPushService appPushService;
    
    /**
     * 发送提醒（多渠道，降级策略）
     */
    public void sendReminder(Appointment appointment) {
        List<String> errors = new ArrayList<>();
        
        // 1. 优先发短信
        try {
            smsService.send(buildSmsContent(appointment));
        } catch (Exception e) {
            errors.add("SMS: " + e.getMessage());
        }
        
        // 2. 同时发小程序订阅消息
        try {
            miniProgramService.sendSubscribe(appointment);
        } catch (Exception e) {
            errors.add("MiniProgram: " + e.getMessage());
        }
        
        // 3. 同时发 App 推送
        try {
            appPushService.push(appointment);
        } catch (Exception e) {
            errors.add("AppPush: " + e.getMessage());
        }
        
        // 4. 至少有一个成功
        if (errors.size() == 3) {
            throw new BizException("所有通知渠道均失败: " + errors);
        }
    }
}
```

```java
// 数据库唯一键保证同一消息只记录一次
```

## 一句话总结

> **预约提醒核心是"延迟消息 + 状态机 + 幂等 + 兜底"：预约时发送延迟消息、消费者检查预约状态后发通知、用 messageId 唯一索引 + 状态字段保证幂等、定时任务兜底补偿丢失的消息，多渠道通知用模板模式+降级策略保证至少一条送达。**

**面试加分话术：**

> "之前项目里出过一个问题：消息队列高峰期延迟，导致用户预约了 2 小时后的号，结果提醒消息延迟到预约时间之后才收到，体验很差。后来我做了几个优化：第一是把提醒消息的 TTL 改成动态的，高峰期自动缩短 TTL 先消费积压；第二是加了兜底定时任务，每 5 分钟扫描一次 `remind_status=PENDING` 且 `remind_time` 已过 30 分钟的记录，主动补发；第三是做了多渠道通知（短信+小程序订阅消息+App推送），三个渠道至少有一个成功，失败了才告警人工介入。现在提醒的送达率做到了 99.9%，基本没收到过用户投诉提醒没收到的反馈。"
