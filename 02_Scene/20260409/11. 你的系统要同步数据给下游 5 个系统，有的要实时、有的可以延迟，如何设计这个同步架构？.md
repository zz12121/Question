### 1. 事件中心（Event Hub）

```java
/**
 * 统一事件模型
 */
@Data
public class BusinessEvent {
    // 事件唯一标识 (雪花算法)
    private String eventId;
    
    // 事件类型 (ORDER_CREATED, ORDER_PAID, INVENTORY_CHANGED...)
    private String eventType;
    
    // 实体类型 (ORDER, PRODUCT, USER...)
    private String entityType;
    
    // 实体ID
    private String entityId;
    
    // 事件数据负载
    private Map<String, Object> payload;
    
    // 事件发生时间
    private Long timestamp;
    
    // 来源系统
    private String source;
    
    // 事件版本 (用于幂等)
    private Long version;
}

/**
 * 事件发布服务
 */
@Service
public class EventPublisher {

    @Autowired
    private RocketMQTemplate rocketMQTemplate;

    @Autowired
    private KafkaTemplate<String, BusinessEvent> kafkaTemplate;

    /**
     * 发布事件（自动分发到多个 Topic）
     */
    public void publish(BusinessEvent event) {
        // 1. 写入本地消息表（事件持久化，binlog 丢失可回溯）
        saveToLocalMessageTable(event);

        // 2. 发布到实时 Topic（实时消费者）
        rocketMQTemplate.convertAndSend("order-topic", event);
        rocketMQTemplate.convertAndSend("inventory-topic", event);

        // 3. 发布到 Kafka（日志流/审计/回溯）
        kafkaTemplate.send("business-events", event.getEntityId(), event);

        log.info("事件已发布, eventId={}, eventType={}", event.getEventId(), event.getEventType());
    }

    private void saveToLocalMessageTable(BusinessEvent event) {
        // 落库保障事件不丢失
    }
}
```

```java
/**
 * Canal 监听器 - 将 DB 变更转为业务事件
 */
@Component
@Slf4j
public class CanalEventListener {

    @Autowired
    private EventPublisher eventPublisher;

    @CanalEntry.Table(tableName = "orders")
    public void onOrderChanged(CanalEntry.RowData rowData) {
        for (CanalEntry.Column column : rowData.getBeforeColumnsList()) {
            if ("status".equals(column.getName())) {
                String eventType = mapStatusToEvent(column.getValue());
                BusinessEvent event = BusinessEvent.builder()
                    .eventId(UUID.randomUUID().toString())
                    .eventType(eventType)
                    .entityType("ORDER")
                    .entityId(rowData.getBeforeColumnsList().stream()
                        .filter(c -> "id".equals(c.getName()))
                        .findFirst().map(CanalEntry.Column::getValue).orElse(""))
                    .payload(extractRowData(rowData))
                    .timestamp(System.currentTimeMillis())
                    .source("canal-cdc")
                    .build();
                
                eventPublisher.publish(event);
            }
        }
    }

    private String mapStatusToEvent(String status) {
        return switch (status) {
            case "CREATED" -> "ORDER_CREATED";
            case "PAID" -> "ORDER_PAID";
            case "CANCELLED" -> "ORDER_CANCELLED";
            default -> "ORDER_STATUS_CHANGED";
        };
    }
}
```

```
│              │ 重试策略: 最多3次，间隔 1s → 5s → 30s                  │

│              │                                                        │
```

```java
/**
 * 实时消费者 - 订单系统
 */
@RocketMQMessageListener(
    topic = "order-topic",
    consumerGroup = "order-consumer-group",
    messageModel = MessageModel.CLUSTERING,  // 集群消费
    consumeThreadMin = 20,
    consumeThreadMax = 100
)
public class OrderRealTimeConsumer implements RocketMQListener<BusinessEvent> {

    @Override
    public void onMessage(BusinessEvent event) {
        log.info("收到订单事件, eventId={}, eventType={}", event.getEventId(), event.getEventType());

        // 幂等检查
        if (!idempotentService.tryAcquire(event.getEventId())) {
            log.info("事件已处理，跳过, eventId={}", event.getEventId());
            return;
        }

        try {
            // 业务处理
            switch (event.getEventType()) {
                case "ORDER_CREATED" -> orderService.syncCreate(event);
                case "ORDER_PAID" -> orderService.syncPaid(event);
                case "ORDER_CANCELLED" -> orderService.syncCancelled(event);
            }
        } catch (Exception e) {
            log.error("处理失败，等待重试, eventId={}", event.getEventId(), e);
            throw e;  // 触发 MQ 重试机制
        }
    }
}
```

```java
/**
 * 延迟消息发布 - 报表系统（每小时同步一次）
 */
@Service
public class DelayMessagePublisher {

    @Autowired
    private RabbitTemplate rabbitTemplate;

    /**
     * 发送延迟消息（报表批量同步）
     * @param delay 延迟时间
     */
    public void publishBatchSync(String bizType, Object data, Duration delay) {
        rabbitTemplate.convertAndSend(
            "biz.delay.exchange",     // 延迟交换机
            "batch.sync." + bizType,  // routing key
            data,
            message -> {
                message.getMessageProperties().setDelay((int) delay.toMillis());
                return message;
            }
        );
        log.info("延迟消息已发送, bizType={}, delay={}", bizType, delay);
    }
}
```

## 六、可靠性保障

```
┌─────────────────────────────────────────────────────────────────────┐
│                        可靠性设计                                    │
│                                                                     │
│  1. 【消息不丢】                                                    │
│     ✅ 事件表持久化（binlog丢了也能回溯）                             │
```

```java
/**
 * 事件回溯服务 - 根据 Event ID 查询事件并重新投递
 */
@Service
public class EventTraceService {

    @Autowired
    private EventRepository eventRepository;

    @Autowired
    private EventPublisher eventPublisher;

    /**
     * 回溯事件（重新投递）
     */
    public void replayEvent(String eventId) {
        BusinessEvent event = eventRepository.findById(eventId)
            .orElseThrow(() -> new RuntimeException("事件不存在"));

        event.setEventId(UUID.randomUUID().toString()); // 新ID
        eventPublisher.publish(event);

        log.info("事件已回溯重投, originalEventId={}, newEventId={}",
                 eventId, event.getEventId());
    }

    /**
     * 时间范围回溯（批量）
     */
    public void replayByTimeRange(Long startTime, Long endTime) {
        List<BusinessEvent> events = eventRepository
            .findByTimestampBetween(startTime, endTime);

        for (BusinessEvent event : events) {
            event.setEventId(UUID.randomUUID().toString());
            eventPublisher.publish(event);
        }

        log.info("时间范围事件已回溯, startTime={}, endTime={}, count={}",
                 startTime, endTime, events.size());
    }
}
```

## 七、面试加分话术

> **面试官问：实时和批量混在一起，如何避免相互影响？**
>
> 两个方向：
> 1. **物理隔离**：实时走 RocketMQ，批量走 Kafka（或者同一 Kafka 不同 Consumer Group）
> 2. **资源隔离**：消费线程池分开，批量消费用小线程池 + 大批量处理，避免抢占 CPU

> **追问：下游处理失败了怎么办？**
>
> 三层保障：
> 1. **MQ 重试**：Exponential Backoff，最多重试 3 次
> 2. **死信队列**：超过重试次数进 DLQ，触发告警
> 3. **事件回溯**：支持按 Event ID 或时间范围手动/自动补偿

> **再问：如何保证数据一致性？**
>
> 不能保证强一致性（分布式链路太长），但可以做到：
> 4. **最终一致性**：事件驱动异步同步，允许短暂延迟
> 5. **每日对账**：T+1 比对源数据和下游数据差异
> 6. **实时监控**：消费延迟 > 阈值立即告警
