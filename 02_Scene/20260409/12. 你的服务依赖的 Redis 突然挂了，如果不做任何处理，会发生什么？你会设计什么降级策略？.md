## 结论先行

**Redis 挂了 = 缓存击穿 → 数据库被打爆 → 服务雪崩 → 系统宕机。**

降级策略核心：**流量拦截 → 熔断 → 本地缓存兜底 → 数据库原生防护**，四层防线层层递减。

### 1. 熔断器降级（Sentienl + Spring Boot）

```java
/**
 * Redis 熔断降级注解
 */
@SentinelResource(
    value = "redis:get",
    blockHandler = "getFromDbFallback",  // 触发限流/熔断时调用
    fallback = "getFromDbFallback"       // 业务异常时调用
)
public String get(String key) {
    // 尝试从 Redis 获取
    String value = redisTemplate.opsForValue().get(key);
    return value;
}

/**
 * 降级兜底方法：走数据库
 */
public String getFromDbFallback(String key, Throwable t) {
    log.warn("Redis 不可用，降级走 DB, key={}, error={}", key, t.getMessage());
    
    // 走数据库
    String value = dbMapper.selectByKey(key);
    
    // 异步回填 Redis（不阻塞主流程）
    CompletableFuture.runAsync(() -> {
        try {
            redisTemplate.opsForValue().set(key, value, Duration.ofMinutes(5));
        } catch (Exception e) {
            log.error("回填 Redis 失败", e);
        }
    });
    
    return value;
}
```

### 2. 本地缓存兜底

```java
/**
 * 多级缓存：Redis + 本地缓存
 */
@Service
public class MultiLevelCacheService {

    // 一级：本地缓存（进程内，最快）
    private final Cache<String, String> localCache = Caffeine.newBuilder()
        .maximumSize(10_000)           // 最多 1 万条
        .expireAfterWrite(Duration.ofMinutes(1))  // 写后 1 分钟过期
        .recordStats()                 // 记录命中率
        .build();

    @Autowired
    private RedisTemplate<String, String> redisTemplate;

    @Autowired
    private ProductMapper productMapper;

    /**
     * 读取：LocalCache → Redis → DB
     */
    public String getProduct(String productId) {
        // 1. 尝试本地缓存
        String result = localCache.getIfPresent(productId);
        if (result != null) {
            log.debug("命中本地缓存, productId={}", productId);
            return result;
        }

        try {
            // 2. 尝试 Redis
            result = redisTemplate.opsForValue().get("product:" + productId);
            if (result != null) {
                // 回填本地缓存
                localCache.put(productId, result);
                return result;
            }
        } catch (Exception e) {
            log.warn("Redis 调用失败, productId={}", productId, e);
            // Redis 挂了，继续往下走
        }

        // 3. 兜底走数据库
        result = loadFromDb(productId);
        return result;
    }

    private String loadFromDb(String productId) {
        log.info("降级：直接从 DB 加载, productId={}", productId);
        
        // 数据库查询
        Product product = productMapper.selectById(productId);
        String value = product != null ? JSON.toJSONString(product) : null;

        // 异步回填 Redis（不阻塞）
        if (value != null) {
            CompletableFuture.runAsync(() -> {
                try {
                    redisTemplate.opsForValue().set("product:" + productId, value, Duration.ofHours(1));
                } catch (Exception e) {
                    log.error("回填 Redis 失败", e);
                }
            });
        }

        return value;
    }

    /**
     * 写入：先写 DB → 再写 Redis → 再写 LocalCache
     */
    public void updateProduct(String productId, Product product) {
        // 1. 写数据库
        productMapper.updateById(product);
        
        String value = JSON.toJSONString(product);

        try {
            // 2. 写 Redis
            redisTemplate.opsForValue().set("product:" + productId, value, Duration.ofHours(1));
        } catch (Exception e) {
            log.warn("写 Redis 失败", e);
        }

        // 3. 写本地缓存
        localCache.put(productId, value);
    }
}
```

### 3. 熔断器配置（Sentinel）

```java
/**
 * Sentinel 熔断规则配置
 */
@Configuration
public class SentinelConfig {

    @PostConstruct
    public void init熔断规则() {
        // 熔断器规则：慢调用比例 / 异常比例 / 异常数
        List<DegradeRule> rules = Arrays.asList(
            // 规则1：异常比例熔断
            new DegradeRule("redisTemplate")
                .setGrade(CircuitBreakerStrategy.ERROR_RATIO_GRADE.getType())  // 异常比例
                .setCount(0.5)                     // 50% 异常比例触发熔断
                .setMinRequestAmount(10)           // 最少 10 个请求才统计
                .setStatIntervalMs(10000)          // 10 秒内统计
                .setSlowRatioThreshold(0.5)       // 慢调用比例阈值
                .setMaxResponseTime(500)            // 超过 500ms 算慢调用
                .setTimeWindow(10),                // 熔断持续 10 秒

            // 规则2：异常数熔断
            new DegradeRule("redisTemplate")
                .setGrade(CircuitBreakerStrategy.ERROR_COUNT_GRADE.getType())  // 异常数
                .setCount(50)                      // 10 秒内超过 50 次异常触发熔断
                .setMinRequestAmount(10)
                .setStatIntervalMs(10000)
                .setTimeWindow(30)                  // 熔断持续 30 秒
        );

        DegradeRuleManager.loadRules(rules);
        log.info("Sentinel 熔断规则已加载");
    }
}
```

### 4. 健康检查 + 自动切换

```java
/**
 * Redis 健康检查 + 自动降级
 */
@Component
@Slf4j
public class RedisHealthChecker {

    @Autowired
    private RedisTemplate<String, String> redisTemplate;

    // Redis 是否可用
    private volatile boolean redisAvailable = true;

    // 最近一次检查时间
    private volatile long lastCheckTime = 0;

    @Scheduled(fixedRate = 1000)  // 每秒检查一次
    public void healthCheck() {
        try {
            // PING 命令，100ms 超时
            redisTemplate.execute(new RedisCallback<String>() {
                @Override
                public String doInRedis(RedisConnection connection) throws DataAccessException {
                    connection.ping();
                    return "PONG";
                }
            });

            // PING 成功
            if (!redisAvailable) {
                log.info("Redis 恢复可用");
                redisAvailable = true;
            }

        } catch (Exception e) {
            if (redisAvailable) {
                log.error("Redis 不可用，触发降级模式", e);
                redisAvailable = false;
            }
        }
    }

    /**
     * 判断当前是否启用降级模式
     */
    public boolean isDegraded() {
        return !redisAvailable;
    }

    /**
     * 降级模式下的操作
     */
    public String getWithDegrade(String key) {
        if (isDegraded()) {
            // 直接返回空或默认值，不去打 Redis
            return null;
        }
        return redisTemplate.opsForValue().get(key);
    }
}
```

## 五、对比表

| 降级策略 | 响应时间 | 数据一致性 | 实现成本 | 适用场景 |
|---------|---------|-----------|---------|---------|
| **本地缓存兜底** | 极快（ns~μs） | 低（TTL 内可能过期） | 低 | 非核心数据（商品信息、配置） |
| **数据库兜底** | 慢（ms~s） | 高 | 中 | 核心链路（订单、支付） |
| **限流拒绝** | 极快（直接返回） | 无数据 | 低 | 保护系统不被压垮 |
| **Sentinel 熔断** | 熔断期快，后续走 DB | 看兜底策略 | 中 | 通用降级 |
| **Redis Sentinel** | 故障转移期慢 | 最终一致 | 高 | 高可用保障 |
| **Redis Cluster** | 同上 | 同上 | 高 | 水平扩展 |

## 七、面试加分话术

> **面试官问：本地缓存和 Redis 数据不一致怎么办？**
>
> 这是 trade-off：
> 1. **本地缓存设置短 TTL**（1~5 分钟），最终一致即可
> 2. **更新时主动失效本地缓存**，而不是更新
> 3. **一致性要求高的场景不放本地缓存**，直接走降级后的 DB 查询

> **追问：降级期间大量请求打 DB，DB 也扛不住怎么办？**
>
> 三层保护：
> 4. **HikariCP 连接池限流**（maxPoolSize=50，超时直接抛异常）
> 5. **SQL 超时熔断**（queryTimeout=3s，防止慢查询拖垮连接池）
> 6. **网关层限流**（超过阈值直接 return 503，保护下游）

> **再问：Redis 恢复后，如何快速预热？**
>
> 7. **懒加载 + 异步回填**：请求触发加载，不阻塞
> 8. **热点数据预加载**：基于历史访问记录，启动时主动加载 Top N 热点
> 9. **旁路预热任务**：定时扫描 DB，全量/增量回填 Redis

## 一句话收尾

**"不怕 Redis 挂，就怕没降级"—— 多级缓存 + 熔断 + 本地兜底 + DB 原生防护，四层防线保你系统不死。**
