## 先说结论

**"附近的人"的核心 = 地理位置索引 + 高效范围查询。最主流的方案是用 Redis 的 GEO 数据结构（底层是 GeoHash），能在毫秒级别完成半径范围内的用户查询。**

```
核心思想：
把二维的经纬度坐标，编码成一维的字符串（GeoHash）
字符串前缀相同的，地理位置就越接近
变二维问题为一维前缀匹配问题，大幅提升查询效率
```

## 🔥 核心概念

### 暴力方案为什么不行？

**最直接的想法：**

```sql

SELECT user_id,
```

**问题：**

```
4. 1亿用户，每秒几千次查询，MySQL 扛不住
```

## 🔥 主流方案：GeoHash + Redis GEO

### 什么是 GeoHash？

```
GeoHash = 把地球的二维坐标映射成一维字符串

原理：
1. 经度范围 [-180, 180] 二分
2. 纬度范围 [-90, 90] 二分
3. 交替编码经纬度，得到 01 二进制串
4. 再转成 Base32 字符串

示例：
北京天安门：116.3912, 39.9071
GeoHash = "wx4g0b"

上海外滩：121.4901, 31.2397
GeoHash = "wtw37q"

字符串前缀越相同，距离越近
"wx..." 都在北京附近
```

**编码过程：**

```
经度 116.3912 编码：
-180          0          180
|-------------|-----------|
           > 0，取1
          |------180|
       < 116.3912？ 取0
...持续二分...

纬度 39.9071 类似操作

交替合并：
经度bit: 1, 0, 1, 1, 1, 0...
纬度bit: 1, 0, 1, 0, 0, 1...
合并:    11 00 11 10 10 01...
转Base32: w  x  4  g  0  b
```

## 🔥 Redis GEO 实现方案

### 核心命令

```bash
# 1. 添加用户位置（用户上线/移动时）
GEOADD user:location 116.3912 39.9071 "user:1001"
GEOADD user:location 116.4112 39.9271 "user:1002"
GEOADD user:location 121.4901 31.2397 "user:1003"

# 2. 查询5km范围内的用户
GEORADIUS user:location 116.3912 39.9071 5 km 
    COUNT 100       # 最多返回100个
    ASC             # 按距离排序
    WITHCOORD       # 返回坐标
    WITHDIST        # 返回距离

# 返回：
# 1) "user:1002" -> 距离: 2.3km
# 2) "user:1001" -> 距离: 0.1km（自己）

# 3. 获取两个用户之间的距离
GEODIST user:location user:1001 user:1002 km

# 4. 获取用户的 GeoHash
GEOHASH user:location user:1001
```

### Java 代码实现

```java
@Service
public class NearbyService {
    
    @Autowired
    private StringRedisTemplate redisTemplate;
    
    private static final String GEO_KEY = "user:location";
    
    // 用户上线/位置更新
    public void updateLocation(Long userId, double longitude, double latitude) {
        redisTemplate.opsForGeo()
            .add(GEO_KEY, 
                new Point(longitude, latitude), 
                "user:" + userId);
    }
    
    // 查询附近5km的用户
    public List<NearbyUser> getNearbyUsers(Long userId, 
                                           double longitude, 
                                           double latitude,
                                           double radiusKm) {
        // 查询半径范围内的用户
        GeoResults<RedisGeoCommands.GeoLocation<String>> results = 
            redisTemplate.opsForGeo()
                .radius(GEO_KEY,
                    new Circle(
                        new Point(longitude, latitude),
                        new Distance(radiusKm, Metrics.KILOMETERS)
                    ),
                    RedisGeoCommands.GeoRadiusCommandArgs
                        .newGeoRadiusArgs()
                        .includeDistance()    // 包含距离
                        .includeCoordinates() // 包含坐标
                        .sortAscending()      // 由近到远
                        .limit(100)           // 最多100个
                );
        
        if (results == null) return Collections.emptyList();
        
        return results.getContent().stream()
            .map(result -> {
                String memberId = result.getContent().getName();
                Long nearbyUserId = Long.parseLong(memberId.replace("user:", ""));
                
                // 过滤掉自己
                if (nearbyUserId.equals(userId)) return null;
                
                double distance = result.getDistance().getValue();
                return new NearbyUser(nearbyUserId, distance);
            })
            .filter(Objects::nonNull)
            .collect(Collectors.toList());
    }
    
    // 用户下线，移除位置
    public void removeLocation(Long userId) {
        redisTemplate.opsForGeo()
            .remove(GEO_KEY, "user:" + userId);
    }
}
```

## 🔥 扩展追问 2：GeoHash 的精度

```
GeoHash 长度 vs 精度：

长度    精度
1      ±2500km
2      ±630km
3      ±78km
4      ±20km
5      ±2.4km   ← 5km 范围建议精度
6      ±610m
7      ±76m
8      ±19m
9      ±2.4m    ← 精确位置

查询 5km 范围：
用精度 5（±2.4km）作为 key 前缀
查询当前格子 + 周围 8 个格子 = 9 个格子
每个格子 ~5km × 5km，完全覆盖 5km 半径
```

**边界问题：**

```
GeoHash 的边界效应：
┌─────┬─────┬─────┐
│ wx4f│ wx4g│ wx4u│
├─────┼─────┼─────┤
│ wx4d│(当前)│ wx4v│
├─────┼─────┼─────┤
│ wx49│ wx4b│ wx4c│
└─────┴─────┴─────┘

当前用户在格子边缘时，
只查自己所在格子会漏掉邻近格子的用户
所以必须查 9 个格子！
```

## 🔥 扩展追问 3：位置更新频率与策略

### 客户端策略

```java
// 智能上报频率（节省流量和电量）
public class LocationReporter {
    
    private static final int FAST_INTERVAL = 5000;   // 移动中，5秒上报
    private static final int SLOW_INTERVAL = 30000;  // 静止时，30秒上报
    private static final double MOVE_THRESHOLD = 50; // 移动超过50米才上报
    
    private Location lastLocation;
    
    public void onLocationChange(Location newLocation) {
        if (lastLocation == null) {
            reportLocation(newLocation);
            return;
        }
        
        float distance = lastLocation.distanceTo(newLocation);
        
        // 移动超过50米才上报
        if (distance > MOVE_THRESHOLD) {
            reportLocation(newLocation);
            lastLocation = newLocation;
        }
    }
}
```

### 服务端策略

```java
// 防止频繁写入 Redis
@Service
public class LocationService {
    
    // 限制同一用户的位置更新频率
    public boolean updateLocation(Long userId, double lon, double lat) {
        String rateKey = "location:rate:" + userId;
        
        // 5秒内只处理一次
        Boolean isNew = redisTemplate.opsForValue()
            .setIfAbsent(rateKey, "1", 5, TimeUnit.SECONDS);
        
        if (Boolean.FALSE.equals(isNew)) {
            return false;  // 频率限制
        }
        
        // 更新 GEO
        geoUpdate(userId, lon, lat);
        
        // 异步持久化到 MySQL
        asyncSaveToDb(userId, lon, lat);
        
        return true;
    }
}
```

### 按 GeoHash 前缀分片

```java
// 根据 GeoHash 前2位分片（32 * 32 = 1024 个分片）
public String getGeoKey(String geoHash) {
    // GeoHash 前2位代表一个大约 1250km x 625km 的区域
    String prefix = geoHash.substring(0, 2);
    int shardIndex = SHARD_MAP.get(prefix);
    return "user:location:shard:" + shardIndex;
}
```

### 跨片查询处理

```java
// 跨城市边界时，查询多个分片
public List<NearbyUser> getNearbyAcrossShards(double lon, double lat, double radius) {
    // 确定影响的城市列表
    List<String> cities = getNearbyCities(lon, lat, radius);
    
    // 并行查询多个分片
    List<CompletableFuture<List<NearbyUser>>> futures = cities.stream()
        .map(city -> CompletableFuture.supplyAsync(
            () -> queryCity(city, lon, lat, radius)))
        .collect(Collectors.toList());
    
    // 合并结果
    return futures.stream()
        .map(CompletableFuture::join)
        .flatMap(List::stream)
        .sorted(Comparator.comparingDouble(NearbyUser::getDistance))
        .limit(100)
        .collect(Collectors.toList());
}
```
