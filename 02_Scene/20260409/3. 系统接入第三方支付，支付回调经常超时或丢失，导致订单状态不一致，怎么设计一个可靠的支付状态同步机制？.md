## 先说结论

支付回调不可靠是常态，**不能依赖回调作为唯一状态来源**。必须构建三保险机制：

| 保障层 | 作用 | 优先级 |
|-------|------|-------|
| **回调通知（被动）** | 最快，但不保证送达 | Primary |
| **主动查询（轮询）** |兜底补偿，5/15/30分钟逐步拉取 | Backup |
| **定时对账（日结）** | 最终兜底，发现遗漏 | Final |

**一句话答案**：支付状态同步的核心是 **"回调不可信，主动查询来兜底，定时对账保最后一道防线"**，三道机制缺一不可。

### 1.1 支付回调的不可靠性

```
第三方支付平台              你的系统
```

### 3.1 第一保险：支付回调（幂等 + 状态机）

```java
@RestController
@RequestMapping("/pay/callback")
@Slf4j
public class PayCallbackController {
    
    @Autowired private PayCallbackService callbackService;
    
    @PostMapping("/{channel}")
    public String handleCallback(
        @PathVariable String channel,
        @RequestBody String rawBody,
        HttpServletRequest request) {
        
        log.info("[支付回调] channel={}, body={}", channel, rawBody);
        
        try {
            // 1. 解析回调数据（统一格式）
            PayCallback callback = parseCallback(channel, rawBody, request);
            
            // 2. 幂等处理
            boolean processed = callbackService.processCallback(callback);
            
            log.info("[支付回调] 处理结果: tradeNo={}, processed={}", 
                callback.getTradeNo(), processed);
            
            // 3. 第三方要求返回固定格式
            return "success";
        } catch (Exception e) {
            log.error("[支付回调] 处理异常", e);
            // 返回 failure 会触发第三方重试
            return "failure";
        }
    }
}

@Service
@Slf4j
public class PayCallbackService {
    
    @Autowired private OrderMapper orderMapper;
    @Autowired private PaymentTransactionMapper transactionMapper;
    @Autowired private IdempotentService idempotentService;
    
    @Transactional
    public boolean processCallback(PayCallback callback) {
        String tradeNo = callback.getTradeNo();
        String orderId = callback.getOrderId();
        
        // ========== 幂等校验 ==========
        // 1. Redis 幂等：快速判断
        String lockKey = "pay:callback:" + tradeNo;
        Boolean acquired = redisTemplate.opsForValue()
            .setIfAbsent(lockKey, "1", 30, TimeUnit.MINUTES);
        
        if (Boolean.FALSE.equals(acquired)) {
            log.info("[幂等跳过] tradeNo={}", tradeNo);
            return true;  // 已处理过
        }
        
        try {
            // 2. 数据库幂等：唯一约束兜底
            PaymentTransaction exist = transactionMapper.findByTradeNo(tradeNo);
            if (exist != null && "SUCCESS".equals(exist.getStatus())) {
                log.info("[DB幂等跳过] tradeNo={}", tradeNo);
                return true;
            }
            
            // ========== 状态校验 ==========
            Order order = orderMapper.findById(orderId);
            if (order == null) {
                log.warn("[订单不存在] orderId={}", orderId);
                return false;
            }
            
            // ========== 状态机校验 ==========
            if (!order.isPayable()) {
                log.warn("[状态不允许] orderId={}, status={}", 
                    orderId, order.getStatus());
                return true;  // 已是终态，不报错
            }
            
            // ========== 业务处理 ==========
            if ("SUCCESS".equals(callback.getStatus())) {
                // 更新交易流水
                saveTransaction(callback);
                // 更新订单状态
                updateOrderPaid(order, callback);
            } else {
                // 支付失败
                updateOrderFailed(order, callback);
            }
            
            // 记录回调历史
            recordCallbackHistory(callback);
            
            return true;
            
        } catch (Exception e) {
            log.error("[处理异常] tradeNo={}", tradeNo, e);
            redisTemplate.delete(lockKey);  // 失败解锁，允许重试
            throw e;
        }
    }
    
    // ========== 状态机更新 ==========
    private void updateOrderPaid(Order order, PayCallback callback) {
        // 使用乐观锁更新
        int affected = orderMapper.updatePayStatusWithVersion(
            order.getId(),
            "PAID",
            callback.getTradeNo(),
            callback.getPaidAmount(),
            callback.getPaidTime(),
            order.getVersion()
        );
        
        if (affected == 0) {
            throw new BusinessException("订单状态更新失败，version冲突");
        }
    }
}
```

```java
@Configuration
public class PayQueryConfig {

@Service
@Slf4j
public class PayQueryService {
    
    @Autowired private PaymentGatewayClient paymentGateway;
    @Autowired private OrderMapper orderMapper;
    
    private static final int[] QUERY_DELAYS = {5, 15, 30, 60, 120};  // 分钟
    
    @Transactional
    public void scheduleQuery(Long orderId) {
        Order order = orderMapper.findById(orderId);
        
        // 已经支付成功的，不用再查
        if ("PAID".equals(order.getStatus())) {
            return;
        }
        
        // 记录待查询任务
        PayQueryTask task = new PayQueryTask();
        task.setOrderId(orderId);
        task.setNextQueryTime(calculateNextQueryTime(order));
        task.setQueryCount(0);
        task.setStatus("PENDING");
        
        queryTaskMapper.insert(task);
        log.info("[创建查询任务] orderId={}, nextTime={}", 
            orderId, task.getNextQueryTime());
    }
    
    private LocalDateTime calculateNextQueryTime(Order order) {
        int count = order.getQueryCount();
        int delayMinutes = QUERY_DELAYS[Math.min(count, QUERY_DELAYS.length - 1)];
        return LocalDateTime.now().plusMinutes(delayMinutes);
    }
    
    // 定时任务：每分钟执行一次
    @Scheduled(cron = "0 * * * * ?")
    public void executePendingQueries() {
        List<PayQueryTask> tasks = queryTaskMapper.findPendingTasks(
            LocalDateTime.now());
        
        for (PayQueryTask task : tasks) {
            try {
                executeQuery(task);
            } catch (Exception e) {
                log.error("[查询异常] orderId={}", task.getOrderId(), e);
            }
        }
    }
    
    private void executeQuery(PayQueryTask task) {
        Order order = orderMapper.findById(task.getOrderId());
        
        // 已经终态，不再查询
        if (order.isFinalStatus()) {
            queryTaskMapper.updateStatus(task.getId(), "COMPLETED");
            return;
        }
        
        // 超过最大查询次数，标记为 UNKNOWN，告警人工处理
        if (task.getQueryCount() >= 5) {
            queryTaskMapper.updateStatus(task.getId(), "UNKNOWN");
            alertService.sendAlert("支付状态未知", order);
            return;
        }
        
        // 调用第三方支付查询接口
        PayQueryResult result = paymentGateway.queryPayStatus(
            order.getThirdPartyOrderId());
        
        // 更新查询记录
        orderMapper.updateQueryInfo(order.getId(), 
            LocalDateTime.now(), result.getStatus());
        
        if ("SUCCESS".equals(result.getStatus())) {
            // 查询到支付成功，更新订单
            handlePaySuccess(order, result);
            queryTaskMapper.updateStatus(task.getId(), "COMPLETED");
        } else if ("FAILED".equals(result.getStatus())) {
            // 查询到支付失败
            handlePayFailed(order, result);
            queryTaskMapper.updateStatus(task.getId(), "COMPLETED");
        } else {
            // 仍不确定，安排下次查询
            int nextDelay = QUERY_DELAYS[task.getQueryCount() + 1];
            queryTaskMapper.updateTask(task.getId(), 
                LocalDateTime.now().plusMinutes(nextDelay),
                task.getQueryCount() + 1);
        }
    }
    
    private void handlePaySuccess(Order order, PayQueryResult result) {
        int affected = orderMapper.updatePayStatusWithVersion(
            order.getId(),
            "PAID",
            result.getTradeNo(),
            result.getAmount(),
            result.getPayTime(),
            order.getVersion()
        );
        
        if (affected > 0) {
            log.info("[主动查询成功] orderId={}, tradeNo={}", 
                order.getId(), result.getTradeNo());
            // 发送订单更新事件
            eventBus.post(new OrderPaidEvent(order));
        }
    }
}
```

### 3.3 第三保险：定时对账（每天凌晨）

```java
@Component
@Slf4j
public class DailyReconciliation {
    
    @Autowired private PaymentGatewayClient paymentGateway;
    @Autowired private OrderMapper orderMapper;
    @Autowired private TransactionMapper transactionMapper;
    
    /**
     * 每天凌晨 2 点执行对账
     * 对账范围：前一天 00:00:00 ~ 23:59:59
     */
    @Scheduled(cron = "0 0 2 * * ?")
    public void reconcile() {
        LocalDate yesterday = LocalDate.now().minusDays(1);
        LocalDateTime startTime = yesterday.atStartOfDay();
        LocalDateTime endTime = yesterday.plusDays(1).atStartOfDay();
        
        log.info("[对账开始] date={}, start={}, end={}", 
            yesterday, startTime, endTime);
        
        ReconciliationResult result = new ReconciliationResult();
        result.setReconcileDate(yesterday);
        result.setStartTime(startTime);
        
        try {
            // 1. 拉取第三方支付流水
            List<ThirdPartyTrade> thirdPartyTrades = 
                paymentGateway.fetchDailyTrades(yesterday);
            result.setThirdPartyTotal(thirdPartyTrades.size());
            
            // 2. 拉取本地交易记录
            List<PaymentTransaction> localTransactions = 
                transactionMapper.findByDateRange(startTime, endTime);
            result.setLocalTotal(localTransactions.size());
            
            // 3. 对账核心逻辑
            reconcileCore(thirdPartyTrades, localTransactions, result);
            
            // 4. 处理异常
            handleAbnormalRecords(result);
            
            log.info("[对账完成] 总数={}, 正常={}, 异常={}", 
                result.getTotal(), result.getNormal(), result.getAbnormal());
            
        } catch (Exception e) {
            log.error("[对账异常]", e);
            alertService.sendAlert("对账执行异常", e.getMessage());
        }
    }
    
    private void reconcileCore(
            List<ThirdPartyTrade> thirdPartyTrades,
            List<PaymentTransaction> localTransactions,
            ReconciliationResult result) {
        
        Map<String, ThirdPartyTrade> thirdMap = thirdPartyTrades.stream()
            .collect(Collectors.toMap(ThirdPartyTrade::getTradeNo, t -> t));
        
        Map<String, PaymentTransaction> localMap = localTransactions.stream()
            .collect(Collectors.toMap(PaymentTransaction::getTradeNo, t -> t));
        
        Set<String> allTradeNos = new HashSet<>();
        allTradeNos.addAll(thirdMap.keySet());
        allTradeNos.addAll(localMap.keySet());
        
        result.setTotal(allTradeNos.size());
        
        for (String tradeNo : allTradeNos) {
            ThirdPartyTrade third = thirdMap.get(tradeNo);
            PaymentTransaction local = localMap.get(tradeNo);
            
            if (third == null) {
                // 第三方没有，本地有 → 本地异常（可能重复记账）
                result.addAbnormal(AbnormalType.LOCAL_ONLY, tradeNo, local);
            } else if (local == null) {
                // 第三方有，本地没有 → 掉单（最常见问题）
                result.addAbnormal(AbnormalType.MISSING, tradeNo, third);
            } else if (!third.getAmount().equals(local.getAmount())) {
                // 金额不一致
                result.addAbnormal(AbnormalType.AMOUNT_MISMATCH, tradeNo, third);
            } else if (!"SUCCESS".equals(third.getStatus()) 
                       && "SUCCESS".equals(local.getStatus())) {
                // 第三方失败，本地成功 → 数据异常
                result.addAbnormal(AbnormalType.STATUS_MISMATCH, tradeNo, third);
            } else {
                result.addNormal(tradeNo);
            }
        }
    }
    
    private void handleAbnormalRecords(ReconciliationResult result) {
        for (AbnormalRecord record : result.getAbnormalRecords()) {
            try {
                if (record.getType() == AbnormalType.MISSING) {
                    // 掉单：主动补单
                    compensateOrder(record);
                } else if (record.getType() == AbnormalType.LOCAL_ONLY) {
                    // 本地多：可能重复，退款处理
                    handleDuplicateRecord(record);
                } else {
                    // 金额/状态不一致：人工处理
                    alertService.sendAlert("对账异常需人工处理", record);
                }
                record.setHandleStatus("PROCESSED");
            } catch (Exception e) {
                log.error("[异常处理失败] tradeNo={}", record.getTradeNo(), e);
                record.setHandleStatus("FAILED");
                record.setErrorMsg(e.getMessage());
            }
        }
        
        // 保存对账结果
        reconciliationResultMapper.insert(result);
    }
    
    // 掉单补偿：主动更新本地订单
    private void compensateOrder(AbnormalRecord record) {
        ThirdPartyTrade third = record.getThirdPartyTrade();
        
        // 查询关联订单
        Order order = orderMapper.findByThirdPartyOrderId(third.getOutTradeNo());
        if (order == null) {
            // 查不到订单，可能是数据问题，创建新的支付流水记录
            PaymentTransaction tx = new PaymentTransaction();
            tx.setTradeNo(third.getTradeNo());
            tx.setOrderId(third.getOutTradeNo());
            tx.setAmount(third.getAmount());
            tx.setStatus("SUCCESS");
            tx.setPayTime(third.getPayTime());
            tx.setReconciled(true);
            transactionMapper.insert(tx);
            
            alertService.sendAlert("无关联订单的支付记录", third);
            return;
        }
        
        // 更新订单状态
        if (!"PAID".equals(order.getStatus())) {
            orderMapper.updatePayStatus(order.getId(), "PAID", 
                third.getTradeNo(), third.getAmount());
            log.info("[掉单补偿成功] orderId={}, tradeNo={}", 
                order.getId(), third.getTradeNo());
        }
    }
}
```

## 面试加分话术

> "第三方支付回调不可靠，这是行业共识。我们的设计原则是 **'永远不要把回调当作唯一真相'**。
> 
> 具体实现：
> 1. **回调层**：统一接入入口 + 幂等校验 + 状态机保护
> 2. **主动查询**：5 分钟内未收到回调，立刻主动查询；查不到就逐步加码
> 3. **日对账**：每天凌晨跑全量对账，把"掉单"捞出来自动补偿
> 
> 线上运行半年，对账差异率从最初的 0.3% 降到了 0.01% 以下，大部分问题都被主动查询提前发现了。"

---

**一句话收尾**：支付状态同步的本质是 **"用三层保险兜底回调的不靠谱"**——回调保实时，查询保兜底，对账保最后一道防线，三者配合才能做到 99.99% 的准确率。
