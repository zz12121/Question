## 先说结论

这是比秒杀更极端的场景——**100 万 QPS 集中在 1 秒内**，核心矛盾是 **"瞬间洪峰 vs 精确库存"**。

| 挑战 | 本质 | 解决方案 |
|-----|------|---------|
| **1 秒抢完** | 流量集中，瞬时洪峰 | 分层拦截 + 预扣 + 异步 |
| **不超发** | 并发扣减，原子性要求极高 | Redis Lua + 数据库乐观锁双重保证 |
| **不崩溃** | 系统过载 | 限流熔断 + 弹性扩容 |

**一句话答案**：通过 CDN 页面分离 → Redis 原子预扣（Lua）→ MQ 异步落库 → 数据库乐观锁兜底，四层防护实现精确发券。

## 二、核心代码实现

### 2.1 Redis Lua 原子扣券（关键！）

```java
@Service
public class CouponGrabService {
    
    @Autowired private RedisTemplate<String, Object> redis;
    @Autowired private RabbitTemplate rabbitTemplate;
    
    // Redis 连接
    @Autowired private RedisConnectionFactory connectionFactory;
    
    /**
     * Lua 脚本：原子性检查 + 扣减 + 标记
     */
    private static final String GRAB_SCRIPT = """
        -- KEYS[1]: 库存 key
        -- KEYS[2]: 已抢用户 set key
        -- ARGV[1]: userId
        -- ARGV[2]: 券模板ID（用于 MQ 消息）
        -- ARGV[3]: 时间戳
        
        -- 1. 检查用户是否已抢过
        local hasBought = redis.call('SISMEMBER', KEYS[2], ARGV[1])
        if hasBought == 1 then
            return {-1, 'ALREADY_GRABBED'}  -- 已抢过
        end
        
        -- 2. 检查库存
        local stock = tonumber(redis.call('GET', KEYS[1]))
        if stock == nil or stock <= 0 then
            return {-2, 'SOLD_OUT'}  -- 已抢完
        end
        
        -- 3. 原子扣减库存
        redis.call('DECR', KEYS[1])
        
        -- 4. 标记用户已抢（不设置过期，抢完后清理）
        redis.call('SADD', KEYS[2], ARGV[1])
        
        -- 5. 返回成功 + 剩余库存
        local remaining = redis.call('GET', KEYS[1])
        return {1, remaining}  -- 成功
        """;
    
    /**
     * 抢券核心逻辑
     */
    public GrabResult grabCoupon(Long userId, Long activityId) {
        String stockKey = "coupon:stock:" + activityId;
        String boughtKey = "coupon:bought:" + activityId;
        String statusKey = "coupon:status:" + activityId;
        
        // 1. 快速检查活动状态
        String status = (String) redis.opsForValue().get(statusKey);
        if (!"STARTED".equals(status)) {
            return GrabResult.fail("活动未开始");
        }
        
        // 2. 执行 Lua 脚本（保证原子性）
        RedisScript<List> script = new RedisScript<>(GRAB_SCRIPT, List.class);
        List result = redis.execute(script, 
            List.of(stockKey, boughtKey), 
            userId.toString(), 
            activityId.toString(),
            String.valueOf(System.currentTimeMillis()));
        
        Integer code = (Integer) result.get(0);
        String message = (String) result.get(1);
        
        if (code == -1) {
            return GrabResult.fail("您已抢过该券");
        }
        if (code == -2) {
            return GrabResult.fail("券已抢完");
        }
        
        // 3. 发送 MQ 消息（异步落库）
        CouponGrabMessage message = new CouponGrabMessage();
        message.setUserId(userId);
        message.setActivityId(activityId);
        message.setGrabTime(System.currentTimeMillis());
        message.setRemainingStock(Integer.parseInt(message));
        
        rabbitTemplate.convertAndSend("coupon.grab.exchange", 
            "coupon.grab." + activityId, message);
        
        return GrabResult.success("抢券成功", Integer.parseInt(message));
    }
}
```

### 2.2 MQ 消费者：异步落库

```java
@Service
@Slf4j
public class CouponConsumerService {
    
    @Autowired private CouponUserMapper couponUserMapper;
    @Autowired private CouponActivityMapper activityMapper;
    @Autowired private RedisTemplate<String, Object> redis;
    
    /**
     * 消费抢券消息，落库
     */
    @RabbitListener(
        queues = "coupon.grab.queue",
        concurrency = "20-50",  // 弹性消费
        prefetch = 100
    )
    public void handleGrabMessage(CouponGrabMessage message, Channel channel,
                                  @Header(AmqpHeaders.DELIVERY_TAG) long tag) {
        try {
            // 1. 幂等检查
            CouponUser existed = couponUserMapper
                .findByUserAndActivity(message.getUserId(), message.getActivityId());
            
            if (existed != null) {
                log.warn("[重复消费，跳过] userId={}, activityId={}", 
                    message.getUserId(), message.getActivityId());
                channel.basicAck(tag, false);
                return;
            }
            
            // 2. 开启事务写入
            CouponUser couponUser = new CouponUser();
            couponUser.setUserId(message.getUserId());
            couponUser.setActivityId(message.getActivityId());
            couponUser.setCouponTemplateId(message.getTemplateId());
            couponUser.setGrabTime(message.getGrabTime());
            couponUser.setStatus("UNUSED");
            couponUser.setCreateTime(LocalDateTime.now());
            
            couponUserMapper.insert(couponUser);
            
            // 3. 更新活动已抢数量
            activityMapper.incrementGrabCount(message.getActivityId());
            
            // 4. 确认消息
            channel.basicAck(tag, false);
            
            log.info("[券落库成功] userId={}, activityId={}, remaining={}", 
                message.getUserId(), message.getActivityId(), message.getRemainingStock());
            
        } catch (DuplicateKeyException e) {
            // 唯一键冲突，说明已落库，直接确认
            channel.basicAck(tag, false);
        } catch (Exception e) {
            log.error("[券落库失败] message={}", message, e);
            // 拒绝消息，重新入队
            channel.basicNack(tag, false, true);
        }
    }
}
```

// 分布式限流（Redis 实现）
@Service
public class DistributedRateLimiter {
    
    private static final String RATE_LIMIT_PREFIX = "rate:limit:coupon:";
    
    /**
     * 令牌桶算法限流
     */
    public boolean tryAcquire(Long userId, Long activityId) {
        String key = RATE_LIMIT_PREFIX + userId + ":" + activityId;
        
        String lua = """
            local key = KEYS[1]
            local limit = tonumber(ARGV[1])
            local window = tonumber(ARGV[2])
            local current = tonumber(redis.call('get', key) or '0')
            
            if current >= limit then
                return 0  -- 被限流
            else
                redis.call('incr', key)
                if current == 0 then
                    redis.call('expire', key, window)
                end
                return 1  -- 通过
            end
            """;
        
        Long result = redis.execute(lua, List.of(key), "1", "1");
        return result == 1;
    }
}

**推荐组合**：Redis Lua + MQ + 乐观锁兜底，理论支持 **50 万+ QPS**

## 面试加分话术

> "100 万张券 1 秒抢完，核心难点是 **把 100 万 QPS 的写操作压缩到 Redis 单线程能处理的量级**。
> 
> 我的方案是 **'三层保险'**：
> 1. **网关限流**：把 100 万挡成 10 万，IP + 用户双重限制
> 2. **Redis Lua 原子扣减**：10 万再压缩到几千，因为库存扣到 0 后直接返回
> 3. **MQ 异步落库**：扣成功的几千条消息，异步写入数据库，不阻塞前端
> 
> 实际压测结果：TPS 从纯数据库的 500 提升到了 Redis Lua 的 **8 万**，核心就是 **让 99% 的请求在 Redis 层就被处理掉**，数据库只处理最后那 1%。"

---

**一句话收尾**：整点抢券的本质是 **"用 Redis 的单线程换精确性，用 MQ 的异步换吞吐量"**，三层拦截 + 三保险防超发 + 熔断降级，才能做到既快又准。
