## 先说结论

**推荐方案：先更新数据库，再删除缓存（Cache Aside + 删除而非更新）**

| 方案 | 原理 | 风险 | 推荐度 |
|-----|------|------|-------|
| **先更缓存再更DB** | 缓存命中高 | 脏数据难处理 | ❌ 不推荐 |
| **先更DB再删缓存** | 保证数据准确 | 可能短暂缓存穿透 | ✅ **推荐** |
| **先删缓存再更DB** | 避免脏读 | 缓存穿透风险高 | ❌ 不推荐 |
| **更新缓存再删DB** | 无实际意义 | 数据丢失 | ❌ 禁止 |

**一句话答案**：先更新 MySQL 成功，再删除 Redis 缓存，即使删失败也有补偿机制（如延迟双删、对账）兜底。

## 二、核心问题：两步之间挂了怎么办？

### 问题场景分析

```
写操作：
① 更新 MySQL 成功
② 删除 Redis 缓存
③ 网络抖动，删除失败

结果：MySQL=新值，Redis=旧值 → 数据不一致！
```

### 解决方案一：延迟双删

```java
@Service
public class CacheService {
    
    @Autowired private RedisTemplate<String, Object> redis;
    @Autowired private ProductMapper productMapper;
    
    /**
     * 延迟双删：解决先更DB再删缓存失败的问题
     */
    @Transactional
    public void updateProduct(Long productId, BigDecimal newPrice) {
        // 1. 先删除缓存（让其他读请求不命中脏缓存）
        String cacheKey = "product:" + productId;
        redis.delete(cacheKey);
        
        // 2. 更新数据库
        productMapper.updatePrice(productId, newPrice);
        
        // 3. 延迟再次删除缓存（异步，防止步骤2执行期间有脏数据写入缓存）
        // 延迟时间：预估最大业务耗时 + 网络延迟
        CompletableFuture.runAsync(() -> {
            try {
                Thread.sleep(500);  // 500ms 延迟
                redis.delete(cacheKey);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        });
    }
}
```

### 解决方案二：消息队列可靠删除

```java
@Service
public class CacheDeleteService {
    
    @Autowired private RocketMQTemplate rocketMQ;
    
    /**
     * 缓存删除失败时，发送 MQ 消息补偿
     */
    public void deleteCacheWithCompensation(String cacheKey) {
        try {
            redisTemplate.delete(cacheKey);
        } catch (Exception e) {
            log.error("[缓存删除失败] key={}, 发送补偿消息", cacheKey, e);
            
            // 发送延迟消息补偿删除
            CacheDeleteMessage message = new CacheDeleteMessage();
            message.setCacheKey(cacheKey);
            message.setRetryCount(0);
            message.setCreateTime(System.currentTimeMillis());
            
            // 延迟 5 秒投递（给系统恢复时间）
            rocketMQ.asyncSend("cache.delete.topic", message, new SendCallback() {
                @Override
                public void onSuccess(SendResult result) {
                    log.info("[缓存删除补偿消息发送成功] key={}", cacheKey);
                }
                
                @Override
                public void onException(Throwable e) {
                    log.error("[缓存删除补偿消息发送失败] key={}", cacheKey, e);
                }
            });
        }
    }
}

/**
 * MQ 消费者：处理缓存删除补偿
 */
@Service
public class CacheDeleteConsumer {
    
    @RocketMQMessageListener(
        topic = "cache.delete.topic",
        consumerGroup = "cache-delete-consumer",
        concurrency = "5"
    )
    public void handleDelete(CacheDeleteMessage message) {
        String cacheKey = message.getCacheKey();
        
        try {
            Boolean deleted = redisTemplate.delete(cacheKey);
            
            if (Boolean.TRUE.equals(deleted)) {
                log.info("[缓存删除成功] key={}", cacheKey);
            } else {
                // 缓存已不存在，忽略
                log.info("[缓存不存在，跳过删除] key={}", cacheKey);
            }
        } catch (Exception e) {
            log.error("[缓存删除失败，重试] key={}, retry={}", 
                cacheKey, message.getRetryCount(), e);
            
            // 重试次数未超限，延迟重发
            if (message.getRetryCount() < 3) {
                message.setRetryCount(message.getRetryCount() + 1);
                // 重新发送消息（指数退避）
                scheduleRetry(message, message.getRetryCount() * 1000);
            } else {
                // 重试次数超限，告警人工处理
                alertService.sendAlert("缓存删除失败", message);
            }
        }
    }
}
```

### 解决方案三：订阅 MySQL Binlog 异步更新缓存

```java
/**
 * Canal 方案：通过 Binlog 同步更新缓存
 * 优点：完全异步，不影响主流程
 * 缺点：引入新组件，增加复杂度
 */
@Component
@Slf4j
public class CanalCacheSyncService {
    
    @Autowired private RedisTemplate<String, Object> redis;
    
    @RabbitListener(queues = "canal.product.queue")
    public void handleBinlogEvent(CanalMessage message) {
        String tableName = message.getTableName();
        String eventType = message.getEventType();  // INSERT/UPDATE/DELETE
        
        if (!"product".equals(tableName)) {
            return;
        }
        
        Map<String, Object> data = message.getData();
        Long productId = (Long) data.get("id");
        String cacheKey = "product:" + productId;
        
        try {
            switch (eventType) {
                case "INSERT":
                case "UPDATE":
                    // 直接写入缓存（Binlog 是数据已落库后的变更事件）
                    redis.opsForValue().set(cacheKey, JSON.toJSONString(data));
                    break;
                    
                case "DELETE":
                    // 删除缓存
                    redis.delete(cacheKey);
                    break;
            }
        } catch (Exception e) {
            log.error("[Binlog同步缓存失败] key={}, event={}", cacheKey, eventType, e);
            // 失败重试或告警
        }
    }
}
```

## 三、完整代码实现

### Cache-Aside 标准实现

```java
@Service
@Slf4j
public class ProductService {
    
    @Autowired private ProductMapper productMapper;
    @Autowired private RedisTemplate<String, Object> redis;
    
    private static final String PRODUCT_KEY_PREFIX = "product:";
    
    /**
     * 查询：Cache Aside 读
     */
    public Product getProduct(Long productId) {
        String cacheKey = PRODUCT_KEY_PREFIX + productId;
        
        // 1. 先查缓存
        Object cached = redis.opsForValue().get(cacheKey);
        if (cached != null) {
            log.debug("[缓存命中] productId={}", productId);
            return JSON.parseObject(cached.toString(), Product.class);
        }
        
        // 2. 缓存未命中，查数据库
        log.debug("[缓存未命中，查DB] productId={}", productId);
        Product product = productMapper.selectById(productId);
        
        if (product != null) {
            // 3. 写入缓存（设置过期时间，防止脏数据永久留存）
            redis.opsForValue().set(cacheKey, JSON.toJSONString(product), 
                30, TimeUnit.MINUTES);
        }
        
        return product;
    }
    
    /**
     * 更新：Cache Aside 写（延迟双删 + MQ 补偿）
     */
    @Transactional
    public void updateProduct(Long productId, BigDecimal newPrice) {
        String cacheKey = PRODUCT_KEY_PREFIX + productId;
        
        try {
            // 1. 删除缓存（让读请求不命中旧缓存）
            redis.delete(cacheKey);
            log.debug("[删除缓存] productId={}", productId);
            
            // 2. 更新数据库
            int affected = productMapper.updatePrice(productId, newPrice);
            if (affected == 0) {
                throw new BizException("商品不存在");
            }
            
            // 3. 延迟再次删除缓存（兜底）
            CompletableFuture.runAsync(() -> {
                try {
                    Thread.sleep(500);
                    redis.delete(cacheKey);
                    log.debug("[延迟删除缓存] productId={}", productId);
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
            });
            
        } catch (BizException e) {
            throw e;
        } catch (Exception e) {
            log.error("[更新商品失败] productId={}", productId, e);
            // 发送 MQ 补偿
            sendCacheDeleteMessage(cacheKey);
            throw new BizException("更新失败");
        }
    }
    
    /**
     * 删除：同样使用延迟双删
     */
    @Transactional
    public void deleteProduct(Long productId) {
        String cacheKey = PRODUCT_KEY_PREFIX + productId;
        
        try {
            redis.delete(cacheKey);
            productMapper.deleteById(productId);
            
            // 延迟双删
            CompletableFuture.runAsync(() -> {
                try {
                    Thread.sleep(500);
                    redis.delete(cacheKey);
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
            });
        } catch (Exception e) {
            log.error("[删除商品失败] productId={}", productId, e);
            sendCacheDeleteMessage(cacheKey);
            throw new BizException("删除失败");
        }
    }
    
    private void sendCacheDeleteMessage(String cacheKey) {
        CacheDeleteMessage msg = new CacheDeleteMessage();
        msg.setCacheKey(cacheKey);
        msg.setRetryCount(0);
        rocketMQ.asyncSend("cache.delete.topic", msg);
    }
}
```

## 五、面试加分话术

> "这是一个经典的缓存一致性问题，我的答案是 **'先更数据库，再删缓存，配合延迟双删兜底'**。
> 
> 为什么不是先更缓存？
> - 因为缓存更新成功后，DB 更新失败，缓存里的数据就是脏数据，而且很难自动恢复
> 
> 为什么删除而不是更新缓存？
> - 因为更新的逻辑复杂（需要先查再写），删除更简单
> - 而且更新容易出现并发问题（ABA 问题）
> 
> 两步之间挂了怎么办？
> - **延迟双删**：500ms 后再删一次
> - **MQ 补偿**：删除失败发消息，消费后重试
> - **定时对账**：兜底方案，凌晨扫数据差异
> 
> 实际项目中，我们用 **延迟双删 + MQ 补偿 + 每日对账** 三重保障，一致性从 99.9% 提升到了 99.99%。"

### Q8-1: 缓存过期时间设置多长合适？

```java
// 缓存过期时间设计
public class CacheExpireStrategy {
    
    // 不同类型数据的过期时间
    public static final Duration USER_SESSION = Duration.ofHours(2);
    public static final Duration PRODUCT_INFO = Duration.ofMinutes(30);
    public static final Duration CONFIG = Duration.ofHours(1);
    public static final Duration HOT_PRODUCT = Duration.ofMinutes(5);  // 热点数据短过期
    
    // 热点数据：过期时间 = 随机 5-10 分钟，防止雪崩
    public Duration getHotProductExpire() {
        return Duration.ofMinutes(5 + new Random().nextInt(5));
    }
    
    // 普通数据：过期时间 = 基准时间 + 随机偏移
    public Duration getNormalExpire(Duration base) {
        long jitter = new Random().nextInt((int) base.toMinutes() / 4);
        return base.plusMinutes(jitter);
    }
}
```

### Q8-2: 如何做缓存一致性监控？

```java
/**
 * 缓存一致性监控：对账任务
 */
@Component
@Slf4j
public class CacheConsistencyMonitor {
    
    @Scheduled(cron = "0 0 3 * * ?")  // 每天凌晨3点
    public void checkConsistency() {
        log.info("[缓存一致性对账开始]");
        
        // 1. 扫描热门商品
        List<Long> hotProductIds = productMapper.selectHotProductIds(1000);
        
        int inconsistent = 0;
        for (Long productId : hotProductIds) {
            // 2. 查缓存
            Product cached = getCachedProduct(productId);
            // 3. 查数据库
            Product dbProduct = productMapper.selectById(productId);
            
            // 4. 比对
            if (!equals(cached, dbProduct)) {
                log.warn("[缓存不一致] productId={}, cache={}, db={}", 
                    productId, cached, dbProduct);
                
                // 5. 修复
                refreshCache(productId);
                inconsistent++;
            }
        }
        
        log.info("[缓存一致性对账完成] 检查={}, 不一致={}", 
            hotProductIds.size(), inconsistent);
        
        // 6. 告警
        if (inconsistent > 100) {
            alertService.sendAlert("缓存不一致率过高", inconsistent);
        }
    }
}
```
