### 1.3 核心代码示例

#### 1. ForkJoinPool 基础用法

```java
import java.util.concurrent.*;

public class ForkJoinDemo {
    
    // ====== 继承 RecursiveTask（有返回值）======
    static class SumTask extends RecursiveTask<Long> {
        
        private final long[] array;
        private final int start;
        private final int end;
        
        private static final int THRESHOLD = 10_000;  // 拆分阈值
        
        public SumTask(long[] array, int start, int end) {
            this.array = array;
            this.start = start;
            this.end = end;
        }
        
        @Override
        protected Long compute() {
            int length = end - start;
            
            // 小任务：直接计算
            if (length <= THRESHOLD) {
                long sum = 0;
                for (int i = start; i < end; i++) {
                    sum += array[i];
                }
                return sum;
            }
            
            // 大任务：拆分
            int mid = (start + end) / 2;
            
            SumTask leftTask = new SumTask(array, start, mid);
            SumTask rightTask = new SumTask(array, mid, end);
            
            // Fork：异步提交左任务到队列
            leftTask.fork();
            
            // Compute：同步计算右任务
            Long rightResult = rightTask.compute();
            
            // Join：等待并合并左任务结果
            Long leftResult = leftTask.join();
            
            return leftResult + rightResult;
        }
    }
    
    // ====== 使用示例 ======
    public static void main(String[] args) throws Exception {
        long[] array = new long[1_000_000];
        for (int i = 0; i < array.length; i++) {
            array[i] = i + 1;
        }
        
        ForkJoinPool pool = new ForkJoinPool();
        ForkJoinTask<Long> task = new SumTask(array, 0, array.length);
        
        Long result = pool.invoke(task);
        System.out.println("Sum: " + result);  // 500000500000
        
        pool.shutdown();
    }
}
```

#### 2. RecursiveAction（无返回值）

```java
import java.util.concurrent.*;

public class RecursiveActionDemo {
    
    // ====== 并行数组排序 ======
    static class SortTask extends RecursiveAction {
        
        private final long[] array;
        private final int start;
        private final int end;
        
        private static final int THRESHOLD = 1000;
        
        public SortTask(long[] array, int start, int end) {
            this.array = array;
            this.start = start;
            this.end = end;
        }
        
        @Override
        protected void compute() {
            int length = end - start;
            
            if (length <= THRESHOLD) {
                Arrays.sort(array, start, end);
                return;
            }
            
            int mid = (start + end) / 2;
            
            SortTask leftTask = new SortTask(array, start, mid);
            SortTask rightTask = new SortTask(array, mid, end);
            
            // 并行执行两个子任务
            invokeAll(leftTask, rightTask);
        }
    }
}
```

```java
import java.util.stream.*;
import java.util.concurrent.*;

public class CommonPoolDemo {
    
    public static void main(String[] args) {
        
        // ====== 使用 commonPool() ======
        // Stream 并行操作默认使用 commonPool
        ForkJoinPool commonPool = ForkJoinPool.commonPool();
        System.out.println("CommonPool size: " + commonPool.getParallelism());
        // 输出：Runtime.getRuntime().availableProcessors() - 1
        
        // ====== 并行 Stream 使用 commonPool ======
        long sum = LongStream.rangeClosed(1, 1_000_000)
            .parallel()  // 使用 ForkJoinPool.commonPool()
            .filter(n -> n % 2 == 0)
            .sum();
        
        // ====== 计算密集型任务 ======
        List<Integer> result = IntStream.range(0, 1000)
            .parallel()
            .mapToObj(i -> expensiveComputation(i))
            .collect(Collectors.toList());
        
        // commonPool 会自动用于所有并行 Stream
    }
    
    private static Integer expensiveComputation(int i) {
        // 计算密集型逻辑
        return i * i;
    }
}
```

```
│  │                         双端队列操作                                      │   │
│  │                                                                          │   │
│  │   Thread-1 的 Deque:                                                     │   │
│  │                                                                          │   │
│  │   push()      pop()        steal()                                      │   │
│  │     │          │              │                                         │   │
```

**为什么从尾部偷：**

```
偷取策略原理：
┌─────────────────────────────────────────────────────────────────────────────────┐
│                                                                                  │
│  假设队列中有 [大任务A, 子任务1, 子任务2, 子任务3, 子任务4]                        │
│                      HEAD                                    TAIL                │
│                                                                                  │
│  从 TAIL 偷的好处：                                                              │
│  • TAIL 是最后加入的任务                                                         │
│  • 大任务被 fork() 后，子任务在队列尾部                                         │
│  • 从 TAIL 偷更容易拿到完整的子任务                                              │
│  • 窃取者拿到子任务后，自己的队列也倾向于从 TAIL 追加                              │
│                                                                                  │
│  结果：                                                                          
│  • Worker-1 执行 HEAD（子任务2）                                                │
│  • Worker-2 从 Worker-1 TAIL 偷到子任务3                                        │
│  • 双方都拿到了相对完整的任务，负载均衡                                           │
│                                                                                  │
└─────────────────────────────────────────────────────────────────────────────────┘
```

### 2.2 🔥 追问 2：ForkJoinPool.commonPool() 有什么坑？

> **A:**

**commonPool 的五大坑：**

```

│                         commonPool() 常见问题                                    │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                  │
│  ┌─────────────────────────────────────────────────────────────────────────┐   │
│  │  坑1: 线程数固定为 CPU核心数-1，无法配置                                  │   │
│  │                                                                          │   │

│  │                                                                          │   │

│                                                                                  │
│  ┌─────────────────────────────────────────────────────────────────────────┐   │

│  └─────────────────────────────────────────────────────────────────────────┘   │
│                                                                                  │

│  │  坑3: commonPool 默认是 daemon 线程                                     │   │

│  │  • UncaughtExceptionHandler                                              │   │

│  │                                                                          │   │
│  └─────────────────────────────────────────────────────────────────────────┘   │
│                                                                                  │
│  ┌─────────────────────────────────────────────────────────────────────────┐   │
│  │  坑5: 阻塞任务会导致其他任务饥饿                                          │   │
│  │                                                                          │   │
│  │  commonPool 线程数有限，如果某个任务内部有 BlockingIO                      │   │
```

```java
public class CommonPoolBestPractice {
    
    // ✅ 推荐：自定义 ForkJoinPool
    private static final ForkJoinPool pool = new ForkJoinPool(
        Runtime.getRuntime().availableProcessors(),  // 线程数
        ForkJoinPool.defaultForkJoinWorkerThreadFactory,  // 线程工厂
        null,  // UncaughtExceptionHandler
        false  // asyncMode=false（任务公平排队）
    );
    
    public static <T> T invoke(Task<T> task) {
        try {
            return pool.invoke(task);
        } finally {
            // 根据需要决定是否 shutdown
            // pool.shutdown();
        }
    }
}
```

### 2.3 🔥 追问 3：什么任务不适合 ForkJoinPool？

> **A:**

**不适合场景分析：**

| 场景 | 原因 | 替代方案 |
|------|------|---------|
| **IO密集型任务** | 线程阻塞等待IO，不释放，浪费资源 | ThreadPoolExecutor + 更大线程数 |
| **小数据量/小任务** | 任务拆分和调度开销 > 并行收益 | 单线程顺序执行 |
| **任务有强依赖** | 无法分叉，后续任务必须等前序完成 | 单线程或固定依赖链 |
| **任务执行时间差异大** | 短任务线程等长任务，效率低 | 预估时间后分组 |
| **需要线程上下文** | ThreadLocal 数据丢失 | ThreadPoolExecutor |

**代码示例：**

```java
public class UnsuitableTasks {
    
    // ====== ❌ 不适合：IO密集型 ======
    static class IOTask extends RecursiveTask<String> {
        @Override
        protected String compute() {
            // 模拟网络IO
            try {
                String result = httpCall();  // 阻塞 100ms
                return result;
            } catch (Exception e) {
                return "";
            }
        }
        // ⚠️ 线程在等待IO时无法做其他事，浪费线程资源
    }
    
    // ✅ 正确做法：
    // 使用 ThreadPoolExecutor + 更大的线程数
    ExecutorService ioExecutor = new ThreadPoolExecutor(
        50, 100, 60L, TimeUnit.SECONDS,
        new LinkedBlockingQueue<>(1000)
    );
    
    
    // ====== ❌ 不适合：强依赖任务 ======
    static class DependentTask extends RecursiveTask<Integer> {
        @Override
        protected Integer compute() {
            // 任务2必须等任务1完成
            Task1Result r1 = task1.compute();  // 必须先完成
            
            Task2Result r2 = task2.compute();  // 依赖 r1
            // ⚠️ task2.compute() 已经在同步等待，无法分叉
            
            return combine(r1, r2);
        }
    }
    
    
    // ====== ❌ 不适合：小任务拆分 ======
    static class TinyTask extends RecursiveTask<Integer> {
        @Override
        protected Integer compute() {
            if (任务太小) {
                // 直接计算，别拆了
                return 直接计算();
            }
            
            // 拆分
            leftTask.fork();
            rightTask.compute();
            // ⚠️ 拆分开销 > 并行收益
        }
    }
    
    
    // ====== ✅ 适合：计算密集型分治 ======
    static class ComputeTask extends RecursiveTask<Long> {
        @Override
        protected Long compute() {
            if (数据量 <= 阈值) {
                return 直接计算();  // 纯CPU计算
            }
            
            leftTask.fork();    // 异步
            rightTask.compute(); // 同步
            return leftTask.join() + rightTask.compute();
        }
    }
}
```

### 2.4 🔥 追问 4：ForkJoinPool 和 ThreadPoolExecutor 如何选择？

> **A:**

**选型决策树：**

```
┌─────────────────────────────────────────────────────────────────────────────────┐
```

**详细对比：**

| 维度 | ForkJoinPool | ThreadPoolExecutor |
|------|-------------|-------------------|
| **设计理念** | 分治并行 + 工作窃取 | 生产者-消费者 |
| **任务队列** | 每线程独立 Deque | 共享任务队列 |
| **任务类型** | ForkJoinTask（可分叉） | Runnable/Callable |
| **适用场景** | 递归分治、MapReduce | 任意异步任务 |
| **线程饥饿** | 不容易（工作窃取） | 容易（队列为空） |
| **适合任务大小** | 计算密集、长时间 | 混合类型 |
| **线程利用率** | 高（互相偷） | 一般 |

**实际选型建议：**

```java
public class SelectionGuide {
    
    // 场景1: 并行归约/累加（求和、找最大/最小）
    // ✅ ForkJoinPool
    Long sum = LongStream.rangeClosed(1, 1_000_000)
        .parallel()
        .sum();
    
    // 场景2: 批量HTTP请求
    // ❌ ForkJoinPool → ✅ ThreadPoolExecutor
    ExecutorService httpPool = new ThreadPoolExecutor(
        50, 200, 60L, TimeUnit.SECONDS,
        new LinkedBlockingQueue<>(1000)
    );
    
    // 场景3: 数据库批量处理
    // ✅ ThreadPoolExecutor（IO等待）
    // 或分批 ForkJoinPool（每批足够大）
    
    // 场景4: 树形结构遍历
    // ✅ ForkJoinPool
    new ForkJoinPool().invoke(new TreeWalkTask(root));
    
    // 场景5: 异步事件处理
    // ✅ ThreadPoolExecutor
    @Async
    public void handleEvent() { }
```

**使用示例：**

```java
public class AsyncModeDemo {
    
    public static void main(String[] args) {
        
        // ====== LIFO 模式（默认）：父子任务强依赖 ======
        ForkJoinPool pool1 = new ForkJoinPool();
        // Task A fork()
        // Task B fork()
        // Task C fork()
        // 执行顺序：C → B → A（后进先出，父子任务保证）
        
        
        // ====== FIFO 模式：独立任务流水线 ======
        ForkJoinPool pool2 = new ForkJoinPool(
            Runtime.getRuntime().availableProcessors(),
            ForkJoinPool.defaultForkJoinWorkerThreadFactory,
            null,
            true  // asyncMode = true
        );
        
        // 适用场景：事件流处理、消息处理
        pool2.submit(() -> {
            processStream(Stream.iterate(0, i -> i + 1).limit(1000));
        });
    }
    
    // ====== 流水线处理（适合 FIFO）======
    static void processStream(Stream<Integer> stream) {
        ForkJoinPool pool = new ForkJoinPool(4);
        
        pool.submit(() -> 
            stream.parallel()
                .filter(i -> i % 2 == 0)
                .map(i -> i * 2)
                .forEach(System.out::println)
        ).join();
    }
}
```

## 3. 一句话总结

> **ForkJoinPool 是专为"分治并行"设计的线程池，每个工作线程有独立的双端队列，通过工作窃取算法（空闲线程从其他线程队列尾部偷任务）实现负载均衡。commonPool() 是全局共享的默认池，线程数为 CPU-1，存在线程饥饿、daemon 线程等问题。ForkJoinPool 适合计算密集型递归任务，不适合 IO 密集型或任务间有强依赖的场景。**

## 4. 面试加分话术

> "ForkJoinPool 我之前用它优化过一个报表计算系统。
> 
> 需求是：一个大数据聚合查询，要统计 1000 万用户的行为数据，拆分成多个子任务并行计算。用 ThreadPoolExecutor 的话，所有任务都在一个共享队列里，抢任务时有锁竞争，CPU 利用率上不去。
> 
> 后来改用 ForkJoinPool，把数据按用户 ID 范围拆分：
> ```java
> @Override
> protected ReportResult compute() {
>     if (rangeSize < 阈值) {
>         return 直接计算();
>     }
>     int mid = start + rangeSize / 2;
>     leftTask.fork();   // 异步提交
>     rightTask.compute(); // 同步计算
>     return leftTask.join() + rightTask.compute();
> }
> ```
> 
> 优化后计算时间从 45 秒降到了 12 秒。
> 
> 但这里有个坑：**commonPool() 不要用于生产环境**。之前团队有个同学用并行流做了个批量处理，结果所有接口的并行流都竞争同一个 commonPool，导致响应时间不稳定。后来改成自定义 ForkJoinPool就好了。
> 
> 还有个小技巧：**阈值设置很重要**。阈值太小会导致任务太碎，调度开销大于计算收益；阈值太大会导致并行度不够。经验值是每个子任务至少 1000~5000 个元素。"
