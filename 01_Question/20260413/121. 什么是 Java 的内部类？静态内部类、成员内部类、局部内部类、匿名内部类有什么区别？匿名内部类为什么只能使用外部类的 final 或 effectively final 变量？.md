## 1. 基础流程

### 1.1 内部类分类概览

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                              Java 内部类体系                                      │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                  │
│                              内部类 (Inner Class)                                │
│                                     │                                            │
│         ┌───────────────────────────┼───────────────────────────┐               │
│         │                           │                           │               │
│         ▼                           ▼                           ▼               │
│  ┌──────────────┐         ┌──────────────┐           ┌──────────────┐          │
│  │  静态内部类    │         │  非静态内部类  │           │   局部内部类  │          │
│  │ (成员级别)    │         │              │           │              │          │
│  │              │         │  ┌──────────┐│           │              │          │
│  │ 不需要外部    │         │  │成员内部类││           │  定义在方法   │          │
│  │ 实例即可访问  │         │  └──────────┘│           │  内部         │          │
│  └──────────────┘         └──────────────┘           └──────┬───────┘          │
│                                                            │                     │
│                                                           ▼                     │
│                                                   ┌──────────────┐            │
│                                                   │  匿名内部类   │            │
│                                                   │              │            │
│                                                   │ 无名字的局部  │            │
│                                                   │ 内部类        │            │
│                                                   └──────────────┘            │
│                                                                                  │
└─────────────────────────────────────────────────────────────────────────────────┘
```

### 1.2 四种内部类对比

| 维度 | 静态内部类 | 成员内部类 | 局部内部类 | 匿名内部类 |
|------|-----------|-----------|-----------|-----------|
| **定义位置** | 类方法外，`static` 修饰 | 类方法外，无 `static` | 方法/代码块内部 | 方法内部，无名字 |
| **外部类引用** | ❌ 不持有 | ✅ 持有（隐式引用） | ✅ 持有 | ✅ 持有 |
| **访问修饰符** | `public/protected/private/默认` | 同上 | 无（默认包级） | 无 |
| **创建方式** | `new Outer.Inner()` | `outer.new Inner()` | 方法内 `new` | 接口/类后直接 `{}` |
| **访问外部成员** | 只能访问静态成员 | 所有成员（含 private） | 所有成员 | 所有成员 |
| **能否声明static** | ✅ 可以有静态成员 | ❌ 不能有静态成员 | ❌ 不能有静态成员 | ❌ 不能有静态成员 |
| **编译产物** | `Outer$Inner.class` | `Outer$Inner.class` | `Outer$1Inner.class` | `Outer$1.class` |

### 1.3 核心代码示例

#### 1. 静态内部类

```java
public class Outer {
    
    private static int staticOuterField = 100;
    private int instanceOuterField = 200;
    
    /**
     * 静态内部类：
     * - 不依赖外部类实例，可直接创建
     * - 只能访问外部类的静态成员
     */
    public static class StaticInner {
        private int innerField = 300;
        
        public void test() {
            // ✅ 访问外部静态成员
            System.out.println(staticOuterField);  // OK
            
            // ❌ 编译错误！不能访问实例成员
            // System.out.println(instanceOuterField);  // Error!
        }
        
        // ✅ 静态内部类可以有静态方法
        public static void staticMethod() {
            System.out.println("静态方法");
        }
    }
}

// 创建方式：不依赖外部类实例
Outer.StaticInner inner = new Outer.StaticInner();
inner.test();
Outer.StaticInner.staticMethod();
```

#### 2. 成员内部类（非静态内部类）

```java
public class Outer {
    
    private int outerField = 100;
    private static int staticField = 200;
    
    /**
     * 成员内部类：
     * - 必须依赖外部类实例存在
     * - 持有外部类的引用（Outer.this）
     * - 可访问外部类的所有成员（包括 private）
     */
    public class MemberInner {
        private int innerField = 300;
        
        public void test() {
            // ✅ 访问外部类实例成员
            System.out.println(outerField);
            
            // ✅ 访问外部类静态成员
            System.out.println(staticField);
            
            // ✅ 访问外部类 private 成员
            outerPrivateMethod();
            
            // ✅ 显式引用外部类实例
            System.out.println(Outer.this.outerField);
        }
    }
    
    private void outerPrivateMethod() {
        System.out.println("Outer private method");
    }
}

// 创建方式：必须先有外部类实例
Outer outer = new Outer();
Outer.MemberInner inner = outer.new MemberInner();
inner.test();
```

#### 3. 局部内部类

```java
public class Outer {
    
    private int outerField = 100;
    
    public void method() {
        final int localVar = 200;  // final 或 effectively final
        int anotherLocal = 300;    // effectively final
        
        /**
         * 局部内部类：
         * - 定义在方法或代码块内部
         * - 作用域限定在方法内
         * - 可以访问外部类的所有成员
         * - 可以访问方法的局部变量（必须是 final 或 effectively final）
         */
        class LocalInner {
            private int innerField = 400;
            
            public void test() {
                // ✅ 访问外部类成员
                System.out.println(outerField);
                
                // ✅ 访问方法的局部变量
                System.out.println(localVar);
                System.out.println(anotherLocal);
            }
        }
        
        // 在方法内创建和使用
        LocalInner inner = new LocalInner();
        inner.test();
    }
    
    // 局部内部类也可以定义在静态方法中
    public static void staticMethod() {
        class StaticLocalInner {
            public void test() {
                // ❌ 不能访问外部实例成员
                // System.out.println(outerField);  // Error!
                
                // ✅ 可以访问静态成员
                System.out.println("Static context");
            }
        }
        
        StaticLocalInner inner = new StaticLocalInner();
        inner.test();
    }
}
```

#### 4. 匿名内部类

```java
public class AnonymousTest {
    
    public static void main(String[] args) {
        
        // ============ 匿名内部类实现接口 ============
        // 匿名内部类：没有名字的类，直接在创建实例时定义
        Runnable runnable = new Runnable() {
            @Override
            public void run() {
                System.out.println("Runnable 实现");
            }
        };
        runnable.run();
        
        // ============ 匿名内部类继承类 ============
        Thread thread = new Thread("MyThread") {
            @Override
            public void run() {
                System.out.println("继承 Thread 的匿名类");
            }
        };
        thread.start();
        
        // ============ 匿名内部类常见用法 ============
        // 事件监听
        button.addActionListener(new ActionListener() {
            @Override
            public void actionPerformed(ActionEvent e) {
                System.out.println("按钮点击");
            }
        });
        
        // 简化写法（Java 8+ Lambda）
        button.addActionListener(e -> System.out.println("按钮点击"));
    }
    
    // 作为返回值
    public Comparator<String> getComparator() {
        return new Comparator<String>() {
            @Override
            public int compare(String s1, String s2) {
                return s1.length() - s2.length();
            }
        };
        // Lambda 简化
        // return Comparator.comparingInt(String::length);
    }
}
```

## 2. 扩展追问

### 2.1 🔥 追问 1：匿名内部类为什么只能访问外部的 final 或 effectively final 变量？

> **A:**

**核心原因：生命周期不同 + 拷贝机制**

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                    匿名内部类捕获变量的原理                                      │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                  │
│  方法栈帧                                                               堆内存    │
│  ┌─────────────────────┐                                    ┌─────────────────┐ │
│  │  method() {         │                                    │                 │ │
│  │    int localVar = 10│  ──拷贝─→                          │  匿名内部类对象  │ │
│  │                      │                                    │  ┌─────────────┐│ │
│  │    new Runnable() {  │                                    │  │localVar=10  ││ │
│  │      run() {        │                                    │  │ (值拷贝)    ││ │
│  │        print(local) │                                    │  └─────────────┘│ │
│  │      }              │                                    │                 │ │
│  │    }                │                                    │  ───引用──────────│ │
│  │  }                  │                                    │                 │ │
│  └─────────────────────┘                                    └─────────────────┘ │
│       方法结束，栈帧销毁                                                常驻堆中   │
│       localVar 不存在                                                    可能被调用 │
│                                                                                  │
└─────────────────────────────────────────────────────────────────────────────────┘
```

**Java 的实现机制（字节码层面）：**

```java
// 源代码
public void method() {
    int localVar = 10;  // 局部变量
    
    Runnable r = new Runnable() {
        @Override
        public void run() {
            System.out.println(localVar);  // 使用外部变量
        }
    };
}
```

```java
// 编译后的等效代码（反编译示意）
public void method() {
    // 编译器生成"桥接变量"
    final int localVar$1 = 10;  // 匿名内部类能访问的版本
    
    Runnable r = new Runnable() {
        // 匿名内部类内部：不是直接引用局部变量，而是引用桥接变量
        private final int val$localVar;
        
        // 编译器添加的构造方法参数
        public void <init>() {
            this.val$localVar = localVar$1;
        }
        
        @Override
        public void run() {
            // 实际使用的是 val$localVar
            System.out.println(this.val$localVar);
        }
    };
}
```

**为什么必须是 final 或 effectively final？**

```
┌──────────────────────────────────────────────────────────────────────┐
│                      为什么不能修改捕获的变量？                        │
├──────────────────────────────────────────────────────────────────────┤
│                                                                       │
│  如果允许修改（假设）：                                              │
│                                                                       │
│  ┌─────────────────────────────────────────────────────────────┐     │
│  │  void method() {                                            │     │
│  │    int localVar = 10;                                        │     │
│  │                                                               │     │
│  │    new Runnable() {                                          │     │
│  │      run() { print(localVar); }  // 读                      │     │
│  │    };                                                         │     │
│  │                                                               │     │
│  │    localVar = 20;  // ← 假设允许修改                         │     │
│  │    // 此时匿名内部类看到的值是多少？10 还是 20？             │     │
│  │  }                                                            │     │
│  └─────────────────────────────────────────────────────────────┘     │
│                                                                       │
│  问题：                                                                │
│  - 方法栈帧和堆对象生命周期不同步                                      │
│  - 内部类持有的是值的拷贝                                             │
│  - 如果允许修改，会产生"写后读"的歧义                                  │
│  - JVM 实现复杂度大幅增加                                            │
│                                                                       │
│  解决方案：                                                           │
│  - 强制要求 final（Java 8 之前）                                      │
│  - 或 effectively final（Java 8+，编译时检查）                        │
│                                                                       │
└──────────────────────────────────────────────────────────────────────┘
```

**effectively final 规则（Java 8+）：**

```java
public void method() {
    int a = 10;        // effectively final（没有修改过）
    int b = 20;        // effectively final
    
    // ✅ OK - a 和 b 没有被修改
    Runnable r1 = () -> System.out.println(a + b);
    
    // ❌ Error - b 被修改了，不是 effectively final
    b = 30;  // 这行导致 b 不再是 effectively final
    Runnable r2 = () -> System.out.println(b);
    
    // ❌ Error - 匿名内部类中引用的变量必须是 effectively final
    Runnable r3 = new Runnable() {
        @Override
        public void run() {
            System.out.println(a);  // Error!
        }
    };
}
```

### 2.2 🔥 追问 2：内部类会内存泄漏吗？

> **A:**

**会！成员内部类和匿名内部类是常见的内存泄漏来源**

```
┌──────────────────────────────────────────────────────────────────────┐
│                      内部类导致的内存泄漏                              │
├──────────────────────────────────────────────────────────────────────┤
│                                                                       │
│  典型泄漏场景：                                                       │
│                                                                       │
│  ┌────────────────────────────┐                                       │
│  │  Activity / Fragment       │  持有                                 │
│  │                            │ ────┐                                 │
│  │  Handler handler = new     │     │                                 │
│  │    Handler() {            │     │ 隐式持有外部类引用               │
│  │      ...                   │     │                                 │
│  │    }                       │     ▼                                 │
│  │                            │  ┌─────────┐                         │
│  │  Runnable r = new Runnable │  │ Handler │                         │
│  │    { ... }                 │  │ 或      │                         │
│  │                            │  │ Runnable│                         │
│  │  new Thread(r).start()     │  └────┬────┘                         │
│  └────────────────────────────┘       │                              │
│                                       │ 被线程持有                    │
│                                       ▼                               │
│                              ┌────────────────┐                       │
│                              │  Thread        │                       │
│                              │  (长生命周期)   │ ──→ 不会被GC回收     │
│                              └────────────────┘                       │
│                                                                       │
│  结果：Activity/Fragment 无法被 GC，导致内存泄漏                      │
│                                                                       │
└──────────────────────────────────────────────────────────────────────┘
```

**安全写法 vs 泄漏写法：**

```java
public class LeakyActivity extends AppCompatActivity {
    
    // ❌ 泄漏写法 1：非静态内部类隐式持有外部引用
    private Handler leakyHandler = new Handler() {
        @Override
        public void handleMessage(Message msg) {
            // 这里隐式持有 Activity 引用
            textView.setText((String) msg.obj);  // Activity 泄漏
        }
    };
    
    // ❌ 泄漏写法 2：匿名 Runnable
    private Runnable leakyRunnable = new Runnable() {
        @Override
        public void run() {
            // 持有 Activity 引用
            doSomething();
        }
    };
    
    // ✅ 安全写法 1：静态内部类 + 弱引用
    private static class SafeHandler extends Handler {
        private final WeakReference<LeakyActivity> activityRef;
        
        SafeHandler(LeakyActivity activity) {
            this.activityRef = new WeakReference<>(activity);
        }
        
        @Override
        public void handleMessage(Message msg) {
            LeakyActivity activity = activityRef.get();
            if (activity != null) {
                activity.textView.setText((String) msg.obj);
            }
        }
    }
    
    // ✅ 安全写法 2：静态内部类（不持有外部引用时）
    private static class SafeRunnable implements Runnable {
        @Override
        public void run() {
            // 只使用局部变量或参数，不访问外部实例成员
            Log.d("TAG", "安全的方式");
        }
    }
    
    // ✅ 安全写法 3：在生命周期结束时清理
    @Override
    protected void onDestroy() {
        super.onDestroy();
        leakyHandler.removeCallbacksAndMessages(null);  // 清理所有回调
        leakyHandler = null;
    }
}
```

### 2.3 🔥 追问 3：局部内部类和匿名内部类有什么区别？

> **A:**

**核心区别对比：**

| 维度 | 局部内部类 | 匿名内部类 |
|------|-----------|-----------|
| **名字** | ✅ 有名字 | ❌ 无名字 |
| **定义位置** | 方法/代码块内 | 方法内，接口/类声明后直接跟 `{}` |
| **使用次数** | 可多次创建实例 | 只能创建一次（因为没名字） |
| **成员声明** | 可定义属性、方法、构造器 | 只能定义实例成员，不能有构造器 |
| **继承/实现** | 可继承其他类或实现多个接口 | 只能继承一个类或实现一个接口 |
| **代码组织** | 逻辑清晰，可复用 | 一次性使用，代码较短时更简洁 |
| **适用场景** | 类较复杂，需要多次使用 | 简单回调、事件监听、策略模式 |

**代码对比：**

```java
public class Comparison {
    
    public void test() {
        
        // ====== 局部内部类 ======
        // 有名字，可复用
        class LocalInner implements Runnable {
            @Override
            public void run() {
                System.out.println("局部内部类");
            }
            
            // ✅ 可以有独立的方法
            public void cleanup() {
                System.out.println("清理资源");
            }
        }
        
        // 可以创建多个实例
        LocalInner instance1 = new LocalInner();
        LocalInner instance2 = new LocalInner();
        instance1.run();
        instance2.run();
        
        
        // ====== 匿名内部类 ======
        // 无名字，只能创建一次
        Runnable anonymous = new Runnable() {
            @Override
            public void run() {
                System.out.println("匿名内部类");
            }
            
            // ❌ 不能定义额外的方法
            // public void cleanup() {}  // Error!
        };
        
        // 只能使用一次
        anonymous.run();
        // anonymous2 不存在，因为没名字无法再创建
        
        
        // ====== 匿名内部类的高级用法 ======
        // 模拟带参数的匿名内部类（通过实例初始化块）
        Object obj = new Object() {
            private String name;
            
            // 无构造器，但有实例初始化块
            {
                this.name = "init";
            }
            
            @Override
            public String toString() {
                return "MyObject: " + name;
            }
        };
        System.out.println(obj);  // MyObject: init
    }
}
```

### 2.4 🔥 追问 4：内部类编译后是什么？内部类如何访问外部类？

> **A:**

**编译产物：**

```bash
# 编译前
OuterClass.java
├── 静态内部类 StaticInner
├── 成员内部类 MemberInner
├── 局部内部类 LocalInner
└── 匿名内部类

# 编译后
OuterClass.class           # 外部类
OuterClass$StaticInner.class       # 静态内部类
OuterClass$MemberInner.class       # 成员内部类
OuterClass$1LocalInner.class       # 局部内部类（数字编号）
OuterClass$1.class                 # 匿名内部类（数字编号）
OuterClass$2.class                 # 第二个匿名内部类
```

**内部类如何访问外部类（字节码层面）：**

```java
public class Outer {
    private int outerField = 100;
    
    public class Inner {
        private int innerField = 200;
        
        public void accessOuter() {
            // 编译后：实际上是 Outer.this.outerField
            System.out.println(outerField);
        }
    }
}
```

```java
// 反编译 Inner.class 后的等价代码
public class Outer$Inner {
    private int innerField;
    
    // 编译器添加：隐式持有外部类实例
    private final Outer this$0;  // 自动生成的引用
    
    // 编译器添加：构造方法接收外部类实例
    public Outer$Inner(Outer this$0) {
        this.this$0 = this$0;
        this.innerField = 200;
    }
    
    public void accessOuter() {
        // 访问外部类成员：this$0.outerField
        System.out.println(this$0.outerField);
    }
}
```

## 3. 一句话总结

> **Java 内部类分为四种：静态内部类（不持有外部引用、只能访问静态成员）、成员内部类（持有外部引用 `Outer.this`、可访问所有成员）、局部内部类（定义在方法内、作用域受限）、匿名内部类（无名字、只能访问 final/effectively final 变量，因为编译器会做值拷贝）。**

## 4. 面试加分话术

> "Java 内部类是我在项目中也经常用到的特性。
> 
> 比如我之前做过一个图片处理模块，用到**匿名内部类**来处理回调：给图片打水印是异步的，接口返回后我用一个 `Callback<Result>` 的匿名实现类来处理结果，避免了单独创建一个类，代码更紧凑。
> 
> 但这里有个坑必须注意：**内部类可能导致内存泄漏**。之前项目里有个同事在 Activity 里写了 `new Handler() { ... }`，然后 `sendMessageDelayed` 发送了一个延迟消息，Activity 销毁后 Handler 仍然持有引用，导致内存泄漏。后来我们统一要求：Handler 必须用**静态内部类 + 弱引用**，或者在 `onDestroy` 里 `removeCallbacksAndMessages(null)`。
> 
> 关于匿名内部类捕获变量的问题，面试官可能会追问底层原理。核心是**编译器会把捕获的变量拷贝一份到匿名内部类对象里**，所以必须要求是 final，避免值不一致的歧义。Java 8 之后放宽到 effectively final，但本质上还是同一套机制。"
