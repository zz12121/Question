## 1. 基础流程

### 中断机制的核心原理

```
┌─────────────────────────────────────────────────────────────────────┐
│                       线程中断机制                                   │
│                                                                     │
│  每个线程都有一个 interrupt 标志位（boolean）                        │
│                                                                     │
│  ┌──────────────┐                                                  │
│  │ Thread 对象   │                                                  │
│  │ ┌──────────┐  │                                                  │
│  │ │interrupted│ = false  ←─────┐                                  │
│  │ └──────────┘  │           │                                     │
│  └──────────────┘           │                                     │
│                              │                                     │
│         t.interrupt() ──────┘                                     │
│              ↓                                                     │
│         interrupted = true                                        │
│                                                                     │
│  线程检测到中断标志后：                                              │
│  • 可以选择抛出 InterruptedException                                │
│  • 可以选择调用 Thread.interrupted() 清除标志                        │
│  • 也可以完全无视，继续执行                                          │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 三个方法的区别

| 方法 | 作用 | 是否清除中断标志 |
|------|------|----------------|
| `t.interrupt()` | 设置中断标志为 true | ❌ 不清除 |
| `t.isInterrupted()` | 检查中断标志 | ❌ 不清除 |
| `Thread.interrupted()` | 检查并清除中断标志 | ✅ 清除 |

```java
public class InterruptDemo {
    
    public static void main(String[] args) {
        Thread thread = new Thread(() -> {
            while (!Thread.currentThread().isInterrupted()) {
                // 正常工作
                System.out.println("running...");
            }
            // 检测到中断标志，退出循环
            System.out.println("线程被中断，退出");
        }, "my-thread");
        
        thread.start();
        
        // 主线程休眠 1 秒后中断子线程
        try {
            Thread.sleep(1000);
        } catch (InterruptedException e) {}
        
        thread.interrupt();  // 设置中断标志 = true
    }
}
```

### 为什么 interrupt() 不一定能停止线程？

```
┌─────────────────────────────────────────────────────────────────────┐
│                   interrupt() 为什么不停止线程？                      │
│                                                                     │
│  interrupt() 只是设置一个标志位，告诉线程"有人希望你停止"             │
│  但线程是否响应，完全由线程自己决定                                   │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────┐     │
│  │  interrupt() 的效果：                                     │     │
│  │                                                           │     │
│  │  1. 如果线程调用了阻塞方法（sleep/wait/join）             │     │
│  │     → 会立即抛出 InterruptedException，清除中断标志          │     │
│  │                                                           │     │
│  │  2. 如果线程正在执行普通代码                                │     │
│  │     → 仅设置标志位为 true，线程继续运行                     │     │
│  │     → 线程必须主动检查 isInterrupted() 才能响应中断         │     │
│  └───────────────────────────────────────────────────────────┘     │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

```java
// ❌ 不响应中断的写法
Thread thread = new Thread(() -> {
    // 永远无法停止！因为没检查中断标志
    while (true) {
        System.out.println("running...");
    }
});
thread.interrupt();  // 无效！

// ✅ 响应中断的写法
Thread thread = new Thread(() -> {
    while (!Thread.currentThread().isInterrupted()) {
        System.out.println("running...");
    }
});
thread.interrupt();  // 有效，线程会退出循环
```

## 2. 扩展追问

### ## 🔥 扩展追问 1：interrupted() 和 isInterrupted() 的区别？

```java
Thread thread = new Thread(() -> {
    for (int i = 0; i < 1000; i++) {
        System.out.println(i);
    }
});

thread.start();
thread.interrupt();

// 对比：
System.out.println(thread.isInterrupted());  // true（不清除标志）
System.out.println(thread.isInterrupted());  // true（第二次还是 true）

System.out.println(Thread.interrupted());    // false（清除标志！）
System.out.println(Thread.interrupted());    // false（第二次还是 false）
```

> **A:** `Thread.interrupted()` 是静态方法，操作的是**当前线程**的中断标志，并且会**清除**标志；`t.isInterrupted()` 是实例方法，操作的是**特定线程**的中断标志，**不会清除**标志。在实际开发中，`isInterrupted()` 更常用，因为通常检查的是某个特定线程是否被中断。

---

### ## 🔥 扩展追问 2：阻塞方法被中断时会发生什么？

```java
Thread thread = new Thread(() -> {
    try {
        Thread.sleep(10000);  // 阻塞10秒
    } catch (InterruptedException e) {
        // 线程被中断时：
        // 1. 抛出 InterruptedException
        // 2. 清除中断标志（设为 false）
        System.out.println("被中断了！");
    }
});

thread.start();
thread.interrupt();

// 输出：被中断了！
// 注意：此时 thread.isInterrupted() = false（标志被清除了）
```

> **A:** 当线程调用 `sleep()`、`wait()`、`join()` 等阻塞方法时，如果另一个线程对该线程调用 `interrupt()`，会立即抛出 `InterruptedException` 并**清除中断标志**。这就是为什么在 catch 块中，`thread.isInterrupted()` 通常返回 false。

### ## 🔥 扩展追问 3：为什么需要中断机制？不用行吗？

```java
// ❌ 不用中断机制：volatile 方案（不推荐）
volatile boolean running = true;

void stop() {
    running = false;  // 无法中断正在 sleep 的线程！
}

// ✅ 用中断机制（推荐）
void stop() {
    thread.interrupt();  // 可以中断正在 sleep 的线程！
}
```

> **A:** 中断机制解决了两个问题：第一，**可以中断正在阻塞的线程**，volatile 布尔标志无法中断 sleep 中的线程；第二，**是标准的中断协作机制**，JVM 和 JDK 框架（如 Future.cancel()、ExecutorService）都使用中断。正确的响应方式是：catch 到异常后，要么恢复中断（`Thread.currentThread().interrupt()`），要么向上传播异常。

---

### ## 🔥 扩展追问 4：catch 住 InterruptedException 后为什么要恢复中断？

```java
// ❌ 错误：吃掉异常后线程不再响应中断
void run() {
    try {
        Thread.sleep(1000);
    } catch (InterruptedException e) {
        // 吞掉异常，中断标志已被清除
        // 后续代码无法知道线程曾被中断
    }
}

// ✅ 正确：恢复中断状态
void run() {
    try {
        Thread.sleep(1000);
    } catch (InterruptedException e) {
        // 重新设置中断标志，让后续代码能检测到
        Thread.currentThread().interrupt();
        // 然后退出或处理
        return;
    }
}

// ✅ 也正确：让异常传播
void run() throws InterruptedException {
    Thread.sleep(1000);  // 让调用方处理
}
```

> **A:** 因为 `InterruptedException` 会清除中断标志，如果我们在 catch 中只是记录日志然后继续运行，后续代码就无法知道线程曾被中断。正确的做法是：**要么恢复中断**（`Thread.currentThread().interrupt()`），让外层代码可以检测到；**要么让异常传播出去**，把责任交给调用方。

### ## 🔥 扩展追问 5：Future 和 ExecutorService 如何响应中断？

```java
ExecutorService executor = Executors.newSingleThreadExecutor();
Future<?> future = executor.submit(() -> {
    try {
        Thread.sleep(10000);
    } catch (InterruptedException e) {
        Thread.currentThread().interrupt();
        return;
    }
});

// 取消任务
future.cancel(true);  // true = 尝试中断正在执行的线程

// ExecutorService 关闭时
executor.shutdownNow();  // 发送 interrupt 信号给所有线程
```

> **A:** `Future.cancel(true)` 会向正在执行任务的线程发送中断信号，`ExecutorService.shutdownNow()` 会向所有线程发送中断信号。但要注意：只有任务内部正确响应 `InterruptedException` 并恢复中断，这些取消操作才能生效。如果任务在执行不可中断的操作（如同步 I/O），中断信号会被忽略。

---

## 3. 一句话总结

> **线程中断机制只是一个协作式的标志位，`interrupt()` 设置标志，`isInterrupted()` 检查标志（不清除），`interrupted()` 检查并清除标志；只有线程主动检测标志或调用阻塞方法时才会响应中断，调用 `interrupt()` 本身不会强制停止线程。**
