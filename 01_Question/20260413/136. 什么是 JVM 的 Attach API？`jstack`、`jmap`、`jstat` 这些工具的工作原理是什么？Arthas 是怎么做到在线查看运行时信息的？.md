### Attach API 的工作原理

```
┌─────────────────────────────────────────────────────────────────────┐
│                     Attach API 通信流程                             │

│  │                                                              │   │

│  2. 加载 Agent（agentmain/main）                                   │

│  4. 返回结果给外部进程                                              │
│                                                                     │

```

### jstack、jmap、jstat 的原理对比

| 工具 | 底层原理 | 触发方式 |
|------|---------|---------|
| **jstack** | 发送 SIGQUIT 信号，或 Attach API 获取线程栈 | SignalHandler / Attach |
| **jmap** | Attach API + JVM 内部 HeapDumper | Attach |
| **jstat** | Attach API + JMX (PerfData) | Attach |
| **jinfo** | Attach API 读取 JVM 参数 | Attach |
| **jcmd** | Attach API 调用 JVM Command | Attach |

```bash
# jstack 打印线程栈
$ jstack <pid>

# jmap 导出堆
$ jmap -dump:format=b,file=heap.hprof <pid>

# jstat 查看 GC 统计
$ jstat -gcutil <pid> 1000

# jinfo 查看 JVM 参数
$ jinfo -flags <pid>

# jcmd 综合命令
$ jcmd <pid> VM.native_memory summary
```

## 2. 扩展追问

### ## 🔥 扩展追问 1：Attach API 具体是怎么连接目标 JVM 的？

> **A:** Attach API 通过两种方式连接目标 JVM：**信号机制**和**socket 机制**。

```java
/**
 * Attach API 连接目标 JVM 的两种方式
 */
public class AttachMechanism {
    
    // 方式1：Socket 连接
    // • 目标 JVM 启动时在 /tmp/.java_pid<N> 创建 socket 文件
    // • 外部进程连接这个 socket
    // • 双方通过自定义协议通信
    
    // 方式2：信号机制
    // • SIGQUIT (Ctrl+\ 或 jstack) 触发 Thread Dump
    // • SIGUSR1 触发 HPROF 堆转储
    
    // Attach API 源码简化：
    public class VirtualMachineAttacher {
        public static VirtualMachine attach(String pid) {
            // 1. 检查目标 JVM 是否支持 Attach
            //    检查 /tmp/.java_pid<N> 是否存在
            
            // 2. 建立 socket 连接
            //    SocketChannel.connect(new File("/tmp/.java_pid" + pid))
            
            // 3. 握手确认
            //    发送 "1\n" 表示 Attach Protocol Version 1
            
            // 4. 返回 VirtualMachine 实例
            return new VirtualMachineImpl(pid, socket);
        }
    }
}
```

### ## 🔥 扩展追问 2：Arthas 是怎么做到在线查看运行时信息的？

> **A:** Arthas 本质上是一个**Java Agent**，通过 Attach API 连接到目标 JVM，然后加载 Agent jar 包，在目标 JVM 中执行字节码增强来拦截方法、获取数据。

```
┌─────────────────────────────────────────────────────────────────────┐
│                      Arthas 工作原理                                  │
│                                                                     │
│  用户启动 Arthas：                                                   │
│  ┌───────────────────────────────────────────────────────────────┐  │
│  │  java -jar arthas-boot.jar                                   │  │
│  └───────────────────────────────────────────────────────────────┘  │
│        │                                                           │
│        │ 1. 列出所有 Java 进程                                       │
│        ↓                                                           │
│  ┌───────────────────────────────────────────────────────────────┐  │
│  │  [INFO] Found existing java pid: 12345                        │  │
│  │  [INFO] Process 12345: com.example.Application                  │  │
│  └───────────────────────────────────────────────────────────────┘  │
│        │                                                           │
│        │ 2. Attach 到目标进程                                       │
│        ↓                                                           │
│  ┌───────────────────────────────────────────────────────────────┐  │
│  │  VirtualMachine vm = VirtualMachine.attach("12345");          │  │
│  │  vm.loadAgent("/path/to/arthas-agent.jar");                   │  │
│  └───────────────────────────────────────────────────────────────┘  │
│        │                                                           │
│        │ 3. 目标 JVM 执行 agentmain()                               │
│        ↓                                                           │
│  ┌───────────────────────────────────────────────────────────────┐  │
│  │  public static void agentmain(String args, Instrumentation ins) │  │
│  │      // 启动 Arthas Server（默认 8563 端口）                    │  │
│  │      // 注册类转换器（ClassFileTransformer）                      │  │
│  │      // 拦截方法，收集数据                                      │  │
│  └───────────────────────────────────────────────────────────────┘  │
│                                                                     │
│  用户交互：                                                          │
│  Arthas Server ←── HTTP/WebSocket ←── 浏览器/CLI                    │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

```java
 * Arthas Agent 核心逻辑简化

        // → 拦截方法，参数/返回值/异常信息
        
        // trace com.example.Service method01
        // → 方法调用链路耗时统计
        
        // stack com.example.Service method01
        // → 方法调用栈
    }
    
    /**
     * 方法拦截器
     */
    static class MethodProfilerTransformer implements ClassFileTransformer {
        public byte[] transform(ClassLoader loader, String className,
                                Class<?> classBeingRedefined,
                                ProtectionDomain protectionDomain,
                                byte[] classfileBuffer) {
            // 1. 解析字节码
            // 2. 在方法入口/出口插桩（记录时间戳）

            return enhance(classfileBuffer);
        }
```

### ## 🔥 扩展追问 3：Instrumentation API 和 ClassFileTransformer 是什么？

> **A:** Instrumentation 是 JVM 提供的** instrumentation API**，允许在类加载时或运行时**修改类的字节码**。

```java
/**
 * Instrumentation API 示例
 */
public class InstrumentationDemo {
    
    /**
     * premain: JVM 启动时加载 Agent（java -javaagent:agent.jar）
     */
    public static void premain(String args, Instrumentation inst) {
        System.out.println("Agent 加载成功！");
        inst.addTransformer(new MyTransformer());
    }
    
    /**
     * agentmain: JVM 运行时加载 Agent（Attach API）
     */
    public static void agentmain(String args, Instrumentation inst) {
        System.out.println("Agent 动态加载成功！");
        inst.addTransformer(new MyTransformer(), true);  // true = 重新转换已加载的类
    }
    
    static class MyTransformer implements ClassFileTransformer {
        public byte[] transform(ClassLoader loader, String className,
                                Class<?> classBeingRedefined,
                                ProtectionDomain pd,
                                byte[] classfileBuffer) {
            // className = com/example/Service（斜线格式）
            
            // 1. 只处理特定类
            if (!className.equals("com/example/TargetClass")) {
                return null;  // 不修改
            }
            
            // 2. 解析字节码（可以用 ASM）
            ClassReader cr = new ClassReader(classfileBuffer);
            ClassWriter cw = new ClassWriter(cr, ClassWriter.COMPUTE_FRAMES);
            ClassVisitor cv = new MethodTracingAdapter(cw);
            cr.accept(cv, ClassReader.SKIP_FRAMES);
            
            return cw.toByteArray();  // 返回增强后的字节码
        }
    }
}
```

| 方法 | 说明 |
|------|------|
| `inst.addTransformer(t)` | 注册转换器，后续加载的类会被转换 |
| `inst.addTransformer(t, true)` | 第二个参数=true时，触发重新转换已加载的类 |
| `inst.retransformClasses(Class...)` | 强制重新转换指定类 |
| `inst.getAllLoadedClasses()` | 获取所有已加载的类 |
| `inst.getObjectSize(Object)` | 获取对象大小 |

### ## 🔥 扩展追问 5：为什么 jstat 不需要 Attach？它是怎么获取 GC 数据的？

> **A:** jstat 使用的是 **PerfData**（性能数据），JVM 会将运行时统计信息写入共享内存或文件，jstat 直接读取即可，不需要 Attach。

```
┌─────────────────────────────────────────────────────────────────────┐
│                      PerfData + jstat 原理                          │
│                                                                     │
│  JVM 进程：                                                         │
│  ┌───────────────────────────────────────────────────────────────┐  │
│  │  PerfData (写入)                                              │  │
│  │  ┌───────────────────────────────────────────────────────┐   │  │
│  │  │  /tmp/hsperfdata_<user>/<pid>                          │   │  │
│  │  │  • GC 次数/耗时                                         │   │  │
│  │  │  • 堆使用量                                            │   │  │
│  │  │  • 类加载数量                                          │   │  │
│  │  │  • 线程数                                              │   │  │
│  │  └───────────────────────────────────────────────────────┘   │  │
│  └───────────────────────────────────────────────────────────────┘  │
│                                                                     │
│  jstat 进程（读取）：                                               │
│  ┌───────────────────────────────────────────────────────────────┐  │
│  │  1. 打开 /tmp/hsperfdata_<user>/<pid>                        │  │
│  │  2. 解析 PerfData 格式                                       │  │
│  │  3. 读取指定指标                                             │  │
│  └───────────────────────────────────────────────────────────────┘  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 3. 一句话总结

> **JVM Attach API 通过 socket 或信号机制连接目标 JVM 并动态加载 Agent；jstack 基于信号/Socket 获取线程栈，jmap 通过 Attach 触发堆转储，jstat 直接读取 /tmp/hsperfdata 目录的 PerfData 共享内存；Arthas 通过 Attach API 加载 Java Agent，注册 ClassFileTransformer 拦截方法字节码，在目标 JVM 内启动 HTTP Server 供用户交互，实现热更新、方法追踪等在线诊断功能。**
