## 先说结论

**CompletableFuture = Future 的"完全体"，支持链式组合、异步回调、异常处理，让异步编程像写同步代码一样清晰。**

```
Future：只能问"好了没"，拿不到结果只能阻塞等
CompletableFuture：可以"好了告诉我"，还支持多个任务组合编排
```

---

## 🔥 核心概念

### Future 的问题

```java
// Future 只能这样用
ExecutorService executor = Executors.newFixedThreadPool(2);
Future<String> future = executor.submit(() -> {
    Thread.sleep(1000);
    return "结果";
});

// 1. 阻塞等待（主线程卡住）
String result = future.get();  // 阻塞！

// 2. 轮询（浪费CPU）
while (!future.isDone()) {
    Thread.sleep(100);  // 忙等待
}
String result = future.get();

// 3. 无法链式组合
// 想等A完成后再执行B？自己写逻辑
// 想等A和B都完成再执行C？自己写逻辑
// 异常处理？try-catch 包起来
```

**Future 的痛点：**

|痛点|说明|
|---|---|
|阻塞获取|`get()` 会阻塞主线程|
|无法回调|完成时无法自动通知|
|无法组合|多个 Future 难以编排|
|异常处理麻烦|需要手动处理|
|无法取消后操作|取消了就结束了|

## 🔥 CompletableFuture 的优势

### 1. 异步回调（非阻塞）

```java
CompletableFuture<String> future = CompletableFuture.supplyAsync(() -> {
    // 异步执行任务
    return fetchDataFromRemote();
});

// 完成后自动回调，不阻塞！
future.thenAccept(result -> {
    System.out.println("拿到结果：" + result);
});

System.out.println("我可以继续干别的...");
```

### 2. 链式组合

```java
// 任务A -> 任务B -> 任务C，自动串联
CompletableFuture.supplyAsync(() -> fetchUser())
    .thenApply(user -> enrichUser(user))      // 转换
    .thenCompose(user -> fetchOrders(user))   // 再发起异步
    .thenAccept(orders -> saveToCache(orders)); // 消费结果
```

### 3. 多任务组合

```java
// 等A和B都完成
CompletableFuture<String> futureA = fetchA();
CompletableFuture<String> futureB = fetchB();

CompletableFuture.allOf(futureA, futureB)
    .thenAccept(v -> {
        // A和B都完成了
        String a = futureA.join();
        String b = futureB.join();
    });

// 或者A或B完成就行
CompletableFuture.anyOf(futureA, futureB)
    .thenAccept(result -> {
        // 谁先完成用谁
    });
```

### 4. 异常处理

```java
CompletableFuture.supplyAsync(() -> {
    if (random() > 0.5) throw new RuntimeException("出错");
    return "success";
})
.thenApply(result -> result.toUpperCase())
.exceptionally(ex -> {
    // 捕获异常，返回默认值
    System.out.println("出错了：" + ex.getMessage());
    return "DEFAULT";
})
.thenAccept(System.out::println);
```

## 🔥 CompletableFuture 核心 API

### 创建异步任务
```java
// 1. supplyAsync：有返回值
CompletableFuture<String> future = CompletableFuture.supplyAsync(() -> {
    return "Hello";
});

// 2. runAsync：无返回值
CompletableFuture<Void> future = CompletableFuture.runAsync(() -> {
    System.out.println("执行任务");
});

// 3. 指定线程池（默认 ForkJoinPool.commonPool）
ExecutorService executor = Executors.newFixedThreadPool(4);
CompletableFuture<String> future = CompletableFuture.supplyAsync(() -> {
    return "Hello";
}, executor);
```

### 转换与消费


```java
CompletableFuture<String> future = CompletableFuture.supplyAsync(() -> "100");

// thenApply：有入参有返回值（转换）
CompletableFuture<Integer> intFuture = future.thenApply(Integer::parseInt);

// thenAccept：有入参无返回值（消费）
future.thenAccept(System.out::println);

// thenRun：无入参无返回值（纯副作用）
future.thenRun(() -> System.out.println("执行完毕"));

// 带 Async 后缀 = 在另一个线程执行
future.thenApplyAsync(this::heavyComputation);
```

### 组合多个 Future

```java
CompletableFuture<String> f1 = fetchUser();
CompletableFuture<String> f2 = fetchOrder();

// thenCombine：两个都完成，合并结果
f1.thenCombine(f2, (user, order) -> user + ":" + order);

// thenAcceptBoth：两个都完成，消费结果
f1.thenAcceptBoth(f2, (user, order) -> save(user, order));

// applyToEither：谁先完成用谁
f1.applyToEither(f2, result -> result);

// allOf：等所有完成
CompletableFuture<Void> all = CompletableFuture.allOf(f1, f2, f3);

// anyOf：任意一个完成
CompletableFuture<Object> any = CompletableFuture.anyOf(f1, f2, f3);
```

### 异常处理

```java
CompletableFuture.supplyAsync(() -> {
    throw new RuntimeException("出错");
})
.exceptionally(ex -> {
    // 捕获异常，返回默认值
    return "default";
})
.handle((result, ex) -> {
    // 统一处理正常结果和异常
    if (ex != null) return "error: " + ex.getMessage();
    return result;
});
```

## 🔥 扩展追问 1：如何用 CompletableFuture 实现并行调用多个接口？

### 场景：并行调用用户、订单、库存三个接口

```java
public class UserService {
    public User getUser(Long userId) {
        // 模拟RPC调用
        sleep(100);
        return new User(userId, "张三");
    }
}

public class OrderService {
    public List<Order> getOrders(Long userId) {
        sleep(150);
        return Arrays.asList(new Order(1L), new Order(2L));
    }
}

public class InventoryService {
    public Inventory getInventory(Long skuId) {
        sleep(80);
        return new Inventory(skuId, 100);
    }
}
```

### 方案 1：串行调用（慢！）

```java
public UserDetail getUserDetail(Long userId) {
    // 总耗时 = 100 + 150 = 250ms
    User user = userService.getUser(userId);           // 100ms
    List<Order> orders = orderService.getOrders(userId); // 150ms
    return new UserDetail(user, orders);
}
```

### 方案 2：并行调用（CompletableFuture）

```java
public UserDetail getUserDetailParallel(Long userId) {
    // 两个任务同时发起
    CompletableFuture<User> userFuture = CompletableFuture
        .supplyAsync(() -> userService.getUser(userId));
    
    CompletableFuture<List<Order>> orderFuture = CompletableFuture
        .supplyAsync(() -> orderService.getOrders(userId));
    
    // 等两个都完成
    CompletableFuture<UserDetail> resultFuture = userFuture
        .thenCombine(orderFuture, (user, orders) -> {
            return new UserDetail(user, orders);
        });
    
    // 阻塞获取结果（或继续异步处理）
    return resultFuture.join();  // 总耗时 ≈ max(100, 150) = 150ms
}
```

### 方案 3：更优雅的封装

```java
public class AsyncService {
    private final ExecutorService executor = Executors.newFixedThreadPool(10);
    
    public UserDetail getUserDetail(Long userId) {
        return CompletableFuture.supplyAsync(() -> userService.getUser(userId), executor)
            .thenCombine(
                CompletableFuture.supplyAsync(() -> orderService.getOrders(userId), executor),
                UserDetail::new
            )
            .join();
    }
    
    // 批量查询优化
    public List<UserDetail> batchGetUserDetail(List<Long> userIds) {
        List<CompletableFuture<UserDetail>> futures = userIds.stream()
            .map(id -> CompletableFuture.supplyAsync(() -> getUserDetail(id), executor))
            .collect(Collectors.toList());
        
        // 等所有完成
        CompletableFuture.allOf(futures.toArray(new CompletableFuture[0])).join();
        
        return futures.stream()
            .map(CompletableFuture::join)
            .collect(Collectors.toList());
    }
}
```

### 方案 4：带超时的并行调用

```java
public UserDetail getUserDetailWithTimeout(Long userId) {
    CompletableFuture<User> userFuture = CompletableFuture
        .supplyAsync(() -> userService.getUser(userId))
        .orTimeout(200, TimeUnit.MILLISECONDS)  // 超时200ms
        .exceptionally(ex -> User.EMPTY);        // 超时返回空对象
    
    CompletableFuture<List<Order>> orderFuture = CompletableFuture
        .supplyAsync(() -> orderService.getOrders(userId))
        .orTimeout(200, TimeUnit.MILLISECONDS)
        .exceptionally(ex -> Collections.emptyList());
    
    return userFuture
        .thenCombine(orderFuture, UserDetail::new)
        .join();
}
```

### 方案 5：复杂编排（A->B, C->D, 然后 B+D->E）

```java
public Result complexWorkflow() {
    // 步骤1：并行获取配置和用户信息
    CompletableFuture<Config> configFuture = fetchConfig();
    CompletableFuture<User> userFuture = fetchUser();
    
    // 步骤2：根据配置获取商品，根据用户获取订单（依赖步骤1）
    CompletableFuture<Product> productFuture = configFuture
        .thenCompose(config -> fetchProduct(config));
    
    CompletableFuture<List<Order>> orderFuture = userFuture
        .thenCompose(user -> fetchOrders(user));
    
    // 步骤3：合并商品和订单生成结果
    CompletableFuture<Result> resultFuture = productFuture
        .thenCombine(orderFuture, (product, orders) -> {
            return calculateResult(product, orders);
        });
    
    return resultFuture.join();
}
```

**流程图：**

```
fetchConfig() ──┐
                ├──┬─> fetchProduct(config) ──┐
fetchUser() ────┘  │                          ├──> calculateResult()
                   └─> fetchOrders(user) ─────┘
```

## 🔥 扩展追问 2：CompletableFuture 的实现原理

### 核心结构

```java
public class CompletableFuture<T> implements Future<T>, CompletionStage<T> {
    // 结果（CAS 更新）
    volatile Object result;
    
    // 等待栈（链表结构）
    volatile Completion stack;
    
    // 内部类：表示一个回调操作
    abstract static class Completion extends ForkJoinTask<Void> 
        implements Runnable, AsynchronousCompletionTask {
        volatile Completion next;  // 下一个回调
    }
}
```

### 执行流程

```java
// supplyAsync 提交任务
public static <U> CompletableFuture<U> supplyAsync(Supplier<U> supplier) {
    CompletableFuture<U> f = new CompletableFuture<>();
    // 提交到 ForkJoinPool
    ForkJoinPool.commonPool().execute(new AsyncSupply<>(f, supplier));
    return f;
}

// 任务完成时
final boolean completeValue(T t) {
    // CAS 设置结果
    if (UNSAFE.compareAndSwapObject(this, RESULT, null, t)) {
        // 触发所有回调
        postComplete();
        return true;
    }
    return false;
}

// 触发回调
final void postComplete() {
    Completion h = stack;
    while (h != null) {
        Completion next = h.next;
        h.tryFire(mode);  // 执行回调
        h = next;
    }
}
```

## 🔥 扩展追问 3：CompletableFuture vs Future 对比

|特性|Future|CompletableFuture|
|---|---|---|
|阻塞获取|`get()` 阻塞|`get()` 阻塞，但推荐回调|
|非阻塞回调|❌ 不支持|✅ `thenAccept`/`thenApply`|
|链式组合|❌ 手动实现|✅ 内置支持|
|多任务组合|❌ 手动实现|✅ `allOf`/`anyOf`/`thenCombine`|
|异常处理|❌ try-catch 包|✅ `exceptionally`/`handle`|
|超时控制|❌ 不支持|✅ `orTimeout`/`completeOnTimeout`|
|取消后操作|❌ 无|✅ `cancel` 可指定是否中断|

## 🔥 扩展追问 5：常见坑

### 坑 1：忘记处理异常

```java
// 错误：异常被吞掉
CompletableFuture.supplyAsync(() -> {
    throw new RuntimeException("出错");
}).thenAccept(System.out::println);  // 异常了不会打印！

// 正确：加异常处理
CompletableFuture.supplyAsync(() -> {
    throw new RuntimeException("出错");
})
.exceptionally(ex -> {
    log.error("出错", ex);
    return null;
});
```

### 坑 2：在 thenApply 里做阻塞操作

```java
// 错误：阻塞了公共线程池
CompletableFuture.supplyAsync(() -> fetchA())
    .thenApply(result -> {
        // 这里阻塞了！
        Thread.sleep(1000);
        return result;
    });

// 正确：用 thenApplyAsync 切换到另一个线程池
CompletableFuture.supplyAsync(() -> fetchA())
    .thenApplyAsync(result -> {
        Thread.sleep(1000);
        return result;
    }, blockingExecutor);
```

### 坑 3：join() vs get()

```java
// join()：不抛受检异常，更简洁
String result = future.join();

// get()：抛 InterruptedException 和 ExecutionException
try {
    String result = future.get();
} catch (InterruptedException | ExecutionException e) {
    e.printStackTrace();
}
```

## 一句话收尾

> "CompletableFuture 相比 Future 最大的优势是支持链式组合和异步回调，让异步编程像写同步代码一样清晰。实现并行调用多个接口时，用 `supplyAsync` 发起多个异步任务，然后用 `thenCombine` 或 `allOf` 等待它们完成，可以将串行的耗时变成并行的 max(耗时)。实际项目中要注意自定义线程池和异常处理。"
