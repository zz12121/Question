## 先说结论

分布式链路追踪是**追踪请求在分布式系统中完整调用路径**的技术，通过**Trace ID**串联跨服务的调用链，用于**性能分析、故障定位、依赖梳理**。SkyWalking 是**国产 APM 全栈监控**，Zipkin 是**轻量级链路追踪**，Jaeger 是**云原生 CNCF 项目**，三者都基于 OpenTelemetry 标准。

---

## 🔥 核心概念

| 维度 | 说明 |
|------|------|
| **Trace（链路）** | 一次完整请求调用链，由多个 Span 组成 |
| **Span（跨度）** | 一次单独的调用单元（如 HTTP 请求、数据库操作） |
| **Trace ID** | 全局唯一标识，贯穿整个调用链 |
| **Span ID** | 单个 Span 的标识，与 Parent Span ID 构成父子关系 |
| **Baggage** | 随 Trace 传递的上下文数据（如用户 ID） |

## 🔥 扩展追问 1：三者对比选型

| 维度 | SkyWalking | Zipkin | Jaeger |
|------|-----------|--------|--------|
| **功能完整度** | ⭐⭐⭐⭐⭐（全栈 APM） | ⭐⭐⭐（链路追踪） | ⭐⭐⭐⭐（追踪+监控） |
| **侵入性** | ⭐⭐⭐⭐⭐（Agent 无侵入） | ⭐⭐⭐（需集成 SDK） | ⭐⭐⭐（需集成 SDK） |
| **存储扩展** | ⭐⭐⭐⭐（多存储支持） | ⭐⭐⭐ | ⭐⭐⭐⭐ |
| **云原生** | ⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐⭐（K8s 原生） |
| **社区活跃度** | ⭐⭐⭐⭐⭐（国内活跃） | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐（CNCF） |
| **学习成本** | ⭐⭐⭐ | ⭐⭐⭐⭐⭐（最简单） | ⭐⭐⭐⭐ |

**选型建议**：
- **SkyWalking**：需要全栈 APM（链路+指标+日志）、Java 生态、无侵入探针
- **Zipkin**：快速上手、Spring Cloud 生态、轻量级部署
- **Jaeger**：云原生环境、K8s 部署、OpenTelemetry 标准

## 🔥 扩展追问 2：OpenTelemetry 标准

```java
// OpenTelemetry 统一 API（未来趋势）
import io.opentelemetry.api.trace.Tracer;

@Service
public class OrderService {
    
    private final Tracer tracer;
    
    public OrderService(Tracer tracer) {
        this.tracer = tracer;
    }
    
    public void createOrder(Order order) {
        Span span = tracer.spanBuilder("createOrder")
            .setAttribute("order.id", order.getId())
            .setAttribute("user.id", order.getUserId())
            .startSpan();
        
        try (Scope scope = span.makeCurrent()) {
            // 业务逻辑
            validateOrder(order);
            saveOrder(order);
        } catch (Exception e) {
            span.recordException(e);
            throw e;
        } finally {
            span.end();
        }
    }
}
```

## 🔥 扩展追问 3：生产环境最佳实践

### 采样策略

```yaml
# SkyWalking 采样配置
agent:
  sampler:
    type: percentage  # 百分比采样
    percentage: 10    # 10% 采样

# 或动态采样（根据 QPS 自适应）
agent:
  sampler:
    type: adaptive
    max_qps: 1000
```

### 数据存储选型

| 数据量 | 推荐存储 | 原因 |
|--------|---------|------|
| < 100GB/天 | H2/MySQL | 简单、易维护 |
| 100GB-1TB/天 | Elasticsearch | 查询快、易扩展 |
| > 1TB/天 | BanyanDB/TiDB | SkyWalking 原生、高性能 |

### 性能优化

```yaml
# 批量发送配置
agent:
  buffer:
    size: 1000      # 缓冲区大小
    flush_interval: 5s  # 刷新间隔
    
# 异步发送
agent:
  async:
    enabled: true
    queue_size: 10000
```

## 一句话收尾

分布式链路追踪通过 **Trace ID** 串联调用链，**SkyWalking** 是国产全栈 APM（无侵入），**Zipkin** 轻量易用，**Jaeger** 云原生友好，三者都向 **OpenTelemetry** 标准演进，选型根据生态和场景灵活选择。
