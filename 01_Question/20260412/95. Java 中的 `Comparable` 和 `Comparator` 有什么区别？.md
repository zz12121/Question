## 先说结论

`Comparable` 是**内置排序**（对象自己定义怎么比），`Comparator` 是**外部排序**（第三方定义怎么比）；前者改变类本身，后者灵活可插拔，优先用 `Comparator`。

---

## 🔥 核心概念

| 维度 | Comparable | Comparator |
|------|-----------|------------|
| 包 | `java.lang` | `java.util` |
| 方法 | `compareTo(T o)` | `compare(T o1, T o2)` |
| 实现位置 | **类内部**实现 | **类外部**独立定义 |
| 修改类 | 需要修改原类 | 不需要修改原类 |
| 排序规则数量 | 只能有**一种** | 可以有**无数种** |
| 典型场景 | 自然排序（字符串字典序、数字大小） | 多字段排序、第三方类排序 |
| Collections.sort | `Collections.sort(list)` | `Collections.sort(list, comparator)` |

---

## 🔥 详细展开

### 1. Comparable —— 自然顺序，类内定义

```java
// 实体类实现 Comparable，定义"自然顺序"
public class Employee implements Comparable<Employee> {
    private String name;
    private int age;
    private double salary;

    @Override
    public int compareTo(Employee other) {
        // 约定：负数=this<other，0=相等，正数=this>other
        return Integer.compare(this.age, other.age); // 按年龄升序
    }
}

// 使用：无需传 Comparator，直接排
List<Employee> list = Arrays.asList(
    new Employee("张三", 30, 8000),
    new Employee("李四", 25, 12000),
    new Employee("王五", 35, 6000)
);
Collections.sort(list);      // 按 compareTo 定义的自然顺序
list.sort(null);             // 同上，null 表示用自然顺序
TreeSet<Employee> set = new TreeSet<>(list); // TreeSet 也依赖 Comparable
```

---

### 2. Comparator —— 外部定义，灵活可插拔

```java
// 不修改 Employee 类，外部定义多种排序规则
public class Employee {
    String name;
    int age;
    double salary;
    // 没有实现 Comparable
}

// 方式1：匿名类（老写法）
Comparator<Employee> byAge = new Comparator<Employee>() {
    @Override
    public int compare(Employee e1, Employee e2) {
        return Integer.compare(e1.age, e2.age);
    }
};

// 方式2：Lambda（Java 8+，推荐）
Comparator<Employee> byAge     = (e1, e2) -> Integer.compare(e1.age, e2.age);
Comparator<Employee> bySalary  = (e1, e2) -> Double.compare(e1.salary, e2.salary);
Comparator<Employee> byName    = Comparator.comparing(e -> e.name);

// 方式3：Comparator 工厂方法链式调用（最推荐）
Comparator<Employee> complex = Comparator
    .comparingDouble(Employee::getSalary).reversed()  // 先按薪资降序
    .thenComparingInt(Employee::getAge)               // 薪资相同按年龄升序
    .thenComparing(Employee::getName);                // 年龄相同按姓名字典序

list.sort(complex);
```

### 3. 多字段排序 —— Comparator 链式 API

```java
// 场景：订单列表，先按状态排，状态相同按金额降序，金额相同按时间升序
Comparator<Order> orderComparator = Comparator
    .comparingInt(Order::getStatus)              // 主排序：状态升序
    .thenComparingDouble(Order::getAmount)       // 次排序：金额升序
        .reversed()                              // ⚠️ 注意：reversed 翻转整个链！
    .thenComparing(Order::getCreateTime);        // 再排序：时间

// 正确的多字段部分翻转写法
Comparator<Order> correct = Comparator
    .comparingInt(Order::getStatus)                       // 状态升序
    .thenComparing(
        Comparator.comparingDouble(Order::getAmount).reversed()  // 只翻转金额
    )
    .thenComparing(Order::getCreateTime);                 // 时间升序
```

### 4. null 安全排序

```java
// 原生 Comparator 对 null 直接抛 NPE
// 使用 Comparator.nullsFirst / nullsLast
List<String> withNulls = Arrays.asList("banana", null, "apple", null, "cherry");

withNulls.sort(Comparator.nullsFirst(String::compareTo));
// 结果：[null, null, apple, banana, cherry]

withNulls.sort(Comparator.nullsLast(String::compareTo));
// 结果：[apple, banana, cherry, null, null]

// 对象字段为 null 的处理
Comparator<Employee> safe = Comparator.comparing(
    Employee::getDepartment,
    Comparator.nullsLast(String::compareTo) // department 可为 null
).thenComparing(Employee::getName);
```

### 5. compareTo / compare 的返回值约定

```java
// 必须遵守的约定（违反会导致排序结果不稳定）
int result = a.compareTo(b);
// result < 0  → a 排在 b 前面
// result == 0 → a 和 b 相等（建议与 equals 一致）
// result > 0  → a 排在 b 后面

// ⚠️ 经典错误：直接相减（整数溢出！）
// 错误写法
public int compareTo(Employee other) {
    return this.age - other.age; // ← 如果 age 差值超过 Integer.MAX_VALUE 会溢出
}

// 正确写法
public int compareTo(Employee other) {
    return Integer.compare(this.age, other.age); // ← 安全
}

// ⚠️ 与 equals 不一致的坑（TreeSet/TreeMap 会出问题）
BigDecimal a = new BigDecimal("1.0");
BigDecimal b = new BigDecimal("1.00");
System.out.println(a.equals(b));      // false（精度不同）
System.out.println(a.compareTo(b));   // 0（数值相等）

// TreeSet 用 compareTo 判断相等，所以只存一个！
TreeSet<BigDecimal> set = new TreeSet<>();
set.add(a);
set.add(b);
System.out.println(set.size()); // 1，因为 compareTo 返回 0
```

## 🔥 扩展追问 1：TreeMap / TreeSet 用哪个排序？

```java
// TreeMap / TreeSet 默认用 Comparable（自然排序）
// 也可以在构造时传入 Comparator（优先级更高）

// 默认自然排序（要求元素实现 Comparable）
TreeSet<String> naturalSet = new TreeSet<>();
naturalSet.add("banana");
naturalSet.add("apple");
naturalSet.add("cherry");
// 输出：[apple, banana, cherry]（字典序）

// 传入 Comparator 覆盖自然排序
TreeSet<String> reverseSet = new TreeSet<>(Comparator.reverseOrder());
reverseSet.addAll(naturalSet);
// 输出：[cherry, banana, apple]（反字典序）

// TreeMap 按 key 排序
TreeMap<String, Integer> map = new TreeMap<>(String.CASE_INSENSITIVE_ORDER);
map.put("Banana", 1);
map.put("apple", 2);
// key 按忽略大小写的字典序排列
```

## 🔥 扩展追问 2：PriorityQueue（最小堆）如何自定义排序？

```java
// 默认最小堆（元素需实现 Comparable）
PriorityQueue<Integer> minHeap = new PriorityQueue<>();
minHeap.offer(3); minHeap.offer(1); minHeap.offer(2);
System.out.println(minHeap.poll()); // 1（最小值先出）

// 自定义 Comparator 实现最大堆
PriorityQueue<Integer> maxHeap = new PriorityQueue<>(Comparator.reverseOrder());
maxHeap.offer(3); maxHeap.offer(1); maxHeap.offer(2);
System.out.println(maxHeap.poll()); // 3（最大值先出）

// 实战：TopK 问题，维护大小为 K 的最小堆
public List<Integer> topKFrequent(int[] nums, int k) {
    Map<Integer, Integer> freq = new HashMap<>();
    for (int n : nums) freq.merge(n, 1, Integer::sum);

    // 按频率维护最小堆（堆顶是频率最低的，超出K就弹出）
    PriorityQueue<Integer> heap = new PriorityQueue<>(
        Comparator.comparingInt(freq::get)  // 按频率升序 = 最小堆
    );

    for (int num : freq.keySet()) {
        heap.offer(num);
        if (heap.size() > k) heap.poll(); // 淘汰频率最低的
    }
    return new ArrayList<>(heap);
}
```

## 🔥 扩展追问 3：Comparator 的 comparing 系列方法原理

```java
// Comparator.comparing 源码分析
public static <T, U extends Comparable<? super U>> Comparator<T> comparing(
    Function<? super T, ? extends U> keyExtractor) {
    return (c1, c2) -> keyExtractor.apply(c1).compareTo(keyExtractor.apply(c2));
}

// thenComparing 链式组合原理
default Comparator<T> thenComparing(Comparator<? super T> other) {
    return (c1, c2) -> {
        int res = this.compare(c1, c2);   // 先用当前 Comparator 比
        return (res != 0) ? res : other.compare(c1, c2); // 相等再用下一个
    };
}

// reversed 原理
default Comparator<T> reversed() {
    return (c1, c2) -> this.compare(c2, c1); // 参数对调，结果取反
}

// 实际应用：函数式组合，无需写任何 if-else
Comparator<Student> rank = Comparator
    .comparingInt(Student::getScore).reversed()  // 高分在前
    .thenComparing(Student::getName);            // 同分按姓名

students.sort(rank);
```

## 🔥 扩展追问 4：排序稳定性问题

```java
// Java 的 Arrays.sort / Collections.sort 对对象使用 TimSort（稳定排序）
// 对基本类型使用双轴快排（不稳定，但基本类型无需稳定性）

// 稳定排序意味着：相等元素的原始相对顺序保持不变
List<Employee> employees = Arrays.asList(
    new Employee("张三", 25, "研发"),
    new Employee("李四", 25, "产品"),  // 同为 25 岁
    new Employee("王五", 30, "研发")
);

// 按年龄排序后，张三仍然在李四前面（稳定性保证）
employees.sort(Comparator.comparingInt(Employee::getAge));
// [张三(25), 李四(25), 王五(30)] ← 稳定

// 多字段排序可以利用稳定性：先按次要字段排，再按主要字段排
employees.sort(Comparator.comparing(Employee::getDept));  // 先按部门
employees.sort(Comparator.comparingInt(Employee::getAge)); // 再按年龄（稳定）
// 等价于：先年龄，年龄相同保持部门顺序
```

## 一句话收尾

> `Comparable` 是类的"出厂设置"只有一种，`Comparator` 是"外挂插件"可以无限扩展；实际开发中**优先用 `Comparator.comparing().thenComparing()` 链式 API**，既不污染领域类，又能随时切换排序策略，还能处理 `null` 安全和逆序，这才是现代 Java 的正确姿势。

---
几个高频考点补充一下：

**面试必背的坑：**

1. **整数相减溢出** → `return this.age - other.age` 是经典错误，应该用 `Integer.compare(this.age, other.age)`
2. **`BigDecimal` 的 `equals` vs `compareTo`** → `1.0` 和 `1.00` 的 `equals` 是 `false`，`compareTo` 是 `0`，放进 `TreeSet` 只存一个！
3. **`reversed()` 的陷阱** → 链式调用 `.thenComparing().reversed()` 会翻转整个链，而不是只翻转最后一个字段，要精确控制需要嵌套包裹

**实际项目怎么选：**
- 领域对象有明确的"自然顺序"（如订单按 ID 排）→ 实现 `Comparable`
- 需要多种排序策略（列表页各种排序条件）→ 用 `Comparator`
- 第三方类不能改 → 只能用 `Comparator`
- **两者都有时，`Comparator` 优先级更高**（传入排序时覆盖自然顺序）
