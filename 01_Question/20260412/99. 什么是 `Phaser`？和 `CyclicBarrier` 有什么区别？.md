## 先说结论

`Phaser` 是**可复用、可动态增减参与者的多阶段栅栏**，比 `CyclicBarrier` 更灵活；支持多阶段任务、动态注册/注销线程、父子层级结构，是 JDK 7 引入的**下一代同步屏障**。

---

## 🔥 核心概念

| 维度 | `CyclicBarrier` | `Phaser` |
|------|-----------------|----------|
| 阶段支持 | **单阶段**（一轮等齐就完） | **多阶段**（可定义多个阶段，每阶段都等齐） |
| 参与者数量 | **固定**（构造时确定） | **动态**（运行时可注册/注销） |
| 复用性 | 支持（`reset()` 或自动复用） | 支持（阶段自动递增） |
| 中断处理 | 一个线程中断，屏障破坏 | 更灵活，可单独处理 |
| 层级结构 | 无 | **支持父子 Phaser**（树形结构，减少竞争） |
| 使用复杂度 | 简单 | 较复杂 |
| 适用场景 | 固定人数、单轮等待 | 多阶段流水线、动态人数、复杂协调 |

---

## 🔥 详细展开

### 1. CyclicBarrier 回顾（对比基准）

```java
// CyclicBarrier：固定 3 个人，都到齐才能开饭
CyclicBarrier barrier = new CyclicBarrier(3, () -> {
    System.out.println("人齐了，开饭！");
});

for (int i = 0; i < 3; i++) {
    final int id = i;
    new Thread(() -> {
        System.out.println("线程" + id + "到达");
        try {
            barrier.await(); // 等待，3个人都到才继续
        } catch (Exception e) {
            e.printStackTrace();
        }
        System.out.println("线程" + id + "继续执行");
    }).start();
}
// 输出：
// 线程0到达
// 线程1到达
// 线程2到达
// 人齐了，开饭！（回调）
// 线程0继续执行
// 线程1继续执行
// 线程2继续执行
```

**CyclicBarrier 的局限：**
- 人数固定，不能中途加人
- 只有一轮等待，不支持"阶段1等齐 → 阶段2等齐 → 阶段3等齐"

---

### 2. Phaser 基础用法——多阶段任务

```java
// Phaser：支持多阶段，每阶段都等所有人到齐
Phaser phaser = new Phaser(3); // 初始3个参与者

for (int i = 0; i < 3; i++) {
    final int id = i;
    new Thread(() -> {
        // 阶段1：加载数据
        System.out.println("线程" + id + "：阶段1-加载数据");
        phaser.arriveAndAwaitAdvance(); // 到达并等待其他人
        
        // 阶段2：处理数据
        System.out.println("线程" + id + "：阶段2-处理数据");
        phaser.arriveAndAwaitAdvance(); // 再次等待
        
        // 阶段3：保存结果
        System.out.println("线程" + id + "：阶段3-保存结果");
        phaser.arriveAndAwaitAdvance();
        
        System.out.println("线程" + id + "：全部完成！");
        
        // 注销自己（可选，最后阶段可自动注销）
        phaser.arriveAndDeregister();
    }).start();
}

// 输出顺序：
// 线程0：阶段1-加载数据
// 线程1：阶段1-加载数据
// 线程2：阶段1-加载数据
// （等齐后自动进入阶段2）
// 线程0：阶段2-处理数据
// 线程1：阶段2-处理数据
// 线程2：阶段2-处理数据
// （等齐后进入阶段3）
// ...
```

---

### 3. Phaser 核心 API

```java
// 构造
Phaser phaser = new Phaser();           // 0个参与者
Phaser phaser = new Phaser(int parties); // 指定初始参与者数
Phaser phaser = new Phaser(Phaser parent); // 创建子Phaser（层级结构）
Phaser phaser = new Phaser(Phaser parent, int parties);

// 注册/注销
int register()           // 注册一个新参与者，返回当前阶段号
int bulkRegister(int n)  // 批量注册n个参与者
int arriveAndDeregister() // 到达并注销（最后阶段常用）

// 到达/等待
int arrive()             // 到达，不等待，继续执行
int arriveAndAwaitAdvance() // 到达并等待其他参与者（最常用）
int awaitAdvance(int phase) // 等待指定阶段完成

// 查询状态
int getPhase()           // 当前阶段号（从0开始，负数表示终止）
int getRegisteredParties() // 注册参与者总数
int getArrivedParties()   // 已到达的参与者数
int getUnarrivedParties() // 未到达的参与者数
boolean isTerminated()    // 是否已终止

// 终止
void forceTermination()   // 强制终止所有等待
```

---

### 4. 动态注册——运行时增减参与者

```java
// 场景：分治算法，任务动态拆分
Phaser phaser = new Phaser(1); // 主线程注册

void forkJoinTask(int start, int end, Phaser phaser) {
    if (end - start <= 100) {
        // 小任务直接执行
        processRange(start, end);
        phaser.arriveAndDeregister(); // 完成并注销
        return;
    }
    
    // 大任务拆分
    int mid = (start + end) / 2;
    
    // 为子任务注册（动态增加2个参与者）
    phaser.bulkRegister(2);
    
    // 异步执行子任务
    CompletableFuture.runAsync(() -> forkJoinTask(start, mid, phaser));
    CompletableFuture.runAsync(() -> forkJoinTask(mid, end, phaser));
    
    // 当前线程完成自己的部分并注销
    phaser.arriveAndDeregister();
}

// 主线程启动
forkJoinTask(0, 10000, phaser);
phaser.arriveAndAwaitAdvance(); // 等待所有子任务完成
System.out.println("全部完成！");
```

### 5. 父子 Phaser——减少竞争（高级特性）

```java
// 大量线程竞争一个 Phaser 会有性能问题
// 解决方案：树形结构，父 Phaser 管理多个子 Phaser

// 创建根 Phaser
Phaser root = new Phaser();

// 创建多个子 Phaser（每个管理一批线程）
for (int i = 0; i < 10; i++) {
    Phaser child = new Phaser(root); // 子Phaser，自动注册到父
    
    // 每个子Phaser管理100个线程
    for (int j = 0; j < 100; j++) {
        new Thread(() -> {
            child.arriveAndAwaitAdvance(); // 先等子Phaser内齐
            // 子Phaser齐后，会自动通知父Phaser
        }).start();
    }
}

// 等待所有子Phaser完成（即所有1000个线程完成）
root.arriveAndAwaitAdvance();
```

**层级结构优势：**
- 减少根 Phaser 的竞争压力
- 子 Phaser 内部先同步，再与父同步
- 适合超大规模并发（如 MapReduce 场景）

## 🔥 扩展追问 1：Phaser vs CountDownLatch 怎么选？

| 场景 | 推荐方案 |
|------|---------|
| 一次性等待（如启动加载） | `CountDownLatch`（最简单） |
| 固定人数、多轮复用 | `CyclicBarrier` |
| 多阶段流水线、动态人数 | `Phaser` |
| 父子任务、分治算法 | `Phaser`（父子层级） |

```java
// CountDownLatch：一次性，不能复用
CountDownLatch latch = new CountDownLatch(3);
// ... 各线程 countDown()
latch.await(); // 只能等一次

// Phaser：阶段自动递增，天然复用
phaser.arriveAndAwaitAdvance(); // 阶段0
phaser.arriveAndAwaitAdvance(); // 阶段1
phaser.arriveAndAwaitAdvance(); // 阶段2
// 自动进入下一阶段，无需 reset
```

## 🔥 扩展追问 2：Phaser 在 ForkJoinPool 中的应用

```java
// Java 8 的 CompletableFuture 和并行流底层有用到 Phaser 思想
// 自定义 ForkJoin 任务示例

class ParallelQuickSort extends RecursiveAction {
    private final int[] arr;
    private final int left, right;
    private final Phaser phaser;
    
    ParallelQuickSort(int[] arr, int left, int right, Phaser phaser) {
        this.arr = arr;
        this.left = left;
        this.right = right;
        this.phaser = phaser;
        phaser.register(); // 注册自己
    }
    
    @Override
    protected void compute() {
        if (right - left < 1000) {
            Arrays.sort(arr, left, right + 1);
            phaser.arriveAndDeregister(); // 完成并注销
            return;
        }
        
        int pivot = partition(arr, left, right);
        
        // 拆分任务
        phaser.bulkRegister(2); // 为两个子任务注册
        
        ParallelQuickSort leftTask = new ParallelQuickSort(arr, left, pivot - 1, phaser);
        ParallelQuickSort rightTask = new ParallelQuickSort(arr, pivot + 1, right, phaser);
        
        invokeAll(leftTask, rightTask);
        
        phaser.arriveAndDeregister(); // 当前任务完成
    }
}

// 使用
int[] arr = new int[1000000];
// ... 填充数据
Phaser phaser = new Phaser(1); // 主线程
new ParallelQuickSort(arr, 0, arr.length - 1, phaser).fork();
phaser.arriveAndAwaitAdvance(); // 等待全部完成
```

## 🔥 扩展追问 3：Phaser 的终止机制

```java
Phaser phaser = new Phaser(3);

// 方式1：自然终止（所有参与者都注销）
// 当 getRegisteredParties() == 0 时，Phaser 自动终止

// 方式2：强制终止（异常场景）
new Thread(() -> {
    try {
        phaser.arriveAndAwaitAdvance();
    } catch (Exception e) {
        // 某个线程出问题，强制终止整个 Phaser
        phaser.forceTermination();
    }
}).start();

// 检查终止状态
if (phaser.isTerminated()) {
    System.out.println("Phaser 已终止，阶段=" + phaser.getPhase());
    // 终止后 getPhase() 返回负数
}

// 终止后，所有正在 awaitAdvance 的线程会立即返回
// 后续 arrive 操作会返回负数（表示已终止）
```

## 一句话收尾

> `Phaser` 是 `CyclicBarrier` 的**完全上位替代**，支持多阶段流水线、动态增减参与者、父子层级结构；虽然 API 更复杂，但在分治算法、多阶段 MapReduce、动态任务拆分等场景下是唯一选择；简单单轮等待用 `CyclicBarrier`，一次性等待用 `CountDownLatch`，复杂协调用 `Phaser`。

---
几个关键点再强调一下：

**Phaser 相比 CyclicBarrier 的三大优势：**
1. **多阶段** → 可以 phase 0 → phase 1 → phase 2 连续多轮等待
2. **动态人数** → 运行时可以 `register()` / `arriveAndDeregister()` 增减参与者
3. **父子层级** → 大量线程时拆成树形结构减少竞争

**三个同步工具的选择口诀：**
- `CountDownLatch` → **一次性**等待（启动加载）
- `CyclicBarrier` → **固定人数**、**单阶段**、**多轮复用**
- `Phaser` → **多阶段**、**动态人数**、**分治算法**、**父子结构**
