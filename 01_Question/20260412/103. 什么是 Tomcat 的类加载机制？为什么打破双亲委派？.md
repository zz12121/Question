## 先说结论

类加载是**加载 → 链接（验证、准备、解析）→ 初始化**的三部曲；`<clinit>` 是**类构造器**（执行静态变量赋值和静态代码块），`<init>` 是**实例构造器**（执行字段赋值和构造方法）；类在**首次主动使用**时初始化，有 6 种触发场景。

---

## 🔥 核心概念

| 阶段 | 作用 | 关键动作 |
|------|------|---------|
| **加载** | 把 .class 文件读进内存 | 通过全限定名获取二进制流 → 生成 Class 对象 |
| **链接** | 确保类可用且合法 | 验证 → 准备 → 解析 |
| ├─ 验证 | 安全检查 | 文件格式、元数据、字节码、符号引用验证 |
| ├─ 准备 | 分配内存并设置默认值 | 为类变量分配内存，设置零值（0/null/false） |
| └─ 解析 | 符号引用转直接引用 | 把类名/方法名换成内存地址指针 |
| **初始化** | 执行类构造器 | 执行 `<clinit>()` 方法 |

## 🔥 详细展开

### 1. 加载（Loading）

```java
// 类加载器通过全限定名找到 .class 文件
// 1. 从文件系统读取：/com/example/User.class
// 2. 从 jar 包读取：jar:file:/app/lib/xxx.jar!/com/example/User.class
// 3. 从网络读取：http://example.com/User.class
// 4. 动态生成：Proxy、CGLIB、ASM 字节码生成

// 加载完成后，在方法区生成：
// - 类的元数据（字段、方法、父类、接口）
// - 在堆中创建 java.lang.Class 对象（反射的入口）
```

### 2. 链接（Linking）

#### 2.1 验证（Verification）—— 安全第一道防线

```java
// 验证阶段确保字节码合法，防止恶意代码

// 1. 文件格式验证
//    - 魔数是否是 0xCAFEBABE
//    - 版本号是否在当前 JVM 支持范围
//    - 常量池是否有不支持的类型

// 2. 元数据验证
//    - 类是否有父类（除了 Object）
//    - 是否继承了 final 类
//    - 是否实现了接口的所有方法

// 3. 字节码验证（最复杂）
//    - 操作数栈不会溢出/下溢
//    - 跳转指令不会跳到方法体外
//    - 类型转换合法

// 4. 符号引用验证
//    - 引用的类是否存在
//    - 引用的方法/字段是否存在且可访问

// 可以关闭验证（生产不推荐）：
// -Xverify:none  或  -noverify
// 仅对启动类加载器加载的类跳过验证（JDK 9+）
```

#### 2.2 准备（Preparation）—— 分配内存设零值

```java
public class PreparationDemo {
    // 准备阶段：
    private static int a = 123;      // 分配内存，a = 0（不是 123！）
    private static final int B = 456; // 编译期常量，B = 456（准备阶段就赋值）
    private static String s = "abc"; // s = null（不是 "abc"！）
    
    // 123 和 "abc" 的赋值在初始化阶段执行
}

// 准备阶段只设置"零值"：
// 基本类型：0、0L、0.0f、0.0d、false
// 引用类型：null
// 只有 static final 的编译期常量在准备阶段就赋值
```

#### 2.3 解析（Resolution）—— 符号引用转直接引用

```java
// 符号引用：用字符串描述目标
// 直接引用：内存地址指针/偏移量

// 解析前（符号引用）：
// 类：com.example.User
// 方法：java/io/PrintStream.println:(Ljava/lang/String;)V
// 字段：com/example/Demo.a:I

// 解析后（直接引用）：
// 0x7f8a2b3c4d5e  ← 方法区的实际内存地址

// 解析时机：
// - 静态绑定：编译期已知（private、static、final、构造方法），加载时解析
// - 动态绑定：运行时才知道（虚方法），第一次调用时解析

// 动态绑定的优化：虚方法表（vtable）、接口方法表（itable）
```

### 3. 初始化（Initialization）—— 执行 `<clinit>()`

```java
public class InitializationDemo {
    // 静态变量
    private static int a = 10;           // <clinit> 中赋值
    
    // 静态代码块（按声明顺序合并到 <clinit>）
    static {
        System.out.println("静态代码块1, a=" + a);  // 可以访问前面的静态变量
        a = 20;
    }
    
    private static int b = 30;           // <clinit> 中赋值（在代码块2之后）
    
    static {
        System.out.println("静态代码块2, a=" + a + ", b=" + b);
        // b 此时还是 0，因为还没执行到 b=30 的赋值！
    }
    
    // <clinit>() 方法实际执行顺序（编译器生成）：
    // 1. a = 10
    // 2. 静态代码块1: print, a = 20
    // 3. b = 30
    // 4. 静态代码块2: print
}

// 输出：
// 静态代码块1, a=10
// 静态代码块2, a=20, b=0  ← b 还没赋值！
```

## 🔥 扩展追问 1：`<clinit>` vs `<init>` 的区别

| 维度 | `<clinit>()` | `<init>()` |
|------|-------------|-----------|
| 全称 | Class Initializer | Instance Initializer |
| 触发时机 | **类初始化**时（首次主动使用） | **创建对象**时（new/反射/克隆/反序列化） |
| 执行内容 | 静态变量赋值 + 静态代码块 | 字段赋值 + 构造方法 + 实例代码块 |
| 执行次数 | **每个类只执行一次** | 每次创建对象都执行 |
| 线程安全 | JVM 保证**串行执行**，多线程会阻塞 | 无特殊保证 |
| 父类调用 | 自动先执行父类 `<clinit>` | 自动先调用父类 `<init>`（super()） |

```java
public class InitDemo extends Parent {
    private int x = 10;           // <init> 中赋值
    private static int y = 20;    // <clinit> 中赋值
    
    {                             // 实例代码块 → 合并到 <init>
        System.out.println("实例代码块, x=" + x);
    }
    
    static {                      // 静态代码块 → 合并到 <clinit>
        System.out.println("静态代码块, y=" + y);
    }
    
    public InitDemo() {           // 构造方法 → 合并到 <init> 最后
        super();                  // 隐式调用父类构造
        System.out.println("构造方法");
    }
}

// 首次 new InitDemo() 的执行顺序：
// 1. 加载 InitDemo 类
// 2. 链接（验证、准备、解析）
// 3. 初始化 InitDemo（先初始化 Parent）：
//    3.1 执行 Parent.<clinit>()
//    3.2 执行 InitDemo.<clinit>()：y=20 → 静态代码块
// 4. 创建对象，执行 <init>()：
//    4.1 执行 Parent.<init>()
//    4.2 执行 InitDemo.<init>()：x=10 → 实例代码块 → 构造方法
```

## 🔥 扩展追问 2：触发类初始化的 6 种主动使用场景

```java
// JVM 规范规定：只有"首次主动使用"才触发初始化

// 场景1：new 实例
new User();  // 触发 User 初始化

// 场景2：访问静态变量（非 final 编译期常量）
int x = User.age;  // 触发初始化
// 例外：static final 的编译期常量不触发（已在准备阶段赋值）
int y = User.MAX_AGE;  // 如果 MAX_AGE 是编译期常量，不触发初始化

// 场景3：调用静态方法
User.sayHello();  // 触发初始化

// 场景4：反射
Class.forName("com.example.User");  // 触发初始化

// 场景5：初始化子类时，先初始化父类
new Student();  // 先初始化 User，再初始化 Student

// 场景6：JVM 启动时的主类（包含 main 方法的类）
public class Main {  // 先初始化 Main 类
    public static void main(String[] args) { ... }
}

// ========== 不会触发初始化的"被动使用" ==========

// 1. 访问编译期常量
int max = User.MAX_VALUE;  // 常量池直接引用，不初始化

// 2. 创建数组
User[] users = new User[10];  // 只生成数组类，不初始化 User

// 3. 获取 Class 对象
Class<User> clazz = User.class;  // 不初始化

// 4. 访问父类的静态字段
int age = Student.age;  // 如果 age 定义在 User 中，只初始化 User，不初始化 Student
```

// 类加载器层次（JDK 8）：

## 🔥 扩展追问 4：`<clinit>` 的线程安全问题

```java
// JVM 保证 <clinit> 只执行一次，且线程安全（串行执行）

public class Singleton {
    private static Singleton instance;
    
    static {
        // 模拟耗时初始化
        try {
            Thread.sleep(1000);
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
        instance = new Singleton();
        System.out.println("Singleton 初始化完成 by " + Thread.currentThread().getName());
    }
    
    public static Singleton getInstance() {
        return instance;
    }
}

// 多线程并发调用 getInstance()
// 结果：只有一个线程执行 <clinit>，其他线程阻塞等待
// 天然线程安全，但注意：阻塞时间 = 初始化耗时
```

## 一句话收尾

> 类加载是**加载 → 链接（验证准备解析）→ 初始化**的三部曲；`<clinit>` 是类的"静态构造函数"只跑一次，`<init>` 是对象的"实例构造函数"每次 new 都跑；类在**首次主动使用**（new/静态成员/反射/子类初始化等）时才初始化，JVM 保证 `<clinit>` 线程安全串行执行，这是单例模式**静态内部类实现**的底层原理。

---
几个关键点再强调一下：

**链接阶段三步骤的记忆口诀：**
- **验证** → 安检（文件格式、字节码合法性）
- **准备** → 分房（分配内存，先给零值）
- **解析** → 换证（符号引用换成内存地址）

**`<clinit>` vs `<init>` 的核心区别：**
| | `<clinit>` | `<init>` |
|--|-----------|---------|
| 执行时机 | 类初始化 | 创建对象 |
| 执行次数 | **一次** | 每次 new |
| 线程安全 | JVM 保证串行 | 无特殊保证 |
| 内容 | static 变量 + 静态代码块 | 字段赋值 + 实例代码块 + 构造方法 |

**触发初始化的 6 种主动使用（必背）：**
1. `new` 实例
2. 访问**非 final** 静态变量
3. 调用静态方法
4. `Class.forName()`
5. 初始化子类（先初始化父类）
6. 主类（含 `main` 方法）

**不触发的被动使用（陷阱）：**
- `static final` **编译期常量**（准备阶段已赋值）
- 创建数组（`new User[10]` 只创建数组类）
- `User.class` 获取 Class 对象
