### 传入 null 时会选择哪个方法？

```java
public class OverloadDemo {
    
    // 三个重载方法
    public static void print(Object obj) {
        System.out.println("print(Object): " + obj);
    }
    
    public static void print(String str) {
        System.out.println("print(String): " + str);
    }
    
    public static void print(int... arr) {
        System.out.println("print(int...): " + Arrays.toString(arr));
    }
    
    public static void main(String[] args) {
        // ========== 传入 null 时的选择 ==========
        
        print(null);  // ❓ 选哪个？
        
        // 答案：print(String)
        // 原因：编译器选择"最具体"的方法
        // Object < String（String 更具体）
        // Object < int...（数组类型更具体）
        // String vs int... → String 更具体（不是数组）
    }
}
```

```
┌─────────────────────────────────────────────────────────────────────┐
│                      null 值匹配规则                                 │
│                                                                     │
│  传入 null 时的匹配优先级：                                          │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │   │
│  │   print(null)                                               │   │
│  │        │                                                    │   │
│  │        ├── print(String)  ← 最具体，匹配！                   │   │
│  │        │         ↓                                          │   │
│  │        │      Object ← String 的父类                        │   │
│  │        │         ↑                                          │   │
│  │        ├── print(Object)  ← 兜底                            │   │

│  │                     ↓                                        │   │
│  │              Object ← 数组的父类（所有数组的父类是 Object[]）   │   │
│  │                                                             │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  匹配链：                                                          │
│  String 更具体（因为 null 可以赋值给任何引用类型，String 不是数组）  │
```

### Most Specific Method 规则

```java
// ========== 什么是"最具体"？==========
// 编译器会按以下顺序判断：

public class MostSpecificDemo {
    
    // 方法1：参数是 Object
    public static void method(Object obj) { }
    
    // 方法2：参数是 String（String extends Object）
    public static void method(String str) { }
    
    // method(String) 比 method(Object) 更具体
    // 因为 String 能做的事 Object 不一定能做（String 有额外方法）
}

// ========== 更复杂的例子 ==========
public class ComplexDemo {
    
    public static void print(Collection<?> c) { }
    public static void print(List<?> l) { }
    public static void print(ArrayList<?> l) { }
    public static void print(LinkedList<?> l) { }
    
    public static void main(String[] args) {
        LinkedList<String> list = new LinkedList<>();
        
        print(list);  // 匹配 print(LinkedList<?>)
        // 因为 LinkedList 是 Collection/List/LinkedList 中最具体的
    }
}
```

| 方法 A | 方法 B | 谁更具体 | 原因 |
|-------|-------|---------|------|
| `print(String)` | `print(Object)` | String | String 是 Object 的子类 |
| `print(ArrayList)` | `print(List)` | ArrayList | ArrayList 是 List 的子类 |
| `print(int[])` | `print(Object)` | int[] | int[] 是 Object 数组 |
| `print(String)` | `print(int...)` | String | String 不是数组 |

## 2. 扩展追问

### ## 🔥 扩展追问 1：varargs（可变参数）和重载一起时的坑

> **A:** varargs 是重载解析中最"弱"的类型，通常是最后才被匹配的；但多个 varargs 重载可能导致编译器无法决定。

```java
public class VarargsOverload {
    
    // 可变参数方法
    public static void print(String... args) {
        System.out.println("varargs: " + Arrays.toString(args));
    }
    
    // 普通参数方法
    public static void print(String s) {
        System.out.println("single: " + s);
    }
    
    public static void main(String[] args) {
        // ========== varargs 和普通方法的优先级 ==========
        
        print("hello");      // 匹配 print(String)，不是 varargs！
        // 编译器认为普通参数比 varargs 更具体
        
        print();             // 匹配 print(String...)
        // 只有无参时才会匹配 varargs
        
        // ========== 多个 varargs 时的歧义 ==========
    }
}

class AmbiguousDemo {
    public static void print(String... a) { }
    public static void print(String... b) { }  // 编译错误！无法区分
    
    // 两个 varargs 方法签名相同，编译器无法决定
}
```

```
┌─────────────────────────────────────────────────────────────────────┐
│                 varargs 的匹配优先级                                 │
│                                                                     │
│  调用 print("hello") 时：                                          │
│                                                                     │
│  1. 精确匹配 print(String) ──→ 找到！                               │
│                                                                     │
│  2. varargs 展开为数组                                              │
│     print(new String[]{"hello"})                                    │
│     匹配 print(String...)                                           │
│                                                                     │
│  3. 向上转型匹配 print(Object)                                      │
│                                                                     │
│  编译器选择：精确匹配 > varargs > 向上转型                           │
│                                                                     │
│  所以 print("hello") → print(String)                                │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### ## 🔥 扩展追问 2：泛型和重载一起时的歧义问题

> **A:** 泛型擦除后，相同擦除签名的泛型方法不能重载，编译器会报"method is already defined"错误。

```java
// ========== 泛型擦除后的重载问题 ==========
public class GenericOverloadIssue {
    
    // 编译错误！两个方法在擦除后签名相同
    public static void process(List<String> list) { }  // 擦除后 List<Object>
    public static void process(List<Integer> list) { } // 擦除后 List<Object>
    
    // ========== 解决方案：使用泛型方法 ==========
    public static <T> void processGeneric(List<T> list) {
        // 通过泛型方法处理所有类型
    }
    
    // ========== 方法重写时的泛型 ==========
}

class Parent<T> {
    // 父类方法
    public void print(T t) {
        System.out.println("Parent: " + t);
    }
}

class Child extends Parent<String> {
    @Override
    public void print(String t) {  // T 被 String 替换
        System.out.println("Child: " + t);
    }
}
```

### ## 🔥 扩展追问 3：自动装箱和拆箱对重载的影响

> **A:** 自动装箱（boxing）和拆箱（unboxing）会影响方法匹配，但有明确的优先级规则。

```java
public class BoxingOverload {
    
    public static void print(int i) {
        System.out.println("int: " + i);
    }
    
    public static void print(Integer i) {
        System.out.println("Integer: " + i);
    }
    
    public static void print(Object o) {
        System.out.println("Object: " + o);
    }
    
    public static void main(String[] args) {
        int primitive = 10;
        Integer wrapper = 20;
        
        // ========== int 类型参数 ==========
        print(primitive);  // 匹配 print(int)
        // 匹配顺序：int → int(精确) > Integer > Object
        
        print(wrapper);    // 匹配 print(Integer)
        // 匹配顺序：Integer(精确) > int(自动拆箱) > Object
        
        // ========== null 值的匹配 ==========
        Integer nullInt = null;
        print(nullInt);    // 匹配 print(Integer)，null 可匹配任何引用
        
        // ========== 基本类型 vs 包装类型 ==========
        // print(10)   → print(int)      精确匹配 int
        // print(10L)  → print(Object)   long 匹配不上 int/Integer
    }
}
```

| 实参类型 | 优先级 |
|---------|--------|
| `int x` → `print(int)` | 精确匹配 int > Integer > Object |
| `Integer x` → `print(Integer)` | 精确匹配 Integer > int(拆箱) > Object |
| `null` → `print(String)` | String > Object |

### ## 🔥 扩展追问 4：重载和重写的执行优先级

> **A:** **重写（Override）的优先级高于重载（Overload）**，子类重写方法会遮蔽父类的重载方法。

```java
public class OverrideVsOverload {
    
    static class Parent {
        public void print(String s) {
            System.out.println("Parent.print(String): " + s);
        }
    }
    
    static class Child extends Parent {
        // 这不是重写，是重载！因为参数类型不同
        public void print(Object o) {
            System.out.println("Child.print(Object): " + o);
        }
        
        // 如果参数相同，才是重写
        @Override
        public void print(String s) {
            System.out.println("Child.print(String) @Override");
        }
    }
    
    public static void main(String[] args) {
        Parent p = new Child();
        p.print("hello");
        // 调用的是 Child.print(String)（重写）
        // 运行时多态：实际对象是 Child，看 Child 类的方法
        
        p.print((Object)"hello");
        // 调用 Child.print(Object)（重载）
        // 编译时决定：参数是 Object，所以匹配 Parent.print(String) 不行
        // 向上转型后匹配 Child.print(Object)
    }
}
```

```
┌─────────────────────────────────────────────────────────────────────┐
│                 重写 vs 重载：优先级关系                             │
│                                                                     │
│  调用链：                                                          │
│  Parent p = new Child();                                          │
│  p.print("hello");                                                │
│                                                                     │
│  1. 先确定引用类型：Parent                                         │
│                                                                     │
│  2. 编译期方法解析（Overload）：                                     │
│     Parent 类有 print(String)                                      │
│                                                                     │
│  3. 运行时方法分派（Override）：                                     │
│     Child 重写了 print(String)                                      │
│     → 实际调用 Child.print(String)                                 │
│                                                                     │
│  总结：编译期决定"能用哪个方法"，运行时决定"实际用哪个版本"           │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### ## 🔥 扩展追问 5：方法引用和方法重载的交互

> **A:** 方法引用（如 `println`）也会进行重载解析，编译器根据目标类型选择最具体的方法。

```java
import java.util.function.*;

public class MethodRefOverload {
    
    public static void print(String s) {
        System.out.println("print(String): " + s);
    }
    
    public static void print(Object o) {
        System.out.println("print(Object): " + o);
    }
    
    public static void main(String[] args) {
        // ========== 方法引用的重载解析 ==========
        
        // Consumer<T> 接口的 accept(T) 方法
        Consumer<String> c1 = System.out::println;
        // 匹配：void println(String) 
        // 因为 Consumer<String> → accept(String) → println(String)
        
        Consumer<Object> c2 = System.out::println;
        // 匹配：void println(Object) 
        // 因为 Consumer<Object> → accept(Object) → println(Object)
        
        // ========== 方法引用赋值给函数式接口 ==========
        // print 是重载方法
        // 用方法引用赋值时，根据函数式接口类型决定用哪个
        
        ToStringFunction f1 = OverloadDemo::print;
        // ToStringFunction: R apply(T t)
        // 匹配 print(String)
        
        Function<Object, Void> f2 = OverloadDemo::print;
        // Function<Object, Void>: Void apply(Object)
        // 匹配 print(Object)
    }
}
```

## 3. 一句话总结

> **方法重载是编译期的静态分派，传入 null 时编译器选择"最具体"的方法版本（String > Object，String > int...）；Most Specific Method 规则是参数类型范围更小的更具体（子类比父类具体），泛型擦除后相同擦除签名的方法不能重载，varargs 优先级最低但精确匹配优先于 varargs 展开。**

## 4. 面试加分话术

> "我之前踩过重载的坑——当时写了个方法，同时有 `print(String)` 和 `print(Object)` 两个重载，传 null 的时候按预期调用了 print(String)，但后来加了 `print(String... args)` 之后，null 就开始调用 varargs 版本了，排查了半天才发现是参数顺序变了。还有个经验：自动装箱和基本类型混用时要注意，`print(int)` 和 `print(Integer)` 在传数字字面量时优先级不一样。实际项目中尽量避免用 null 作为参数，如果必须这样用，注释要写清楚调用的是哪个方法。另外静态导入或 Lambda 里用方法引用时，函数式接口类型会决定匹配哪个重载版本，这个坑在大项目里很隐蔽。"
