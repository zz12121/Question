```
┌─────────────────────────────────────────────────────────────────────┐
│                      泛型擦除过程                                    │
│                                                                     │
│  Java 源码：                                                         │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  List<String> list1 = new ArrayList<>();                    │   │
│  │  List<Integer> list2 = new ArrayList<>();                    │   │
│  │                                                             │   │
│  │  list1.add("hello");                                        │   │
│  │  list2.add(100);                                            │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                           ↓ 编译（类型擦除）                          │
│                                                                     │
│  编译后字节码：                                                     │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  List list1 = new ArrayList();                              │   │

│                                                                     │

│  ┌─────────────────────────────────────────────────────────────┐   │

│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### List String vs List Integer 的真相

```
┌─────────────────────────────────────────────────────────────────────┐
│            List<String> 和 List<Integer> 在 JVM 层面完全相同！          │
│                                                                     │
│  编译后：                                                           │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  List<String> list1  ──→  List list1                        │   │
│  │  List<Integer> list2 ──→  List list2                        │   │
│  │                                                             │   │
│  │  擦除规则：                                                  │   │
│  │  • 无上限的泛型 → Object                                     │   │
│  │  • 有上限的泛型 → 上限类型                                    │   │
│  │    例: <T extends Number> → Number                          │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  运行时：                                                           │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  list1.getClass() == list2.getClass()  → true！             │   │
│  │  结果都是：java.util.ArrayList                                │   │
│  │                                                             │   │
│  │  isInstance() 检查：                                         │   │
│  │  list1 instanceof List    → true                           │   │
│  │  list1 instanceof List<?>  → true（通配符可以）              │   │
│  │  list1 instanceof List<String> → 编译不通过！                │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 为什么泛型数组不能直接创建？

```java
// ❌ 编译错误
T[] array = new T[10];  // Cannot create a generic array

// ❌ 编译错误
List<String>[] arr = new ArrayList<String>[10];

// ✅ 绕过方式（但不安全）
T[] array = (T[]) new Object[10];  // 强制类型转换，编译器警告

// ✅ 正确做法：传入 Class 对象
public class GenericArray<T> {
    private final T[] array;
    
    public GenericArray(Class<T> clazz, int size) {
        this.array = (T[]) Array.newInstance(clazz, size);
    }
}
```

```
┌─────────────────────────────────────────────────────────────────────┐
│                 为什么泛型数组创建会编译错误？                        │
│                                                                     │
│  问题场景：                                                         │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  public T[] createArray(Class<T> clazz) {                   │   │
│  │      return new T[10];  // 编译错误                          │   │
│  │  }                                                           │   │
│  │                                                             │   │
│  │  // 调用：                                                   │   │
│  │  String[] arr = createArray(String.class);                   │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  原因分析：                                                         │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  1. 泛型在运行时被擦除，T → Object                            │   │
│  │  2. new T[10] 实际变成 new Object[10]                        │   │
│  │  3. Object[] 不能赋值给 String[]（类型不安全）                │   │
│  │  4. 所以编译器直接禁止这种写法                                │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  泛型数组的反面教材：                                               │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  List<String>[] arr = new ArrayList[10];  // 编译通过！      │   │
│  │  arr[0] = new ArrayList<Integer>();  // 运行时才发现错误     │   │
│  │  String s = arr[0].get(0);  // ClassCastException！         │   │
│  │                                                             │   │
│  │  这就是"类型安全"的隐患                                       │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 2. 扩展追问

### ## 🔥 扩展追问 1：泛型擦除后编译器会做什么？

> **A:** 泛型擦除后，编译器会做三件事：**类型转换插入**、**桥接方法生成**、**类型检查保留**。

```java
// ========== 源码 ==========
public class Person implements Comparable<Person> {
    private String name;
    
    public int compareTo(Person other) {
        return this.name.compareTo(other.name);
    }
}

// ========== 编译后（反编译）==========
public class Person implements Comparable {
    private String name;
    
    // 原始方法（编译器保留）
    public int compareTo(Person other) {
        return this.name.compareTo(other.name);
    }
    
    // 桥接方法（编译器生成！）
    public volatile int compareTo(Object other) {
        // Object 是被擦除后的类型
        return this.compareTo((Person) other);  // 自动插入类型转换
    }
}
```

```java
// ========== 类型转换插入示例 ==========
// 源码
List<String> list = new ArrayList<>();
String s = list.get(0);  // 看起来不需要转型

// 编译后字节码
List list = new ArrayList();
String s = (String) list.get(0);  // 编译器自动插入转型！
```

| 编译器行为 | 说明 |
|-----------|------|
| 类型转换插入 | getter 返回 Object，编译器插入 cast |
| 桥接方法生成 | 子类方法签名冲突时生成桥接方法 |
| 类型检查保留 | 泛型检查发生在编译期，运行时不检查 |

### ## 🔥 扩展追问 2：getTypeParameters() 能拿到实际类型吗？

> **A:** **不能**。`getTypeParameters()` 返回的是**泛型声明时的类型参数**，而不是泛型实例化时的实际类型。

```java
// ========== getTypeParameters() 能拿到的 ==========
public class Container<T, K extends Number> {
    // ...
}

TypeVariable<?>[] params = Container.class.getTypeParameters();
// 返回：[T, K]
// 能拿到：参数名、泛型边界
// 拿不到：T 实际是 String 还是 Integer

for (TypeVariable<?> param : params) {
    System.out.println("参数名: " + param.getTypeName());
    System.out.println("上界: " + Arrays.toString(param.getBounds()));
    // 参数名: T
    // 上界: [java.lang.Object]
    // 参数名: K
    // 上界: [java.lang.Number]
}

// ========== 无法获取实际类型 ==========
Container<String, Integer> container = new Container<>();
// 即使有这个实例，也无法通过 container 获取 String 和 Integer

// 因为泛型信息在编译后被擦除！
// 运行时只有：Container.class
```

```java
// ========== 通过反射获取字段的泛型类型（可以！）==========
public class User {
    private Map<String, List<Order>> orderMap;
}

Field field = User.class.getDeclaredField("orderMap");
Type type = field.getGenericType();  // 返回 ParameterizedType

if (type instanceof ParameterizedType) {
    ParameterizedType pt = (ParameterizedType) type;
    Type[] actualTypes = pt.getActualTypeArguments();
    // actualTypes[0] = String.class
    // actualTypes[1] = List<Order>（ParameterizedType 嵌套）
}
```

| 方法 | 能获取什么 | 不能获取什么 |
|------|----------|------------|
| `Class.getTypeParameters()` | 声明时的类型参数（T、K） | 实例化时的实际类型 |
| `Field.getGenericType()` | 字段的泛型类型 | ❌ |
| `getActualTypeArguments()` | 参数化类型的实际类型 | ❌ |

### ## 🔥 扩展追问 3：泛型擦除有什么实际影响？

> **A:** 泛型擦除带来了几个实际限制：**泛型不能用于 instanceof**、**不能 new 泛型对象**、**不能创建泛型数组**。

```java
// ========== 影响 1：泛型不能用于 instanceof ==========
List<String> list = new ArrayList<>();
// ❌ 编译错误
if (list instanceof List<String>) { }

// ✅ 可以用通配符
if (list instanceof List<?>) { }

// ========== 影响 2：泛型不能作为静态变量 ==========
public class Cache<T> {
    // ❌ 编译错误：静态变量不能引用类级别的泛型
    private static T value;
    
    // ✅ 静态方法可以有自己的泛型
    public static <T> void doSomething(T item) { }
}

// ========== 影响 3：泛型不能用于基本类型 ==========
// ❌ 编译错误
List<int> list = new ArrayList<>();

// ✅ 必须用包装类型
List<Integer> list = new ArrayList<>();

// ========== 影响 4：类型擦除后的冲突 ==========
public class MyList extends ArrayList<Integer> {
    // 重写 ArrayList 的 add(E e) 方法
    public void add(Integer e) { }
    
    // 编译器自动生成桥接方法：
    // public void add(Object e) {
    //     return this.add((Integer) e);
    // }
}

// 如果父类没有泛型，子类想加泛型怎么办？
// ❌ 编译错误：无法多态化
public class MyList<T> extends ArrayList<T> {
    @Override
    public void add(T e) { }  // 正常
}
```

### ## 🔥 扩展追问 4：泛型上限（extends）和下限（super）有什么区别？

> **A:** `<? extends T>` 用于**读取**（生产者），`<? super T>` 用于**写入**（消费者），这就是 PECS 原则（Producer Extends, Consumer Super）。

```java
// ========== PECS 原则 ==========
// Producer extends：需要读取数据时用 extends
public void readFrom(List<? extends Number> list) {
    Number n = list.get(0);  // ✅ 可以读，类型是 Number
    // list.add(100);  // ❌ 不能写，编译器不知道具体类型
}

// Consumer super：需要写入数据时用 super
public void writeTo(List<? super Integer> list) {
    list.add(100);  // ✅ 可以写，Integer 是 Integer 或其父类型
    Number n = (Number) list.get(0);  // ❌ 读出来是 Object，需转型
}

// ========== 典型应用场景 ==========
// Collections.copy()
public static <T> void copy(List<? super T> dest, List<? extends T> src) {
    for (T item : src) {    // src 是生产者，读取 T
        dest.add(item);       // dest 是消费者，接收 T
    }
}

// ========== 错误示例 ==========
// ❌ 这是错误的理解
List<Number> numbers = new ArrayList<Integer>();  // 编译错误！
// Integer 是 Number 的子类型，但 List<Integer> 不是 List<Number> 的子类型！

// ✅ 正确：用通配符
List<? extends Number> numbers = new ArrayList<Integer>();  // 编译通过
```

| 通配符 | 别名 | 用途 | 能读？ | 能写？ |
|-------|------|------|--------|--------|
| `<? extends T>` | 生产者-extends | 读取数据 | ✅（返回 T） | ❌ |
| `<? super T>` | 消费者-super | 写入数据 | ❌ | ✅（接收 T） |
| `<?>` | 无界通配符 | 不知道/不关心类型 | ❌（返回 Object） | ❌ |

### ## 🔥 扩展追问 5：为什么需要泛型？不用泛型行不行？

> **A:** 泛型的核心价值是**编译期类型安全**和**代码复用**，不用泛型也可以（Java 1.5 之前就是这样），但会导致大量强制类型转换和运行时 ClassCastException。

```java
// ========== 不用泛型（Java 1.4 风格）==========
ArrayList list = new ArrayList();
list.add("hello");
list.add(100);

String s = (String) list.get(0);  // ✅ 能编译，但运行时 ClassCastException！
Integer i = (Integer) list.get(1);  // 正常

// ========== 用泛型（Java 5+）==========
ArrayList<String> list = new ArrayList<>();
list.add("hello");
// list.add(100);  // 编译错误！类型安全
String s = list.get(0);  // 不需要转型，100% 安全

// ========== 泛型的优势 ==========
// 1. 编译期类型检查
// 2. 消除强制类型转换
// 3. 代码复用（一次定义，多种类型）
// 4. 类型安全（把 ClassCastException 提前到编译期）

// 泛型擦除的本质：运行时退化为 Object，类型检查交给编译器
```

## 3. 一句话总结

> **泛型擦除发生在编译期（运行时不存在），`List<String>` 和 `List<Integer>` 在 JVM 层面都是 `List`；泛型数组不能 `new T[10]` 因为擦除后是 `Object[]`，无法赋值给具体类型的数组；`Class.getTypeParameters()` 只能拿到声明时的类型参数名和边界，拿不到实例化时的实际类型。**
