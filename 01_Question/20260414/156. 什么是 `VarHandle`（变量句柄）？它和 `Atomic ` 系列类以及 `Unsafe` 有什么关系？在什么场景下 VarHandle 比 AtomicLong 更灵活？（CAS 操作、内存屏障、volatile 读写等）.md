### VarHandle 的基本用法

```java
// ========== VarHandle 创建方式 ==========
public class VarHandleDemo {
    
    // 普通字段
    private int value;
    
    // 静态字段
    private static long staticValue;
    
    // 数组字段
    private int[] array = new int[10];
    
    public void demo() throws NoSuchFieldException, IllegalAccessException {
        // 1. 通过 MethodHandles 创建（推荐）
        VarHandle vh = MethodHandles.lookup()
            .in(VarHandleDemo.class)
            .findVarHandle(VarHandleDemo.class, "value", int.class);
        
        // 2. 通过反射创建
        Field field = VarHandleDemo.class.getDeclaredField("value");
        VarHandle vh2 = MethodHandles.lookup().unreflectVarHandle(field);
        
        // 3. 数组元素访问
        VarHandle arrVh = MethodHandles.arrayElementVarHandle(int[].class);
    }
}
```

### CAS 操作对比

```java
// ========== VarHandle vs AtomicInteger CAS 对比 ==========

public class CASComparison {
    private int value = 0;
    private AtomicInteger atomicValue = new AtomicInteger(0);
    
    // AtomicInteger 方式
    public void atomicIncrement() {
        atomicValue.compareAndSet(0, 1);  // CAS
        atomicValue.incrementAndGet();      // ++i
    }
    
    // VarHandle 方式（Java 9+）
    private VarHandle vh;
    
    public void init() throws NoSuchFieldException, IllegalAccessException {
        vh = MethodHandles.lookup()
            .findVarHandle(CASComparison.class, "value", int.class);
    }
    
    public void varhandleIncrement() {
        // CAS
        boolean success = vh.compareAndSet(this, 0, 1);
        
        // 原子更新（类似 incrementAndGet）
        int oldVal = (int) vh.getAndAdd(this, 1);
        int newVal = oldVal + 1;
    }
}
```

## 2. 扩展追问

### ## 🔥 扩展追问 1：VarHandle 和 Unsafe 有什么区别？

> **A:** VarHandle 是 Unsafe 的**安全替代品**，Unsafe 允许任意内存访问和指针操作（危险），VarHandle 只操作 Java 变量且需要通过 MethodHandles 获取。

| 维度 | VarHandle | Unsafe |
|------|----------|--------|
| **安全性** | ✅ 安全，通过 MethodHandles 获取 | ⚠️ 不安全，需启动参数授权 |
| **使用场景** | Java 变量并发访问 | 底层内存操作、CMS 内部 |
| **内存屏障** | 内置支持 | 需手动调用 |
| **学习成本** | 中等 | 高 |
| **稳定性** | 标准 API | JDK 内部 API，可能变更 |

```java
// Unsafe（危险，需 --add-opens）
Unsafe unsafe = Unsafe.getUnsafe();
// 可以：unsafe.objectFieldOffset()、putLong()、copyMemory()

// VarHandle（安全，Java 9+ 标准 API）
VarHandle vh = MethodHandles.lookup().findVarHandle(...);
// 操作边界清晰，类型安全
```

### ## 🔥 扩展追问 2：VarHandle 如何控制内存屏障和可见性？

> **A:** VarHandle 通过不同的方法变体控制内存顺序语义，从最严格到最宽松有多个级别。

```java
public class MemoryBarrierDemo {
    private volatile int value = 0;
    private VarHandle vh;
    
    public void init() throws NoSuchFieldException, IllegalAccessException {
        vh = MethodHandles.lookup()
            .findVarHandle(MemoryBarrierDemo.class, "value", int.class);
    }
    
    public void writeExample(int newValue) {
        // ========== 不同写入顺序 ==========
        
        // 1. 最严格：volatile 语义（StoreLoad 屏障）
        vh.setVolatile(this, newValue);
        // 等价于：this.value = newValue; （volatile 写）
        
        // 2. StoreStore 屏障（禁止之前写重排到这里）
        vh.setRelease(this, newValue);
        
        // 3. 无屏障
        vh.set(this, newValue);
        
        // 4. StoreStore + LoadStore 屏障
        vh.setOpaque(this, newValue);
    }
    
    public int readExample() {
        // ========== 不同读取顺序 ==========
        
        // 1. 最严格：volatile 语义（LoadLoad + LoadStore 屏障）
        return (int) vh.getVolatile(this);
        
        // 2. LoadAcquire 屏障
        return (int) vh.getAcquire(this);
        
        // 3. 无屏障
        return (int) vh.get(this);
        
        // 4. LoadLoad + LoadStore 屏障
        return (int) vh.getOpaque(this);
    }
}
```

| 方法 | 内存屏障 | 说明 |
|------|---------|------|
| `setVolatile / getVolatile` | Full Barrier | volatile 语义，最严格 |
| `setRelease / getAcquire` | StoreLoad / LoadLoad+LoadStore | 释放获取语义 |
| `setOpaque / getOpaque` | StoreStore+LoadStore / LoadLoad+LoadStore | Opaque |
| `set / get` | 无 | 最低保证 |

### ## 🔥 扩展追问 3：VarHandle 比 AtomicLong 更灵活的场景

> **A:** VarHandle 的灵活性体现在：可以**批量操作数组**、**自定义 CAS 条件**、**访问非公开字段**、**细粒度内存顺序控制**。

```java
// ========== 场景1：批量 CAS（数组操作）==========
public class BatchCAS {
    private int[] values = new int[1000];
    private VarHandle arrVh;
    
    public void init() throws NoSuchFieldException, IllegalAccessException {
        arrVh = MethodHandles.arrayElementVarHandle(int[].class);
    }
    
    // 给数组中所有符合条件的元素 +1
    public int batchIncrementIf(int condition) {
        int updated = 0;
        for (int i = 0; i < values.length; i++) {
            while (true) {
                int oldVal = (int) arrVh.getVolatile(values, i);
                if (oldVal < condition) break;  // 不满足条件
                
                // 尝试 CAS
                if (arrVh.compareAndSet(values, i, oldVal, oldVal + 1)) {
                    updated++;
                    break;  // 成功，继续下一个
                }
                // else: 失败，重试
            }
        }
        return updated;
    }
}

// ========== 场景2：自定义 CAS 条件 ==========
public class CustomCAS {
    private long value = 0;
    private VarHandle vh;
    
    // compareAndExchange：返回更新前的值（原子操作）
    public long compareAndExchangeIfGreater(int expected, int newValue) {
        // 语义：如果当前值 > expected，则设置为 newValue
        // 返回：实际被替换的值
        return (long) vh.compareAndExchange(this, (long) expected, (long) newValue);
    }
}

// ========== 场景3：访问父类/其他类的 private 字段 ==========
public class AccessPrivateField {
    static class Inner {
        private int secret = 42;
    }
    
    public static void main(String[] args) throws Throwable {
        // VarHandle 可以访问 private 字段（通过 lookup）
        VarHandle vh = MethodHandles.lookup()
            .findVarHandle(Inner.class, "secret", int.class);
        
        Inner obj = new Inner();
        
        // 写入（绕过 getter/setter）
        vh.set(obj, 100);
        
        // 读取
        int val = (int) vh.get(obj);  // 100
    }
}
```

| 场景 | VarHandle | AtomicLong |
|------|-----------|------------|
| 数组批量操作 | ✅ 支持 | ❌ 不支持 |
| 细粒度内存顺序 | ✅ 多级别可选 | ❌ 固定 volatile |
| 非 volatile 变量 | ✅ 加内存屏障 | ❌ 只能 volatile |
| compareAndExchange | ✅ 返回旧值 | ❌ 返回 boolean |
| 访问 private 字段 | ✅ 通过 MethodHandles | ❌ 不行 |

### ## 🔥 扩展追问 4：VarHandle 的典型使用场景

> **A:** VarHandle 主要用于：**并发框架内部实现**（如 JUC 优化）、**自定义同步器**、**高性能数据结构**、**性能关键的细粒度控制**。

```java
// ========== 场景1：自定义双检查单例 ==========
public class VarHandleSingleton {
    private static volatile VarHandleSingleton instance;
    
    // VarHandle 比 AtomicReference 更灵活
    private static final VarHandle VH = MethodHandles.lookup()
        .findVarHandle(VarHandleSingleton.class, "instance", VarHandleSingleton.class);
    
    public static VarHandleSingleton getInstance() {
        VarHandleSingleton current = (VarHandleSingleton) VH.getAcquire(this);
        if (current == null) {
            synchronized (VarHandleSingleton.class) {
                current = (VarHandleSingleton) VH.get(this);
                if (current == null) {
                    current = new VarHandleSingleton();
                    VH.setRelease(this, current);  // release 语义写入
                }
            }
        }
        return current;
    }
}

// ========== 场景2：JDK 内部的 VarHandle 使用 ==========
// Java 9+ 的 JUC 包大量使用 VarHandle
// 例如：AtomicInteger 的实现内部可能使用 VarHandle

// sun.misc.Contended → @Contended 注解内部使用 VarHandle 实现
// ForkJoinPool 的 WorkQueue 使用 VarHandle
```

### ## 🔥 扩展追问 5：VarHandle 和 juc.atomic 的选择建议

> **A:** 日常开发用 **Atomic*** 系列足够了；需要**细粒度控制**、**数组操作**、**自定义原子操作**时才用 VarHandle。

```java
// ========== 选择建议 ==========

// ✅ 用 AtomicInteger/AtomicLong（简单场景）
AtomicLong counter = new AtomicLong(0);
counter.incrementAndGet();  // 常用操作，一行搞定

// ✅ 用 VarHandle（高级场景）
// 1. 需要细粒度内存顺序
vh.getAcquire(this);  // 非 volatile 读，但有 acquire 屏障

// 2. 需要操作数组大量元素
vh.compareAndSet(array, index, oldVal, newVal);

// 3. 需要自定义复合原子操作
// compareAndExchange 返回旧值，可以做更复杂的判断

// 4. 需要访问受限字段
VarHandle vh = MethodHandles.privateLookupIn(field);
```

| 场景 | 推荐 | 原因 |
|------|------|------|
| 普通计数器 | AtomicLong | API 简洁 |
| 需要 compareAndSet 循环 | AtomicLong | 常用 |
| 细粒度内存屏障 | VarHandle | 可选屏障级别 |
| 数组批量 CAS | VarHandle | 支持数组操作 |
| JDK 内部实现 | VarHandle | 标准 API，稳定 |
| 简单并发修改 | AtomicReference | 简单易用 |

## 3. 一句话总结

> **VarHandle 是 JDK 9 引入的安全替代 Unsafe 的变量访问 API，支持细粒度 CAS 和内存屏障控制（get/set/ACQUIRE/RELEASE/OPAQUE），比 AtomicLong 更灵活在于支持数组元素原子操作、compareAndExchange 返回旧值、自定义内存顺序、访问 private 字段；AtomicLong 适合简单计数器场景，VarHandle 适合并发框架和高性能数据结构内部实现。**
