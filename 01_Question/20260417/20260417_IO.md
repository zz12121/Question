# 理论综合题 - Java IO/NIO（A266-A295）

> 以下为 Java IO/NIO 领域的理论综合题，编号从266起，覆盖 BIO/NIO/AIO 模型、零拷贝、Selector/epoll、序列化、文件操作等核心原理。

---

## IO 模型基础（6题）

266. Java 的 BIO（Blocking IO）、NIO（Non-blocking IO）、AIO（Asynchronous IO）三种 IO 模型的核心区别是什么？BIO 为什么在高并发下不可行？（每个连接占用一个线程，线程资源有限）NIO 是怎么实现"一个线程处理多个连接"的？（Selector 多路复用 + 非阻塞 Channel）AIO 和 NIO 的本质区别在哪里？（NIO 是同步非阻塞——数据就绪后仍需用户线程自己读写；AIO 是异步——内核完成读写后回调通知用户线程）为什么 Linux 下 Netty 用的还是 NIO 而不是 AIO？（Linux 的 AIO 实现不完善，epoll + NIO 性能更好）

267. 操作系统层面的 IO 模型有哪五种？（阻塞 IO / 非阻塞 IO / IO 多路复用 / 信号驱动 IO / 异步 IO）它们和 Java 的 BIO/NIO/AIO 是怎么对应的？（BIO → 阻塞 IO；NIO → IO 多路复用；AIO → 异步 IO）"同步"和"异步"的区别是什么？"阻塞"和"非阻塞"的区别是什么？这两个维度是正交的吗？（是的，同步/异步关注"谁来做实际的 IO 操作"，阻塞/非阻塞关注"调用方是否等待"）

268. 什么是"用户空间"和"内核空间"？为什么 IO 操作需要在这两个空间之间拷贝数据？（CPU 的保护模式，用户程序不能直接操作硬件，必须通过系统调用进入内核态）一次普通的文件读取（`FileInputStream.read()`）涉及几次数据拷贝？（磁盘 → 内核缓冲区 → 用户缓冲区，2 次拷贝；如果再写网卡：用户缓冲区 → Socket 缓冲区 → 网卡，共 4 次拷贝）上下文切换有几次？（用户态→内核态 read() → 用户态 → 内核态 write() → 用户态，4 次切换）

269. 什么是"零拷贝"（Zero-Copy）？它解决的是什么问题？（减少用户空间和内核空间之间的数据拷贝次数，降低 CPU 开销和上下文切换）Java 中有哪些零拷贝的实现方式？（`FileChannel.transferTo()` / `FileChannel.transferFrom()` / `DirectByteBuffer` / `mmap` Memory-Mapped File）`sendfile()` 系统调用是怎么工作的？（数据直接从内核文件缓冲区到内核 Socket 缓冲区，不经过用户空间）`mmap()` 和 `sendfile()` 的区别是什么？（mmap 将文件映射到用户空间，用户程序可以直接操作；sendfile 完全在内核空间完成）

270. Java NIO 的三大核心组件是什么？Buffer、Channel、Selector 各自的作用是什么？（Buffer：数据的容器，读写的中转站 / Channel：双向的数据通道，可读可写 / Selector：多路复用器，一个线程监控多个 Channel 的 IO 事件）Buffer 的核心属性有哪些？（capacity / position / limit / mark）`flip()` 方法的作用是什么？（将 Buffer 从写模式切换到读模式：position→0, limit→原 position）`clear()` 和 `compact()` 的区别是什么？

271. Buffer 的类型有哪些？（ByteBuffer / CharBuffer / ShortBuffer / IntBuffer / LongBuffer / FloatBuffer / DoubleBuffer）ByteBuffer 又分"堆内缓冲"（HeapByteBuffer）和"直接缓冲"（DirectByteBuffer），两者的区别是什么？（HeapByteBuffer 在 JVM 堆上，GC 管理 / DirectByteBuffer 在堆外内存，通过 `Unsafe` 分配，不受 GC 直接管理）DirectByteBuffer 有什么优势和劣势？（优势：减少一次内核→用户的拷贝 / 劣势：分配和回收开销大、内存不受 JVM 堆大小限制、可能内存泄漏）什么场景下应该用 DirectByteBuffer？（大量 IO 操作 / Netty 的 ByteBuf 默认用池化的 DirectByteBuf）

## Selector 与 epoll（5题）

272. Java NIO 的 Selector 是怎么工作的？核心流程是什么？（创建 Selector → 将 Channel 注册到 Selector 并指定感兴趣的事件 → `select()` 阻塞等待事件 → 遍历 SelectedKeys 处理事件）Selector 支持哪四种事件？（OP_READ / OP_WRITE / OP_CONNECT / OP_ACCEPT）一个 Selector 最多能管理多少个 Channel？（受操作系统限制，Linux 默认 `fs.file-max` 约 10 万+，但实际受 `ulimit -n` 限制）为什么 NIO 模型下不需要为每个连接创建一个线程？

273. Java NIO 的 Selector 在 Linux 底层使用的是什么系统调用？（epoll）epoll 相比 select/poll 有什么优势？（epoll 基于"事件通知"而非"轮询" / epoll 只返回就绪的 fd，不需要遍历所有 fd / epoll 使用红黑树管理 fd，没有 fd 数量限制）select 的最大 fd 数量限制是多少？（1024，`FD_SETSIZE`）poll 有没有这个限制？（没有，但仍然需要遍历所有 fd）epoll 的 LT（Level Triggered，水平触发）和 ET（Edge Triggered，边缘触发）模式有什么区别？Java NIO 用的是哪种？（LT）

274. epoll 的"惊群效应"（Thundering Herd）是什么？在什么场景下会出现？（多个线程/进程同时 `epoll_wait` 同一个 fd，当 fd 就绪时所有线程都被唤醒但只有一个能处理）Linux 后来的版本是怎么解决的？（`EPOLLEXCLUSIVE` 标志，只唤醒一个线程）Nginx 是怎么处理这个问题的？（加锁 `accept_mutex`，同一时刻只有一个 worker 进程监听新连接）Java NIO 的 Selector 有这个问题吗？（在多 Selector 场景下可能出现，Netty 通过"单 Reactor 线程接收连接"来规避）

275. Java NIO 的 Selector 有一个"空轮询"（Epoll Bug）问题，是什么？（`Selector.select()` 应该阻塞等待事件，但某些情况下会立即返回 0，导致 CPU 100%空转）这个 Bug 在什么条件下触发？（epoll 的 LT 模式下，如果 fd 被关闭但没有从 epoll 中移除，可能导致 `EPOLLHUP` 事件不断触发）Netty 是怎么解决这个问题的？（检测 `select()` 返回 0 的次数，超过阈值（默认 512 次）后重建 Selector）重建 Selector 的开销大吗？需要迁移所有已注册的 Channel 吗？

276. 什么是 Reactor 模式？单 Reactor 单线程、单 Reactor 多线程、主从 Reactor 多线程三种模式分别是什么？Netty 采用的是哪种？（主从 Reactor 多线程：Boss Group 接收连接，Worker Group 处理 IO）为什么"接收连接"和"处理 IO"要分开？（接收连接很快不会阻塞，但 IO 处理可能耗时）Reactor 模式和 Proactor 模式的区别是什么？（Reactor 是同步非阻塞——事件就绪后用户线程读写 / Proactor 是异步——内核完成读写后回调通知）

## 文件 IO 与序列化（6题）

277. Java 的 `FileInputStream` / `FileOutputStream` 和 `FileChannel` 有什么区别？（Stream 是面向字节的阻塞 IO / Channel 是面向块的 NIO，支持零拷贝和内存映射）`FileChannel` 的 `read()` 和 `write()` 方法是阻塞的吗？（是的，即使 Channel 注册到 Selector，FileChannel 也不支持非阻塞模式）为什么 FileChannel 不支持非阻塞？（文件的 IO 不像网络 IO 那样需要等待远端响应，磁盘 IO 速度可预测）`FileChannel` 的 `force()` 方法做什么？（将 Channel 的数据强制刷到磁盘，类似 `fsync`）

278. 什么是"内存映射文件"（Memory-Mapped File / mmap）？Java 中怎么使用？（`FileChannel.map(MapMode.READ_WRITE, position, size)`）mmap 的原理是什么？（将文件的一块区域映射到用户空间的虚拟内存地址，访问这块内存等于访问文件，由 OS 负责页面的换入换出）mmap 和普通的 `FileChannel.read()` 有什么区别？（mmap 减少了一次"内核缓冲区→用户缓冲区"的拷贝）mmap 有什么限制？（映射区域大小受限 / 修改后什么时候刷盘由 OS 决定 / 大文件映射可能占用大量虚拟地址空间）

279. Java 的序列化（Serialization）有哪些方式？JDK 原生序列化、JSON（Jackson/Gson）、Protobuf、Hessian、Kryo 各自的优缺点是什么？（JDK：慢、体积大、不跨语言 / JSON：可读、慢、体积大 / Protobuf：快、体积小、需 IDL / Hessian：支持复杂对象、中等 / Kryo：快、不支持跨语言）为什么生产环境不推荐用 JDK 原生序列化？（`serialVersionUID` 兼容性问题 / 反序列化漏洞 / 性能差 / 体积大）`Serializable` 和 `Externalizable` 的区别是什么？

280. Protobuf 的编解码原理是什么？为什么它比 JSON 快这么多？（二进制编码 / 字段编号代替字段名 / Varint 变长编码 / 不需要解析字符串）Protobuf 的 `proto3` 和 `proto2` 有什么区别？（proto3 去掉了 required/optional 关键字 / 去掉了默认值 / 新增了 JSON 映射 / 更简洁但灵活性降低）Protobuf 的"向后兼容"是怎么保证的？（新增字段用新编号，旧代码忽略未知字段 / 删除字段用 `reserved` 保留编号）Protobuf 的缺点是什么？（需要 IDL 定义和代码生成 / 不支持直接查看数据 / 调试不方便）

281. Java 的 `Files` 类（JDK 7+）和传统的 `File` 类有什么区别？（`Files` 基于 NIO.2，支持异步、符号链接、文件属性等 / `File` 是传统 IO，功能有限）`Files.walk()` 和 `Files.list()` 的区别是什么？（`walk` 递归遍历子目录 / `list` 只列当前目录）`Files.walk()` 返回的是 Stream，如果目录层级很深或有大量文件，会不会 OOM？（不会 OOM，Stream 是惰性求值，但需要及时关闭防止资源泄露）`Files.lines()` 读取大文件时是怎么工作的？（惰性读取，每次只加载一行到内存）

282. Java 的 `WatchService`（JDK 7+）是怎么工作的？它用于什么场景？（监控文件/目录的创建、修改、删除事件）底层在不同 OS 上的实现有什么差异？（Linux: inotify / macOS: kqueue / Windows: ReadDirectoryChangesW）`WatchService` 有什么局限性？（事件可能丢失 / 无法获取变更内容 / 事件顺序不保证 / 大量文件时性能下降）在生产环境中如何实现更可靠的"文件变更通知"机制？（定时轮询 + 文件 hash 比对 / 自研基于 inotify 的方案 / 使用消息队列通知）

## Netty 核心（6题）

283. Netty 的线程模型是怎样的？Boss Group 和 Worker Group 分别做什么？（Boss Group: 负责 Accept 新连接 / Worker Group: 负责 IO 读写和业务处理）Boss Group 为什么通常只需要 1 个线程？（一个端口只需要一个 Selector 接收连接）Worker Group 的线程数怎么设置？（CPU 核数 × 2，如果是 IO 密集型可以更多）如果业务处理很耗时，怎么避免阻塞 Worker 线程？（添加业务线程池 / 使用 `DefaultEventExecutorGroup` 处理耗时 Handler）

284. Netty 的 ChannelPipeline 和 ChannelHandler 机制是怎样的？ChannelPipeline 是什么？（Handler 的链式容器，类似责任链模式）InboundHandler 和 OutboundHandler 的区别是什么？（Inbound 处理读入数据，方向 head→tail / Outbound 处理写出数据，方向 tail→head）一个请求的完整流转路径是怎样的？（Socket 读 → HeadContext → Decoder → BusinessHandler → Encoder → HeadContext → Socket 写）如果在 BusinessHandler 中调用了 `ctx.writeAndFlush()`，数据会经过哪些 Handler？（从当前 Handler 向前找 OutboundHandler）

285. Netty 的 ByteBuf 和 JDK 的 ByteBuffer 有什么区别？（ByteBuf 支持"读写双指针"，不需要 flip() / ByteBuf 支持池化 / ByteBuf 支持组合缓冲区 CompositeByteBuf / ByteBuf 支持引用计数）ByteBuf 的"池化"（PooledByteBufAllocator）和"非池化"（UnpooledByteBufAllocator）有什么区别？（池化使用 `jemalloc` 算法管理内存，减少 GC 压力 / 非池化每次创建新对象）ByteBuf 的引用计数（Reference Counting）是怎么工作的？（`refCnt` 引用计数，`release()` 减到 0 时回收内存）如果忘记 `release()` 会怎样？（内存泄漏，Netty 的 `ResourceLeakDetector` 会检测并告警）

286. Netty 的"粘包/半包"问题是什么？为什么 TCP 会产生这个问题？（TCP 是流式协议，没有消息边界 / 发送方可能将多条消息合并发送（粘包） / 接收方可能一次只收到半条消息（半包））Netty 提供了哪些解码器来解决？（`FixedLengthFrameDecoder` 固定长度 / `DelimiterBasedFrameDecoder` 分隔符 / `LengthFieldBasedFrameDecoder` 长度字段 / `LineBasedFrameDecoder` 换行符）`LengthFieldBasedFrameDecoder` 的参数含义是什么？（maxFrameLength / lengthFieldOffset / lengthFieldLength / lengthAdjustment / initialBytesToStrip）

287. Netty 的"空闲检测"和"心跳机制"是怎么实现的？`IdleStateHandler` 的作用是什么？（检测 Channel 的读空闲、写空闲、读写空闲时间）如果在指定时间内没有读写事件，`IdleStateHandler` 会触发什么？（触发 `userEventTriggered()` 方法，传入 `IdleStateEvent`）如何实现"服务端空闲超时后主动关闭连接"？（在 `userEventTriggered()` 中判断空闲类型，调用 `ctx.close()`）客户端如何实现"空闲时发送心跳"？（在 `userEventTriggered()` 中判断写空闲，发送心跳包）Netty 的心跳和 TCP 的 `SO_KEEPALIVE` 有什么区别？（TCP KEEPALIVE 在内核层检测，默认 2 小时才触发 / Netty 心跳在应用层，可自定义间隔和逻辑）

288. Netty 的 `EpollEventLoopGroup` 和 `NioEventLoopGroup` 有什么区别？（EpollEventLoopGroup 使用 Netty 自己的 JNI 封装的 epoll / NioEventLoopGroup 使用 JDK 的 Selector）为什么 Netty 要自己实现 epoll 而不直接用 JDK 的？（JDK 的 Selector 有 Epoll Bug / Netty 的 epoll 实现暴露了更多 epoll 特性如 `EPOLLET` 边缘触发 / 性能更好 / 支持 `tcp-cork`、`tcp-quickack` 等 TCP 优化参数）`EpollEventLoopGroup` 只能在 Linux 上使用吗？（是的，其他平台用 `KQueueEventLoopGroup`（macOS）或 `NioEventLoopGroup`）

## IO 调优与底层机制（4题）

289. Linux 的 `epoll` 事件通知机制在底层是怎么实现的？（`epoll_create` 创建 epoll 实例（红黑树 + 就绪链表）/ `epoll_ctl` 注册/修改/删除 fd（插入红黑树）/ `epoll_wait` 检查就绪链表，有数据则返回，无数据则阻塞）epoll 为什么比 select 高效？（不需要每次调用都传入所有 fd / 内核只返回就绪的 fd / 红黑树管理 fd 插入删除 O(logN)）epoll 的"回调"机制是什么？（当网卡收到数据，中断处理程序将 fd 加入就绪链表并唤醒 `epoll_wait`）

290. TCP 的 `TCP_NODELAY` 参数是什么？为什么要禁用 Nagle 算法？（Nagle 算法：将小包攒成大包再发送，减少网络中小包数量 / 禁用 Nagle 后小包立即发送，降低延迟）在什么场景下应该禁用 Nagle？（实时通信 / RPC 调用 / 游戏服务器——对延迟敏感的场景）`TCP_CORK` 和 `TCP_NODELAY` 的区别是什么？（`TCP_CORK` 是"塞住"发送，等到攒够数据或超时再发 / `TCP_NODELAY` 是立即发送）Netty 默认设置了 `TCP_NODELAY` 吗？（是的，`childOption(ChannelOption.TCP_NODELAY, true)`）

291. 什么是"IO 密集型"和"CPU 密集型"任务？如何判断一个任务是 IO 密集型还是 CPU 密集型？（IO 密集型：大部分时间在等待 IO（网络/磁盘）/ CPU 密集型：大部分时间在计算）线程池大小应该怎么设置？（CPU 密集型：核数+1 / IO 密集型：核数 × 2 或更多，公式 `线程数 = CPU核数 × (1 + IO等待时间/CPU计算时间)`）IO 密集型任务为什么需要更多线程？（线程在等待 IO 时让出 CPU，其他线程可以利用 CPU 做计算）虚拟线程（JDK 21）如何改变这个模型？（IO 密集型不再需要大量平台线程，虚拟线程自动挂起和恢复）

292. Linux 的 PageCache 是什么？它对文件 IO 性能有什么影响？（PageCache 是内核维护的磁盘缓存，最近读取的文件内容缓存在内存中，避免重复读磁盘）`FileChannel.read()` 和 `FileChannel.write()` 是否经过 PageCache？（是的，标准文件 IO 都经过 PageCache）`mmap` 和 PageCache 是什么关系？（mmap 映射的内存区域和 PageCache 共享同一份物理内存页）如何强制绕过 PageCache？（`O_DIRECT` 直接 IO，绕过 PageCache，数据库系统常用）Java 的 `FileChannel` 支持直接 IO 吗？（不直接支持，需要通过 JNI 或第三方库）

## NIO.2 与异步 IO（3题）

293. Java 7 引入的 NIO.2（JSR 203）主要新增了哪些特性？（`Path` 和 `FileSystem` API / `Files` 工具类 / `AsynchronousFileChannel` / `WatchService` / `SeekableByteChannel`）`Path` 和传统的 `File` 有什么区别？（`Path` 是接口，可扩展 / `File` 是具体类 / `Path` 支持符号链接和相对路径解析 / `Path` 提供更丰富的路径操作方法）`AsynchronousFileChannel` 和 NIO 的 `FileChannel` 有什么区别？（`AsynchronousFileChannel` 支持异步读写，操作完成后通过 `Future` 或 `CompletionHandler` 回调通知 / `FileChannel` 是同步的）

294. `AsynchronousFileChannel` 的 `Future` 模式和 `CompletionHandler` 模式有什么区别？（Future 模式：调用 `read()` 返回 `Future<Integer>`，通过 `future.get()` 阻塞等待结果 / CompletionHandler 模式：传入回调对象，操作完成后自动调用 `completed()` 或 `failed()`）`CompletionHandler` 的回调在哪个线程执行？（由 `AsynchronousChannelGroup` 管理的线程池中的线程）如果回调中执行了耗时操作，会怎样？（占用回调线程，影响其他异步操作的回调执行）`AsynchronousChannelGroup` 是怎么工作的？（内部维护一个线程池，所有注册到该 Group 的异步 Channel 共享线程池）

295. Java AIO 在 Linux 和 Windows 上的实现有什么差异？（Windows：使用 IOCP（I/O Completion Port），真正的异步 IO / Linux：使用 epoll 模拟，底层仍然是同步非阻塞）为什么 Linux 下的 Java AIO 性能不如 NIO？（因为 Linux 的 AIO 实现是基于 epoll 模拟的，多了一层封装的开销）为什么 Netty 在 Linux 下选择 NIO 而不是 AIO？（性能更好 / 更可控 / 避免 Linux AIO 的坑）在什么场景下应该考虑使用 Java AIO？（Windows 平台 / 文件异步 IO / 不需要极致性能但需要简化异步编程模型）
