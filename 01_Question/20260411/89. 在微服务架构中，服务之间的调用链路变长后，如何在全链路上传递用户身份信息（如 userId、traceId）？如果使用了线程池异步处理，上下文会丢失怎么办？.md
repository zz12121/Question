## 2. 核心代码示例

### 2.1 上下文 holder 定义

```java
// ========== 上下文 Holder ==========
public class TraceContext {
    
    private static final ThreadLocal<TraceInfo> CONTEXT = new ThreadLocal<>();
    
    @Data
    @AllArgsConstructor
    public static class TraceInfo {
        private String traceId;      // 全链路追踪ID
        private String userId;       // 用户ID
        private String userName;     // 用户名
        private String requestId;    // 请求唯一ID
        private Map<String, String> headers;  // 其他头信息
    }
    
    // 设置上下文
    public static void set(String traceId, String userId) {
        TraceInfo info = new TraceInfo();
        info.setTraceId(traceId);
        info.setUserId(userId);
        CONTEXT.set(info);
    }
    
    // 获取 traceId
    public static String getTraceId() {
        TraceInfo info = CONTEXT.get();
        return info != null ? info.getTraceId() : null;
    }
    
    // 获取 userId
    public static String getUserId() {
        TraceInfo info = CONTEXT.get();
        return info != null ? info.getUserId() : null;
    }
    
    // 获取完整上下文
    public static TraceInfo get() {
        return CONTEXT.get();
    }
    
    // 清理（重要！防止内存泄漏）
    public static void clear() {
        CONTEXT.remove();
    }
}
```

### 2.2 网关层：生成并传递 traceId

```java
// ========== 网关 / Filter 拦截器 ==========
@Component
@Order(Ordered.HIGHEST_PRECEDENCE)
public class TraceFilter extends OncePerRequestFilter {
    
    @Autowired
    private Tracer tracer;  // Sleuth 或 Skywalking 的 Tracer
    
    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                    HttpServletResponse response,
                                    FilterChain chain) throws Exception {
        try {
            // 优先使用上游传入的 traceId，没有则生成新的
            String traceId = request.getHeader("X-Trace-Id");
            if (traceId == null || traceId.isEmpty()) {
                traceId = tracer.nextSpan().context().traceId();
            }
            
            // 从 JWT/Token 中解析 userId
            String userId = extractUserId(request);
            
            // 设置到 ThreadLocal
            TraceContext.set(traceId, userId);
            
            // 传递给下游（通过 OpenFeign 拦截器 / RestTemplate 拦截器）
            MDC.put("traceId", traceId);  // 日志用
            
            chain.doFilter(request, response);
        } finally {
            TraceContext.clear();
            MDC.clear();
        }
    }
}
```

### 2.3 OpenFeign 拦截器：向下游传递

```java
// ========== Feign 传递上下文 ==========
@Component
public class FeignTraceInterceptor implements RequestInterceptor {
    
    @Override
    public void apply(RequestTemplate template) {
        TraceContext.TraceInfo info = TraceContext.get();
        if (info != null) {
            // 传递链路追踪ID
            template.header("X-Trace-Id", info.getTraceId());
            // 传递用户身份
            template.header("X-User-Id", info.getUserId());
            template.header("X-User-Name", info.getUserName());
            // 透传其他头信息
            if (info.getHeaders() != null) {
                info.getHeaders().forEach(template::header);
            }
        }
    }
}
```

### 2.4 线程池场景：自定义 Runnable 包装

```java
// ========== 方案一：手动包装 Runnable ==========
public class TraceRunnable implements Runnable {
    
    private final Runnable delegate;
    private final TraceContext.TraceInfo context;
    
    public TraceRunnable(Runnable delegate, TraceContext.TraceInfo context) {
        this.delegate = delegate;
        this.context = context;
    }
    
    @Override
    public void run() {
        try {
            // 复制上下文到子线程
            TraceContext.set(context.getTraceId(), context.getUserId());
            MDC.put("traceId", context.getTraceId());
            
            delegate.run();
        } finally {
            TraceContext.clear();
            MDC.clear();
        }
    }
}

// 使用方式
@Service
public class OrderService {
    
    @Autowired
    private ExecutorService asyncExecutor;
    
    public void createOrder(OrderDTO dto) {
        // 主线程上下文
        TraceContext.TraceInfo context = TraceContext.get();
        
        asyncExecutor.submit(new TraceRunnable(() -> {
            // 子线程中依然可以获取到 traceId 和 userId
            String traceId = TraceContext.getTraceId();
            String userId = TraceContext.getUserId();
            
            log.info("异步处理订单, traceId={}, userId={}", traceId, userId);
        }, context));
    }
}
```

### 2.5 线程池封装：自动传递上下文

```java
// ========== 方案二：封装线程池，自动传递 ==========
@Configuration
public class TraceThreadPoolConfig {
    
    @Bean("traceExecutor")
    public Executor traceExecutor() {
        ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
        executor.setCorePoolSize(10);
        executor.setMaxPoolSize(20);
        executor.setQueueCapacity(100);
        executor.setThreadNamePrefix("trace-");
        executor.setTaskDecorator(runnable -> {
            // 从主线程复制上下文
            TraceContext.TraceInfo context = TraceContext.get();
            return () -> {
                try {
                    if (context != null) {
                        TraceContext.set(context.getTraceId(), context.getUserId());
                        MDC.put("traceId", context.getTraceId());
                    }
                    runnable.run();
                } finally {
                    TraceContext.clear();
                    MDC.clear();
                }
            };
        });
        executor.initialize();
        return executor;
    }
}

// 使用
@Service
public class NotificationService {
    
    @Autowired
    @Qualifier("traceExecutor")
    private Executor executor;
    
    public void sendNotification(Order order) {
        executor.execute(() -> {
            // 自动继承了 traceId 和 userId
            log.info("发送通知给用户: {}", TraceContext.getUserId());
        });
    }
}
```

## 🔥 扩展追问 1：Sleuth/Brave 是怎么做的？

> **A:** Spring Cloud Sleuth 基于上述原理做了封装。它内部用 `Tracer` 和 `Span` 封装链路信息，核心也是 ThreadLocal + 请求头透传。Sleuth 2.x 用 `FeignInterceptor` 自动透传，到了 Sleuth 3.x 用 `BRAVE` 做链路追踪，支持 [b3 propagation](https://github.com/openzipkin/b3-propagation) 协议。

**常用 header 格式（b3-propagation）：**
```
X-B3-TraceId: 4bf92f3577b34da6a3ce929d0e0e4736
X-B3-SpanId: 00f067aa0ba902b7
X-B3-ParentSpanId: 4bf92f3577b34da6a3ce929d0e0e4736
X-B3-Sampled: 1
```

## 🔥 扩展追问 2：MDC 和 TraceContext 的区别？

> **A:** 两者分工不同。MDC 是日志框架 Logback/Log4j2 的上下文，专门给日志输出用的，比如 `%X{traceId}` 可以自动打印 traceId。TraceContext 是业务代码中随时获取链路信息的，比如在 Service 里取 userId 做逻辑判断。

**最佳实践：两者同步更新：**
```java
public static void set(String traceId, String userId) {
    TraceInfo info = new TraceInfo(traceId, userId);
    CONTEXT.set(info);
    MDC.put("traceId", traceId);  // 同步到 MDC
    MDC.put("userId", userId);
}
```

## 🔥 扩展追问 3：Hystrix/Seata 线程隔离模式下怎么传？

> **A:** Hystrix 的 THREAD 模式下，子线程和主线程完全隔离，ThreadLocal 肯定丢失。解决方案：

```java
// Hystrix 线程隔离模式
@HystrixCommand(groupKey = "order", threadPoolKey = "order-pool")
public Order createOrder() {
    // 在 HystrixCommand 之前把上下文拿到
    TraceContext.TraceInfo context = TraceContext.get();
    
    return new OrderCommand(context).execute();
}

class OrderCommand extends HystrixCommand<Order> {
    private final TraceContext.TraceInfo context;
    
    public OrderCommand(TraceContext.TraceInfo context) {
        super(HystrixCommandGroupKey.Factory.asKey("order"));
        this.context = context;
    }
    
    @Override
    protected Order run() {
        // 恢复上下文
        TraceContext.set(context.getTraceId(), context.getUserId());
        // 执行业务
        return orderService.process();
    }
}

// Seata 的 AT 模式会自动处理 Connection，但 ThreadLocal 上下文仍需手动传递
```

## 🔥 扩展追问 4：Dubbo 怎么透传 traceId？

> **A:** Dubbo 有内置的 Attachment 机制，可以在 RPC 调用时透传隐式参数：

```java
// 服务消费者
RpcContext.getContext()
    .setAttachment("traceId", TraceContext.getTraceId())
    .setAttachment("userId", TraceContext.getUserId());

// 服务提供者
String traceId = RpcContext.getContext().getAttachment("traceId");

// 或者用 Filter 自动透传
@Activate(group = Constants.CONSUMER_GROUP, order = -10000)
public class TraceIdFilter implements Filter {
    @Override
    public Result invoke(Invoker<?> invoker, Invocation invocation) {
        RpcContext.getContext()
            .setAttachment("traceId", TraceContext.getTraceId())
            .setAttachment("userId", TraceContext.getUserId());
        return invoker.invoke(invocation);
    }
}
```

## 🔥 扩展追问 5：上下文泄漏（内存泄漏）问题怎么避免？

> **A:** ThreadLocal 最大的坑就是忘记 remove。必须在线程使用完毕后清理，有几种方式：

```java
// 方式一：Filter 中 finally 清理（最外层）
finally {
    TraceContext.clear();
}

// 方式二：使用装饰器模式自动清理（推荐）
// 已在 TraceRunnable 和 TaskDecorator 中用 finally 清理

// 方式三：使用 Apache Commons-pool 等带清理的线程池
// 或使用 TransmittableThreadLocal（阿里开源）
// TTL 可以跨线程传递，并且在线程池复用时自动清理

// 使用 TTL
<dependency>
    <groupId>com.alibaba</groupId>
    <artifactId>transmittable-thread-local</artifactId>
    <version>2.14.2</version>
</dependency>

// TTL 简化写法
TransmittableThreadLocal<String> traceId = new TransmittableThreadLocal<>();
// 它会自动在线程池中传递和清理
```

## 一句话总结

> **用 ThreadLocal 存储链路上下文，通过网关 Filter 生成 traceId、OpenFeign 拦截器透传请求头、线程池包装类/TtlLibrary 自动复制上下文到子线程，配合 finally 块清理，避免内存泄漏。**
