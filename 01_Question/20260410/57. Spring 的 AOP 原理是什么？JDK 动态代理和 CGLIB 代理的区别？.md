## 先说结论

Spring AOP 基于**代理模式**实现，运行时通过动态代理在目标方法前后织入增强逻辑。默认策略：**目标类实现了接口用 JDK 动态代理，否则用 CGLIB 生成子类代理**。

## 🔥 核心概念

### 解决什么问题

- **横切关注点分离**：日志、事务、权限、监控等代码散落在各处
- **代码复用**：统一的地方定义，自动应用到目标方法
- **开闭原则**：不修改原有代码，通过配置增强功能

### AOP 核心术语

| 术语 | 含义 |
|------|------|
| **Aspect** | 切面，包含通知和切点的模块 |
| **Join Point** | 连接点，程序执行过程中的点（方法调用、异常抛出等） |
| **Pointcut** | 切点，定义哪些连接点会被拦截 |
| **Advice** | 通知，拦截后执行的增强逻辑 |
| **Weaving** | 织入，将通知应用到目标对象的过程 |

### 2. 通知类型（Advice）

```java
@Aspect
@Component
public class LogAspect {
    
    // 切点表达式： execution(修饰符 返回类型 包名.类名.方法名(参数))
    @Pointcut("execution(* com.example.service.*.*(..))")
    public void servicePointcut() {}
    
    @Before("servicePointcut()")           // 方法执行前
    public void before(JoinPoint jp) {
        System.out.println("前置通知: " + jp.getSignature());
    }
    
    @AfterReturning(pointcut = "servicePointcut()", returning = "result")
    public void afterReturn(Object result) {  // 方法正常返回后
        System.out.println("返回通知: " + result);
    }
    
    @AfterThrowing(pointcut = "servicePointcut()", throwing = "ex")
    public void afterThrow(Exception ex) {    // 方法抛出异常后
        System.out.println("异常通知: " + ex.getMessage());
    }
    
    @After("servicePointcut()")            // 方法执行后（无论是否异常）
    public void after() {
        System.out.println("后置通知");
    }
    
    @Around("servicePointcut()")           // 环绕通知（最强）
    public Object around(ProceedingJoinPoint pjp) throws Throwable {
        System.out.println("环绕前");
        Object result = pjp.proceed();     // 执行目标方法
        System.out.println("环绕后");
        return result;
    }
}
```

## 🔥 扩展追问 1：JDK 动态代理 vs CGLIB 代理

### 1. 核心区别对比

| 特性 | JDK 动态代理 | CGLIB 代理 |
|------|-------------|-----------|
| **实现原理** | 基于接口，反射调用 | 基于继承，生成子类 |
| **依赖** | 必须实现接口 | 不需要接口 |
| **生成方式** | `java.lang.reflect.Proxy` | `net.sf.cglib.proxy.Enhancer` |
| **性能** | 反射调用稍慢 | 使用 FastClass 优化，较快 |
| **final 方法** | 支持 | 不支持（无法重写） |
| **final 类** | 支持 | 不支持（无法继承） |
| **Spring 默认** | 有接口时优先使用 | 无接口时使用 |

### 2. JDK 动态代理原理

```java
// 目标接口
public interface UserService {
    void login(String username);
}

// 目标实现
public class UserServiceImpl implements UserService {
    @Override
    public void login(String username) {
        System.out.println(username + " 登录成功");
    }
}

// JDK 代理实现
public class JdkProxyDemo {
    public static void main(String[] args) {
        UserService target = new UserServiceImpl();
        
        UserService proxy = (UserService) Proxy.newProxyInstance(
            target.getClass().getClassLoader(),
            target.getClass().getInterfaces(),  // 必须实现接口

                System.out.println("前置日志");

        proxy.login("admin");  // 调用代理对象

}
```

### 3. CGLIB 代理原理

```java
// 目标类（无需接口）
public class OrderService {
    public void createOrder() {
        System.out.println("创建订单");
    }
    
    public final void cancelOrder() {  // final 方法无法被代理
        System.out.println("取消订单");
    }
}

// CGLIB 代理实现
public class CglibProxyDemo {
    public static void main(String[] args) {
        Enhancer enhancer = new Enhancer();
        enhancer.setSuperclass(OrderService.class);  // 设置父类
        enhancer.setCallback((MethodInterceptor) (obj, method, args, proxy) -> {
            System.out.println("前置日志");
            // 调用父类方法（非反射，使用 FastClass 优化）
            Object result = proxy.invokeSuper(obj, args);
            System.out.println("后置日志");
            return result;
        });
        
        OrderService proxy = (OrderService) enhancer.create();
        proxy.createOrder();
    }
}
```

```java
// 伪代码：CGLIB 生成的代理类

        // 调用 MethodInterceptor.intercept()
        callback.intercept(this, createOrderMethod, null, proxy);
    }
    
    // FastClass 机制：为每个方法生成索引，避免反射
    public Object invoke(int methodIndex, Object[] args) {
        switch(methodIndex) {
            case 1: return super.createOrder();
            // ...
        }
```

### 3. 方法调用拦截链

```java
// ReflectiveMethodInvocation：责任链模式
public class ReflectiveMethodInvocation implements ProxyMethodInvocation {
    private int currentInterceptorIndex = -1;
    private List<Object> interceptorsAndDynamicMethodMatchers;
    
    public Object proceed() throws Throwable {
        // 所有拦截器执行完毕，调用目标方法
        if (currentInterceptorIndex == interceptorsAndDynamicMethodMatchers.size() - 1) {
            return invokeJoinpoint();  // 执行目标方法
        }
        
        // 获取下一个拦截器
        Object interceptor = interceptorsAndDynamicMethodMatchers.get(++currentInterceptorIndex);
        
        // 调用拦截器（可能是 Advice 或 Interceptor）
        return ((MethodInterceptor) interceptor).invoke(this);
    }
}
```

```
├─────────────────────────────────────────┤

├─────────────────────────────────────────┤
│  AspectJAfterThrowingAdvice             │ ← @AfterThrowing
├─────────────────────────────────────────┤
│  AspectJAroundAdvice                    │ ← @Around（proceed() 前后）
├─────────────────────────────────────────┤

       ↓
返回结果
```

| **织入时机** | 运行时 | 编译时、类加载时 |
| **性能** | 有代理开销 | 无运行时开销 |
| **功能** | 仅方法级别 | 方法、字段、构造器、静态初始化 |
| **依赖** | 纯 Spring | 需 AspectJ 编译器或 LTW |
| **使用场景** | 大多数业务场景 | 更细粒度的切面需求 |

---

## 一句话收尾

Spring AOP 通过**JDK 动态代理**（基于接口）或 **CGLIB**（基于继承）在运行时生成代理对象，将通知织入目标方法；理解代理选择策略、拦截链执行顺序以及同类调用不走代理的坑，是正确使用 AOP 的关键。
