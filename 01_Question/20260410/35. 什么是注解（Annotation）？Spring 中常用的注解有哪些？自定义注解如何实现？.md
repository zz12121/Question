## 先说结论

**注解**是 Java 的元数据机制，提供代码的额外信息供编译器或运行时工具处理，本身不直接影响程序逻辑。Spring 常用注解包括 **@Component/@Service/@Repository/@Controller**（ Bean 定义）、**@Autowired/@Resource**（依赖注入）、**@Transactional**（事务）、**@RequestMapping/@GetMapping**（ MVC 路由）等。自定义注解通过 `@interface` 定义元注解（@Target、@Retention 等），配合 **反射** 或 **APT（注解处理器）** 实现处理逻辑。

---

## 🔥 核心概念

**解决什么问题：**
- 减少样板代码（XML 配置 → 注解配置）
- 声明式编程（声明意图，框架处理细节）
- 编译期代码生成（Lombok、Dagger 等）

**处理时机：**
- **编译期：** 注解处理器（APT）生成代码
- **类加载期：** 字节码转换（AspectJ LTW）
- **运行期：** 反射读取处理（Spring 主要方式）

## 🔥 详细展开

### 1. 元注解（定义注解的注解）

```java
// 自定义注解示例
@Target({ElementType.TYPE, ElementType.METHOD})  // 作用目标
@Retention(RetentionPolicy.RUNTIME)              // 保留策略
@Documented                                       // 包含在 Javadoc
@Inherited                                        // 子类继承
@Repeatable(Schedules.class)                      // 可重复（Java 8+）
public @interface Scheduled {
    String cron() default "";      // 成员，有默认值
    long fixedRate() default -1;
    String zone() default "";
}
```

| 元注解 | 作用 |
|--------|------|
| **@Target** | 限定注解使用位置（类、方法、字段等） |
| **@Retention** | 保留策略（SOURCE/CLASS/RUNTIME） |
| **@Documented** | 是否包含在 Javadoc |
| **@Inherited** | 子类是否继承该注解 |
| **@Repeatable** | 同一位置可重复使用 |

**@Target 取值：**

```java
ElementType.TYPE          // 类、接口、枚举
ElementType.FIELD         // 字段
ElementType.METHOD        // 方法
ElementType.PARAMETER     // 参数
ElementType.CONSTRUCTOR   // 构造器
ElementType.LOCAL_VARIABLE // 局部变量
ElementType.ANNOTATION_TYPE // 注解
ElementType.PACKAGE       // 包
ElementType.TYPE_PARAMETER // 泛型参数（Java 8）
ElementType.TYPE_USE      // 类型使用（Java 8）
```

**@Retention 策略：**

| 策略 | 说明 | 使用场景 |
|------|------|----------|
| **SOURCE** | 源码级，编译后丢弃 | @Override、@SuppressWarnings |
| **CLASS** | 编译到 class，JVM 不保留 | 默认，字节码处理 |
| **RUNTIME** | 运行时保留，可反射读取 | Spring、JPA 等框架注解 |

#### 3.2 AOP 处理注解（Spring 方式）

```java
// 3. AOP 拦截处理
@Aspect
@Component
public class LogOperationAspect {
    
    @Autowired
    private OperationLogRepository logRepository;
    
    // 定义切点
    @Pointcut("@annotation(com.example.annotation.LogOperation)")
    public void logPointcut() {}
    
    @Around("logPointcut()")
    public Object around(ProceedingJoinPoint point) throws Throwable {
        MethodSignature signature = (MethodSignature) point.getSignature();
        Method method = signature.getMethod();
        LogOperation annotation = method.getAnnotation(LogOperation.class);
        
        // 记录开始时间
        long startTime = System.currentTimeMillis();
        
        // 获取请求参数
        Object[] args = point.getArgs();
        String params = annotation.saveParams() ? 
            truncate(JsonUtils.toJson(args), annotation.maxParamLength()) : "";
        
        Object result = null;
        boolean success = true;
        String errorMsg = null;
        
        try {
            result = point.proceed();
            return result;
        } catch (Exception e) {
            success = false;
            errorMsg = e.getMessage();
            throw e;
        } finally {
            // 保存操作日志
            OperationLog log = new OperationLog();
            log.setOperation(annotation.value());
            log.setMethod(method.getDeclaringClass().getName() + "." + method.getName());
            log.setParams(params);
            log.setResult(annotation.saveResult() ? JsonUtils.toJson(result) : "");
            log.setSuccess(success);
            log.setErrorMsg(errorMsg);
            log.setDuration(System.currentTimeMillis() - startTime);
            log.setCreateTime(LocalDateTime.now());
            
            logRepository.save(log);
        }
    }
    
    private String truncate(String str, int maxLength) {
        return str.length() > maxLength ? str.substring(0, maxLength) + "..." : str;
    }
}
```

@Retention(RetentionPolicy.RUNTIME)
@Constraint(validatedBy = PhoneValidator.class)  // 指定校验器
public @interface Phone {

    Class<?>[] groups() default {};
    Class<? extends Payload>[] payload() default {};
}

// 2. 实现校验器
public class PhoneValidator implements ConstraintValidator<Phone, String> {
    
    private static final Pattern PHONE_PATTERN = 
        Pattern.compile("^1[3-9]\\d{9}$");
    
    @Override
    public boolean isValid(String value, ConstraintValidatorContext context) {
        if (value == null || value.isEmpty()) {
            return true;  // 空值由 @NotNull 处理
        }
        return PHONE_PATTERN.matcher(value).matches();
    }
}

// 3. 使用
public class UserRegisterRequest {
    
    @NotNull(message = "手机号不能为空")
    @Phone(message = "请输入正确的手机号")
    private String phone;

#### 3.4 编译期注解处理器（APT）

```java
// 1. 定义注解
@Target(ElementType.TYPE)
@Retention(RetentionPolicy.SOURCE)  // 只在编译期使用
public @interface Builder {
}

// 2. 实现注解处理器
@SupportedAnnotationTypes("com.example.annotation.Builder")
@SupportedSourceVersion(SourceVersion.RELEASE_11)
public class BuilderProcessor extends AbstractProcessor {
    
    @Override
    public boolean process(Set<? extends TypeElement> annotations, RoundEnvironment roundEnv) {
        for (Element element : roundEnv.getElementsAnnotatedWith(Builder.class)) {
            if (element.getKind() != ElementKind.CLASS) {
                processingEnv.getMessager().printMessage(
                    Diagnostic.Kind.ERROR, "@Builder can only be applied to classes");
                continue;
            }
            
            TypeElement classElement = (TypeElement) element;
            String className = classElement.getQualifiedName().toString();
            String packageName = processingEnv.getElementUtils()
                .getPackageOf(classElement).toString();
            
            // 生成 Builder 类代码
            String builderClassName = classElement.getSimpleName() + "Builder";
            String builderCode = generateBuilderCode(classElement, packageName, builderClassName);
            
            // 写入文件
            try {
                JavaFileObject file = processingEnv.getFiler()
                    .createSourceFile(packageName + "." + builderClassName);
                try (Writer writer = file.openWriter()) {
                    writer.write(builderCode);
                }
            } catch (IOException e) {
                processingEnv.getMessager().printMessage(
                    Diagnostic.Kind.ERROR, "Failed to generate builder: " + e.getMessage());
            }
        }
        return true;
    }
    
    private String generateBuilderCode(TypeElement classElement, String packageName, String builderName) {
        // 生成 Builder 模式代码...
        return "package " + packageName + ";\n\npublic class " + builderName + " { ... }";
    }
}

// 3. 注册处理器（META-INF/services/javax.annotation.processing.Processor）
// 文件内容：com.example.processor.BuilderProcessor

// 4. 使用
@Builder
public class User {
    private String name;
    private int age;
    // 编译后自动生成 UserBuilder 类
}
```

## 🔥 扩展追问 1：注解和 XML 配置怎么选？

| 维度 | 注解配置 | XML 配置 |
|------|----------|----------|
| 可读性 | 与代码在一起，直观 | 集中管理，清晰 |
| 灵活性 | 编译期固定 | 运行期可修改 |
| 侵入性 | 有（代码依赖 Spring） | 无 |
| 第三方库 | 难配置 | 易配置 |
| 版本控制 | 分散，冲突多 | 集中，易合并 |

**最佳实践：**
- 应用内部：注解配置（简洁）
- 基础设施：Java Config（类型安全）
- 第三方集成：XML（灵活）

## 🔥 扩展追问 3：Spring Boot 自动配置的注解原理

```java
// @SpringBootApplication = @Configuration + @EnableAutoConfiguration + @ComponentScan
@SpringBootApplication
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}

// @EnableAutoConfiguration
@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
@Documented
@Inherited
@AutoConfigurationPackage
@Import(AutoConfigurationImportSelector.class)  // 关键！
public @interface EnableAutoConfiguration {
}

// AutoConfigurationImportSelector
public class AutoConfigurationImportSelector implements DeferredImportSelector {
    
    @Override
    public String[] selectImports(AnnotationMetadata metadata) {
        // 1. 读取 META-INF/spring.factories
        List<String> configurations = SpringFactoriesLoader.loadFactoryNames(
            EnableAutoConfiguration.class, getBeanClassLoader());
        
        // 2. 过滤（@ConditionalOnClass 等）
        configurations = filter(configurations, metadata);
        
        return configurations.toArray(new String[0]);
    }
}
```

## 一句话收尾

注解是 Java 的元数据机制，Spring 通过运行时反射读取 @Component/@Autowired 等注解实现 IoC 和 AOP；自定义注解用 @interface + 元注解定义，配合 AOP 或 APT 处理；编译期注解（如 Lombok）通过 APT 生成代码，运行时注解（如 Spring）通过反射处理，面试要区分清楚！
