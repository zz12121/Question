## 先说结论

Spring Boot Starter 是**约定优于配置**思想的产物，通过引入一个依赖即可自动完成组件的自动配置和依赖管理。核心机制是 **META-INF/spring.factories**（Spring Boot 2.7+ 为 **META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports**）中注册的自动配置类。

---

## 🔥 核心概念

### 解决什么问题

| 传统方式 | Starter 方式 |
|---------|-------------|
| 引入依赖 + 写大量配置类 | 引入 starter 即可 |
| 版本冲突手动解决 | 依赖版本统一管理 |
| 每个项目重复造轮子 | 开箱即用，约定配置 |

### Starter 命名规范

| 类型 | 命名格式 | 示例 |
|------|---------|------|
| 官方 Starter | `spring-boot-starter-*` | `spring-boot-starter-web` |
| 第三方 Starter | `*-spring-boot-starter` | `mybatis-spring-boot-starter` |

## 🔥 详细展开

### 1. Starter 自动配置原理

```
┌─────────────────────────────────────────────────────────────────┐
│                    Spring Boot 自动配置流程                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 引入 Starter 依赖                                            │
│     └── pom.xml: mybatis-spring-boot-starter                    │
│                                                                 │
│  2. 扫描 spring.factories（或 AutoConfiguration.imports）         │
│     └── META-INF/spring/org.springframework.boot.autoconfigure    │
│         .AutoConfiguration.imports                               │
│     └── 内容: com.xxx.MybatisAutoConfiguration                  │
│                                                                 │
│  3. 条件注解过滤（@Conditional 家族）                              │
│     └── @ConditionalOnClass(SqlSessionFactory.class)            │
│     └── @ConditionalOnMissingBean(DataSource.class)             │
│     └── @ConditionalOnProperty(prefix="mybatis", ...)           │
│                                                                 │
│  4. 执行自动配置类                                                │
│     └── 创建 SqlSessionFactory、SqlSessionTemplate 等 Bean      │
│                                                                 │
│  5. 应用配置文件（application.yml）绑定                            │
│     └── @ConfigurationProperties(prefix="mybatis")              │
│     └── MybatisProperties 类接收配置                             │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. 核心注解详解

```java
// @Conditional 家族：条件装配
@ConditionalOnClass          // 类存在时生效
@ConditionalOnMissingClass   // 类不存在时生效
@ConditionalOnBean           // Bean 存在时生效
@ConditionalOnMissingBean    // Bean 不存在时生效（常用！）
@ConditionalOnProperty       // 配置属性满足条件时生效
@ConditionalOnWebApplication // Web 应用时生效
@ConditionalOnExpression     // SpEL 表达式满足时生效

// 示例
@Configuration
@ConditionalOnClass(DataSource.class)  // 类路径有 DataSource
@ConditionalOnMissingBean(DataSource.class)  // 容器中没有 DataSource Bean
@EnableConfigurationProperties(MybatisProperties.class)
public class MybatisAutoConfiguration {
    
    @Bean
    @ConditionalOnMissingBean  // 用户没定义时才创建
    public SqlSessionFactory sqlSessionFactory(DataSource dataSource) {
        // 创建 SqlSessionFactory
    }
}
```

### 3. 配置属性绑定

```java
// 1. 定义配置属性类
@ConfigurationProperties(prefix = "mybatis")
public class MybatisProperties {
    
    private String configLocation;
    private String[] mapperLocations;
    private String typeAliasesPackage;
    
    // getter/setter
}

// 2. application.yml 配置
mybatis:
  config-location: classpath:mybatis-config.xml
  mapper-locations: classpath:mapper/**/*.xml
  type-aliases-package: com.example.entity

// 3. 启用配置属性（两种方式）
// 方式1：@EnableConfigurationProperties
@Configuration
@EnableConfigurationProperties(MybatisProperties.class)
public class MyConfig {}

// 方式2：在类上加 @ConfigurationProperties + @Component
// 或在配置类加 @ConfigurationPropertiesScan
```

## 🔥 扩展追问 1：如何自定义一个 Starter

### 完整示例：自定义短信发送 Starter

**项目结构：**
```
my-sms-spring-boot-starter/
├── pom.xml
└── src/main/java/com/example/sms/
    ├── auto/
    │   └── SmsAutoConfiguration.java      # 自动配置类
    ├── core/
    │   ├── SmsService.java                # 核心服务接口
    │   ├── AliyunSmsService.java          # 阿里云实现
    │   └── TencentSmsService.java         # 腾讯云实现
    ├── properties/
    │   └── SmsProperties.java             # 配置属性
    └── META-INF/
        └── spring/
            └── org.springframework.boot.autoconfigure.AutoConfiguration.imports
```

**1. pom.xml**
```xml
<?xml version="1.0" encoding="UTF-8"?>
<project>
    <parent>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-parent</artifactId>
        <version>2.7.x</version>
    </parent>
    
    <artifactId>my-sms-spring-boot-starter</artifactId>
    
    <dependencies>
        <!-- 必须：自动配置 -->
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-autoconfigure</artifactId>
        </dependency>
        
        <!-- 可选：配置处理器，生成元数据 -->
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-configuration-processor</artifactId>
            <optional>true</optional>
        </dependency>
        
        <!-- 短信 SDK -->
        <dependency>
            <groupId>com.aliyun</groupId>
            <artifactId>aliyun-java-sdk-dysmsapi</artifactId>
            <version>2.2.1</version>
            <optional>true</optional>  <!-- 可选依赖 -->
        </dependency>
    </dependencies>
</project>
```

**2. 配置属性类**
```java
@ConfigurationProperties(prefix = "sms")
public class SmsProperties {
    
    /**
     * 短信服务商：aliyun/tencent
     */
    private String provider = "aliyun";
    
    /**
     * 阿里云配置
     */
    private Aliyun aliyun = new Aliyun();
    
    /**
     * 腾讯云配置
     */
    private Tencent tencent = new Tencent();
    
    // 内部类
    public static class Aliyun {
        private String accessKeyId;
        private String accessKeySecret;
        private String signName;
        // getter/setter
    }
    
    public static class Tencent {
        private String secretId;
        private String secretKey;
        private String appId;
        // getter/setter
    }
    
    // getter/setter
}
```

**3. 核心服务接口与实现**
```java
// 接口
public interface SmsService {
    /**
     * 发送短信
     * @param phone 手机号
     * @param templateCode 模板编码
     * @param params 模板参数
     */
    SendResult send(String phone, String templateCode, Map<String, String> params);
}

// 阿里云实现
public class AliyunSmsService implements SmsService {
    
    private final SmsProperties properties;
    
    public AliyunSmsService(SmsProperties properties) {
        this.properties = properties;
    }
    
    @Override
    public SendResult send(String phone, String templateCode, Map<String, String> params) {
        // 调用阿里云 SDK 发送短信
        System.out.println("阿里云发送短信到: " + phone);
        return SendResult.success();
    }
}

// 腾讯云实现
public class TencentSmsService implements SmsService {
    // 类似实现...
}
```

**4. 自动配置类**
```java
@Configuration
@ConditionalOnClass(SmsService.class)  // 类路径有 SmsService
@EnableConfigurationProperties(SmsProperties.class)  // 启用配置属性
public class SmsAutoConfiguration {
    
    @Bean
    @ConditionalOnMissingBean  // 用户没定义时才创建
    @ConditionalOnProperty(prefix = "sms", name = "provider", havingValue = "aliyun", matchIfMissing = true)
    public SmsService aliyunSmsService(SmsProperties properties) {
        return new AliyunSmsService(properties);
    }
    
    @Bean
    @ConditionalOnMissingBean
    @ConditionalOnProperty(prefix = "sms", name = "provider", havingValue = "tencent")
    public SmsService tencentSmsService(SmsProperties properties) {
        return new TencentSmsService(properties);
    }
}
```

**5. 注册自动配置（Spring Boot 2.7+）**
```properties
# META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports
com.example.sms.auto.SmsAutoConfiguration
```

**Spring Boot 2.6 及以下：**
```properties
# META-INF/spring.factories
org.springframework.boot.autoconfigure.EnableAutoConfiguration=\
com.example.sms.auto.SmsAutoConfiguration
```

    }
}

## 🔥 扩展追问 4：常见面试考点

| 问题 | 答案 |
|------|------|
| Starter 和 AutoConfiguration 的关系？ | Starter 是依赖包，包含 AutoConfiguration 类和默认配置 |
| 为什么用 `spring.factories`？ | SPI 机制，Spring Boot 启动时扫描并加载 |
| `@ConditionalOnMissingBean` 作用？ | 用户自定义 Bean 优先，实现可覆盖 |
| 配置属性提示怎么实现？ | `spring-boot-configuration-processor` 编译期生成元数据 |
| 如何控制自动配置顺序？ | `@AutoConfigureBefore` / `@AutoConfigureAfter` |

---

## 一句话收尾

Spring Boot Starter 通过 **spring.factories SPI 机制** 加载自动配置类，配合 **@Conditional 条件注解** 实现智能装配，让组件真正做到**"引入依赖，开箱即用"**；自定义 Starter 需遵循**配置属性 → 核心服务 → 自动配置 → 注册文件**四步规范。
