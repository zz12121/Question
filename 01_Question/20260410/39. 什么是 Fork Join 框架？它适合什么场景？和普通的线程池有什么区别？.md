## 先说结论

**Fork/Join 框架**是 Java 7 引入的并行计算框架，基于**分治思想**（Divide and Conquer）将大任务递归拆分为小任务，在**工作窃取（Work Stealing）**线程池中执行。适合**可分解的递归任务**（如归并排序、MapReduce、斐波那契计算）。与普通线程池相比，ForkJoinPool 采用**双端队列 + 工作窃取**机制，线程空闲时会从其他线程队列"偷"任务，提高 CPU 利用率。

---

## 🔥 核心概念

**解决什么问题：**
- 多核 CPU 利用率不足（传统线程池线程可能空闲等待）
- 递归分治任务的并行执行

**核心思想：**
- Fork：拆分任务为子任务
- Join：等待子任务结果并合并
- Work Stealing：负载均衡，减少线程空闲

### 2. 工作窃取（Work Stealing）机制

```java
// ForkJoinPool 的工作队列设计
public class ForkJoinPool {
    
    // 每个工作线程有自己的双端队列（Deque）
    volatile WorkQueue[] workQueues;
    
    class WorkQueue {
        // 本地任务：LIFO（栈）- 自己执行
        // 窃取任务：FIFO（队列）- 其他线程偷
        ForkJoinTask<?>[] array;
        int base;   // 窃取端（FIFO）
        int top;    // 本地端（LIFO）
    }
}
```

**工作窃取流程：**

```
线程 A 的队列：[大任务] → [子任务1] → [子任务2] → [子任务3]
                      ↑top              ↑base

线程 A 执行：
  1. 从 top 取子任务3（LIFO，栈顶）执行
  2. 从 top 取子任务2 执行
  
线程 B 空闲：
  1. 从线程 A 的 base 端窃取子任务1（FIFO，队列头）
  2. 执行子任务1
  
优势：
  - 线程 A 操作自己的栈（无竞争）
  - 线程 B 从队列头窃取（减少冲突）
  - 大任务在队列尾，优先被本地执行
```

**为什么用双端队列？**

| 操作 | 位置 | 优势 |
|------|------|------|
| 本地任务 | top（栈顶，LIFO） | 缓存友好，刚创建的任务优先执行 |
| 窃取任务 | base（队列头，FIFO） | 大任务在队列尾，减少窃取大任务的概率 |

### 3. 与普通线程池的区别

| 特性 | ThreadPoolExecutor | ForkJoinPool |
|------|-------------------|--------------|
| **任务类型** | Runnable/Callable | RecursiveTask/RecursiveAction |
| **任务拆分** | 不支持 | 支持递归拆分 |
| **负载均衡** | 任务分配给固定线程 | 工作窃取，动态负载均衡 |
| **线程空闲** | 等待新任务 | 窃取其他线程任务 |
| **适用场景** | 独立、同质任务 | 可分治的递归任务 |
| **线程数** | 固定或可配置 | 默认 CPU 核心数 |

### 2. 工作窃取（Work Stealing）算法

```java
// ForkJoinWorkerThread 的核心逻辑
public class ForkJoinWorkerThread extends Thread {
    
    // 每个线程有自己的双端队列（Deque）
    ForkJoinPool.WorkQueue queue;
    
    void runTask(ForkJoinTask<?> task) {
        // 1. 自己执行的任务从队列头部取（LIFO）
        // 2. 执行完自己的，从其他线程队列尾部偷（FIFO）
    }
}
```

**工作窃取图解：**

```
线程 1 的队列（双端）：
头部 → [大任务] [子任务1] [子任务2] [子任务3] ← 尾部
       ↑ 自己从头部取（LIFO，最新任务）
       
线程 2 的队列：
头部 → [大任务] [子任务A] ← 尾部
       （任务少，很快执行完）

线程 2 空闲后：
    ↓
从线程 1 队列尾部窃取 [子任务3]（FIFO，最老任务）
    ↓
线程 1 从头部取 [子任务2]
    ↓
两个线程并行执行，无竞争
```

**为什么从尾部偷？**

- 线程自己从头部取（LIFO）：最新提交的任务，缓存友好（刚创建的子任务数据在缓存中）
- 其他线程从尾部偷（FIFO）：最老的任务，减少与所有者线程的竞争

### 3. 与普通线程池的对比

| 特性 | ThreadPoolExecutor | ForkJoinPool |
|------|-------------------|--------------|
| **核心类** | `ThreadPoolExecutor` | `ForkJoinPool` |
| **任务类型** | `Runnable`/`Callable` | `ForkJoinTask`（`RecursiveTask`/`RecursiveAction`） |
| **任务队列** | 阻塞队列（共享） | 双端队列（每个线程独立） |
| **负载均衡** | 任务分配后固定 | 工作窃取，动态平衡 |
| **适用场景** | 独立、同质任务 | 可分解的递归任务 |
| **线程数** | 可配置 | 默认 CPU 核心数 |
| **阻塞处理** | 线程阻塞等待 | 工作窃取，减少阻塞 |
