## 先说结论

Spring IOC 容器初始化遵循**"定位 → 加载 → 注册 → 实例化 → 依赖注入 → 初始化"**流程；**BeanDefinition**是 Bean 的**元数据描述对象**，存储类名、作用域、依赖关系、初始化方法等配置信息，是 Spring 管理 Bean 的**核心数据结构**。

## 🔥 详细展开

### 1. BeanDefinition 详解

```java
// BeanDefinition 是 Bean 的元数据描述接口
public interface BeanDefinition extends AttributeAccessor, BeanMetadataElement {
    
    // 作用域
    String SCOPE_SINGLETON = ConfigurableBeanFactory.SCOPE_SINGLETON;  // 单例
    String SCOPE_PROTOTYPE = ConfigurableBeanFactory.SCOPE_PROTOTYPE;  // 原型
    
    // 获取/设置 Bean 类名
    void setBeanClassName(String beanClassName);
    String getBeanClassName();
    
    // 作用域
    void setScope(String scope);
    String getScope();
    
    // 是否懒加载
    void setLazyInit(boolean lazyInit);
    boolean isLazyInit();
    
    // 依赖的 Bean
    String[] getDependsOn();
    void setDependsOn(String... dependsOn);
    
    // 是否是主候选
    void setPrimary(boolean primary);
    boolean isPrimary();
    
    // 初始化/销毁方法
    void setInitMethodName(String initMethodName);
    String getInitMethodName();
    void setDestroyMethodName(String destroyMethodName);
    String getDestroyMethodName();
    
    // 构造参数
    ConstructorArgumentValues getConstructorArgumentValues();
    
    // 属性值
    MutablePropertyValues getPropertyValues();
}
```

**BeanDefinition 的核心属性：**

| 属性 | 说明 | 示例 |
|------|------|------|
| `beanClass` | Bean 的 Class 对象或类名 | `com.example.UserService` |
| `scope` | 作用域：singleton/prototype/request/session | `singleton` |
| `lazyInit` | 是否延迟初始化 | `false` |
| `dependsOn` | 依赖的其他 Bean | `["dataSource"]` |
| `autowireMode` | 自动装配模式：byName/byType/constructor | `AUTOWIRE_BY_TYPE` |
| `initMethodName` | 初始化方法名 | `"init"` |
| `destroyMethodName` | 销毁方法名 | `"destroy"` |
| `propertyValues` | 属性值集合 | `{name: "Alice", age: 25}` |
| `constructorArgs` | 构造参数 | `[dataSource]` |

### 2. 容器初始化的核心源码

```java
// AbstractApplicationContext.refresh() - 容器刷新入口
public void refresh() throws BeansException, IllegalStateException {
    synchronized (this.startupShutdownMonitor) {
        // 1. 准备刷新（设置启动时间、状态等）
        prepareRefresh();
        
        // 2. 获取/刷新 BeanFactory（加载 BeanDefinition）
        ConfigurableListableBeanFactory beanFactory = obtainFreshBeanFactory();
        
        // 3. 准备 BeanFactory（设置类加载器、添加后置处理器等）
        prepareBeanFactory(beanFactory);
        
        try {
            // 4. 子类扩展：允许在 BeanFactory 创建后做自定义处理
            postProcessBeanFactory(beanFactory);
            
            // 5. 调用 BeanFactoryPostProcessor（如配置占位符替换）
            invokeBeanFactoryPostProcessors(beanFactory);
            
            // 6. 注册 BeanPostProcessor（用于 Bean 初始化前后处理）
            registerBeanPostProcessors(beanFactory);
            
            // 7. 初始化消息源（国际化）
            initMessageSource();
            
            // 8. 初始化事件广播器
            initApplicationEventMulticaster();
            
            // 9. 子类扩展：初始化其他特殊 Bean
            onRefresh();
            
            // 10. 注册监听器
            registerListeners();
            
            // 11. 实例化所有非懒加载的单例 Bean（核心！）
            finishBeanFactoryInitialization(beanFactory);
            
            // 12. 完成刷新，发布事件
            finishRefresh();
        }
        // ...
    }
}
```

### 4. BeanDefinition 的来源

```java
// 不同配置方式对应的 BeanDefinitionReader

    new ClassPathXmlApplicationContext("applicationContext.xml");

    new AnnotationConfigApplicationContext(AppConfig.class);
```

## 🔥 扩展追问 1：BeanFactory vs ApplicationContext

| 特性 | BeanFactory | ApplicationContext |
|------|-------------|-------------------|
| **功能** | 基础 IOC 容器 | 高级 IOC 容器 |
| **Bean 加载** | 延迟加载（getBean 时才创建） | 默认预加载单例 Bean |
| **AOP 支持** | 不支持 | 支持 |
| **事件机制** | 不支持 | 支持 ApplicationEvent |
| **国际化** | 不支持 | 支持 MessageSource |
| **资源加载** | 基础 | 支持通配符、多资源 |
| **典型实现** | DefaultListableBeanFactory | ClassPathXmlApplicationContext |

```java
// BeanFactory 基础用法
BeanFactory factory = new XmlBeanFactory(new ClassPathResource("beans.xml"));
UserService service = factory.getBean(UserService.class);  // 此时才创建 Bean

// ApplicationContext 高级用法
ApplicationContext ctx = new ClassPathXmlApplicationContext("beans.xml");
// 容器初始化时就创建了所有单例 Bean
UserService service = ctx.getBean(UserService.class);  // 直接从缓存获取
```

## 🔥 扩展追问 2：循环依赖的解决

```java
// 循环依赖场景
@Component
public class A {
    @Autowired
    private B b;
}

@Component
public class B {
    @Autowired
    private A a;
}
```

```
┌─────────────────────────────────────────────────────────────────┐
│                  三级缓存解决循环依赖                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  DefaultSingletonBeanRegistry 中的三个 Map：                      │
│                                                                  │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │  singletonObjects      - 一级缓存：成品单例 Bean            │   │
│  │  earlySingletonObjects - 二级缓存：早期暴露的 Bean（已实例化）│   │
│  │  singletonFactories    - 三级缓存：ObjectFactory 工厂       │   │
│  └──────────────────────────────────────────────────────────┘   │
│                                                                  │
│  创建 A 的流程：                                                  │
│  1. 实例化 A（调用构造方法）                                       │
│  2. 将 A 的 ObjectFactory 放入三级缓存                              │
│  3. 填充 A 的属性，发现需要 B                                       │
│       ↓                                                          │
│  4. 创建 B                                                        │
│     4.1 实例化 B                                                  │
│     4.2 将 B 的 ObjectFactory 放入三级缓存                          │
│     4.3 填充 B 的属性，发现需要 A                                   │
│          ↓                                                       │
│     4.4 从三级缓存获取 A 的 ObjectFactory，获取早期引用               │
│     4.4 完成 B 的创建，放入一级缓存                                  │
│       ↓                                                          │
│  5. 继续完成 A 的创建                                              │
│  6. A 放入一级缓存                                                 │
│                                                                  │
│  关键：通过三级缓存提前暴露"半成品"Bean，打破循环                    │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 🔥 扩展追问 3：BeanPostProcessor 扩展点

```java
// BeanPostProcessor 是 Spring 扩展的核心机制

@Component
public class MyBeanPostProcessor implements BeanPostProcessor {
    
    // 初始化前处理
    @Override
    public Object postProcessBeforeInitialization(Object bean, String beanName) {
        System.out.println("Before init: " + beanName);
        // 可以修改或替换 Bean
        return bean;
    }
    
    // 初始化后处理
    @Override
    public Object postProcessAfterInitialization(Object bean, String beanName) {
        System.out.println("After init: " + beanName);
        // AOP 代理在此创建
        return bean;
    }
}

// 常用 BeanPostProcessor：
// • AutowiredAnnotationBeanPostProcessor - 处理 @Autowired
// • CommonAnnotationBeanPostProcessor - 处理 @PostConstruct/@PreDestroy
// • AbstractAutoProxyCreator - 创建 AOP 代理
// • ApplicationContextAwareProcessor - 处理 Aware 接口
```

## 一句话收尾

Spring IOC 容器初始化经过**定位资源 → 加载 BeanDefinition → 注册到容器 → 实例化 → 依赖注入 → 初始化**六个阶段；**BeanDefinition**是 Bean 的**元数据蓝图**，存储类名、作用域、依赖、生命周期回调等配置，Spring 通过**三级缓存**解决循环依赖，通过**BeanPostProcessor**提供扩展点。
