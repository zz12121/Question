## 先说结论

**泛型擦除**是 Java 泛型的实现机制，编译时将泛型类型参数替换为边界类型（无边界则替换为 Object），并插入强制类型转换。运行时泛型信息**大部分被擦除**，但可通过反射获取部分声明信息。`List<String>` 和 `List<Integer>` 在运行时**完全相同**，都是 `List`，无法区分。

---

## 🔥 核心概念

**解决什么问题：**
- 编译期类型检查，提前发现类型错误
- 减少强制类型转换，代码更简洁安全
- 保持与旧版本 Java 的兼容性（二进制兼容）

**擦除的本质：**
- 编译期：泛型参与类型检查
- 运行期：泛型信息被擦除，转为原始类型

## 🔥 详细展开

### 1. 泛型擦除的过程

```java
// 源代码
public class GenericExample<T> {
    private T value;
    
    public void setValue(T value) {
        this.value = value;
    }
    
    public T getValue() {
        return value;
    }
    
    public void printList(List<String> list) {
        for (String s : list) {
            System.out.println(s);
        }
    }
}
```

**编译后（反编译）：**

```java
public class GenericExample {
    private Object value;  // T 被替换为 Object
    
    public void setValue(Object value) {
        this.value = value;
    }
    
    public Object getValue() {
        return value;
    }
    
    public void printList(List list) {  // 泛型参数被擦除
        for (Object s : (String) list.get(0)) {  // 插入强制类型转换
            System.out.println((String) s);
        }
    }
}
```

### 2. 不同类型参数的擦除

```java
// 无边界类型参数 → 擦除为 Object
public class Box<T> {
    private T value;  // 运行时：Object value
}

// 有边界类型参数 → 擦除为边界类型
public class NumberBox<T extends Number> {
    private T value;  // 运行时：Number value
}

// 多边界 → 擦除为第一个边界
public class MultiBound<T extends Number & Comparable> {
    private T value;  // 运行时：Number value
    // Comparable 信息丢失，但编译期可用
}
```

### 3. 运行时泛型信息

**大部分被擦除：**

```java
List<String> stringList = new ArrayList<>();
List<Integer> intList = new ArrayList<>();

System.out.println(stringList.getClass());  // class java.util.ArrayList
System.out.println(intList.getClass());     // class java.util.ArrayList

// 运行时完全相同！
System.out.println(stringList.getClass() == intList.getClass());  // true
```

**无法区分：**

```java
// 编译错误：方法签名冲突（擦除后相同）
public void process(List<String> list) { }
public void process(List<Integer> list) { }  // ❌ 编译错误：和上面是同一个方法
```

### 4. 但部分泛型信息可通过反射获取

```java
// 1. 类声明中的泛型信息（未被完全擦除）
public class StringList extends ArrayList<String> {
}

Type genericSuperclass = StringList.class.getGenericSuperclass();
if (genericSuperclass instanceof ParameterizedType) {
    ParameterizedType paramType = (ParameterizedType) genericSuperclass;
    Type[] actualTypeArgs = paramType.getActualTypeArguments();
    for (Type typeArg : actualTypeArgs) {
        System.out.println(typeArg);  // class java.lang.String
    }
}

// 2. 方法参数中的泛型
public class Example {
    public void process(List<String> list) { }
}

Method method = Example.class.getMethod("process", List.class);
Type[] paramTypes = method.getGenericParameterTypes();
for (Type paramType : paramTypes) {
    if (paramType instanceof ParameterizedType) {
        System.out.println(paramType);  // java.util.List<java.lang.String>
    }
}

// 3. 字段中的泛型
public class Example {
    private List<String> names;
}

Field field = Example.class.getDeclaredField("names");
Type genericType = field.getGenericType();
System.out.println(genericType);  // java.util.List<java.lang.String>
```

**关键限制：**

```java
// 无法获取实际对象的泛型类型
List<String> list = new ArrayList<>();
// list.getClass() 只能得到 ArrayList，得不到 String

// 匿名子类技巧（Gson 等库使用）
TypeToken<List<String>> token = new TypeToken<List<String>>() {};
// 创建匿名子类，保留泛型信息
```

### 5. 桥接方法（Bridge Method）

**问题：** 擦除后与父类方法签名不一致

```java
// 父类（擦除后）
public class Node {
    private Object data;
    public void setData(Object data) { this.data = data; }
}

// 子类
public class MyNode extends Node {
    private Integer data;
    
    @Override
    public void setData(Integer data) {  // 编译期类型是 Integer
        this.data = data;
    }
}
```

**编译器生成的桥接方法：**

```java
public class MyNode extends Node {
    private Integer data;
    
    // 编译器生成的桥接方法
    @Override
    public void setData(Object data) {  // 与父类匹配
        setData((Integer) data);  // 调用实际方法
    }
    
    // 实际方法
    public void setData(Integer data) {
        this.data = data;
    }
}
```

验证：

```java
Method[] methods = MyNode.class.getDeclaredMethods();
for (Method m : methods) {
    System.out.println(m.getName() + " - " + Arrays.toString(m.getParameterTypes()) 
        + " - synthetic: " + m.isSynthetic() + " - bridge: " + m.isBridge());
}

// 输出：
// setData - [class java.lang.Object] - synthetic: true - bridge: true
// setData - [class java.lang.Integer] - synthetic: false - bridge: false
```

### 6. 泛型数组的限制

```java
// 无法创建泛型数组（编译错误）
List<String>[] array = new List<String>[10];  // ❌ 编译错误

// 原因：擦除后 List<String>[] 和 List<Integer>[] 都是 List[]
// 如果允许，会导致类型安全问题：
Object[] objArray = new List<String>[10];
objArray[0] = new ArrayList<Integer>();  // 运行时不会报错，但类型混乱！

// 解决方案 1：使用原始类型（有警告）
List<?>[] array = new List<?>[10];

// 解决方案 2：使用集合代替数组
List<List<String>> list = new ArrayList<>();

// 解决方案 3：强制转换（不推荐）
List<String>[] array = (List<String>[]) new List[10];
```

## 🔥 扩展追问 1：为什么 Java 选择擦除而不是具体化？

| 方案 | 优点 | 缺点 |
|------|------|------|
| **擦除（Java）** | 兼容旧代码、JVM 改动小 | 运行时丢失类型信息 |
| **具体化（C#）** | 运行时保留类型信息 | 需要新 JVM、不兼容旧代码 |

**Java 选择擦除的主要原因：**

1. **向后兼容性：** Java 5 引入泛型时，需要兼容已有的大量非泛型代码
2. **JVM 稳定性：** 不需要修改 JVM 字节码格式
3. **迁移平滑：** 可以逐步将旧代码迁移到泛型

```java
// 兼容示例：原始类型和泛型可以互操作
List rawList = new ArrayList();      // 旧代码
List<String> genericList = rawList;  // 可以赋值（有警告）
rawList = genericList;               // 也可以反向赋值
```

## 🔥 扩展追问 2：如何绕过泛型检查？

```java
// 通过原始类型绕过
List<Integer> intList = new ArrayList<>();
List rawList = intList;  // 原始类型
rawList.add("hello");    // 可以插入 String！

Integer num = intList.get(0);  // 运行时 ClassCastException！

// 通过反射绕过
List<Integer> list = new ArrayList<>();
Method addMethod = List.class.getMethod("add", Object.class);
addMethod.invoke(list, "hello");  // 成功插入 String

// 警告：这是未检查行为，会导致类型安全问题
```

## 🔥 扩展追问 3：通配符和边界

```java
// ? extends T（上界）- 只能读，不能写
public void printList(List<? extends Number> list) {
    for (Number n : list) {  // ✅ 可以读为 Number
        System.out.println(n);
    }
    // list.add(1);  // ❌ 编译错误，不能写入
}

// ? super T（下界）- 只能写，不能读（只能读为 Object）
public void addNumbers(List<? super Integer> list) {
    list.add(1);  // ✅ 可以写入 Integer
    list.add(2);
    // Integer i = list.get(0);  // ❌ 编译错误
    Object o = list.get(0);  // ✅ 只能读为 Object
}

// PECS 原则：Producer-Extends, Consumer-Super
```

**擦除后的边界：**

```java
// ? extends Number 擦除后就是 Number
public void printList(List list) {  // 运行时
    for (Number n : (List<Number>) list) { }
}
```

## 🔥 扩展追问 4：Kotlin 的泛型具体化（reified）

```kotlin
// Kotlin 支持具体化类型参数
inline fun <reified T> isOfType(value: Any): Boolean {
    return value is T  // ✅ 可以检查类型
}

// 编译后（Java 伪代码）
public static final boolean isOfType(Object value) {
    return value instanceof String;  // 类型被内联
}

// 使用
isOfType<String>("hello")  // 编译时确定类型
```

**原理：** Kotlin 使用 `inline` 函数，在编译时将类型参数替换为具体类型。

## 一句话收尾

Java 泛型擦除是编译期检查、运行期擦除为 Object（或边界），`List<String>` 和 `List<Integer>` 运行时完全相同；部分声明信息可通过反射获取，桥接方法保证多态正确性；设计选择擦除是为了兼容旧代码，代价是运行时无法区分泛型类型！
