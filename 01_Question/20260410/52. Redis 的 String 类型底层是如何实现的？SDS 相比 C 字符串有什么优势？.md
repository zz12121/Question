## 先说结论

Redis String 底层使用 **SDS（Simple Dynamic String，简单动态字符串）** 实现；相比 C 字符串，SDS 具有**O(1) 获取长度、杜绝缓冲区溢出、减少内存重分配、二进制安全、兼容 C 字符串 API** 五大优势。

## 🔥 详细展开

### 1. SDS 的五种 Header 类型

```c
// Redis 3.2+ 根据字符串长度选择不同 header 类型，节省内存

#define SDS_TYPE_5  0   // 5位长度，无 alloc 字段（已废弃）
#define SDS_TYPE_8  1   // 8位长度，最大 255 字节
#define SDS_TYPE_16 2   // 16位长度，最大 65535 字节
#define SDS_TYPE_32 3   // 32位长度，最大 4GB
#define SDS_TYPE_64 4   // 64位长度，超大字符串

// 类型选择逻辑
static inline char sdsReqType(size_t string_size) {
    if (string_size < 1<<8) return SDS_TYPE_8;    // < 256
    if (string_size < 1<<16) return SDS_TYPE_16;  // < 64K
    if (string_size < 1ll<<32) return SDS_TYPE_32; // < 4G
    return SDS_TYPE_64;
}
```

### 2. SDS 相比 C 字符串的五大优势

#### 优势 1：O(1) 获取字符串长度

```c
// C 字符串：O(n)
size_t strlen(const char *s) {
    size_t len = 0;
    while (s[len] != '\0') len++;  // 遍历到 \0
    return len;
}

// SDS：O(1)
static inline size_t sdslen(const sds s) {
    // 通过 flags 确定 header 类型，直接读取 len 字段
    unsigned char flags = s[-1];  // flags 在 buf 前面 1 字节
    switch(flags & SDS_TYPE_MASK) {
        case SDS_TYPE_8:  return ((struct sdshdr8*)s)->len;
        case SDS_TYPE_16: return ((struct sdshdr16*)s)->len;
        // ...
    }
}
```

**应用场景：**

```
┌─────────────────────────────────────────────────────────────────┐
│  STRLEN 命令实现：                                                │
│                                                                  │
│  客户端：STRLEN mykey                                             │
│       ↓                                                          │
│  Redis：直接读取 SDS 的 len 字段                                  │
│       ↓                                                          │
│  返回：O(1) 完成，无论字符串多长                                  │
│                                                                  │
│  如果是 C 字符串，1MB 的字符串需要遍历 100 万次！                   │
└─────────────────────────────────────────────────────────────────┘
```

#### 优势 2：杜绝缓冲区溢出

```c
// C 字符串：容易溢出
char buf[10] = "Hello";
strcat(buf, " World!");  // 溢出！需要 12 字节，但只有 10 字节

// SDS：自动扩容
sds s = sdsnew("Hello");
s = sdscat(s, " World!");  // 自动检查并扩容
// 内部逻辑：
// 1. 检查现有空间：alloc - len >= 需要的长度？
// 2. 不够则重新分配内存
// 3. 执行拼接
```

```c
// SDS 扩容源码
sds sdsMakeRoomFor(sds s, size_t addlen) {
    struct sdshdr *sh = (void*)(s - sizeof(struct sdshdr));
    size_t free = sdsavail(s);  // 获取剩余空间
    
    if (free >= addlen) return s;  // 空间足够，直接返回
    
    // 空间不够，重新分配
    size_t len = sdslen(s);
    size_t newlen = len + addlen;
    
    // 预分配策略：如果新长度 < 1MB，分配 2*newlen
    // 否则分配 newlen + 1MB
    if (newlen < SDS_MAX_PREALLOC)
        newlen *= 2;
    else
        newlen += SDS_MAX_PREALLOC;
    
    // 重新分配内存
    // ...
}
```

#### 优势 4：二进制安全

```c
// C 字符串：遇到 \0 就结束，不能存储二进制数据
char *str = "Hello\0World";  // strlen(str) = 5，World 被截断

// SDS：通过 len 字段确定长度，可以存储任意二进制数据
sds s = sdsnewlen("Hello\0World", 11);  // 明确指定长度 11
// sdslen(s) = 11，完整保存所有字节

// Redis 使用场景：
SET key "\x00\x01\x02\x03"  // 存储二进制数据
GET key                     // 返回 4 字节，不是空字符串
```

#### 优势 5：兼容 C 字符串 API

```c
// SDS 的 buf 末尾仍然以 \0 结尾
// 可以直接使用 C 字符串函数

sds s = sdsnew("Hello");
printf("%s\n", s);  // 直接使用 printf，输出 Hello

// 内部保证：
sds s = sdsnew("Redis");
// 实际存储：'R' 'e' 'd' 'i' 's' '\0'
// sdslen(s) = 5，但 buf[5] = '\0'

// 这样可以直接重用 C 标准库函数
strcpy(dst, s);     // 安全
strcat(dst, s);     // 安全（如果 dst 空间足够）
```

## 🔥 扩展追问 2：SDS 的内存分配源码

```c
// 创建 SDS
sds sdsnewlen(const void *init, size_t initlen) {
    void *sh;
    sds s;
    char type = sdsReqType(initlen);  // 根据长度选择类型
    
    // 计算 header 大小
    int hdrlen = sdsHdrSize(type);
    
    // 分配内存：header + 数据 + \0
    sh = s_malloc(hdrlen + initlen + 1);
    
    // 初始化 header
    // ...
    
    // 拷贝数据
    if (initlen && init)
        memcpy(s, init, initlen);
    
    s[initlen] = '\0';  // 添加 C 字符串结尾
    return s;
}

// 释放 SDS
void sdsfree(sds s) {
    if (s == NULL) return;
    s_free((char*)s - sdsHdrSize(s[-1]));  // 释放整个 header + buf
}
```

## 一句话收尾

Redis String 底层用 **SDS** 实现，相比 C 字符串具有**O(1) 取长度、防溢出、预分配减内存重分配、二进制安全、兼容 C API** 五大优势，并通过 **int/embstr/raw** 三种编码根据数据特性自动选择最优存储方式。
