## 二、核心区别详解

### 2.1 exceptionally——只管异常

```java
/**
 * exceptionally()：只有异常时才触发
 * 
 * 特点：
 * 1. 只接收 Throwable，忽略正常值
 * 2. 只能返回 **默认值**，无法处理正常值
 * 3. 异常被消费后，后续链正常执行
 */
public class ExceptionallyDemo {
    
    public static void main(String[] args) {
        CompletableFuture<String> future = CompletableFuture
            .supplyAsync(() -> {
                if (Math.random() > 0.5) {
                    throw new RuntimeException("Oops!");
                }
                return "正常结果";
            })
            .exceptionally(ex -> {
                // ex: Throwable（异常对象）
                // 返回值：作为整个 CF 的结果
                System.out.println("捕获异常: " + ex.getMessage());
                return "默认结果";  // ← 必须有返回值
            })
            .thenApply(result -> {
                // 如果前面异常 → result = "默认结果"（默认值）
                // 如果前面正常 → result = "正常结果"
                return "处理: " + result;
            });
        
        System.out.println(future.join());
    }
}
```

**exceptionally 的局限性**：

```java
// ❌ exceptionally 无法区分是正常还是异常后的默认值
// 问题：后续代码不知道是真实值还是默认值

CompletableFuture<Integer> future = CompletableFuture
    .supplyAsync(() -> {
        if (Math.random() > 0.5) throw new RuntimeException("fail");
        return 100;
    })
    .exceptionally(ex -> {
        return 0;  // ← 异常时返回 0
    })
    .thenApply(x -> {
        // ❌ 问题：x 可能是 0（异常恢复）也可能是 100（正常）
        // 无法区分！
        if (x == 0) {
            // 这是异常恢复？还是真的返回 0？
            // 不知道！
        }
        return x * 2;
    });
```

### 2.2 handle——同时处理正常和异常

```java
/**
 * handle()：无论正常还是异常都会触发
 * 
 * 特点：
 * 1. BiFunction 接收两个参数：(result, ex)
 * 2. 正常时：result = 值，ex = null
 * 3. 异常时：result = null，ex = Throwable
 * 4. 可以根据是否异常返回不同的值
 */
public class HandleDemo {
    
    public static void main(String[] args) {
        CompletableFuture<String> future = CompletableFuture
            .supplyAsync(() -> {
                if (Math.random() > 0.5) {
                    throw new RuntimeException("Oops!");
                }
                return "正常结果";
            })
            .handle((result, ex) -> {
                // ⭐ 关键：result 和 ex 互斥
                // - 正常：result != null, ex == null
                // - 异常：result == null, ex != null
                
                if (ex != null) {
                    // 处理异常
                    System.out.println("捕获: " + ex.getMessage());
                    return "异常恢复值";
                } else {
                    // 处理正常值
                    return "处理: " + result;
                }
            })
            .thenApply(x -> {
                // x 一定是处理后的值（不再是 null）
                return x + " -> 再处理";
            });
        
        System.out.println(future.join());
    }
}
```

**handle 的优势**：

```java
// ✅ handle 可以区分正常值和异常恢复值
CompletableFuture<Integer> future = CompletableFuture
    .supplyAsync(() -> {
        if (Math.random() > 0.5) throw new RuntimeException("fail");
        return 100;
    })
    .handle((result, ex) -> {
        if (ex != null) {
            return new int[]{0, 1};  // [默认值, isError]
        } else {
            return new int[]{result, 0};  // [真实值, isError]
        }
    })
    .thenApply(arr -> {
        // arr[0] = 值，arr[1] = 是否异常恢复
        if (arr[1] == 1) {
            System.out.println("异常恢复");
        }
        return arr[0] * 2;
    });
```

### 2.3 whenComplete——纯副作用，不影响结果

```java
/**
 * whenComplete()：纯观察，不改变结果
 * 
 * 特点：
 * 1. BiConsumer 接收两个参数：(result, ex)
 * 2. 返回值类型是 void（不改变 CF 结果）
 * 3. 适合做日志记录、监控、清理等副作用
 * 4. 通常配合其他异常处理方法使用
 */
public class WhenCompleteDemo {
    
    public static void main(String[] args) {
        CompletableFuture<String> future = CompletableFuture
            .supplyAsync(() -> {
                if (Math.random() > 0.5) {
                    throw new RuntimeException("Oops!");
                }
                return "正常结果";
            })
            .whenComplete((result, ex) -> {
                // ⭐ whenComplete 不返回值，只是观察
                // 后续链仍然使用原始 result/ex
                
                if (ex != null) {
                    System.out.println("发生异常: " + ex.getMessage());
                    // 不能修改 result！
                    // result = "xxx";  ← 编译错误
                } else {
                    System.out.println("正常结果: " + result);
                }
            })
            .exceptionally(ex -> {
                // ⭐ whenComplete 之后仍然可以加 exceptionally
                // because whenComplete 没有消费异常
                return "默认值";
            });
        
        System.out.println(future.join());
    }
}
```

## 三、三者对比表格

| 特性 | exceptionally | handle | whenComplete |
|-----|--------------|--------|--------------|
| **方法签名** | `Function<Throwable, U>` | `BiFunction<T, Throwable, U>` | `BiConsumer<T, Throwable>` |
| **正常时触发** | ❌ 不触发 | ✅ 触发 | ✅ 触发 |
| **异常时触发** | ✅ 触发 | ✅ 触发 | ✅ 触发 |
| **返回值类型** | `U`（必须有） | `U`（必须有） | `void` |
| **是否消费异常** | ✅ 消费 | ✅ 消费 | ❌ 不消费 |
| **改变 CF 结果** | ✅ 可以 | ✅ 可以 | ❌ 不能 |
| **典型场景** | 返回默认值 | 统一处理 | 日志/监控 |

### 4.1 异常传播代码示例

```java
public class ExceptionPropagation {
    
    public static void main(String[] args) {
        // 场景 1：有 exceptionally，后续继续执行
        CompletableFuture<String> future1 = CompletableFuture
            .supplyAsync(() -> {
                System.out.println("Step 1: 正常");
                return "A";
            })
            .thenApply(x -> {
                throw new RuntimeException("Step 2 异常!");
            })
            .thenApply(x -> {
                System.out.println("Step 3: 不会执行");
                return x + " -> B";
            })
            .exceptionally(ex -> {
                System.out.println("Exceptionally: 捕获 " + ex.getMessage());
                return "默认值";
            })
            .thenApply(x -> {
                System.out.println("Step 4: 正常执行，x=" + x);
                return x + " -> C";
            });
        
        System.out.println("Result: " + future1.join());
        /**
         * 输出：
         * Step 1: 正常
         * Exceptionally: 捕获 Step 2 异常!
         * Step 4: 正常执行，x=默认值
         * Result: 默认值 -> C
         */
        
        System.out.println("\n" + "=".repeat(50) + "\n");
        
        // 场景 2：无 exceptionally，异常会传播
        CompletableFuture<String> future2 = CompletableFuture
            .supplyAsync(() -> "A")
            .thenApply(x -> {
                throw new RuntimeException("Step 2 异常!");
            })
            .thenApply(x -> {
                System.out.println("Step 3: 不会执行");
                return x;
            });
        
        try {
            future2.join();
        } catch (CompletionException e) {
            System.out.println("捕获到异常: " + e.getCause().getMessage());
        }
        /**
         * 输出：
         * 捕获到异常: Step 2 异常!
         */
    }
}
```

### 5.1 方法一：orTimeout（推荐，JDK 9+）

```java
import java.util.concurrent.*;
import java.util.concurrent.CompletableFuture;

/**
 * orTimeout + exceptionally 实现超时返回默认值
 */
public class TimeoutDemo {
    
    public static void main(String[] args) {
        CompletableFuture<String> future = CompletableFuture
            .supplyAsync(() -> {
                // 模拟耗时操作
                try {
                    Thread.sleep(5000);  // 5 秒
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
                return "真实结果";
            })
            // ⭐ 关键：设置 3 秒超时
            .orTimeout(3, TimeUnit.SECONDS)
            // ⭐ 超时或异常时返回默认值
            .exceptionally(ex -> {
                if (ex instanceof TimeoutException) {
                    return "超时默认值";
                }
                return "异常默认值: " + ex.getMessage();
            });
        
        System.out.println("结果: " + future.join());
        // 输出：结果: 超时默认值
    }
}
```

### 5.4 方法四：自定义超时 CompletableFuture

```java
/**
 * JDK 8 兼容的超时实现
 */
public class TimeoutSupport {
    
    /**
     * 为 Future 添加超时逻辑
     * 超时后返回默认值，不阻塞
     */
    public static <T> CompletableFuture<T> withTimeout(
            CompletableFuture<T> future, 
            long timeout, 
            TimeUnit unit,
            T defaultValue) {
        
        // 超时 Future
        CompletableFuture<T> timeoutFuture = CompletableFuture
            .supplyAsync(() -> {
                try {
                    Thread.sleep(unit.toMillis(timeout));
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
                throw new TimeoutException("操作超时");
            });
        
        // 任一完成即返回
        return future
            .applyToEither(timeoutFuture, Function.identity())
            .exceptionally(ex -> {
                // 捕获超时/异常，返回默认值
                if (ex instanceof TimeoutException || 
                    ex.getCause() instanceof TimeoutException) {
                    return defaultValue;
                }
                throw (RuntimeException) ex;
            });
    }
    
    // 使用示例
    public static void main(String[] args) {
        CompletableFuture<String> dbQuery = CompletableFuture
            .supplyAsync(() -> {
                // 模拟数据库查询
                try { Thread.sleep(5000); } catch (InterruptedException e) {}
                return "数据库结果";
            });
        
        CompletableFuture<String> result = withTimeout(
            dbQuery, 
            3, TimeUnit.SECONDS,  // 3 秒超时
            "缓存默认值"           // 超时默认值
        );
        
        System.out.println(result.join());
        // 3 秒后输出：缓存默认值
    }
}
```

```java
 * 
 * 价格服务（必须，2秒超时）→ 超时抛异常
 * 库存服务（可选，1秒超时）→ 超时返回"有货"
 * 评论服务（可选，3秒超时）→ 超时返回空列表
 */
public class ProductDetailService {
    
    public ProductDetail query(Long productId) {
        CompletableFuture<BigDecimal> priceFuture = CompletableFuture
            .supplyAsync(() -> priceService.getPrice(productId))
            .orTimeout(2, TimeUnit.SECONDS)
            .exceptionally(ex -> {
                throw new ServiceException("价格服务超时");
            });
        
        CompletableFuture<StockInfo> stockFuture = CompletableFuture
            .supplyAsync(() -> stockService.getStock(productId))
            .orTimeout(1, TimeUnit.SECONDS)
            .exceptionally(ex -> {
                log.warn("库存服务超时，返回默认库存");
                return new StockInfo(true);  // 超时默认有货
            });
        
        CompletableFuture<List<Review>> reviewFuture = CompletableFuture
            .supplyAsync(() -> reviewService.getReviews(productId))
            .orTimeout(3, TimeUnit.SECONDS)
            .exceptionally(ex -> {
                log.warn("评论服务超时，返回空列表");
                return Collections.emptyList();
            });
        
        // 合并结果
        try {
            BigDecimal price = priceFuture.join();  // 主服务，可能抛异常
            StockInfo stock = stockFuture.join();
            List<Review> reviews = reviewFuture.join();
            
            return new ProductDetail(productId, price, stock, reviews);
        } catch (Exception e) {
            throw new ServiceException("商品查询失败", e);
        }
    }
}
```

## 🔥 扩展追问 1：handle vs whenComplete + thenApply 的区别？

> **A:** 本质区别在于 **异常是否被消费**：

```java
// 方式 1：handle 同时处理
future.handle((result, ex) -> {
    if (ex != null) return "默认值";
    return result.toUpperCase();  // 正常时处理
})
.thenApply(x -> x + "!");  // 接收 handle 返回值

// 方式 2：whenComplete + thenApply（等价）
future.whenComplete((result, ex) -> {
    // 纯观察，不改变 CF 结果
    if (ex != null) log.error(ex);
})
.thenApply(result -> {
    // 如果前面异常，这里 result = null！
    // 且异常没有被消费，抛 CompletionException！
    return result.toUpperCase();  // ⚠️ NullPointerException!
});

// 结论：whenComplete 不消费异常，需要后面加 exceptionally
```

## 🔥 扩展追问 2：exceptionally vs handle vs exceptionallyCompose？

> **A:** `exceptionallyCompose` 用于异常时切换到另一个 Future：

```java
/**
 * exceptionally：同步返回默认值
 * exceptionallyCompose：异步切换到另一个 Future
 */
public class ComposeDemo {
    
    public static void main(String[] args) {
        CompletableFuture<String> future = CompletableFuture
            .supplyAsync(() -> {
                if (Math.random() > 0.5) throw new RuntimeException("fail");
                return "主流程结果";
            })
            .exceptionallyCompose(ex -> {
                // ⭐ 返回 CompletableFuture，不是普通值
                System.out.println("异常，切换到降级流程");
                return CompletableFuture
                    .supplyAsync(() -> "降级结果");
            });
        
        System.out.println(future.join());
    }
}
```

## 🔥 扩展追问 3：get() vs join() 的区别？

> **A:** 检查异常 vs 非检查异常：

```java
/**
 * get()：throws InterruptedException, ExecutionException（受检）
 * join()：抛出 CompletionException（非受检）
 * 
 * 通常在 lambda 中用 join()，外层用 get()
 */

// get() 需要 try-catch
CompletableFuture<String> f = CompletableFuture.supplyAsync(...);
try {
    String result = f.get();  // ⚠️ 需要处理受检异常
} catch (InterruptedException | ExecutionException e) {
    throw new RuntimeException(e);
}

// join() 抛出非受检异常
String result = f.join();  // ✅ 不用处理
```

## 一句话总结

> **exceptionally 只管异常（同步返回默认值），handle 同时管正常+异常（BiFunction），whenComplete 纯观察不改结果；链式调用中异常会跳过中间环节直到被异常处理方法消费；超时处理用 `orTimeout + exceptionally` 或 `completeOnTimeout` 返回默认值不阻塞。**

```java
// 常用超时配置
ExecutorService executor = Executors.newFixedThreadPool(10);

      .exceptionally(ex -> "默认值");

future.completeOnTimeout("默认值", 3, TimeUnit.SECONDS);

// 3. 自定义超时任务
CompletableFuture<String> withTimeout(Supplier<String> supplier, 
                                       long timeout, 
                                       TimeUnit unit, 
                                       String defaultValue) {
    CompletableFuture<String> timeoutFuture = CompletableFuture
        .supplyAsync(() -> {
            Thread.sleep(unit.toMillis(timeout));
            throw new TimeoutException();
        });
    
    return CompletableFuture
        .supplyAsync(supplier)
        .applyToEither(timeoutFuture, Function.identity())
        .exceptionally(ex -> {
            if (ex instanceof TimeoutException) return defaultValue;
            throw (RuntimeException) ex;
        });
}
```
