## 二、LinkedHashMap 节点结构

```java
/**
 * LinkedHashMap 继承 HashMap，复用 HashMap 的 Node
 * 但实际使用的是 LinkedHashMapEntry
 */

// HashMap 的 Node（单向链表）
static class Node<K,V> implements Map.Entry<K,V> {
    final int hash;
    final K key;
    V value;
    Node<K,V> next;
}

// LinkedHashMap 的 Entry（双向链表）
static class LinkedHashMapEntry<K,V> extends HashMap.Node<K,V> {
    LinkedHashMapEntry<K,V> before;  // 前驱
    LinkedHashMapEntry<K,V> after;   // 后继
}
```

```

    │

    - 新增 before/after 双向指针
    - 重写了 afterNodeAccess() 和 afterNodeInsertion()
```

## 三、`accessOrder=true` 实现 LRU 缓存

```java
/**
 * LinkedHashMap 实现 LRU（Least Recently Used）缓存
 * 
 * 关键参数：
 * - accessOrder = false（默认）：按插入顺序
 * - accessOrder = true：按访问顺序（get/put 都会移动到链表尾部）
 */
```

### 3.1 LRU 实现原理

```
初始状态（插入顺序）：

header ──► [A] ──► [B] ──► [C] ──► [D] ──► null
     ◄───────────────────────────────
              双向链表

accessOrder=true，调用 get(B) 后：

header ──► [A] ──► [C] ──► [D] ──► [B] ──► null
     ◄───────────────────────────────
              B 被移动到尾部（最近访问）

满容量后，删除最老的（A）：

header ──► [C] ──► [D] ──► [B] ──► null
     ◄─────────────────────────
              A 被移除
```

### 3.2 完整 LRU 实现代码

```java
import java.util.*;

public class LRUCache<K, V> extends LinkedHashMap<K, V> {
    
    private final int capacity;
    
    public LRUCache(int capacity) {
        // ⭐ 关键：accessOrder = true，按访问顺序
        super(capacity, 0.75f, true);
        this.capacity = capacity;
    }
    
    @Override
    protected boolean removeEldestEntry(Map.Entry<K, V> eldest) {
        // 当 size > capacity 时，删除最老的 entry
        return size() > capacity;
    }
    
    // 测试
    public static void main(String[] args) {
        LRUCache<String, Integer> cache = new LRUCache<>(3);
        
        cache.put("A", 1);
        cache.put("B", 2);
        cache.put("C", 3);
        
        System.out.println("初始: " + cache.keySet());  // [A, B, C]
        
        cache.get("A");  // 访问 A，A 移到尾部
        
        cache.put("D", 4);  // 插入 D，触发删除最老的 B
        
        System.out.println("访问后: " + cache.keySet());  // [A, C, D]
        // A 被访问过，所以保留
        // B 最老，被删除
    }
}
```

### 3.3 核心源码分析

```java
// LinkedHashMap.get() → 最终调用 afterNodeAccess()

void afterNodeAccess(Node<K,V> e) { 
    LinkedHashMapEntry<K,V> last;
    // accessOrder=true 时，将当前节点移到链表尾部
    if (accessOrder && (last = tail) != e) {
        LinkedHashMapEntry<K,V> p =
            (LinkedHashMapEntry<K,V>)e, b = p.before, a = p.after;
        p.after = null;
        if (b == null) head = a;
        else b.after = a;
        if (a != null) a.before = b;
        else last = b;
        
        // ⭐ 核心：将节点移到尾部
        if (last == null) head = p;
        else {
            p.before = last;
            last.after = p;
        }
        tail = p;
        ++modCount;
    }
}

// LinkedHashMap.put() → afterNodeInsertion() 删除最老的

void afterNodeInsertion(boolean evict) {
    LinkedHashMapEntry<K,V> first;
    // evict=true（插入模式），删除最老的
    if (evict && (first = head) != null && removeEldest) {
        K key = first.key;
        removeNode(hash(key), key, null, false, false);
    }
}
```

## 四、HashSet 底层实现

```java
/**
 * HashSet 就是一个"只有 key 的 HashMap"
 * 
 * 底层：
 * HashSet<E>  →  HashMap<E, Object>（PRESENT 是固定值对象）
 */
```

**源码证据**：

```java
public class HashSet<E> extends AbstractSet<E> 
    implements Set<E>, Cloneable, java.io.Serializable {
    
    // ⭐ HashSet 内部就是一个 HashMap，PRESENT 是固定的 value
    private transient HashMap<E, Object> map;
    
    // 所有 value 都用这个对象（无实际意义，只做占位）
    private static final Object PRESENT = new Object();
    
    // add → put
    public boolean add(E e) {
        return map.put(e, PRESENT) == null;  // key=e, value=PRESENT
    }
    
    // remove → remove
    public boolean remove(Object o) {
        return map.remove(o) == PRESENT;
    }
    
    // contains → containsKey
    public boolean contains(Object o) {
        return map.containsKey(o);
    }
    
    // size → size
    public int size() {
        return map.size();
    }
}
```

**内存结构对比**：

```
HashSet<Integer> set = new HashSet<>();
set.add(1);
set.add(2);
set.add(3);

底层等价于：

map.put(1, PRESENT);   // PRESENT = new Object()
map.put(2, PRESENT);
map.put(3, PRESENT);

│  HashSet                                    HashMap     │
│  ┌─────────────────────────────────────────────────┐   │
│  │  HashMap<E, Object>                            │   │

│  │  │ 1  │ 2  │ 3  │null│  key = 1,2,3              │   │
│  │  │ ❌ │ ❌ │ ❌ │    │  value = Object() x3     │   │
```

**为什么 HashSet 不允许重复？**

```java
// HashMap.put() 返回 null 表示新插入
// 返回非 null 表示 key 已存在（覆盖旧值）
public boolean add(E e) {
    return map.put(e, PRESENT) == null;  // null=新插入成功，非null=已存在
}

// HashMap.put() 核心逻辑
public V put(K key, V value) {
    if (key == null) {
        // 处理 null key...
    }
    int hash = hash(key);
    int i = indexFor(hash, table.length);
    
    // 遍历链表/红黑树，查找是否存在
    for (Entry<K,V> e = table[i]; e != null; e = e.next) {
        if (e.hash == hash && eq(k, e.key)) {
            V oldValue = e.value;
            e.value = value;  // 覆盖
            return oldValue;  // 返回旧值（非null，说明已存在）
        }
    }
    
    addToSize(1);
    return null;  // 新插入，返回 null
}
```

## 五、TreeSet 底层实现

```java
/**
 * TreeSet = NavigableMap（红黑树）实现
 * 
 * 底层：
 * TreeSet<E>  →  TreeMap<E, Object>（PRESENT 占位）
 *            →  Red-Black Tree（自平衡二叉搜索树）
 */
```

```java
public class TreeSet<E> extends AbstractSet<E> 
    implements NavigableSet<E>, Cloneable, java.io.Serializable {
    
    // ⭐ TreeSet 内部是 TreeMap
    private transient NavigableMap<E, Object> m;
    
    private static final Object PRESENT = new Object();
    
    // 构造方法
    public TreeSet() {
        this(new TreeMap<E, Object>());  // 默认使用 TreeMap
    }
    
    // 指定 Comparator
    public TreeSet(Comparator<? super E> comparator) {
        this(new TreeMap<>(comparator));
    }
    
    public boolean add(E e) {
        return m.put(e, PRESENT) == null;
    }
}
```

## 六、HashSet vs TreeSet 性能对比

| 对比维度 | HashSet | TreeSet |
|---------|---------|---------|
| **底层结构** | HashMap（哈希表） | TreeMap（红黑树） |
| **时间复杂度** | O(1) 平均 | O(log n) |
| **是否有序** | ❌ 无序 | ✅ 自然顺序/自定义顺序 |
| **null 支持** | ✅ 可存 1 个 null | ❌ 不能存 null（需 Comparator 允许） |
| **线程安全** | ❌ 不安全 | ❌ 不安全 |
| **Comparable** | 不需要 | 需要实现或提供 Comparator |
| **内存占用** | 较高（哈希表） | 较低（树节点） |
| **迭代器** | 弱一致性 | fail-fast 有序迭代 |

```java
// 性能测试示例
public class SetPerformance {
    public static void main(String[] args) {
        int n = 1_000_000;
        
        // HashSet
        HashSet<Integer> hashSet = new HashSet<>();
        long start = System.nanoTime();
        for (int i = 0; i < n; i++) hashSet.add(i);
        System.out.println("HashSet add: " + (System.nanoTime() - start) / 1_000_000 + "ms");
        
        // TreeSet
        TreeSet<Integer> treeSet = new TreeSet<>();
        start = System.nanoTime();
        for (int i = 0; i < n; i++) treeSet.add(i);
        System.out.println("TreeSet add: " + (System.nanoTime() - start) / 1_000_000 + "ms");
        
        // 结果：HashSet 通常比 TreeSet 快 3-5 倍
    }
}
```

## 七、什么时候用 TreeSet？

```java
/**
 * 需要有序场景 → TreeSet
 * 高性能查找 → HashSet
 */
```

| 场景 | 推荐 | 原因 |
|-----|------|-----|
| 存储唯一 ID，快速查找 | HashSet | O(1) 性能最优 |
| 需要遍历有序 | TreeSet | 支持 first/last/lower/higher |
| 范围查询（找 <5 或 >10 的元素） | TreeSet | subSet(headSet/tailSet) |
| 排行榜/排名系统 | TreeSet | O(log n) 插入，支持有序遍历 |
| 存储用户唯一访问时间 | LinkedHashMap | 保持插入/访问顺序 |

```java
// TreeSet 特有功能
TreeSet<Integer> set = new TreeSet<>();
set.add(5);
set.add(2);
set.add(8);
set.add(1);
set.add(9);

System.out.println(set.first());        // 1 最小
System.out.println(set.last());         // 9 最大
System.out.println(set.lower(5));       // 2 小于 5 的最大
System.out.println(set.higher(5));     // 8 大于 5 的最小
System.out.println(set.floor(5));      // 5 小于等于 5 的最大
System.out.println(set.ceiling(5));    // 5 大于等于 5 的最小

// 范围查询
System.out.println(set.subSet(2, 8));  // [2, 5) → [2, 5]
System.out.println(set.headSet(5));    // < 5
System.out.println(set.tailSet(5));     // >= 5
```

## 一句话总结

> **LinkedHashMap = HashMap + 双向链表，`accessOrder=true` 时每次访问/插入会把节点移到链表尾部，实现 LRU；HashSet 底层就是 `HashMap<K, Object>`（PRESENT 占位），TreeSet 底层是 `TreeMap<K, Object>`（红黑树），TreeSet 多 O(log n) 开销但支持有序操作。**
