## 基础流程

### 1. 什么是 Work-Stealing（工作窃取）？

Work-Stealing 是 ForkJoinPool 使用的**动态负载均衡算法**，核心思想是：**每个工作线程有自己的任务队列（Deque），空闲时从其他线程队列尾部"偷"任务执行。**

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Work-Stealing 算法原理                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  初始状态:                                                         │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │   │
│  │   Worker-1 (Deque)          Worker-2 (Deque)              │   │
│  │   [A] [B] [C] [D] [E]              [X] [Y] [Z]             │   │
│  │    ▲ head                      ▲ head                      │   │
│  │    └─ pop() 执行               └─ steal() 窃取            │   │
│  │                                                             │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  Worker-2 空闲时，从 Worker-1 的尾部偷取:                          │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                             │   │
│  │   Worker-1 (Deque)          Worker-2 (Deque)              │   │
│  │   [A] [B] [C] [D]             [X] [Y] [Z] [E]              │   │
│  │    ▲ head                     ▲ head                       │   │
│  │    └─ 尾部被窃取              └─ 尾部新增（E）             │   │
│  │                                                             │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  关键设计:                                                         │
│  ├─ 头部执行（push/pop）：保证 LIFO，最新任务先执行                  │
│  └─ 尾部窃取（steal）：保证 FIFO，避免与原线程冲突                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

**为什么从尾部窃取？**

```
┌────────────────────────────────────────────────────────────────┐
│              从尾部窃取的三个理由                                 │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  理由1：减少缓存污染                                            │
│  ┌──────────────────────────────────────────────────────────┐ │
│  │                                                          │ │
│  │  ForkJoin 的任务通常有"父子依赖"                         │ │
│  │  [父任务] [子任务A] [子任务B] [子任务C]                   │ │
│  │      ↑ head        ↑ 从尾部偷走 C                        │ │
│  │      │                                                    │ │
│  │      │ C 执行时可能 fork 新的 子任务                       │ │
│  │      │ → 加入到队列头部                                   │ │
│  │      │ → 本地线程优先执行（缓存友好）                      │ │
│  └──────────────────────────────────────────────────────────┘ │
│                                                                │
│  理由2：减少窃取冲突                                            │
│  ┌──────────────────────────────────────────────────────────┐ │
│  │                                                          │ │
│  │  如果从头部窃取：                                          │ │
│  │  Worker-1: pop() ←─┐  两个线程同时操作头部                │ │
│  │  Worker-2: steal()←┘  → 需要 CAS 保护 → 性能下降        │ │
│  │                                                          │ │
│  │  从尾部窃取：                                              │ │
│  │  Worker-1: pop(head)    线程A 独享头部                   │ │
│  │  Worker-2: steal(tail)  线程B 独享尾部                   │ │
│  │  → 无竞争！                                               │ │
│  └──────────────────────────────────────────────────────────┘ │
│                                                                │
│  理由3：实现任务亲和性                                          │
│  ┌──────────────────────────────────────────────────────────┐ │
│  │                                                          │ │
│  │  最新 fork 的子任务放在队列头部                           │ │
│  │  → 本地线程优先执行                                      │ │
│  │  → 最新任务往往与其他最新任务有关联                       │ │
│  │  → 提高缓存命中率                                        │ │
│  └──────────────────────────────────────────────────────────┘ │
└────────────────────────────────────────────────────────────────┘
```

### 2. Deque 双端队列工作原理

```java
public class ForkJoinDeque {

    public static void main(String[] args) {
        
        ForkJoinPool pool = ForkJoinPool.commonPool();
        
        // ForkJoinPool 使用内部类 ForkJoinTask<?>[] 作为 Deque
        // 但 JDK 源码展示逻辑：
        
        /*
         * 伪代码：每个 Worker 线程的 Deque 操作
         * 
         * // 头部操作（本地线程执行）
         * void push(Task t) {
         *     deque[++top] = t;  // 添加到头部
         * }
         * 
         * Task pop() {
         *     if (top > base) {
         *         return deque[top--];  // 从头部取
         *     }
         *     return null;
         * }
         * 
         * // 尾部操作（其他线程窃取）
         * Task steal() {
         *     if (base > 0) {
         *         return deque[--base];  // 从尾部窃取
         *     }
         *     return null;
         * }
         * 
         * 关键点：
         * - push/pop 只操作 head，由本地线程执行，无竞争
         * - steal 只操作 tail，由其他线程执行，无竞争
         * - top 和 base 是 volatile，保证可见性
         */
    }
}
```

**Deque 结构图示**：

```
┌────────────────────────────────────────────────────────────────┐
│              Deque 双端队列操作示意                              │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  Deque 数组 (可动态扩展):                                      │
│  ┌────┬────┬────┬────┬────┬────┬────┬────┬────┬────┐         │
│  │ T1 │ T2 │ T3 │ T4 │ T5 │ T6 │ T7 │ T8 │ T9 │T10 │         │
│  └────┴────┴────┴────┴────┴────┴────┴────┴────┴────┘         │
│     ▲                                                    ▲     │
│     │ base                                              │ top  │
│     │ (窃取端)                                          │(执行端)│
│                                                                │
│  ┌──────────────────────────────────────────────────────────┐ │
│  │ push(task):                                             │ │
│  │   deque[++top] = task;  // 从头部入队                    │ │
│  │   top++;                                                 │ │
│  └──────────────────────────────────────────────────────────┘ │
│                                                                │
│  ┌──────────────────────────────────────────────────────────┐ │
│  │ pop():                                                  │ │
│  │   if (top > base) {                                     │ │
│  │       return deque[top--];  // 从头部出队                │ │
│  │   }                                                     │ │
│  └──────────────────────────────────────────────────────────┘ │
│                                                                │
│  ┌──────────────────────────────────────────────────────────┐ │
│  │ steal() by other thread:                                │ │
│  │   if (base < top) {                                     │ │
│  │       return deque[--base];  // 从尾部出队              │ │
│  │   }                                                     │ │
│  └──────────────────────────────────────────────────────────┘ │
│                                                                │
│  线程安全：                                                    │
│  ├─ push/pop 由同一线程操作 → 无需同步                        │
│  └─ steal 由其他线程操作 → 操作不同端 → 无竞争                │
└────────────────────────────────────────────────────────────────┘
```

### 3. ForkJoinPool vs ThreadPoolExecutor

```java
import java.util.concurrent.*;

public class ForkJoinVsThreadPool {

    public static void main(String[] args) throws Exception {
        
        // ============ 场景：递归计算 ============
        int size = 10_000_000;
        
        // ============ ThreadPoolExecutor ============
        ExecutorService tpe = Executors.newFixedThreadPool(4);
        
        long start1 = System.nanoTime();
        Future<Long> future = tpe.submit(() -> sumRange(0, size));
        Long result1 = future.get();
        long time1 = System.nanoTime() - start1;
        
        System.out.println("ThreadPoolExecutor: " + result1 + " in " + time1/1_000_000 + "ms");
        tpe.shutdown();
        
        // ============ ForkJoinPool ============
        ForkJoinPool fjp = ForkJoinPool.commonPool();
        
        long start2 = System.nanoTime();
        ForkJoinTask<Long> task = new RecursiveSumTask(0, size);
        Long result2 = fjp.invoke(task);
        long time2 = System.nanoTime() - start2;
        
        System.out.println("ForkJoinPool:       " + result2 + " in " + time2/1_000_000 + "ms");
        
        // 典型结果：ForkJoin 更适合递归分治
    }

    // ThreadPoolExecutor 任务（简单提交）
    static long sumRange(long start, long end) {
        long sum = 0;
        for (long i = start; i < end; i++) {
            sum += i;
        }
        return sum;
    }

    // ForkJoin 任务（递归分治）
    static class RecursiveSumTask extends RecursiveTask<Long> {
        private final long start, end;
        private static final long THRESHOLD = 100_000;

        RecursiveSumTask(long start, long end) {
            this.start = start;
            this.end = end;
        }

        @Override
        protected Long compute() {
            if (end - start <= THRESHOLD) {
                return sumRange(start, end);
            }

            long mid = (start + end) / 2;
            RecursiveSumTask left = new RecursiveSumTask(start, mid);
            RecursiveSumTask right = new RecursiveSumTask(mid, end);

            left.fork();                    // 异步执行左半部分
            Long rightResult = right.compute();  // 同步执行右半部分
            Long leftResult = left.join();       // 等待左半部分结果

            return leftResult + rightResult;
        }
    }
}
```

**核心区别对照表**：

| 维度 | ForkJoinPool | ThreadPoolExecutor |
|------|-------------|-------------------|
| **队列模型** | 每个线程独立 Deque（无锁） | 共享 LinkedBlockingQueue |
| **任务窃取** | ✅ 自动 Work-Stealing | ❌ 无（任务只在一个队列） |
| **执行模型** | fork/join 父子任务协同 | 独立任务，无父子关系 |
| **线程饥饿** | 极低（窃取平衡负载） | 可能（任务分布不均） |
| **适用场景** | 递归分治、分解聚合 | 独立任务、无依赖 |
| **任务类型** | ForkJoinTask（RecursiveTask/Action） | Runnable/Callable |
| **invoke/execute** | 两者都有 | 只有 execute/submit |

```
┌────────────────────────────────────────────────────────────────┐
│              线程模型对比                                       │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  ThreadPoolExecutor (共享队列):                                │
│  ┌──────────────────────────────────────────────────────────┐ │
│  │                                                          │ │
│  │   [Task Queue - LinkedBlockingQueue]                    │ │
│  │              ↑ poll()                                    │ │
│  │   Thread-1 ──┘                                           │ │
│  │   Thread-2 ──┐ poll()                                    │ │
│  │   Thread-3 ──┘                                           │ │
│  │   问题：多线程竞争同一个队列                               │ │
│  └──────────────────────────────────────────────────────────┘ │
│                                                                │
│  ForkJoinPool (每个线程独立 Deque):                             │
│  ┌──────────────────────────────────────────────────────────┐ │
│  │                                                          │ │
│  │   Deque-1        Deque-2        Deque-3                 │ │
│  │   [A][B][C]       [X][Y]         [P][Q][R][S]           │ │
│  │     ↑push/pop       ↑push/pop     ↑push/pop            │ │
│  │     ↑steal(其他)    ↑steal(其他)  ↑steal(其他)         │ │
│  │                                                          │ │
│  │   优势：本地操作无锁，只有窃取时跨线程                     │ │
│  └──────────────────────────────────────────────────────────┘ │
└────────────────────────────────────────────────────────────────┘
```

### 4. 什么任务适合/不适合 ForkJoin？

```java
public class SuitableScenarios {

    // ============ ✅ 适合 ForkJoin 的场景 ============
    
    // 场景1：递归分治算法
    static class QuickSortTask extends RecursiveTask<int[]> {
        private final int[] arr;
        private final int low, high;

        @Override
        protected int[] compute() {
            if (low < high) {
                int pivot = partition(arr, low, high);
                
                RecursiveTask<int[]> left = new QuickSortTask(arr, low, pivot - 1);
                RecursiveTask<int[]> right = new QuickSortTask(arr, pivot + 1, high);
                
                left.fork();              // 并行执行
                int[] rightResult = right.compute();
                int[] leftResult = left.join();
                
                return merge(leftResult, rightResult);
            }
            return new int[]{arr[low]};
        }
        
        private int partition(int[] arr, int low, int high) { return low; }
        private int[] merge(int[] a, int[] b) { return a; }
    }

    // 场景2：MapReduce 风格聚合
    static class WordCountTask extends RecursiveTask<Map<String, Long>> {
        private final String[] texts;
        
        @Override
        protected Map<String, Long> compute() {
            if (texts.length < 1000) {
                return countWords(texts);
            }
            
            int mid = texts.length / 2;
            WordCountTask left = new WordCountTask(Arrays.copyOfRange(texts, 0, mid));
            WordCountTask right = new WordCountTask(Arrays.copyOfRange(texts, mid, texts.length));
            
            left.fork();
            Map<String, Long> rightResult = right.compute();
            Map<String, Long> leftResult = left.join();
            
            return mergeMaps(leftResult, rightResult);
        }
        
        private Map<String, Long> countWords(String[] texts) { return null; }
        private Map<String, Long> mergeMaps(Map<String, Long> a, Map<String, Long> b) { return null; }
    }

    // ============ ❌ 不适合 ForkJoin 的场景 ============
    
    // 反模式1：任务间有依赖
    static class DependentTasks {
        public void badExample() throws Exception {
            // ❌ 任务B 依赖任务A 的结果，不适合 fork
            ForkJoinTask<String> taskA = ForkJoinPool.commonPool()
                .submit(() -> fetchFromDB());
            
            // taskB 必须等待 taskA 完成
            String result = taskA.get();  // 阻塞在这里
            
            ForkJoinTask<Integer> taskB = ForkJoinPool.commonPool()
                .submit(() -> process(result));
        }
    }

    // 反模式2：I/O 密集且阻塞时间长
    static class BlockingIO {
        public void badExample() {
            // ❌ 每个任务阻塞 5 秒，窃取效率极低
            ForkJoinPool.commonPool().submit(() -> {
                Thread.sleep(5000);  // 阻塞！
            });
        }
    }

    // 反模式3：任务粒度过细
    static class FineGrained {
        public void badExample() {
            ForkJoinTask<Void> task = ForkJoinPool.commonPool()
                .submit(() -> {
                    for (int i = 0; i < 1_000_000; i++) {
                        // ❌ 每个任务只做一次加法
                        new FineGrainedTask().compute();
                    }
                });
        }
        
        static class FineGrainedTask extends RecursiveTask<Void> {
            @Override
            protected Void compute() {
                return null;  // 几乎无开销
            }
        }
    }
}
```

## 🔥 扩展追问 1：ForkJoinPool.commonPool() 的默认行为？

```java
public class CommonPoolBehavior {

    public static void main(String[] args) {
        
        // ============ commonPool 默认配置 ============
        ForkJoinPool common = ForkJoinPool.commonPool();
        
        System.out.println("Parallelism: " + common.getParallelism());
        // 默认 = max(1, Runtime.getRuntime().availableProcessors() - 1)
        // 8 核 CPU → 7 个线程
        
        System.out.println("Pool size: " + common.getPoolSize());
        System.out.println("Running threads: " + common.getRunningThreadCount());
        
        // ============ commonPool 的特点 ============
        /*
         * 1. 共享实例：整个 JVM 只有一个 commonPool
         * 2. 非守护线程：但没有未完成任务时不会阻止 JVM 退出
         * 3. 自动管理：不需要显式关闭
         * 4. ForkJoinTask 自动使用
         *    - Arrays.parallelSort()
         *    - Stream.parallel()
         *    - CompletableFuture 指定 executor 时
         */
        
        // ============ 自定义 ForkJoinPool ============
        ForkJoinPool customPool = new ForkJoinPool(
            4,                              // parallelism
            ForkJoinPool.defaultForkJoinWorkerThreadFactory, // factory
            null,                           // uncaughtExceptionHandler
            true                            // asyncMode (FIFO vs LIFO)
        );
        
        /*
         * asyncMode = true: 队列采用 FIFO，适合"消费者"模式
         * asyncMode = false: 队列采用 LIFO（默认），适合"生产者+消费者"
         */
        
        customPool.shutdown();
    }
}
```

> **A:** `ForkJoinPool.commonPool()` 特点：
> 1. **并行度**：`availableProcessors() - 1`（留一个给主线程）
> 2. **共享**：`Stream.parallel()` 和 `Arrays.parallelSort()` 自动使用
> 3. **不需要关闭**：没有未完成任务时允许 JVM 退出
> 4. **可替换**：通过 `System.setProperty("java.util.concurrent.ForkJoinPool.common.parallelism", "8")` 修改

## 🔥 扩展追问 2：RecursiveAction vs RecursiveTask vs CountedCompleter？

```java
public class TaskTypes {

    public static void main(String[] args) throws Exception {
        
        ForkJoinPool pool = ForkJoinPool.commonPool();
        
        // ============ RecursiveTask<T>：有返回值 ============
        RecursiveTask<Long> sumTask = new RecursiveTask<Long>() {
            @Override
            protected Long compute() {
                return 1L;
            }
        };
        Long result = pool.invoke(sumTask);
        
        // ============ RecursiveAction：无返回值 ============
        RecursiveAction action = new RecursiveAction() {
            @Override
            protected void compute() {
                System.out.println("Doing work");
            }
        };
        pool.invoke(action);
        
        // ============ CountedCompleter<T>：完成后触发回调 ============
        CountedCompleter<Void> completer = new CountedCompleter<Void>() {
            @Override
            public void compute() {
                System.out.println("Main task done");
                // 可以设置 onComplete 回调
            }
            
            @Override
            public void onCompletion(CountedCompleter<?> caller) {
                System.out.println("All sub-tasks completed");
            }
        };
        pool.invoke(completer);
    }

    // ============ 实战：使用 CountedCompleter 做树形聚合 ============
    static class TreeAggregator<E> extends CountedCompleter<E> {
        private final Node<E> node;
        private E result;
        private static final int THRESHOLD = 10;

        TreeAggregator(Node<E> root) {
            this(null, root);
        }

        TreeAggregator(CountedCompleter<?> parent, Node<E> node) {
            super(parent, node.children.size());
            this.node = node;
        }

        @Override
        public void compute() {
            if (node.children.isEmpty()) {
                result = node.value;
                tryComplete();  // 通知父任务
                return;
            }

            for (Node<E> child : node.children) {
                child.fork();  // fork 子任务
            }

            for (Node<E> child : node.children) {
                child.join();  // 等待完成
            }
            
            result = aggregate(node.children);
            tryComplete();
        }

        private E aggregate(List<Node<E>> children) { return null; }

        static class Node<E> extends CountedCompleter<E> {
            private final E value;
            private final List<Node<E>> children;
            
            Node(E value, List<Node<E>> children) {
                this.value = value;
                this.children = children;
            }
            
            @Override
            public void compute() { }
            
            public void fork() { ForkJoinPool.commonPool().execute(this); }
            public E join() { return join(); }
        }
    }
}
```

> **A:** 三种 ForkJoinTask 选择：
> 1. **RecursiveTask T**：需要返回值，典型递归分治
> 2. **RecursiveAction**：无返回值，只需执行副作用
> 3. **CountedCompleter**：完成后需要触发回调，适合树形聚合

## 🔥 扩展追问 3：ForkJoinPool 的线程创建机制？

```java
public class ThreadCreation {

    public static void main(String[] args) {
        
        // ============ ForkJoinWorkerThreadFactory ============
        ForkJoinPool pool = new ForkJoinPool(
            4,
            new ForkJoinPool.ForkJoinWorkerThreadFactory() {
                private int count = 0;
                
                @Override
                public ForkJoinWorkerThread newThread(ForkJoinPool pool) {
                    ForkJoinWorkerThread t = new ForkJoinWorkerThread(pool) {
                        @Override
                        protected void onTermination(Throwable exception) {
                            System.out.println("Thread " + getName() + " terminated");
                            super.onTermination(exception);
                        }
                    };
                    t.setName("Custom-Fork-" + count++);
                    return t;
                }
            },
            null,
            false
        );

        pool.shutdown();
    }
}
```

> **A:** ForkJoin 线程的特点：
> 1. **继承 `ForkJoinWorkerThread`**：比普通 Thread 多一个 `pool` 字段
> 2. **每个线程有独立 Deque**：工作窃取的队列基础
> 3. **支持 `onTermination` 回调**：线程结束前可做清理
> 4. **不可复用**：每个任务对应一个 worker 线程

## 🔥 扩展追问 4：ForkJoin 的取消和异常处理？

```java
public class CancelAndException {

    public static void main(String[] args) throws Exception {
        
        ForkJoinPool pool = ForkJoinPool.commonPool();
        
        // ============ 取消任务 ============
        RecursiveTask<Long> task = new RecursiveTask<Long>() {
            @Override
            protected Long compute() {
                for (int i = 0; i < 1_000_000; i++) {
                    if (Thread.currentThread().isInterrupted()) {
                        System.out.println("Task cancelled");
                        return null;
                    }
                    // 检查取消状态
                    if (isCancelled()) {
                        return null;
                    }
                }
                return 1L;
            }
        };
        
        pool.execute(task);
        Thread.sleep(10);
        task.cancel(true);  // 中断线程
        
        // ============ ForkJoinTask 的特殊异常处理 ============
        RecursiveTask<Integer> taskWithException = new RecursiveTask<Integer>() {
            @Override
            protected Integer compute() {
                throw new RuntimeException("Task failed!");
            }
        };
        
        pool.invoke(taskWithException);
        
        // 获取异常（会在 join() 时抛出）
        try {
            taskWithException.join();  // 抛出 RuntimeException
        } catch (Exception e) {
            System.out.println("Caught: " + e.getCause().getMessage());
        }
        
        // 或者静默获取
        if (taskWithException.isCompletedAbnormally()) {
            Throwable ex = taskWithException.getException();
            System.out.println("Exception: " + ex.getMessage());
        }
    }
}
```

> **A:** ForkJoin 异常处理：
> 1. **异常不会立即抛出**：在 `join()` 时才抛出
> 2. **getException()**：静默获取异常，不抛
> 3. **isCompletedAbnormally()**：检查是否异常完成
> 4. **取消**：`cancel(true)` 中断线程

```java
         *    - true (FIFO): 纯消费者，事件处理

```

> **A:** ForkJoinPool 调优要点：
> 1. **Steal Count**：监控窃取频率，高说明负载均衡好
> 2. **Parallelism**：I/O 密集可增加，并行度 > CPU 核心数
> 3. **asyncMode**：LIFO 适合分治，FIFO 适合纯消费
> 4. **队列大小**：`getQueuedTaskCount()` 监控任务积压

## 一句话总结

> **ForkJoinPool 的 Work-Stealing = 每个线程独立 Deque + 本地 push/pop 无锁 + 从尾部窃取无竞争，完美适配递归分治任务；ThreadPoolExecutor 用共享队列，多线程竞争是性能瓶颈。**

---

## 面试加分话术

> "ForkJoinPool 的 Work-Stealing 是我研究过最优雅的并发算法之一。关键设计是**从尾部窃取**：因为 ForkJoin 的任务有父子依赖关系，最近 fork 的子任务在队列头部，本地线程优先执行最新任务能提高缓存命中率；而其他线程从尾部偷走的是较早的父子任务，减少缓存污染。更妙的是，本地线程只操作头部（push/pop），窃取线程只操作尾部，两者**永远不会竞争同一个位置**，连 CAS 都不需要，所以叫'无锁队列'。我们在用 Stream.parallel() 处理归并排序时，后台就是用的 commonPool，实测 1000 万元素排序，8 核机器上比单线程快 6.5 倍，几乎线性扩展。"
