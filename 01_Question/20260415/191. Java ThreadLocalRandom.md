## 基础流程

### 1. ThreadLocalRandom 是什么？

`ThreadLocalRandom` 是 JDK 7 引入的**线程本地随机数生成器**，解决高并发下 `Random` 的 CAS 竞争问题。

```
┌─────────────────────────────────────────────────────────────────────┐
│                    三种随机数生成器对比                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  Random:                                                          │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  全局共享 seed + CAS 竞争                                     │   │
│  │                                                             │   │
│  │  Thread-1 ──┐                                                │   │
│  │  Thread-2 ──┼──► CAS(nextseed) ──► Random instance          │   │
│  │  Thread-3 ──┘                                                │   │
│  │                      ↑ 竞争点：AtomicLong                    │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  ThreadLocalRandom:                                               │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  每个线程独立种子，无竞争                                     │   │
│  │                                                             │   │
│  │  Thread-1 ──► ThreadLocalRandom-1 (seed-1)                 │   │
```

**基本用法**：

```java
import java.util.concurrent.ThreadLocalRandom;

public class ThreadLocalRandomBasics {

    public static void main(String[] args) {
        
        // ============ ThreadLocalRandom 的使用 ============
        // 获取当前线程的随机数生成器
        ThreadLocalRandom random = ThreadLocalRandom.current();
        
        // 生成随机数（与 Random API 相同）
        int i = random.nextInt();           // 任意 int
        int i100 = random.nextInt(100);      // [0, 100) 的 int
        long l = random.nextLong();          // 任意 long
        long l1000 = random.nextLong(1000); // [0, 1000) 的 long
        double d = random.nextDouble();      // [0.0, 1.0) 的 double
        double d10 = random.nextDouble(10); // [0.0, 10.0) 的 double
        boolean b = random.nextBoolean();    // true 或 false
        
        // 生成高斯分布随机数
        double gaussian = random.nextGaussian();  // μ=0, σ=1
        
        System.out.println("int: " + i + ", int(100): " + i100);
        System.out.println("double: " + d + ", gaussian: " + gaussian);
        
        // ============ 典型场景：多线程生成随机 ID ============
        ExecutorService executor = Executors.newFixedThreadPool(10);
        
        IntStream.range(0, 100).forEach(i2 -> {
            executor.submit(() -> {
                int randomId = ThreadLocalRandom.current().nextInt(1_000_000);
                System.out.println("Thread-" + Thread.currentThread().getName() 
                    + " generated: " + randomId);
            });
        });
        
        executor.shutdown();
    }
}
```

### 2. ThreadLocalRandom vs Random vs SplittableRandom

```java
import java.util.Random;
import java.util.SplittableRandom;
import java.util.concurrent.ThreadLocalRandom;
import java.util.concurrent.atomic.AtomicLong;

public class RandomComparison {

    public static void main(String[] args) throws Exception {
        
        int iterations = 10_000_000;
        int threads = 8;
        
        // ============ 性能对比 ============
        
        // 1. Random（共享，CAS 竞争）
        Random sharedRandom = new Random();
        long start1 = System.nanoTime();
        AtomicLong sum1 = new AtomicLong();
        CountDownLatch latch1 = new CountDownLatch(threads);
        
        for (int t = 0; t < threads; t++) {
            new Thread(() -> {
                for (int i = 0; i < iterations; i++) {
                    sum1.addAndGet(sharedRandom.nextInt());
                }
                latch1.countDown();
            }).start();
        }
        latch1.await();
        long time1 = System.nanoTime() - start1;
        
        // 2. ThreadLocalRandom（线程本地，无竞争）
        long start2 = System.nanoTime();
        AtomicLong sum2 = new AtomicLong();
        CountDownLatch latch2 = new CountDownLatch(threads);
        
        for (int t = 0; t < threads; t++) {
            new Thread(() -> {
                ThreadLocalRandom random = ThreadLocalRandom.current();
                for (int i = 0; i < iterations; i++) {
                    sum2.addAndGet(random.nextInt());
                }
                latch2.countDown();
            }).start();
        }
        latch2.await();
        long time2 = System.nanoTime() - start2;
        
        // 3. SplittableRandom（可分裂）
        long start3 = System.nanoTime();
        AtomicLong sum3 = new AtomicLong();
        CountDownLatch latch3 = new CountDownLatch(threads);
        
        for (int t = 0; t < threads; t++) {
            new Thread(() -> {
                SplittableRandom random = new SplittableRandom();
                for (int i = 0; i < iterations; i++) {
                    sum3.addAndGet(random.nextInt());
                }
                latch3.countDown();
            }).start();
        }
        latch3.await();
        long time3 = System.nanoTime() - start3;
        
        System.out.println("Random (shared+CAS):         " + time1/1_000_000 + " ms");
        System.out.println("ThreadLocalRandom:            " + time2/1_000_000 + " ms");
        System.out.println("SplittableRandom (new):      " + time3/1_000_000 + " ms");
        System.out.println();
        System.out.println("ThreadLocalRandom 加速比:     " + 
            String.format("%.1fx", (double) time1 / time2));
        
        // 典型结果：
        // Random (shared+CAS):         ~2500 ms
        // ThreadLocalRandom:            ~180 ms  (快 14x)
        // SplittableRandom (new):       ~150 ms
    }
}
```

**详细对比表**：

| 维度 | Random | ThreadLocalRandom | SplittableRandom |
|------|--------|-------------------|------------------|
| **线程安全** | 线程安全（CAS） | 线程安全（ThreadLocal） | 线程安全（可分裂） |
| **竞争点** | 全局 AtomicLong | 无竞争 | 无竞争 |
| **种子共享** | 共享 | 每个线程独立 | 可分裂继承 |
| **性能** | 高并发差 | 高并发最优 | ForkJoin 最优 |
| **适用场景** | 单线程/低并发 | 多线程独立随机 | ForkJoin 分治 |
| **状态继承** | ❌ | ❌ | ✅ 分裂时继承 |
| **API** | nextInt/bound | nextInt/bound | nextInt/bound |
| **并行度支持** | 差 | 好 | 极好 |

### 3. ThreadLocalRandom 如何避免 CAS 竞争？

```java
// ThreadLocalRandom 的内部实现原理

public class TLRImplementation {

    public static void main(String[] args) {
        
        // 关键源码解析：
        
        /*
         * // ThreadLocalRandom.java (JDK 内部实现)
         * 
         * public class ThreadLocalRandom {
         *     
         *     // ThreadLocal 机制：每个线程持有自己的 ThreadLocalRandom 实例
         *     private static final ThreadLocal<ThreadLocalRandom> local =
         *         new ThreadLocal<ThreadLocalRandom>() {
         *             protected ThreadLocalRandom initialValue() {
         *                 return new ThreadLocalRandom();
         *             }
         *         };
         *     
         *     // 每个实例有自己的种子，不共享！
         *     private long threadLocalRandomSeed;
         *     
         *     // 当前线程的 ThreadLocalRandom 实例
         *     public static ThreadLocalRandom current() {
         *         if (U.getObject(t = Thread.currentThread(), 
         *                         UNSAFE.objectFieldOffset(t, "threadLocalRandomSeed")) 
         *             == 0)
         *             localRandom.init();  // 延迟初始化
         *         return localRandom;
         *     }
         *     
         *     // 生成随机数（无需 CAS！）
         *     final int nextInt(int bound) {
         *         long seed = threadLocalRandomSeed;
         *         long halfs = seed ^ (seed >>> 33);
         *         long hashed = halfs * 0x62A9D9D8C8A5E8F1L;
         *         threadLocalRandomSeed = hashed;
         *         
         *         int r = (int) ((hashed >>> 33) % bound);  // LCG 变种
         *         return r >= 0 ? r : r + bound;  // 保证正数
         *     }
         * }
         * 
         * 关键点：
         * 1. 每个线程有自己的 ThreadLocalRandom 实例
         * 2. 种子存储在线程对象上（threadLocalRandomSeed）
         * 3. 生成随机数时，只需写入线程自己的字段
         * 4. 没有任何共享状态！无 CAS 竞争！
         */
    }
}
```

### 4. 高并发下 Random 的性能瓶颈

```java
import java.util.Random;
import java.util.concurrent.CountDownLatch;
import java.util.concurrent.atomic.AtomicLong;

public class RandomBottleneck {

    public static void main(String[] args) throws Exception {
        
        // ============ Random 内部实现 ============
        /*
         * public class Random {
         *     
         *     private final AtomicLong seed;  // ← 全局唯一的种子！
         *     
         *     protected int next(int bits) {
         *         long oldSeed, newSeed;
         *         AtomicLong seed = this.seed;
         *         do {
         *             oldSeed = seed.get();           // 1. 读取
         *             newSeed = oldSeed * multiplier + addend;  // 2. 计算
         *         } while (!seed.compareAndSet(oldSeed, newSeed)); // 3. CAS
         *         //     ↑ 竞争点！其他线程可能抢先修改
         *         return (int) (newSeed >>> (48 - bits));
         *     }
         * }
         * 
         * 问题：
         * 1. 所有线程共享同一个 AtomicLong seed
         * 2. CAS 失败 → 自旋重试 → 浪费 CPU
         * 3. 线程越多 → CAS 失败率越高 → 性能急剧下降
         */
        
        // ============ 实验：CAS 失败率随线程数增长 ============
        System.out.println("=== Random CAS 竞争实验 ===");
        
        for (int threadCount : new int[]{1, 2, 4, 8, 16, 32}) {
            Random random = new Random();
            int iterations = 1_000_000;
            
            CountDownLatch latch = new CountDownLatch(threadCount);
            AtomicLong totalOps = new AtomicLong();
            long start = System.nanoTime();
            
            for (int t = 0; t < threadCount; t++) {
                new Thread(() -> {
                    for (int i = 0; i < iterations; i++) {
                        random.nextInt();
                        totalOps.incrementAndGet();
                    }
                    latch.countDown();
                }).start();
            }
            
            latch.await();
            long duration = System.nanoTime() - start;
            double opsPerMs = totalOps.get() / (duration / 1_000_000.0);
            
            System.out.printf("线程数=%2d, 吞吐量=%.0f ops/ms%n", 
                threadCount, opsPerMs);
        }
        
        // 典型结果：
        // 线程数= 1, 吞吐量=150000 ops/ms
        // 线程数= 2, 吞吐量=120000 ops/ms
        // 线程数= 4, 吞吐量= 80000 ops/ms
        // 线程数= 8, 吞吐量= 45000 ops/ms
        // 线程数=16, 吞吐量= 20000 ops/ms
        // 线程数=32, 吞吐量= 10000 ops/ms
        // 
        // 16 线程时，吞吐量只有单线程的 7%！
    }
}
```

```java
import java.util.SplittableRandom;
import java.util.concurrent.ForkJoinPool;
import java.util.concurrent.RecursiveTask;

public class SplittableRandomUsage {

    // ============ 核心特性：可分裂 ============
    public static void main(String[] args) {
        
        SplittableRandom root = new SplittableRandom(12345L);
        
        // 分裂：创建两个独立的随机数生成器
        SplittableRandom child1 = root.split();
        SplittableRandom child2 = root.split();
        
        // child1 和 child2 有独立的种子
        // 但种子与 root 有数学关系，保证随机性
        System.out.println("Root: " + root.nextInt(1000));
        System.out.println("Child1: " + child1.nextInt(1000));
        System.out.println("Child2: " + child2.nextInt(1000));
        
        // 分裂后可以继续分裂（树形结构）
        SplittableRandom child1_1 = child1.split();
        SplittableRandom child1_2 = child1.split();
        System.out.println("Child1-1: " + child1_1.nextInt(1000));
    }

    // ============ 场景1：ForkJoin 分治任务 ============
    static class MonteCarloPiTask extends RecursiveTask<Double> {
        private final SplittableRandom random;
        private final int samples;
        private static final int THRESHOLD = 10_000;
        
        MonteCarloPiTask(SplittableRandom random, int samples) {
            this.random = random;
            this.samples = samples;
        }
        
        @Override
        protected Double compute() {
            if (samples < THRESHOLD) {
                return monteCarloPi(samples);
            }
            
            // 分裂：创建两个子任务的随机数生成器
            SplittableRandom r1 = random.split();
            SplittableRandom r2 = random.split();
            
            MonteCarloPiTask task1 = new MonteCarloPiTask(r1, samples / 2);
            MonteCarloPiTask task2 = new MonteCarloPiTask(r2, samples / 2);
            
            task1.fork();
            double result2 = task2.compute();
            double result1 = task1.join();
            
            return (result1 + result2) / 2;
        }
        
        private double monteCarloPi(int samples) {
            int count = 0;
            for (int i = 0; i < samples; i++) {
                double x = random.nextDouble();
                double y = random.nextDouble();
                if (x * x + y * y <= 1) count++;
            }
            return 4.0 * count / samples;
        }
    }

    // ============ 场景2：并行数据生成 ============
    static void parallelDataGeneration() {
        int totalRecords = 1_000_000;
        int batchSize = 100_000;
        int batches = totalRecords / batchSize;

        // 并行生成数据，每批次用独立的随机数生成器
        ForkJoinPool pool = ForkJoinPool.commonPool();
        
        Double result = pool.invoke(new RecursiveTask<Double>() {
            @Override
            protected Double compute() {
                if (batches <= 1) {
                    // 最后一个批次
                    return generateBatch(root, batchSize);
                }
                
                int mid = batches / 2;
                SplittableRandom r1 = root.split();
                SplittableRandom r2 = root.split();
                
                RecursiveTask<Double> task1 = new RecursiveTask<Double>() {
                    @Override
                    protected Double compute() {
                        return generateBatch(r1, batchSize * mid);
                    }
                };
                
                RecursiveTask<Double> task2 = new RecursiveTask<Double>() {
                    @Override
                    protected Double compute() {
                        return generateBatch(r2, batchSize * (batches - mid));
                    }
                };
                
                task1.fork();
                double result2 = task2.compute();
                double result1 = task1.join();
                
                return result1 + result2;
            }
        });
    }
    
    private static Double generateBatch(SplittableRandom random, int count) {
        return 0.0; // 实际生成数据
    }
```

## 🔥 扩展追问 1：ThreadLocalRandom 的边界问题（nextInt(100)）？

```java
public class BoundProblem {

    public static void main(String[] args) {
        
        // ============ 经典问题：nextInt(bound) 的分布不均匀 ============
        // 这是一个 JDK 遗留 bug，直到 JDK 8 才修复
        
        /*
         * JDK 7 的问题：
         * 
         * nextInt(bound) {
         *     int r = next(31);  // [0, 2^31)
         *     int m = bound - 1;
         *     if ((bound & m) == 0) {
         *         r = (int) ((bound * (long) r) >>> 31);
         *     } else {
         *         // 处理非 2^n 情况
         *         int limit = (int) (NEED_CACHED_VALUE / bound) * bound - 1;
         *         while (r > limit) {  // ← 可能死循环！
         *             r = next(31);
         *         }
         *         r %= bound;
         *     }
         * }
         */
        
        // JDK 8+ 的改进：Horspool 算法
        /*
         * JDK 8 的改进实现：
         * 
         * int nextInt(int bound) {
         *     long seed = getAndAddSeed(addend);  // CAS
         *     long oldseed = seed;
         *     long mask = this.mask;  // 2^48 - 1
         *     
         *     // 生成随机数
         *     for (;;) {
         *         long r = (oldseed * multiplier + addend) & mask;
         *         long d = bound - 1;
         *         if ((bound & d) == 0) {
         *             return (int) ((bound * r) >>> 31);
         *         }
         *         int u = (int) (r >>> 33) + oldseed;
         *         int i = u % bound;
         *         if (u - i + d >= 0) {
         *             return i;
         *         }
         *     }
         * }
         */
        
        // 验证分布
        int bound = 7;
        int[] counts = new int[bound];
        ThreadLocalRandom random = ThreadLocalRandom.current();
        
        for (int i = 0; i < 1_000_000; i++) {
            counts[random.nextInt(bound)]++;
        }
        
        System.out.println("分布检验（期望每个约 142857）:");
        for (int i = 0; i < bound; i++) {
            System.out.printf("  [%d]: %d%n", i, counts[i]);
        }
    }
}
```

> **A:** 
> - JDK 7 的 `nextInt(bound)` 有**分布不均匀**问题，尤其在 bound 不是 2^n 时
> - JDK 8+ 使用改进的 **Horspool 算法**，保证均匀分布
> - 建议始终使用 JDK 8+ 的 `ThreadLocalRandom`

## 🔥 扩展追问 2：ThreadLocalRandom 的序列化问题？

```java
public class SerializationIssue {

    public static void main(String[] args) throws Exception {
        
        // ============ ThreadLocalRandom 不能序列化 ============
        ThreadLocalRandom random = ThreadLocalRandom.current();
        
        // ⚠️ ThreadLocalRandom 实现了 java.io.Serializable
        // 但序列化后再反序列化会出问题！
        
        ByteArrayOutputStream baos = new ByteArrayOutputStream();
        ObjectOutputStream oos = new ObjectOutputStream(baos);
        oos.writeObject(random);
        
        ByteArrayInputStream bais = new ByteArrayInputStream(baos.toByteArray());
        ObjectInputStream ois = new ObjectInputStream(bais);
        ThreadLocalRandom restored = (ThreadLocalRandom) ois.readObject();
        
        // restored 是新的 ThreadLocalRandom 实例
        // 与原实例完全独立！
        
        System.out.println("Original: " + System.identityHashCode(random));
        System.out.println("Restored: " + System.identityHashCode(restored));
        
        // 如果需要跨 JVM 传输随机种子，使用 SplittableRandom 或自己实现
    }
}
```

> **A:** 
> - `ThreadLocalRandom` **不建议序列化**
> - 序列化后反序列化会丢失与原线程的关联
> - 如果需要跨 JVM 共享随机状态，传递**种子值**而非 `ThreadLocalRandom` 实例

## 🔥 扩展追问 3：SecureRandom vs ThreadLocalRandom？

```java
public class SecureVsThreadLocal {

    public static void main(String[] args) {
        
        // ============ SecureRandom：密码学安全 ============
        /*
         * SecureRandom 使用：
         * 1. /dev/urandom (Linux) 或 CryptGenRandom (Windows)
         * 2. 种子来自操作系统熵源
         * 3. 密码学安全，不可预测
         * 
         * 性能：极慢（约 1000 倍慢于 ThreadLocalRandom）
         * 用途：密钥生成、Token 生成、安全随机数
         */
        
        SecureRandom secure = new SecureRandom();
        
        // 生成安全的随机字节
        byte[] bytes = new byte[32];
        secure.nextBytes(bytes);
        
        // 生成安全的 UUID
        String secureToken = new UUID(
            secure.nextLong(), secure.nextLong()
        ).toString();
        
        System.out.println("Secure token: " + secureToken);
        
        // ============ 选择建议 ============
        /*
         * ┌──────────────────────────────────────────────────────┐
         * │ ThreadLocalRandom: 普通业务随机、测试数据、负载均衡    │
         * ├──────────────────────────────────────────────────────┤
         * │ SecureRandom: 密码学用途、密钥、Token、安全随机数     │
         * └──────────────────────────────────────────────────────┘
         */
    }
}
```

> **A:** 
> - **ThreadLocalRandom**：性能优先，用于普通业务（负载均衡、抽样、测试数据）
> - **SecureRandom**：安全优先，用于密码学（密钥、Token、盐值）
> - 性能差距巨大：`SecureRandom` 比 `ThreadLocalRandom` 慢 **1000-10000 倍**

## 🔥 扩展追问 4：Random 的 seed 和 LCG 算法？

```java
public class LCGAlgorithm {

    public static void main(String[] args) {
        
        // ============ Random 的线性同余生成器（LCG）===========
        /*
         * Random 使用经典的 LCG 算法：
         * 
         * seed(n+1) = (seed(n) * multiplier + addend) & mask
         * 
         * JDK 实现的参数：
         * - multiplier = 0x5DEECE66DL (固定常数)
         * - addend = 0xBL (固定常数)
         * - mask = (1L << 48) - 1 (48 位掩码)
         * 
         * 特点：
         * - 周期：2^48（早期版本）
         * - 高位有规律，不适合高精度统计
         */
        
        // ============ 手动实现 LCG ============
        class SimpleLCG {
            private long seed;
            private static final long MULTIPLIER = 0x5DEECE66DL;
            private static final long ADDEND = 0xBL;
            private static final long MASK = (1L << 48) - 1;
            
            SimpleLCG(long seed) {
                this.seed = seed;
            }
            
            int nextInt() {
                seed = (seed * MULTIPLIER + ADDEND) & MASK;
                return (int) (seed >>> 16);
            }
        }
        
        SimpleLCG lcg = new SimpleLCG(12345);
        System.out.println("手动 LCG 序列:");
        for (int i = 0; i < 10; i++) {
            System.out.print(lcg.nextInt() + " ");
        }
    }
}
```

> **A:** 
> - Random 使用 **LCG（线性同余生成器）**，周期 2^48
> - `ThreadLocalRandom` 和 `SplittableRandom` 使用改进的 **XorShift** 或其他算法
> - LCG 高位有规律，但 `ThreadLocalRandom` 对普通业务足够

## 🔥 扩展追问 5：并行流中的随机数使用？

```java
public class ParallelStreamRandom {

    public static void main(String[] args) {
        
        // ============ 错误做法：并行流中共享 Random ============
        Random shared = new Random();
        
        // ⚠️ 危险！多线程共享 Random，有 CAS 竞争
        // List<Integer> bad = IntStream.range(0, 100_000)
        //     .parallel()
        //     .map(i -> shared.nextInt())
        //     .boxed()
        //     .toList();
        
        // ============ 正确做法1：ThreadLocalRandom ============
        List<Integer> good1 = IntStream.range(0, 100_000)
            .parallel()
            .map(i -> ThreadLocalRandom.current().nextInt())
            .boxed()
            .toList();
        
        // ============ 正确做法2：在 forEach 中使用 ============
        List<Integer> good2 = new CopyOnWriteArrayList<>();
        IntStream.range(0, 100_000)
            .parallel()
            .forEach(i -> {
                good2.add(ThreadLocalRandom.current().nextInt());
            });
        
        // ============ 正确做法3：SplittableRandom（ForkJoin）===========
        // Arrays.parallelSort 内部使用 ForkJoin + SplittableRandom
        // 但 API 不暴露 SplittableRandom
        
        System.out.println("并行流随机数生成完成");
    }
}
```

> **A:** 
> - **禁止**在并行流中共享 `Random` 实例
> - 使用 `ThreadLocalRandom.current()` 自动获取线程本地实例
> - ForkJoin 任务使用 `SplittableRandom`，通过分裂继承独立种子
