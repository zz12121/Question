## 基础流程

### 1. 什么是结构化并发？

结构化并发是 **JDK 21 预览特性**（JEP 437→453），核心思想来自 **Alice K. Smith** 的论文和 Go/Rust 的实践：**子任务的生命周期必须嵌套在父任务内，父任务不结束，子任务不能独立存在。**

```
┌─────────────────────────────────────────────────────────────────────┐
│                    结构化并发 vs 传统并发                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  传统并发（线程逃逸）:                                              │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │   main() ──► executor.submit(task1) ──┐                   │ │
│  │              executor.submit(task2) ──┼──► [task1] (线程)  │ │
│  │              executor.submit(task3) ──┼──► [task2] (线程)  │ │
│  │                                         └──► [task3] (线程)  │ │
│  │   问题:                                                    │ │
│  │   - main() 返回后，线程仍在运行！                            │ │
│  │   - 异常无法聚合传播                                        │ │
│  │   - 无法追踪任务关系                                        │ │
│  │   - 线程泄漏风险                                            │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  结构化并发（生命周期嵌套）:                                         │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                             │ │
│  │   try (scope = new StructuredTaskScope<Object>()) {       │ │
│  │       Future<String> f1 = scope.fork(task1);              │ │
│  │       Future<String> f2 = scope.fork(task2);              │ │
│  │       scope.join();                                         │ │
│  │   } // ← 所有子任务在此刻必须完成/终止                        │ │
│  │                                                             │ │
│  │   优势:                                                    │ │
│  │   ✓ 父任务等待所有子任务完成                                  │ │
│  │   ✓ 异常自动聚合                                            │ │
│  │   ✓ 取消自动传播                                            │ │
│  │   ✓ 无线程泄漏                                              │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

**核心理念**：

```java
public class StructuredConcurrencyConcept {

    public static void main(String[] throws InterruptedException) {
        
        // 结构化并发三原则：
        
        // 1. 任务必须在作用域内启动
        // try (StructuredTaskScope<String> scope = ...) {
        //     scope.fork(task);  // ✓ 作用域内
        // }
        
        // 2. 作用域关闭前必须等待所有子任务
        // scope.join();  // ✓ 阻塞等待

    }
}
```

### 2. StructuredTaskScope 工作原理

```java
import java.util.concurrent.StructuredTaskScope;

public class StructuredTaskScopeDemo {

    // ============ 基础用法 ============
    public static String fetchUserData(Long userId) 
        throws InterruptedException {
        
        try (var scope = new StructuredTaskScope<String>()) {
            
            // fork: 启动子任务（类似 executor.submit）
            Future<String> userFuture = scope.fork(() -> fetchUser(userId));
            Future<String> orderFuture = scope.fork(() -> fetchOrders(userId));
            Future<String> profileFuture = scope.fork(() -> fetchProfile(userId));
            
            // join: 等待所有子任务完成
            scope.join();
            
            // 结果收集（在 join 之后）
            String user = userFuture.resultNow();
            String orders = orderFuture.resultNow();
            String profile = profileFuture.resultNow();
            
            return user + " | " + orders + " | " + profile;
        }
        // ← 作用域结束：自动等待所有任务 + 取消未完成任务
    }

    // ============ 作用域内异常处理 ============
    public static void exceptionHandling() 
        throws InterruptedException {
        
        try (var scope = new StructuredTaskScope<String>()) {
            
            scope.fork(() -> { throw new RuntimeException("task1 failed"); });
            scope.fork(() -> "task2 success");
            scope.fork(() -> "task3 success");
            
            scope.join();  // 会抛出 ExecutionException("task1 failed")
            
        } catch (ExecutionException e) {
            System.out.println("聚合异常: " + e.getCause().getMessage());
            // 输出: 聚合异常: task1 failed
        }
    }

    // ============ 并行超时控制 ============
    public static void withTimeout() 
        throws InterruptedException, TimeoutException {
        
        try (var scope = new StructuredTaskScope<String>()) {
            
            scope.fork(() -> slowOperation());
            scope.fork(() -> fastOperation());
            
            // 带超时的 join
            scope.joinUntil(Instant.now().plusSeconds(5));
            
        } catch (TimeoutException e) {
            System.out.println("超时，部分任务被取消");
        }
    }
}
```

**StructuredTaskScope 内部状态机**：

```
┌────────────────────────────────────────────────────────────────┐
│              StructuredTaskScope 生命周期                        │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  new StructuredTaskScope()                                    │
│       │                                                       │
│       ▼                                                       │
│  ┌─────────┐                                                  │
│  │  OPEN   │ ← 初始状态，可 fork 新任务                        │
│  └────┬────┘                                                  │
│       │                                                       │
│  join() ─┤                                                    │
│       │                                                       │
│       ├─► 所有任务完成 ──────────────────────► ┌────────┐     │
│       │                                         │ OPEN   │     │

│       │                                                       │
```

#### 3.1 任务取消传播

```java
public class CancellationPropagation {

    // ============ 问题：传统方式无法自动取消 ============
    static class TraditionalProblem {
        
        public static String fetchAll(Long userId) throws Exception {
            ExecutorService executor = Executors.newVirtualThreadPerTaskExecutor();
            
            try {
                // 启动三个任务
                Future<String> f1 = executor.submit(() -> fetchUser(userId));
                Future<String> f2 = executor.submit(() -> fetchOrders(userId));  // 慢
                Future<String> f3 = executor.submit(() -> fetchProfile(userId));
                
                // 如果 f1 失败，f2/f3 仍在运行！需要手动取消
                String r1 = f1.get();
                
                // ❌ f2/f3 仍在运行，造成资源浪费
                
                return r1;
            } finally {
                executor.shutdownNow();  // 手动关闭
            }
        }
    }

    // ============ 结构化并发：自动取消 ============
    static class StructuredSolution {
        
        public static String fetchAll(Long userId) 
            throws InterruptedException {
            
            try (var scope = new StructuredTaskScope.ShutdownOnFailure<String>()) {
                
                Future<String> f1 = scope.fork(() -> fetchUser(userId));
                Future<String> f2 = scope.fork(() -> fetchOrders(userId));
                Future<String> f3 = scope.fork(() -> fetchProfile(userId));
                
                scope.join();  // 等待所有任务
                
                // 如果任一任务失败，自动取消其他任务
                // scope.shutdown() 被隐式调用
                
                return STR."\{f1.resultNow()} | \{f2.resultNow()} | \{f3.resultNow()}";
            }
        }
    }

    private static String fetchUser(Long id) { return "User-" + id; }
    private static String fetchOrders(Long id) { 
        try { Thread.sleep(1000); } catch (Exception e) {}
        return "Orders"; 
    }
    private static String fetchProfile(Long id) { return "Profile"; }
}
```

```java
public class ExceptionAggregation {

    public static void main(String[] args) throws Throwable {
        
        try (var scope = new StructuredTaskScope<String>()) {
            
            scope.fork(() -> { throw new RuntimeException("任务1失败"); });
            scope.fork(() -> { throw new RuntimeException("任务2失败"); });
            scope.fork(() -> "任务3成功");
            
            scope.join();
            
        } catch (ExecutionException e) {
            // e.getCause() = 第一个异常
            // e.getSuppressed() = 其他异常
            System.out.println("主异常: " + e.getCause().getMessage());
            for (Throwable t : e.getSuppressed()) {
                System.out.println("  聚合异常: " + t.getMessage());
            }
            // 输出:
            // 主异常: 任务1失败
            //   聚合异常: 任务2失败
        }
    }
}
```

#### 3.3 防止线程泄漏

```java
public class ThreadLeakPrevention {

    // ============ 问题：线程泄漏 ============
    static class LeakProblem {
        
        public static void main(String[] args) {
            ExecutorService executor = Executors.newFixedThreadPool(4);
            
            // 业务逻辑
            try {
                // 抛出异常时，executor 可能未被关闭
                throw new RuntimeException("业务异常");  // ⚠️ executor 泄漏！
            } finally {
                // 需要记得 finally
                executor.shutdown();
            }
        }
    }

    // ============ 结构化并发：自动清理 ============
    static class LeakSolution {
        
        public static void main(String[] args) {
            
            // try-with-resources 自动关闭
            try (var scope = new StructuredTaskScope<String>()) {
                
                scope.fork(() -> task1());
                scope.fork(() -> task2());
                
                scope.join();  // 自动等待
                
                // 即使抛异常，scope.close() 仍会执行
                // → 等待未完成任务 → 取消 → 关闭
                throw new RuntimeException("业务异常");
                
            }  // ← 这里自动清理，无泄漏
        }
        
        private static String task1() { return "t1"; }
        private static String task2() { return "t2"; }
    }
}
```

### 4. ShutdownOnFailure vs ShutdownOnSuccess

```java
public class ShutdownPolicies {

    // ============ ShutdownOnFailure ============
    // 策略：任一任务失败 → 取消其他任务
    static class ShutdownOnFailureExample {
        
        public static void main(String[] args) throws Throwable {
            
            try (var scope = new StructuredTaskScope.ShutdownOnFailure<String>()) {
                
                Future<String> f1 = scope.fork(() -> fetchUser(1L));   // 成功
                Future<String> f2 = scope.fork(() -> fetchUser(2L));   // 失败
                Future<String> f3 = scope.fork(() -> fetchUser(3L));   // 可能成功/失败
                
                scope.join();  // 等待所有任务
                
                // 如果任一失败，抛出 ExecutionException
                // 其他任务被自动取消
                scope.throwIfFailed();  // 显式抛出异常
                
                // 安全获取结果
                String r1 = f1.resultNow();
                String r2 = f2.resultNow();  // 如果失败可能抛出
                String r3 = f3.resultNow();
                
                System.out.println(r1 + ", " + r2 + ", " + r3);
            }
        }
    }

    // ============ ShutdownOnSuccess ============
    // 策略：任一任务成功 → 立即返回，取消其他任务
    static class ShutdownOnSuccessExample {
        
        public static String findFirst(Long id1, Long id2) 
            throws InterruptedException {
        
            try (var scope = new StructuredTaskScope.ShutdownOnSuccess<String>()) {
                
                scope.fork(() -> queryPrimary(id1));    // 优先
                scope.fork(() -> querySecondary(id2));  // 备选
                
                scope.join();  // 等待任一成功
                
                // 返回第一个成功的结果
                return scope.result();
                
                // ⚠️ 如果所有任务都失败，抛出 ExecutionException
            }
        }
        
        private static String queryPrimary(Long id) {
            if (Math.random() > 0.3) throw new RuntimeException("主库故障");
            return "Primary-" + id;
        }
        
        private static String querySecondary(Long id) {
            return "Secondary-" + id;
        }
    }

    // ============ 两者对比 ============
    static void policyComparison() {
        /*
         * ┌──────────────────────────────────────────────────────────┐
         * │                    策略对比                               │
         * ├──────────────────┬───────────────────────────────────────┤
         * │ ShutdownOnFailure│ ShutdownOnSuccess                     │
         * ├──────────────────┼───────────────────────────────────────┤
         * │ 触发条件         │ 任一任务失败                            │
         * │ 取消时机         │ 失败时立即取消其他                       │
         * │ 返回值           │ 所有任务结果（需自行处理失败）            │
         * │ 典型场景         │ "全部成功才继续"                        │
         * ├──────────────────┼───────────────────────────────────────┤
         * │ ShutdownOnSuccess│ 首个任务成功                            │
         * │ 触发条件         │ 任一任务成功                            │
         * │ 取消时机         │ 成功后立即取消其他                       │
         * │ 返回值           │ 第一个成功的结果                        │
         * │ 典型场景         │ "竞速：谁先到谁赢"                      │
         * └──────────────────┴───────────────────────────────────────┘
         */
    }
}
```

```
┌────────────────────────────────────────────────────────────────┐
│            ShutdownOnFailure vs ShutdownOnSuccess                │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  ShutdownOnFailure（全部成功才继续）:                            │
│  ┌──────────────────────────────────────────────────────────┐ │
│  │                                                          │ │
│  │  [Task1] ──► ✓                                          │ │
```

```java
public class StructuredAndVirtualThreads {

    public static void main(String[] args) throws Throwable {
        
        // 结构化并发必须运行在虚拟线程上！
        // 如果在平台线程上使用，会抛出 IllegalStateException
        
        // ============ 自动使用虚拟线程 ============
        try (var scope = new StructuredTaskScope<String>()) {
            
            // fork() 内部自动创建虚拟线程
            scope.fork(() -> {
                System.out.println("Running on: " + Thread.currentThread());
                // 输出: Running on: VirtualThread[#XX]/...
                return "done";
            });
            
            scope.join();
        }
        
        // ============ 继承性 ============
        // 父任务的上下文自动传递给子任务
        // try (var scope = StructuredTaskScope.where(Thread.ofVirtual())) { ... }
        // 指定子任务运行的载体
    }

    // ============ 手动指定调度器（可选）===========
    static void customScheduler() throws Throwable {
        
        ExecutorService scheduler = Executors.newFixedThreadPool(2);
        
        try (var scope = new StructuredTaskScope<String>()
                .forkingThread(scheduler)) {  // 指定 fork 使用的调度器
            
            scope.fork(() -> "result");
            scope.join();
            
        } finally {
            scheduler.shutdown();
        }
    }
}
```

> **A:** 
> - `StructuredTaskScope` **必须与虚拟线程配合**使用
> - `fork()` 内部使用虚拟线程执行任务
> - 虚拟线程的轻量级使得可以创建大量子任务
> - 两者结合：**10 万级并发任务 + 生命周期管理**

## 🔥 扩展追问 2：如何自定义 StructuredTaskScope？

```java
public class CustomScope {

    // ============ 自定义作用域：限制并发数 ============
    static class BoundedScope<T> extends StructuredTaskScope<T> {
        
        private final Semaphore semaphore;
        private final CountDownLatch latch = new CountDownLatch(1);
        
        public BoundedScope(int maxConcurrency) {
            this.semaphore = new Semaphore(maxConcurrency);
        }
        
        @Override
        protected void handleComplete(Future<T> future) {
            semaphore.release();  // 任务完成后释放信号量
            if (latch.getCount() > 0 && isShutdown()) {
                latch.countDown();
            }
        }
        
        public void awaitShutdown() throws InterruptedException {
            latch.await();
        }
    }

    // ============ 自定义作用域：收集结果 ============
    static class CollectingScope<T> extends StructuredTaskScope<T> {
        
        private final List<T> results = Collections.synchronizedList(new ArrayList<>());
        private volatile Throwable exception;
        
        @Override
        protected void handleComplete(Future<T> future) {
            if (future.isDone() && !future.isCancelled()) {
                try {
                    results.add(future.resultNow());
                } catch (Throwable t) {
                    exception = t;
                }
            }
        }
        
        public List<T> results() {
            return List.copyOf(results);
        }
    }

    // ============ 使用自定义作用域 ============
    public static void main(String[] args) throws Throwable {
        
        try (var scope = new BoundedScope<String>(3)) {
            
            // 最多 3 个并发
            for (int i = 0; i < 10; i++) {
                int taskId = i;
                scope.fork(() -> {
                    System.out.println("Task " + taskId + " started");
                    Thread.sleep(100);
                    return "Task " + taskId;
                });
            }
            
            scope.join();
        }
    }
}
```

> **A:** 自定义 `StructuredTaskScope`：
> 1. **继承 `StructuredTaskScope<T>`**
> 2. **覆盖 `handleComplete(Future<T>)`**：任务完成时回调
> 3. **实现自己的结果收集策略**：如 BoundedScope 限制并发数
> 4. **使用 `subtask()` 方法**：获取所有子任务列表

## 🔥 扩展追问 3：与 Project Reactor / Mutiny 的对比？

```java
public class ComparisonWithReactive {

    // ============ 结构化并发 vs 响应式编程 ============
    static void comparison() {
        /*
         * 结构化并发：                         
         *   - 命令式代码风格，易读易写
         *   - 自然异常处理（try-catch）
         *   - 线程本地变量继承
         *   - 适合 I/O 密集型
         *                                             
         * 响应式编程（Reactor/Mutiny）：        
         *   - 声明式流水线
         *   - 强大的背压支持
         *   - 操作符组合丰富
         *   - 适合流式数据处理
         */
    }

    // ============ 混合使用 ============
    static Mono<String> structuredInReactive() throws Throwable {
        
        try (var scope = new StructuredTaskScope.ShutdownOnSuccess<String>()) {
            
            scope.fork(() -> Mono.just("async1").block());
            scope.fork(() -> Mono.just("async2").block());
            
            scope.join();
            
            // 将结果转为 Mono
            return Mono.just(scope.result());
        }
    }
}
```

> **A:** 
> - **结构化并发**：更适合简单的并行任务聚合，代码直观
> - **响应式编程**：适合复杂的数据流处理、背压、错误恢复
> - 两者可以**互补**：在响应式流水线内部用结构化并发聚合多个调用
> - **趋势**：Java 官方正在提供**轻量级替代**复杂响应式框架的方案

## 🔥 扩展追问 4：异常处理的最佳实践？

```java
public class ExceptionHandlingBestPractices {

    // ============ 方法1：聚合异常 ============
    static void aggregateExceptions() throws Throwable {
        
        try (var scope = new StructuredTaskScope<String>()) {
            
            scope.fork(() -> task1());
            scope.fork(() -> task2());
            scope.fork(() -> task3());
            
            scope.join();
            
        } catch (ExecutionException e) {
            // 展开聚合异常
            Throwable cause = e.getCause();
            System.out.println("失败原因: " + cause.getMessage());
            
            // 遍历所有异常
            System.out.println("所有异常:");
            System.out.println("  - " + cause.getMessage());
            for (Throwable t : e.getSuppressed()) {
                System.out.println("  - " + t.getMessage());
            }
        }
    }

    // ============ 方法2：自定义异常处理 ============
    static class CustomScope extends StructuredTaskScope<String> {
        
        private final List<Throwable> errors = new CopyOnWriteArrayList<>();
        
        @Override
        protected void handleComplete(Future<String> future) {
            if (future.isCompletedExceptionally()) {
                future.exceptionNow();  // 触发异常
                errors.add(future.exceptionNow());
            }
        }
        
        public List<Throwable> getErrors() {
            return List.copyOf(errors);
        }
    }

    // ============ 方法3：部分容忍失败 ============
    static Map<String, Object> partialFailure() throws Throwable {
        
        try (var scope = new StructuredTaskScope<String>()) {
            
            Future<String> user = scope.fork(() -> fetchUser(1L));
            Future<String> order = scope.fork(() -> fetchOrders(2L));
            Future<String> extra = scope.fork(() -> fetchExtra(3L));
            
            scope.join();
            
            Map<String, Object> results = new HashMap<>();
            results.put("user", user.resultNow());
            results.put("order", order.resultNow());
            
            // extra 失败不影响整体
            if (extra.state() == Future.State.SUCCESS) {
                results.put("extra", extra.resultNow());
            } else {
                results.put("extra", "N/A");
            }
            
            return results;
        }
    }
}
```

> **A:** 异常处理策略：
> 1. **聚合模式**（`ShutdownOnFailure`）：任一失败，全部抛异常
> 2. **容忍模式**：捕获异常，检查 `Future.state()`
> 3. **自定义 Scope**：收集所有异常后再处理
> 4. **关键**：`join()` 后再检查结果，避免 `IllegalStateException`

## 一句话总结

> **结构化并发 = 用 try-with-resources 管理任务生命周期，自动聚合异常 + 取消传播 + 防止泄漏；`ShutdownOnFailure` 适合"全成功才继续"，`ShutdownOnSuccess` 适合"竞速取首个成功"。**

## 面试加分话术

> "结构化并发是我认为 Java 并发编程的范式转变。之前的 `CompletableFuture.allOf()` 有个致命缺陷：异常会丢失，只有第一个失败的异常能传播出来。我们团队之前用 `allOf` 并行查多个数据源，结果接口偶发 500，最后查了一周才发现是某个冷门数据源超时被静默吞掉了。换成 `StructuredTaskScope.ShutdownOnFailure`，所有异常都被聚合到 `ExecutionException.getSuppressed()` 里，根本不会再漏。另外在微服务聚合场景，`ShutdownOnSuccess` 的'谁先到谁赢'策略非常适合主备切换，我们把原来 10 个微服务的串行降级改成了并行竞速，P99 从 800ms 降到 120ms。"
