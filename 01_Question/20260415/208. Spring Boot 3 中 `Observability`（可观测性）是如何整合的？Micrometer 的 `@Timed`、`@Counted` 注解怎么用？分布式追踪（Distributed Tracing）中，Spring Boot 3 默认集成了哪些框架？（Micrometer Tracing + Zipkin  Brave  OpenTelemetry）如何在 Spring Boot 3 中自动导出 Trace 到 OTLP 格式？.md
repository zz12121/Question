## 2. @Timed 和 @Counted 注解

### 2.1 @Timed 注解

```java
// @Timed：自动记录方法执行时间和频率
@Service
@Timed(value = "user.service", description = "User Service Operations")
public class UserService {

    // 1. 方法级别的 @Timed
    @Timed(value = "user.getById", 
           description = "Time to fetch user by ID",
           percentiles = {0.5, 0.9, 0.95, 0.99})  // 百分位数
    public User getUserById(Long id) {
        return userRepository.findById(id);
    }

    // 2. 带条件：只有成功才记录
    @Timed(value = "user.update",
           description = "User update timing",
           when = "success")  // only_record_failures / always
    public User updateUser(User user) {
        return userRepository.save(user);
    }

    // 3. 长任务 Timer（异步任务监控）
    @Timed(value = "export.task",
           description = "Export task",
           longTask = true)  // 单独的长任务 Timer
    public void exportData() {
        // 耗时很长的导出任务
    }

    // 4. 自定义 Tags
    @Timed(value = "order.create",
           description = "Create order",
           extraTags = {
               @Tag(name = "region", value = "us-east"),
               @Tag(name = "environment", value = "production")
           })
    public Order createOrder(OrderRequest request) {
        return orderService.process(request);
    }
}
```

```java
// Controller 层使用
@RestController
@RequestMapping("/api/users")
@Timed(value = "api.users", 
       description = "User API Timing",
       percentiles = 0.5,  // 支持单值写法
       histogram = true)  // 开启直方图（Prometheus bucket）
public class UserController {

    @GetMapping("/{id}")
    @Timed(value = "api.users.getById", 
           description = "Get user by ID",
           percentiles = {0.5, 0.90, 0.95, 0.99})
    public User getUser(@PathVariable Long id) {
        return userService.getUserById(id);
    }

    @PostMapping
    @Timed(value = "api.users.create",
           description = "Create user")
    @Counted(value = "api.users.create.count",
             description = "Create user count",
             recordFailuresOnly = true)  // 只记录失败次数
    public User createUser(@RequestBody UserRequest request) {
        return userService.createUser(request);
    }
}
```

### 2.2 @Counted 注解

```java
// @Counted：记录方法调用次数
@Service
public class PaymentService {

    // 1. 基本计数
    @Counted(value = "payment.total",
             description = "Total payment requests")
    public PaymentResult processPayment(PaymentRequest request) {
        return paymentGateway.process(request);
    }

    // 2. 只记录失败次数
    @Counted(value = "payment.failures",
             description = "Payment failures",
             recordFailuresOnly = true)
    public PaymentResult processPayment(PaymentRequest request) {
        try {
            return paymentGateway.process(request);
        } catch (PaymentException e) {
            throw e;  // 异常被记录
        }
    }

    // 3. 带自定义失败判定
    @Counted(value = "payment.declined",
             description = "Payment declined",
             extraTags = {"type", "declined"},
             failureMessage = "Payment was declined")
    public PaymentResult processPayment(PaymentRequest request) {
        // 自定义失败逻辑
        return null;
    }

    // 4. 自定义 Tags
    @Counted(value = "api.request",
             description = "API request count",
             extraTags = {
                 @Tag(name = "method", value = "POST"),
                 @Tag(name = "endpoint", value = "/api/payment")
             })
    public PaymentResult handleRequest(PaymentRequest request) {
        return processPayment(request);
    }
}
```

### 2.3 手动埋点（不使用注解时）

```java
@Service
public class ManualMetricsService {

    private final MeterRegistry meterRegistry;
    private final Tracer tracer;

    public ManualMetricsService(MeterRegistry meterRegistry, Tracer tracer) {
        this.meterRegistry = meterRegistry;
        this.tracer = tracer;
    }

    public void processOrder(Order order) {
        // 1. 创建 Span（分布式追踪）
        Span span = tracer.nextSpan().name("processOrder");
        try (Tracer.SpanInScope inScope = tracer.withSpanInScope(span)) {
            // 添加 Tags
            span.tag("order.id", order.getId().toString());
            span.tag("order.amount", order.getAmount().toString());
            
            // 2. 手动 Timer
            Timer.Sample sample = Timer.start(meterRegistry);
            
            // 3. Counter
            Counter.builder("order.processed")
                   .tag("status", "success")
                   .register(meterRegistry)
                   .increment();
            
            // 业务逻辑
            doProcess(order);
            
            // 记录耗时
            sample.stop(Timer.builder("order.process.time")
                             .publishPercentiles(0.5, 0.9, 0.99)
                             .register(meterRegistry));
            
        } catch (Exception e) {
            span.error(e);  // 记录异常
            span.tag("error", "true");
            throw e;
        } finally {
            span.end();  // 结束 Span
        }
    }
}
```

### 3.2 Spring Boot 3 追踪配置

```yaml
# application.yml
management:
  tracing:
    # 采样策略
    sampling:
      probability: 1.0    # 采样率 100%（生产建议 0.1-0.2）
      # 固定采样器
      # probability: 0.1
      # 速率限制采样器（每秒最多 N 个 trace）
      # rate: 100
      
    # Baggage（随 Trace 传递的上下文）
    baggage:
      # 需要在 HTTP Header 中传递的 baggage key
      propagation:
        correlation:
          fields:
            - user-id
            - request-id
      # 本地 baggage（不传播）
      local:
        fields:
          - tenant-id

    # Span 配置
    span:
      # 最大长度限制
      max-length: 1024

    # 启用 MDC 集成（自动将 traceId/s panId 写入日志）
    enabled: true
```

## 🔥 扩展追问 1：Micrometer 的 `Observation` vs `Tracer` 有什么区别？应该用哪个？

> A: **`Observation` 是更高层的抽象，同时管理 Metrics 和 Traces；`Tracer` 只管理 Traces。Spring Boot 3 推荐优先用 `Observation`。**

```
对比：

┌────────────────────────────────────────────────────────────────────────────┐
│  Tracer（低级 API）                                                        │
│  ┌──────────────────────────────────────────────────────────────────────┐ │
│  │  只负责：Span 管理                                                     │ │
│  │  Span span = tracer.nextSpan().name("op").start();                  │ │
│  │  span.tag("key", "value");                                           │ │
│  │  span.end();                                                         │ │
│  │                                                                      │ │
│  │  问题：不产生任何 Metrics 指标                                        │ │
│  └──────────────────────────────────────────────────────────────────────┘ │
└────────────────────────────────────────────────────────────────────────────┘

┌────────────────────────────────────────────────────────────────────────────┐
│  Observation（高级 API，Spring Boot 3 推荐）                               │
│  ┌──────────────────────────────────────────────────────────────────────┐ │
│  │                                                                      │ │
│  │  同时产生：                                                          │ │
│  │    • Traces (Span) ←── 自动记录到 OTLP/Zipkin                       │ │
│  │    • Metrics (Timer/Counter) ←── 自动记录到 Prometheus/OTLP         │ │
│  │                                                                      │ │
│  │  @Observed 注解用法：                                                │ │
│  │                                                                      │ │
│  │  @Service                                                          │ │
│  │  public class OrderService {                                        │ │
│  │                                                                      │ │
│  │    @Observed(name = "order.create",                                │ │
│  │               contextualName = "order-service.create",              │ │
│  │               lowCardinalityKeyValues = {                           │ │
│  │                 @Cardinality(tag = "type", value = "vip")          │ │
│  │               })                                                    │ │
│  │    public Order createOrder(CreateOrderRequest request) {           │ │
│  │      // 业务逻辑                                                     │ │
│  │      return order;                                                  │ │
│  │    }                                                                │ │
│  │  }                                                                  │ │
│  │                                                                      │ │
│  │  自动产生：                                                          │ │
│  │    • trace.order.create —— span                                     │ │
│  │    • order.create —— timer (含 P50/P90/P99)                         │ │
│  │    • order.create —— counter                                        │ │
│  └──────────────────────────────────────────────────────────────────────┘ │
└────────────────────────────────────────────────────────────────────────────┘
```

## 🔥 扩展追问 2：如何在 Spring Boot 3 中让 RestTemplate / WebClient 自动传播 Trace？

> A: **Spring Boot 3 通过 `ObservationRegistry` 自动完成上下文传播，只需引入相应依赖并使用自动配置即可。**

```java
// 1. RestTemplate（需要 spring-boot-starter-web + micrometer-tracing-bridge-brave）
@Configuration
public class RestTemplateConfig {

    @Bean
    public RestTemplate restTemplate(RestTemplateBuilder builder) {
        return builder
                // 自动添加 traceId 到 HTTP Header
                .additionalInterceptors(
                    new TracingClientHttpRequestInterceptor(
                        tracer(),
                        new BravePropagationFactory()
                    )
                )
                .build();
    }

    @Bean
    public Tracer tracer(ObservationRegistry observationRegistry) {
        return Tracing.newClient()
                .build()
                .getTracer();
    }
}
```

```java
// 2. WebClient（响应式，推荐）
@Configuration
public class WebClientConfig {

    @Bean
    public WebClient webClient(WebClient.Builder builder,
                                ExchangeFilterFunction traceFilter) {
        return builder
                .filter(traceFilter)  // 自动 Trace 过滤
                .baseUrl("http://order-service")
                .build();
    }

    @Bean
    public ExchangeFilterFunction traceFilter(Tracer tracer) {
        return (request, next) -> {
            Span span = tracer.nextSpan().name(
                request.method() + " " + request.url());
            try (Tracer.SpanInScope scope = tracer.withSpanInScope(span)) {
                span.tag("http.method", request.method().name());
                span.tag("http.url", request.url().toString());
                return next.exchange(request);
            } finally {
                span.end();
            }
        };
    }
}
```

## 🔥 扩展追问 3：Spring Boot 3 如何将日志和 Trace 关联？（MDC + TraceId）

```xml
        [%thread]
        [%X{traceId:-NONE}]
        [%X{spanId:-NONE}]
        %-5level
        %logger{36}
        - %msg%n
    </pattern>
</encoder>
```

## 🔥 扩展追问 4：@Timed 的 `histogram` 和 `percentiles` 有什么区别？如何配置？

> A: **`percentiles` 是预计算的百分位数值，`histogram=true` 生成 Prometheus bucket，适合 Prometheus 告警和 Grafana 看板。**

```
两种方式对比：

┌────────────────────────────────────────────────────────────────────────────┐
│  percentiles（预计算）                                                      │
│                                                                              │
│  @Timed(percentiles = {0.5, 0.9, 0.95, 0.99})                              │
│                                                                              │
│  输出到 Prometheus：                                                        │
│  ┌──────────────────────────────────────────────────────────────────────┐ │
│  │ http_server_requests_seconds{...,quantile="0.5"} 0.023               │ │
│  │ http_server_requests_seconds{...,quantile="0.9"} 0.056               │ │
│  │ http_server_requests_seconds{...,quantile="0.95"} 0.089              │ │
│  │ http_server_requests_seconds{...,quantile="0.99"} 0.150               │ │
│  └──────────────────────────────────────────────────────────────────────┘ │
│  优点：直接是最终数值，Prometheus 存储更少                                  │
│  缺点：无法动态计算其他百分位                                                │
├────────────────────────────────────────────────────────────────────────────┤
│  histogram=true（桶统计）                                                    │
│                                                                              │
│  @Timed(histogram = true,                                                │
│         percentiles = {0.5, 0.9, 0.99})                                   │
│                                                                              │
│  输出到 Prometheus：                                                        │
│  ┌──────────────────────────────────────────────────────────────────────┐ │
│  │ http_server_requests_seconds_bucket{...,le="0.01"} 100             │ │
│  │ http_server_requests_seconds_bucket{...,le="0.05"} 500             │ │
│  │ http_server_requests_seconds_bucket{...,le="0.1"} 1000            │ │
│  │ http_server_requests_seconds_bucket{...,le="0.5"} 5000            │ │
│  │ http_server_requests_seconds_bucket{...,le="1.0"} 9000            │ │
│  │ http_server_requests_seconds_bucket{...,le="+Inf"} 10000          │ │
│  │ http_server_requests_seconds_sum{...} 500.5                          │ │
│  │ http_server_requests_seconds_count{...} 10000                        │ │
│  └──────────────────────────────────────────────────────────────────────┘ │
│  优点：可用 histogram_quantile() 任意百分位、Grafana 动态配置               │
│  缺点：bucket 数量多，存储开销稍大                                           │
└────────────────────────────────────────────────────────────────────────────┘
```

## 🔥 扩展追问 5：Spring Boot 3 如何实现无代码侵入的全局链路追踪？（AOP 自动拦截）

> A: **Spring Boot 3 通过 `WebMvcObservationFilter` 和 `WebFluxObservationFilter` 自动拦截所有 HTTP 请求，无需额外代码。**

```java
// 自动拦截的链路（无需手动添加任何代码）

@RestController  ← WebMvcObservationFilter 自动拦截
public class UserController {
    
    @GetMapping("/{id}")
    @Timed  ← 自动记录 Timer + Span
    public User getUser(@PathVariable Long id) {
        
        userService.getUserById(id);  ← Feign 自动传播
        
        return user;
    }
}

@Service  ← Spring AOP 拦截
public class UserService {
    
    @Observed(name = "user.fetch")
    @Cacheable("users")
    public User getUserById(Long id) {
        
        userRepository.findById(id);  ← JPA 自动拦截
        redisTemplate.opsForValue().get("user:" + id);  ← Redis 自动拦截
        
        return user;
    }
}
```

## 6. 面试加分话术

> "我们项目从 Spring Boot 2 升级到 3 之后，可观测性架构简单了特别多。之前 Sleuth + Micrometer 两套配置，版本不对齐就各种问题。现在统一走 Micrometer，所有埋点只加注解：Controller 层用 `@Timed(percentiles = {0.5, 0.9, 0.99}, histogram = true)` 记录接口耗时，Service 层用 `@Observed` 同时出 Trace 和 Metrics，对外暴露 `/actuator/prometheus` 和 `/actuator/otlp/traces`，Grafana 一套看板同时看 QPS 和 P99 耗时。生产配置了 `probability: 0.1` 采样率，1% 数据量足够分析问题了。日志 MDC 自动带了 traceId，`grep traceId xxx.log` 就能拉出完整调用链。升级时踩过一个坑：`spring-cloud-sleuth` 在 Boot 3 里已经废弃了，必须换成 `micrometer-tracing-bridge-brave`，而且之前用 `brave.Tracer` 的代码要改用 `io.micrometer.tracing.Tracer`，包名完全不一样。"
