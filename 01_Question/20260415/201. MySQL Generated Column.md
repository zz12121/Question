## 1. 基础流程

### 什么是 Generated Column？

Generated Column 是 MySQL 5.7+ 引入的特性，列的值由表达式**自动计算得出**，而不是存储用户插入的值。

```
┌─────────────────────────────────────────────────────────────┐
│              Generated Column 与普通列对比                      │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  【普通列】                                                 │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  CREATE TABLE orders (                              │   │
│  │      id INT PRIMARY KEY,                            │   │
│  │      price DECIMAL(10,2),    ← 用户插入的值        │   │
│  │      quantity INT             ← 用户插入的值        │   │
│  │  );                                                │   │
│  │                                                       │   │
│  │  INSERT INTO orders VALUES (1, 100.00, 3);         │   │
│  │                                                       │   │
│  │  total_price = 100.00 × 3 = 300.00                 │   │
│  │  这个值需要应用层计算                                │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  【Generated Column】                                       │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  CREATE TABLE orders (                              │   │
│  │      id INT PRIMARY KEY,                            │   │
│  │      price DECIMAL(10,2),                            │   │
│  │      quantity INT,                                    │   │
│  │      total_price DECIMAL(10,2) AS (price * quantity)  │   │
│  │           ← 表达式自动计算，存储在列中               │   │
│  │  );                                                │   │
│  │                                                       │   │
│  │  INSERT INTO orders (id, price, quantity) VALUES (1, 100, 3);  │   │
│  │       ↑                                              │   │
│  │       不需要插入 total_price，数据库自动计算          │   │
│  │                                                       │   │
│  │  total_price = 100.00 × 3 = 300.00 (自动得出)       │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### VIRTUAL 和 STORED 的区别

```
┌─────────────────────────────────────────────────────────────┐
│              VIRTUAL vs STORED 生成列对比                     │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────────┬───────────────┬───────────────┐       │
│  │     特性        │    VIRTUAL    │    STORED     │       │
│  ├─────────────────┼───────────────┼───────────────┤       │
│  │  存储方式       │  不存储        │  物理存储     │       │
│  ├─────────────────┼───────────────┼───────────────┤       │

│  ├─────────────────┼───────────────┼───────────────┤       │
│  │  读取性能       │  每次读取计算  │  直接读取     │       │
│  ├─────────────────┼───────────────┼───────────────┤       │
│  │  写入性能       │  立即返回      │  需要计算+写入│       │
│  ├─────────────────┼───────────────┼───────────────┤       │
│  │  索引支持       │  ✅ 可以       │  ✅ 可以      │       │
│  ├─────────────────┼───────────────┼───────────────┤       │
│  │  外键引用       │  ❌ 不支持     │  ✅ 支持      │       │
│  ├─────────────────┼───────────────┼───────────────┤       │
│  │  默认值结合     │  ❌ 不支持     │  ✅ 支持      │       │
│  └─────────────────┴───────────────┴───────────────┘       │
│                                                             │
│  【VIRTUAL 列工作原理】                                      │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                                                       │   │
│  │  表结构:                                              │   │
│  │  ┌─────────────────────────────────────────────┐   │   │
│  │  │  id | price | quantity | total_price (V)   │   │   │
│  │  │  ├───┼──────┼──────────┼─────────────────┤   │   │
│  │  │  1 | 100 | 3         │ [计算得出]       │   │   │
│  │  └─────────────────────────────────────────────┘   │   │
│  │                                                       │   │
│  │  INSERT:                                             │   │
│  │    不计算，直接返回                                  │   │
│  │                                                       │   │
│  │  SELECT:                                             │   │
│  │    读取时动态计算 (on-the-fly)                      │   │
│  │    SELECT total_price FROM orders WHERE id=1;       │   │
│  │         ↓                                            │   │
│  │    计算 price × quantity                            │   │
│  │                                                       │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  【STORED 列工作原理】                                      │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                                                       │   │
│  │  表结构:                                              │   │
│  │  ┌─────────────────────────────────────────────┐   │   │
│  │  │  id | price | quantity | total_price (S)   │   │   │
│  │  │  ├───┼──────┼──────────┼─────────────────┤   │   │
│  │  │  1 | 100 | 3         | 300.00           │   │   │
│  │  └─────────────────────────────────────────────┘   │   │
│  │              ↑                                      ↑   │
│  │              └── 物理存储 ── 实际写入磁盘            │   │
│  │                                                       │   │
│  │  INSERT:                                             │   │
│  │    1. 计算 price × quantity                        │   │
│  │    2. 将结果写入 total_price 列                     │   │
│  │                                                       │   │
│  │  UPDATE:                                             │   │
│  │    1. 重新计算                                       │   │
│  │    2. 更新存储的值                                   │   │
│  │                                                       │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 生成列上建索引

```sql
-- ========== 1. VIRTUAL 列 + 索引（最常用） ==========
CREATE TABLE orders (
    id INT PRIMARY KEY,
    price DECIMAL(10,2),
    quantity INT,
    
    -- VIRTUAL 生成列 + 索引
    total_price DECIMAL(10,2) AS (price * quantity) VIRTUAL,
    
    INDEX idx_total_price (total_price)
);

-- 查询可以利用索引
EXPLAIN SELECT * FROM orders WHERE total_price > 500;
-- | type | possible_keys    | key              |
-- | ref  | idx_total_price | idx_total_price  |  ← 使用索引

-- ========== 2. STORED 列 + 索引 ==========
CREATE TABLE products (
    id INT PRIMARY KEY,
    name VARCHAR(100),
    category VARCHAR(50),
    created_at DATETIME,
    
    -- STORED 生成列 + 索引
    category_year VARCHAR(100) AS (CONCAT(category, '-', YEAR(created_at))) STORED,
    
    INDEX idx_category_year (category_year)
);

-- ========== 3. 基于 JSON 的生成列 + 索引 ==========
CREATE TABLE user_profiles (
    id INT PRIMARY KEY,
    data JSON,
    
    -- 从 JSON 提取字段作为生成列
    nickname VARCHAR(100) AS (data->>'$.nickname') VIRTUAL,
    age INT AS (data->>'$.age') VIRTUAL,
    status VARCHAR(20) AS (data->>'$.status') VIRTUAL,
    
    INDEX idx_nickname (nickname),
    INDEX idx_age (age),
    INDEX idx_status (status)
);

-- 查询示例
INSERT INTO user_profiles (data) VALUES (
    '{"nickname": "张三", "age": 28, "status": "active"}'
);

-- 可以直接用索引查询
SELECT * FROM user_profiles WHERE nickname = '张三';
SELECT * FROM user_profiles WHERE age > 25;
```

## 2. 扩展追问

### 🔥 扩展追问 1：Generated Column 和函数索引（Functional Index）有什么异同？

> A: **MySQL 8.0.13+ 的函数索引是 Generated Column 的语法糖，本质相同但使用方式不同。**

```sql
-- ========== 函数索引语法 (MySQL 8.0.13+) ==========
-- 传统 Generated Column 方式
ALTER TABLE orders 
    ADD COLUMN total_price DECIMAL(10,2) AS (price * quantity) VIRTUAL,
    ADD INDEX idx_total_price (total_price);

-- 函数索引方式（更简洁）
CREATE INDEX idx_total_price ON orders ((price * quantity));
--           ↑                              ↑
--           └──── 括号包裹表达式 ──────────┘

-- 两者生成的执行计划完全相同
EXPLAIN SELECT * FROM orders WHERE price * quantity > 500;
-- 两者都使用 idx_total_price 索引
```

```
┌─────────────────────────────────────────────────────────────┐
│              Generated Column vs 函数索引异同                 │
├─────────────────────────────────────────────────────────────┤
│                                                             │

│  ┌─────────────────────────────────────────────────────┐   │
│  │  - 底层实现相同：都是通过隐藏的生成列实现            │   │
```

### 🔥 扩展追问 2：Generated Column 的限制和注意事项？

> A: **Generated Column 有一些重要限制，包括表达式必须是确定性等。**

```sql
-- ========== 限制1：必须是确定性表达式 ==========
-- ✗ 以下表达式不允许（包含非确定性函数）
CREATE TABLE t (
    id INT,
    now_col DATETIME AS (NOW())  -- ✗ NOW() 非确定性
);

CREATE TABLE t (
    id INT,
    rand_col INT AS (RAND() * 100)  -- ✗ RAND() 非确定性
);

-- ✓ 允许的确定性表达式
CREATE TABLE t (
    id INT,
    pi DECIMAL(10,5) AS (3.14159),  -- 常量
    year_col INT AS (YEAR('2024-01-01')),  -- 确定日期
    concat_col VARCHAR(100) AS (CONCAT('a', 'b'))  -- 确定函数
);

-- ========== 限制2：不能引用其他生成列 ==========
CREATE TABLE t (
    a INT,
    b INT,
    c INT AS (a + b),      -- ✓ 引用普通列
    d INT AS (c * 2)        -- ✗ 不能引用其他生成列
);

-- ========== 限制3：VIRTUAL 列不支持某些功能 ==========
-- ✗ VIRTUAL 列不能作为外键
CREATE TABLE parent (
    id INT PRIMARY KEY,
    computed_val INT AS (id * 2) VIRTUAL  -- ✗ 不能被外键引用
);

-- ✓ STORED 列可以
CREATE TABLE parent (
    id INT PRIMARY KEY,
    computed_val INT AS (id * 2) STORED
);

-- child 表可以引用
CREATE TABLE child (
    id INT,
    parent_id INT,
    FOREIGN KEY (parent_id) REFERENCES parent(id)
);

-- ========== 限制4：表达式复杂度 ==========
-- 复杂表达式可能导致写入性能下降
CREATE TABLE large_table (
    id INT,
    -- STORED 列：每次 INSERT/UPDATE 都要计算
    computed STORED AS (
        col1 + col2 + col3 + col4 + col5 + col6 + col7 + col8
    )
);
```

### 🔥 扩展追问 3：Generated Column 的性能测试对比？

### 🔥 扩展追问 4：Generated Column 在实际业务中的应用案例？

> A: **电商订单、用户标签、搜索优化等场景。**

```sql
-- ========== 案例1：电商订单计算 ==========
CREATE TABLE ecommerce_orders (
    id BIGINT PRIMARY KEY,
    user_id BIGINT,
    original_amount DECIMAL(12,2),           -- 原价
    discount_amount DECIMAL(12,2),           -- 优惠
    tax_rate DECIMAL(5,4) DEFAULT 0.06,      -- 税率
    shipping_fee DECIMAL(10,2) DEFAULT 0,     -- 运费
    
    -- 生成列：计算最终价格
    final_amount DECIMAL(12,2) AS (
        (original_amount - discount_amount) * (1 + tax_rate) + shipping_fee
    ) STORED,
    
    -- 生成列：是否包邮
    is_free_shipping TINYINT AS (
        CASE WHEN shipping_fee = 0 THEN 1 ELSE 0 END
    ) VIRTUAL,
    
    created_at DATETIME,
    updated_at DATETIME,
    
    INDEX idx_user_id (user_id),
    INDEX idx_final_amount (final_amount),
    INDEX idx_created_at (created_at)
);

-- 查询高价值订单（利用索引）
SELECT * FROM ecommerce_orders 
WHERE final_amount > 1000 
ORDER BY created_at DESC;

-- ========== 案例2：用户标签管理 ==========
CREATE TABLE user_tags (
    id BIGINT PRIMARY KEY,
    user_id BIGINT,
    tags JSON,  -- ["vip", "active", "big_spender"]
    
    -- 生成列：提取关键标签
    is_vip TINYINT AS (
        JSON_CONTAINS(tags, '"vip"')
    ) VIRTUAL,
    
    is_active TINYINT AS (
        JSON_CONTAINS(tags, '"active"')
    ) VIRTUAL,
    
    tag_count INT AS (
        JSON_LENGTH(tags)
    ) VIRTUAL,
    
    INDEX idx_user_id (user_id),
    INDEX idx_is_vip (is_vip),
    INDEX idx_tag_count (tag_count)
);

-- 查询特定用户群
SELECT * FROM user_tags 
WHERE is_vip = 1 AND tag_count >= 3;

-- ========== 案例3：搜索优化 ==========
CREATE TABLE articles (
    id BIGINT PRIMARY KEY,
    title VARCHAR(200),
    content TEXT,
    author_id BIGINT,
    
    -- 生成列：计算长度
    title_len INT AS (CHAR_LENGTH(title)) VIRTUAL,
    content_len INT AS (CHAR_LENGTH(content)) VIRTUAL,
    
    -- 生成列：搜索友好标识
    search_key VARCHAR(255) AS (
        CONCAT_WS(' ', title, LEFT(content, 100))
    ) VIRTUAL,
    
    INDEX idx_author (author_id),
    INDEX idx_title_len (title_len),
    FULLTEXT INDEX idx_search (search_key)
);
```

## 3. 一句话总结

> **Generated Column 是 MySQL 自动计算并存储/计算列值的功能，VIRTUAL 不占存储空间但每次读取计算，STORED 物理存储但写入时计算；两者都支持建索引实现高效查询；MySQL 8.0.13+ 的函数索引是 Generated Column 的语法糖，本质相同但更简洁；典型场景是 JSON 字段提取+索引、复杂表达式预计算、数据一致性保证。**

## 4. 面试加分话术

> "我们之前做过订单查询的优化：业务需要按订单总价筛选和排序，订单总价 = 商品小计 + 运费 + 税费 - 优惠。原来方案是在应用层算好存一个字段，结果多次发现前后端计算不一致的 bug（浮点数精度问题）。后来改用 STORED 生成列，`final_amount DECIMAL(12,2) AS ((original + tax) * rate - discount) STORED`，结果完全一致，还省掉了应用层的计算逻辑。另外还有个小技巧：对于 JSON 类型的配置字段，我们用 VIRTUAL 生成列提取常用查询字段再建索引，比如 `status VARCHAR(20) AS (data->>'$.status') VIRTUAL`，这样可以用普通 SQL 查询 JSON 字段，还能走索引，性能比 `JSON_EXTRACT` 方案快 10 倍以上。MySQL 8.0 的函数索引也用过，直接 `CREATE INDEX idx_expr ON orders ((price * quantity))`，底层实现跟 Generated Column + INDEX 一模一样，但语法更简洁。"

**追问预判：**
- "Generated Column 能替代触发器吗？" → 部分场景可以，但触发器能做更多事情（如跨表操作）
- "表达式太复杂会影响性能吗？" → VIRTUAL 列不影响写入，STORED 列会影响写入性能
