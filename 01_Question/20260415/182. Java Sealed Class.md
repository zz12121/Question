## 基础流程

### 1. 什么是 Sealed Class？

密封类是 **JDK 17 正式特性**（JDK 15 预览），用于 **限制类的继承层次**，让开发者明确声明"哪些类可以扩展当前类"。

```
┌─────────────────────────────────────────────────────────────┐
│                    Sealed Class 继承体系                     │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌─────────────────┐                                       │
│   │  sealed class   │  ← 基类：声明 sealed                  │
│   │    Shape        │                                       │
│   │  permits         │                                       │
│   │  Circle,         │                                       │
│   │  Rectangle,      │                                       │
│   │  Triangle        │                                       │
│   └────────┬────────┘                                       │
│            │                                                 │
│    ┌───────┼───────┐                                         │
│    │       │       │                                         │
│    ▼       ▼       ▼                                         │
│ ┌──────┐ ┌─────────┐ ┌──────────┐                           │
│ │final │ │sealed   │ │non-sealed│                           │
│ │Circle│ │Rectangle│ │Triangle  │                           │
```

### 2. permits 子句的作用

```java
// sealed 修饰符 + permits 明确授权子类
public sealed class Shape
    permits Circle, Rectangle, Triangle {
    // Shape 的实现
}

// Circle: final → 不能再被继承
public final class Circle extends Shape {
    private final double radius;
}

// Rectangle: sealed → 允许有限扩展（自身也是密封的）
public sealed class Rectangle extends Shape
    permits Square {
    private final double width, height;
}

// Square: final → Rectangle 的唯一允许子类
public final class Square extends Rectangle {
    private final double side;
}

// Triangle: non-sealed → 恢复开放继承（解除封印）
public non-sealed class Triangle extends Shape {
    private final double a, b, c;
}

// 编译器错误！Triangle 是 non-sealed，其子类不受限制
class MyTriangle extends Triangle { }  // ✓ 合法
```

**permits 的核心约束**：

| 约束项 | 说明 |
|--------|------|
| 同一编译单元 | permits 列出的类必须在同一个 `.java` 文件中，或同一模块中 |
| 完整性 | 同一编译单元内的所有允许子类必须列出 |
| 兼容性 | 配合 `switch` 模式匹配实现编译期穷举检查 |

### 3. sealed / non-sealed / final 三者关系

```
┌────────────────────────────────────────────────────────────┐
│              sealed 族三兄弟的继承关系图谱                    │
├────────────────────────────────────────────────────────────┤
│                                                            │
│                 ┌─────────────────┐                        │
│                 │   sealed class  │                        │
│                 │       A         │                        │
│                 └────────┬────────┘                        │
│                          │                                 │
│           ┌──────────────┼──────────────┐                  │
│           │              │              │                  │
│           ▼              ▼              ▼                  │
│    ┌────────────┐ ┌───────────┐ ┌──────────────┐          │
│    │   final    │ │  sealed   │ │ non-sealed   │          │
│    │    B       │ │    C      │ │     D        │          │
│    ├────────────┤ ├───────────┤ ├──────────────┤          │
│    │ 不可继承    │ │ 可继承     │ │  恢复开放     │          │
│    │ 继承树终止   │ │ 但需 permits│ │  任意扩展     │          │
│    └────────────┘ └─────┬─────┘ └──────────────┘          │
│                         │                                   │
│                         ▼                                   │
│                  ┌────────────┐                            │
│                  │  sealed/   │                            │
│                  │  final     │                            │
│                  │ (子类定义)  │                            │
│                  └────────────┘                            │
└────────────────────────────────────────────────────────────┘
```

| 修饰符 | 含义 | 继承状态 | 典型场景 |
|--------|------|----------|----------|
| `final` | 最终类 | **禁止**任何子类 | 不可变数据、终结继承树 |
| `sealed` | 密封类 | **允许**受限子类（需声明 permits） | 有限状态机、受控扩展 |
| `non-sealed` | 非密封类 | **解除封印**，恢复开放继承 | 框架开放点、部分实现 |

**关键规则**：
```java
// sealed 类的子类必须是 sealed / final / non-sealed 之一
sealed class A permits B, C, D { }

// B: final → 终止
final class B extends A { }

// C: sealed → 继续受限
sealed class C extends A permits E, F { }
final class E extends C { }
final class F extends C { }

// D: non-sealed → 开放（解除封印）
non-sealed class D extends A { }
// D 的子类可以任意扩展
class AnySubclass extends D { }
```

### 4. Switch 模式匹配穷举检查（JDK 21）

```java
// JDK 17: 基础 switch 表达式
sealed interface Expr permits NumberExpr, BinaryExpr { }

record NumberExpr(int value) implements Expr { }
sealed interface BinaryExpr permits AddExpr, MulExpr { }
record AddExpr(Expr left, Expr right) implements BinaryExpr { }
record MulExpr(Expr left, Expr right) implements BinaryExpr { }

// JDK 21: 增强的 switch + 模式匹配
public static String eval(Expr expr) {
    return switch (expr) {
        // 编译器检查：是否覆盖了所有 sealed 子类？
        case NumberExpr(int n) -> "数字: " + n;
        case AddExpr(var l, var r) -> "( " + eval(l) + " + " + eval(r) + " )";
        case MulExpr(var l, var r) -> "( " + eval(l) + " * " + eval(r) + " )";
        // ⚠️ 编译器错误：missing implementation for MulExpr
        // ⚠️ 编译器错误：cannot exhaustively cover interface Expr
        // （如果后续添加了新子类）
    };
}
```

```
┌─────────────────────────────────────────────────────────────┐
│          Switch 穷举检查原理（编译器级别）                    │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  sealed class Shape permits Circle, Rectangle, Triangle     │
│                                                             │
│  String describe(Shape s) {                                 │
│      return switch(s) {                                     │
│          case Circle c    -> "圆";     // ✓                 │
│          case Rectangle r -> "矩形";   // ✓                 │
│          case Triangle t  -> "三角";   // ✓                 │
│      };  // ✓ 编译通过：覆盖全部 permits                     │
│  }                                                          │
│                                                             │
│  // 如果漏掉任何一个分支：                                     │
│  String describe(Shape s) {                                │
│      return switch(s) {                                     │
│          case Circle c -> "圆";       // ✓                 │
│          case Rectangle r -> "矩形";  // ✓                 │
│          // ❌ 漏掉 Triangle → 编译器错误                    │

│          //  all possible input values"                    │
│      };                                                    │
│  }                                                          │
│                                                             │

```

### 5. 代数数据类型（ADT）建模

**ADT 两大类型**：

```
┌─────────────────────────────────────────────────────────────┐
│                   代数数据类型 (ADT)                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────────────┐     ┌─────────────────────┐       │
│  │   Sum Type (和类型)  │     │ Product Type (积类型) │       │
│  │   枚举类型           │     │  组合类型             │       │
│  │   "A 或 B 或 C"      │     │  "A 且 B 且 C"        │       │
│  └─────────────────────┘     └─────────────────────┘       │
│            │                           │                    │
│            ▼                           ▼                    │
│  ┌─────────────────────┐     ┌─────────────────────┐       │
│  │  sealed class       │     │  record              │       │
│  │  Expr permits        │     │  Person(             │       │
│  │  Literal,           │     │    String name,      │       │
│  │  BinaryOp           │     │    int age           │       │
│  └─────────────────────┘     │  )                   │       │
│            │                 └─────────────────────┘       │
│            │                           │                    │
│            ▼                           │                    │
│  sealed interface Expr  ←─────────────┘                    │
│  permits Literal, BinaryOp    ↑                             │
│                               │ 组合使用                     │
│  record Literal(int v) implements Expr                     │
│  record BinaryOp(String op, Expr l, Expr r) implements Expr │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**经典案例：二叉树建模**

```java
// 传统方式：多层继承 + 抽象方法
public abstract class Tree<T> {
    public abstract T accept(Visitor<T> v);
}

// Sealed 方式：简洁、安全、编译器穷举检查
public sealed interface Tree<T>
    permits Tree.Empty, Tree.Leaf, Tree.Node {

    // 嵌套类定义子类型（同一文件内）
    final class Empty<T> implements Tree<T> { }

    final class Leaf<T> implements Tree<T> {
        private final T value;
        public Leaf(T value) { this.value = value; }
        public T value() { return value; }
    }

    final class Node<T> implements Tree<T> {
        private final Tree<T> left;
        private final Tree<T> right;
        public Node(Tree<T> left, Tree<T> right) {
            this.left = left;
            this.right = right;
        }
        public Tree<T> left() { return left; }
        public Tree<T> right() { return right; }
    }
}

// 编译器保证：所有分支都被覆盖
public <T> int size(Tree<T> tree) {
    return switch (tree) {
        case Tree.Empty<T>() -> 0;
        case Tree.Leaf<T>(var v) -> 1;
        case Tree.Node<T>(var l, var r) -> 1 + size(l) + size(r);
    };
}
```

**sealed + record 组合的完美性**：

```java
// 组合拳：sealed 限制 + record 简洁 + switch 穷举
public sealed interface Result<T, E>
    permits Result.Ok, Result.Err, Result.Pending {

    record Ok<T, E>(T value) implements Result<T, E> { }
    record Err<T, E>(E error) implements Result<T, E> { }
    record Pending<T, E>() implements Result<T, E> { }

    // 编译器穷举：添加新子类时强制检查所有 switch
    public <R> R match(
        Function<T, R> onOk,
        Function<E, R> onErr,
        Supplier<R> onPending
    ) {
        return switch (this) {
            case Ok<T, E>(var v) -> onOk.apply(v);
            case Err<T, E>(var e) -> onErr.apply(e);
            case Pending<T, E>() -> onPending.get();
        };
    }
}
```

## 🔥 扩展追问 1：sealed 与 `instanceof` 模式匹配的关系？

```java
// JDK 16+: instanceof 模式匹配（自动强转）
public static String describe(Object obj) {
    if (obj instanceof sealed class Shape s) {
        // s 自动强转为 Shape 类型
        return switch (s) {
            case Circle c -> "半径=" + c.radius();
            case Rectangle r -> "宽高=" + r.width() + "x" + r.height();
            case Triangle t -> "三边=" + t.a() + "," + t.b() + "," + t.c();
        };
    }
    return "未知形状";
}
```

> **A:** `instanceof` 模式匹配在 JDK 16 正式稳定，配合 sealed class 可以：
> 1. 自动完成类型强转（不再需要 `(Circle) obj`）
> 2. 结合 sealed 穷举检查，**双重保证**所有分支覆盖
> 3. `Pattern instanceof` 返回 `boolean` 并绑定变量，双重作用

## 🔥 扩展追问 2：sealed class 的反射限制？

```java
// 反射 API 新增方法
sealed class Base permits Derived { }

// 查询密封信息
SealedPermission sp = Base.class.getAnnotation(SealedPermission.class);
Class<?>[] permitted = Base.class.getPermittedSubclasses();

// 检查某类是否是密封类的合法子类
boolean isPermitted = sp != null; // 配合 Class Permission 检查

// JVM 级别约束
// 尝试非法继承 → ClassFormatError
class Illegal extends Base { } // 运行时抛出
```

> **A:** 
> - **编译期检查**：继承 sealed class 必须在 permits 列表中
> - **运行时保护**：JVM 在 `ClassFile` 层面验证 permits 合法性
> - **反射查询**：`getPermittedSubclasses()` 可在运行时获取授权子类列表
> - **组合使用**：适合实现**权限模型**——哪些类可以实例化、哪些插件被允许加载

## 🔥 扩展追问 3：sealed interface 与 sealed class 的区别？

```java
// sealed interface：更适合建模"行为类型"
public sealed interface Comparable<T>
    permits Integer, String, LocalDate {
    int compareTo(T other);
}

// sealed class：更适合建模"数据实体"
public sealed class Shape permits Circle, Rectangle { }

// 嵌套混合：sealed interface + record 实现
public sealed interface Expr permits Expr.Literal, Expr.Add { }

record Literal(int val) implements Expr { }
record Add(Expr left, Expr right) implements Expr { }

// 两者语法几乎一致，核心区别：
// - class: 默认 extends（只能单继承）
// - interface: 默认 implements（可多实现）
```

> **A:** 
> - **sealed interface**：适合定义行为抽象，强调"能做什么"
> - **sealed class**：适合定义数据实体，强调"是什么"
> - 两者可以**嵌套组合**：`sealed interface Expr permits Expr.Literal, Expr.Add {}` + `record Literal(int val) implements Expr {}`
> - `permits` 列表可以包含**嵌套类、record、enum**

## 🔥 扩展追问 4：sealed class 对 `Optional` / `Stream` 处理的帮助？

```java
// 问题：多层嵌套 Optional 导致代码臃肿
String result = optUser
    .flatMap(User::getAddress)
    .flatMap(Address::getCity)
    .map(City::getName)
    .orElse("未知");

// sealed class + switch 模式匹配：清晰处理多状态
public sealed interface ApiResult<T>
    permits ApiResult.Success, ApiResult.Error, ApiResult.Loading {
    
    record Success<T>(T data) implements ApiResult<T> { }
    record Error<T>(String message, int code) implements ApiResult<T> { }
    record Loading<T>() implements ApiResult<T> { }
}

// Stream 中处理多状态
List<String> names = results.stream()
    .map(r -> switch (r) {
        case ApiResult.Success<T>(var data) -> data.toString();
        case ApiResult.Error<T>(var msg, _) -> "错误: " + msg;
        case ApiResult.Loading<T>() -> "加载中...";
    })
    .toList();
```

> **A:** sealed class 在处理**多状态返回值**时特别有用：
> 1. **替代 Optional**：明确表达 Success/Error/Loading 三种状态，而非 null
> 2. **替代 Boolean**：避免 true/false 语义模糊（如 isActive = false 是"禁用"还是"未激活"？）
> 3. **Stream 友好**：`switch (r) { case X -> ... }` 可以直接作为 lambda 表达式传入 `map()`

## 🔥 扩展追问 5：sealed class 在 Spring 生态中的应用？

```java
// 1. Spring Security: 认证结果建模
public sealed interface AuthenticationResult
    permits AuthenticationResult.Success,
            AuthenticationResult.Fail,
            AuthenticationResult.MfaRequired {
    
    record Success(UserDetails user, String token) implements AuthenticationResult {}
    record Fail(String reason) implements AuthenticationResult {}
    record MfaRequired(String mfaToken) implements AuthenticationResult {}
}

// 2. Spring Data: 领域事件
public sealed interface DomainEvent
    permits OrderPlaced, OrderPaid, OrderCancelled {
    
    record OrderPlaced(Long orderId, Instant occurredAt) implements DomainEvent {}
    record OrderPaid(Long orderId, String paymentId) implements DomainEvent {}
    record OrderCancelled(Long orderId, String reason) implements DomainEvent {}
}

@Service
public class EventHandler {
    public void handle(DomainEvent event) {
        return switch (event) {
            case OrderPlaced(var id, _) -> onOrderPlaced(id);
            case OrderPaid(var id, var pid) -> onOrderPaid(id, pid);
            case OrderCancelled(var id, _) -> onOrderCancelled(id);
        };
    }
}
```

> **A:** Spring 生态中 sealed class 的应用场景：
> 1. **认证/授权结果**：`Success/Fail/MFA` 多状态，比 Optional 更语义清晰
> 2. **领域事件驱动**：`OrderPlaced/Paid/Cancelled` 穷举，编译器防止漏处理
> 3. **策略模式替代**：传统 if-else 改为 sealed + switch，**新增状态时编译器强制检查所有分支**
> 4. **DDD 聚合根状态机**：聚合根内部状态用 sealed class 建模，外部只暴露事件
