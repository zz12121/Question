**代码对比**：

```java
public class Comparison {

    public int add(int a, int b) { return a + b; }

    public static void main(String[] args) throws Throwable {
        
        // ============ 反射方式 ============
        Method method = Comparison.class.getMethod("add", int.class, int.class);
        method.setAccessible(true); // 跳过访问检查
        
        // 调用（每次调用都是 reflect 调用）
        long start = System.nanoTime();
        for (int i = 0; i < 1_000_000; i++) {
            Integer result = (Integer) method.invoke(new Comparison(), 1, 2);
        }
        long reflectTime = System.nanoTime() - start;

        // ============ MethodHandle 方式 ============
        MethodHandles.Lookup lookup = MethodHandles.lookup();
        MethodType mt = MethodType.methodType(int.class, int.class, int.class);
        MethodHandle mh = lookup.findVirtual(Comparison.class, "add", mt);
        
        // 调用（ JIT 可内联优化）
        long start2 = System.nanoTime();
        for (int i = 0; i < 1_000_000; i++) {
            int result = (int) mh.invokeExact(new Comparison(), 1, 2);
        }
        long handleTime = System.nanoTime() - start2;

        // ============ 直接调用 ============
        long start3 = System.nanoTime();
        Comparison obj = new Comparison();
        for (int i = 0; i < 1_000_000; i++) {
            int result = obj.add(1, 2);
        }
        long directTime = System.nanoTime() - start3;

        System.out.println("反射: " + reflectTime/1_000_000 + " ms");
        System.out.println("MH:   " + handleTime/1_000_000 + " ms");
        System.out.println("直接: " + directTime/1_000_000 + " ms");
        // 典型结果：反射 ~800ms, MH ~100ms, 直接 ~30ms
    }
}
```

**性能差异原因**：

| 维度 | 反射 | MethodHandle |
|------|------|--------------|
| 调用开销 | 每次通过 `MethodAccessor` 反射调用 | JIT 内联后接近直接调用 |
| 安全检查 | `setAccessible` 跳过模块检查 | `Lookup` 创建时一次检查，后续无开销 |
| 内存占用 | `Method` 对象持有注解等元数据 | 仅持有类型签名+方法指针 |
| JIT 友好 | 对象开销大，难以内联 | 类似函数指针，JIT 可优化 |

### 2. invokeExact vs invoke 的区别

```java
public class InvokeDemo {

    public int add(int a, int b) { return a + b; }
    public int add(Integer a, Integer b) { return a + b; } // 重载
    public long add(long a, long b) { return a + b; }      // 返回类型不同

    public static void main(String[] args) throws Throwable {
        MethodHandles.Lookup lookup = MethodHandles.lookup();
        MethodType mt = MethodType.methodType(int.class, int.class, int.class);
        MethodHandle mh = lookup.findVirtual(InvokeDemo.class, "add", mt);

        InvokeDemo obj = new InvokeDemo();

        // ============ invokeExact ============
        // 必须精确匹配签名：int add(int, int)
        int r1 = (int) mh.invokeExact(obj, 1, 2);  // ✓ 正确
        
        // ⚠️ 编译错误：类型不匹配（找不到方法）
        // int r2 = (int) mh.invokeExact(obj, 1L, 2L);  // ❌
        
        // ⚠️ 编译错误：返回类型不匹配
        // long r3 = mh.invokeExact(obj, 1, 2);  // ❌

        // ============ invoke（允许适配）===========
        // invokeExact 抛出 WrongMethodTypeException 时，JVM 尝试类型适配
        int r4 = (int) mh.invoke(obj, 1, 2);      // ✓ 自动适配
        // 实际调用时，int → Integer 自动装箱后再调用
        
        // 适配开销：invoke 比 invokeExact 稍慢（约 5-10%）
    }
}
```

```

│              invokeExact vs invoke 调用链                        │
├────────────────────────────────────────────────────────────────┤
│                                                                │

│  ┌──────────────────────────────────────────────────────────┐ │
```

### 3. MethodHandles.Lookup 权限检查机制

```java
public class LookupMechanism {

    // ============ Lookup 创建的三种方式 ============
    
    // 1. MethodHandles.lookup() - 从调用位置获取权限
    public static void fromCaller() throws Throwable {
        // 在 ClassA 中调用，从 ClassA 的视角创建 Lookup
        MethodHandles.Lookup lookup = MethodHandles.lookup();
        // lookup 可以访问 ClassA 能访问的所有成员
    }

    // 2. MethodHandles.publicLookup() - 仅公共成员
    public static void publicOnly() throws Throwable {
        MethodHandles.Lookup lookup = MethodHandles.publicLookup();
        // 只能访问 public 方法
        // MethodHandle mh = lookup.findStatic(String.class, "valueOf", ...);
    }

    // 3. Lookup.defineHiddenClass() - 动态类访问
    public static void hiddenClass() throws Throwable {
        // 定义隐藏类（相当于 unsafe defineAnonymousClass）
        MethodHandles.Lookup lookup = MethodHandles.lookup();
        
        // 1. 生成隐藏类字节码
        byte[] bytes = ...; 
        
        // 2. 定义隐藏类 + 获取私有访问权限
        MethodHandles.Lookup hiddenLookup = lookup.defineHiddenClass(
            bytes, true, MethodHandles.Lookup.ClassOption.NESTMATE
        ).lookup();
        
        // 3. 隐藏类没有正式类名，但有私有访问能力
        // 用于 LambdaMetafactory 动态生成实现类
    }

    // ============ Lookup 权限层级 ============
    static class AccessLevels {
        // 权限由创建 Lookup 时的上下文决定
        // lookupModes() 返回可用的权限位掩码

        void demonstrate() throws Throwable {
            MethodHandles.Lookup lookup = MethodHandles.lookup();
            int modes = lookup.lookupModes(); // 权限位掩码
            
            // 检查权限位
            boolean hasPrivateAccess   = (modes & MethodHandles.Lookup.PRIVATE) != 0;
            boolean hasPackageAccess   = (modes & MethodHandles.Lookup.PACKAGE) != 0;
            boolean hasModuleAccess    = (modes & MethodHandles.Lookup.MODULE) != 0;
            boolean hasPublicAccess    = (modes & MethodHandles.Lookup.PUBLIC) != 0;
            
            // Lookup 本身是 "有权限访问某个类的查找器"
            Class<?> lookupClass = lookup.lookupClass(); // 谁创建的
            
            System.out.println("Lookup 拥有者: " + lookupClass);
            System.out.println("权限位: " + Integer.toBinaryString(modes));
        }
    }

    // ============ findVirtual / findStatic / findGetter ============
    public static void findExamples() throws Throwable {
        MethodHandles.Lookup lookup = MethodHandles.lookup();

        // 查找实例方法
        MethodHandle mh = lookup.findVirtual(
            String.class,        // 目标类
            "length",            // 方法名
            MethodType.methodType(int.class)  // (int) → int
        );
        int len = (int) mh.invokeExact("hello"); // 5

        // 查找静态方法
        MethodHandle sqrt = lookup.findStatic(
            Math.class,
            "sqrt",
            MethodType.methodType(double.class, double.class)
        );
        double result = (double) sqrt.invokeExact(4.0); // 2.0

        // 访问字段（getter）
        MethodHandle getAge = lookup.findGetter(
            Person.class,
            "age",
            int.class
        );

        // 访问私有方法
        MethodHandle privateMethod = lookup.findSpecial(
            Outer.class,
            "privateMethod",
            MethodType.methodType(void.class),
            Inner.class  // 限定 this 类型
        );
    }
}
```

**权限检查的核心原理**：

```
┌────────────────────────────────────────────────────────────────┐
│              MethodHandles.Lookup 权限模型                       │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  创建 Lookup 时检查（一次）：                                    │
│  ┌──────────────────────────────────────────────────────────┐ │
│  │ 1. 从当前栈帧获取 caller 类                                │ │
│  │ 2. 检查 caller 对目标类的访问权限                          │ │

└────────────────────────────────────────────────────────────────┘
```

## 🔥 扩展追问 1：MethodHandle 链式调用（fold/dropArguments/insertArguments）？

```java
public class MethodHandleChain {

    public static void main(String[] args) throws Throwable {
        MethodHandles.Lookup lookup = MethodHandles.lookup();

        // 基础 MH
        MethodHandle mh = lookup.findStatic(
            Integer.class, "sum", 
            MethodType.methodType(int.class, int.class, int.class)
        );

        // 1. insertArguments：预填充参数
        MethodHandle mhBound = mh.insertArguments(0, 10); // 固定第一个参数为 10
        int r1 = (int) mhBound.invokeExact(20); // 相当于 sum(10, 20) = 30
        System.out.println("insertArguments: " + r1);

        // 2. dropArguments：丢弃多余参数
        MethodHandle mhDropped = MethodHandles.dropArguments(mh, 0, String.class);
        // 签名变为 (String, int, int) → int
        int r2 = (int) mhDropped.invokeExact("ignored", 10, 20);
        System.out.println("dropArguments: " + r2);

        // 3. guardWithTest：条件执行（类似三元运算符）
        MethodHandle guard = lookup.findStatic(
            Objects.class, "nonNull",
            MethodType.methodType(boolean.class, Object.class)
        );
        MethodHandle ifTrue = lookup.findStatic(
            System.class, "currentTimeMillis",
            MethodType.methodType(long.class)
        );
        MethodHandle ifFalse = MethodHandles.constant(long.class, -1L);
        
        MethodHandle mhConditional = MethodHandles.guardWithArguments(
            guard, ifTrue, ifFalse
        );
        long r3 = (long) mhConditional.invokeExact((Object) "hello");
        System.out.println("guardWithTest (not null): " + r3);

        // 4. filterArguments：参数预处理
        MethodHandle append = lookup.findVirtual(
            StringBuilder.class, "append",
            MethodType.methodType(StringBuilder.class, String.class)
        );
        MethodHandle toUpperCase = lookup.findVirtual(
            String.class, "toUpperCase",
            MethodType.methodType(String.class)
        );
        
        // 将 append 的第一个参数先经过 toUpperCase 处理
        MethodHandle mhFiltered = MethodHandles.filterArguments(append, 1, toUpperCase);
        
        StringBuilder sb = new StringBuilder("Hello ");
        String result = (String) mhFiltered.invoke(sb, "world");
        System.out.println("filterArguments: " + result);
    }
}
```

> **A:** MethodHandle 提供丰富的组合器：
> 1. **insertArguments**：预填充参数，固定部分值
> 2. **dropArguments**：增加假参数以适配签名
> 3. **guardWithArguments**：条件执行（if-else 逻辑）
> 4. **filterArguments**：参数预处理（类似 AOP 前置通知）
> 5. **foldArguments**：将返回值作为下一个调用的额外参数

## 🔥 扩展追问 2：Reflection vs MethodHandle vs invokedynamic 性能对比？

```java
public class PerformanceComparison {

    static final int ITERATIONS = 1_000_000;

    public int method(int x) { return x * 2; }

    public static void main(String[] args) throws Throwable {
        PerformanceComparison obj = new PerformanceComparison();

        // ============ 1. 直接调用（基准）===========
        long t0 = System.nanoTime();
        for (int i = 0; i < ITERATIONS; i++) {
            int r = obj.method(i);
        }
        System.out.println("Direct call:       " + (System.nanoTime() - t0) / 1_000_000);

        // ============ 2. MethodHandle invokeExact ============
        MethodHandle mh = MethodHandles.lookup()
            .findVirtual(PerformanceComparison.class, "method",
                MethodType.methodType(int.class, int.class));
        long t1 = System.nanoTime();
        for (int i = 0; i < ITERATIONS; i++) {
            int r = (int) mh.invokeExact(obj, i);
        }
        System.out.println("MethodHandle exact: " + (System.nanoTime() - t1) / 1_000_000);

        // ============ 3. MethodHandle invoke（类型适配）===========
        long t2 = System.nanoTime();
        for (int i = 0; i < ITERATIONS; i++) {
            int r = (int) mh.invoke(obj, Integer.valueOf(i));
        }
        System.out.println("MethodHandle invoke:" + (System.nanoTime() - t2) / 1_000_000);

        // ============ 4. 反射 ============
        Method m = PerformanceComparison.class.getMethod("method", int.class);
        long t3 = System.nanoTime();
        for (int i = 0; i < ITERATIONS; i++) {
            int r = (int) m.invoke(obj, i);
        }
        System.out.println("Reflection invoke:  " + (System.nanoTime() - t3) / 1_000_000);

        // ============ 5. 反射 + setAccessible ============
        m.setAccessible(true);
        long t4 = System.nanoTime();
        for (int i = 0; i < ITERATIONS; i++) {
            int r = (int) m.invoke(obj, i);
        }
        System.out.println("Reflection + acc:  " + (System.nanoTime() - t4) / 1_000_000);
    }
}
```

**典型结果（JDK 17，关闭分层编译）**：

```
Direct call:        ~30 ms
MethodHandle exact: ~80 ms   (2.7x 直接调用)
MethodHandle invoke: ~120 ms (4x 直接调用)
Reflection invoke:   ~600 ms  (20x 直接调用)
Reflection + acc:    ~800 ms  (27x 直接调用)

说明：
- MethodHandle 虽仍有开销，但 JIT 可内联优化
- 反射开销来自每次调用的包装（MethodAccessor）
- JDK 9+ 反射有 inflation 机制，首次调用生成字节码后加速
```

> **A:** 
> - 反射每次调用都经过 `MethodAccessor`（反射调用器），即使有 inflation 优化也有开销
> - MethodHandle 编译后 JIT 可内联，接近直接调用
> - invokedynamic 更进一步：**调用点可重绑定**，运行时可换掉目标实现（如多态调度、lambda）

## 🔥 扩展追问 4：Lookup.defineHiddenClass 的实际应用？

```java
public class HiddenClassApplication {

    // ============ 场景1：动态生成数据类 ============
    static class DynamicClassFactory {

        public static Class<?> createClass(String... fieldNames) 
            throws Throwable {
            
            // 生成类字节码
            ClassWriter cw = new ClassWriter(ClassWriter.COMPUTE_FRAMES);
            cw.visit(0, 0x1000, "HiddenClass", null, "java/lang/Object", null);
            
            // 添加构造函数
            MethodVisitor mv = cw.visitMethod(ACC_PUBLIC, "<init>", "()V", null, null);
            mv.visitCode();
            mv.visitVarInsn(ALOAD, 0);
            mv.visitMethodInsn(INVOKESPECIAL, "java/lang/Object", "<init>", "()V");
            mv.visitInsn(RETURN);
            mv.visitMaxs(0, 0);
            mv.visitEnd();
            
            cw.visitEnd();
            
            // 定义隐藏类
            return MethodHandles.lookup()
                .defineHiddenClass(cw.toByteArray(), true)
                .lookupClass();
        }
    }

    // ============ 场景2：Lambda 表达式底层（等价于 JDK 内部实现）===========
    static <T> T lambdaToMethodHandle(
        MethodHandles.Lookup lookup,
        Class<T> functionalInterface,
        MethodHandle implMethod
    ) throws Throwable {
        
        MethodType samType = MethodType.methodType(
            functionalInterface
        );
        MethodType implType = implMethod.type();
        
        CallSite cs = LambdaMetafactory.metafactory(
            lookup,
            functionalInterface.getSimpleName(),  // "apply"
            samType,
            implType.erase(),                       // 擦除到 Object
            implMethod,
            implType
        );
        
        return (T) cs.dynamicInvoker().invoke();
    }
}
```

> **A:** `defineHiddenClass` 的核心应用：
> 1. **LambdaMetafactory 内部**：JDK 用它生成 lambda 实现类
> 2. **动态代理**：可替代 `Proxy.newProxyInstance()` 的字节码生成
> 3. **规则引擎**：运行时生成自定义决策类
> 4. **序列化**：动态生成字段读写器
> 5. **特点**：无正式类名、不占用 ClassLoader、不参与类卸载检查

## 🔥 扩展追问 5：MethodHandle 能否替代反射做通用工具库？

```java
// 通用对象映射器：用 MethodHandle 替代反射
public class ReflectionAlternative {

    // 用 MethodHandle 实现通用的 getter/setter
    static class PropertyHandle {
        private final Class<?> clazz;
        private final MethodHandle getter;
        private final MethodHandle setter;
        
        PropertyHandle(MethodHandles.Lookup lookup, 
                       Class<?> clazz, String name) throws Throwable {
            this.clazz = clazz;
            
            // 查找 getter
            String getName = "get" + capitalize(name);
            this.getter = lookup.findVirtual(clazz, getName, 
                MethodType.methodType(Object.class, clazz));
            
            // 查找 setter
            String setName = "set" + capitalize(name);
            this.setter = lookup.findVirtual(clazz, setName, 
                MethodType.methodType(void.class, clazz, Object.class));
        }
        
        Object get(Object obj) throws Throwable {
            return getter.invoke(obj); // 无需 setAccessible！
        }
        
        void set(Object obj, Object value) throws Throwable {
            setter.invoke(obj, value);
        }
    }
    
    // 性能优势
    // - 无需 setAccessible（Lookup 创建时已验证权限）
    // - JIT 可内联 getter.invoke / setter.invoke
    // - 比 Field.setAccessible + Field.get 更安全
}
```

> **A:** MethodHandle **可以替代反射**做通用工具库，优势：
> 1. **权限模型更精确**：`Lookup` 创建时嵌入权限，调用时无额外检查
> 2. **性能更好**：JIT 可内联，减少方法调用开销
> 3. **更安全**：不需要 `--add-opens` 绕过模块边界
> 4. **可组合**：filter/fold/guardWith 等组合器实现复杂逻辑
> 
> **限制**：API 复杂度高，不适合简单场景；MethodHandle 更"底层"

## 一句话总结

> **`MethodHandle` = JVM 原生方法引用，JIT 友好且可组合；`invokedynamic` = 字节码级延迟绑定，lambda 表达式底层用它动态生成实现类；`Reflection` = 运行时元数据 API，功能全但性能开销大。**
